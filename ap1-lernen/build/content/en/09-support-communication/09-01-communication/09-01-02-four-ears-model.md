# The Four-Ears Model (Schulz von Thun)

## Learning Objectives

After this section, you will be able to:
- Explain the four sides of a message
- Apply the model to IT support situations
- Recognize and avoid misunderstandings
- Communicate more consciously

---

## Core Content

### Basic Idea of the Model

Every message has **four sides** (or "ears") on which it can be sent and received.

```
┌─────────────────────────────────────────────────────────────────────┐
│           THE FOUR-EARS MODEL                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│                         FACTUAL CONTENT                              │
│                        (What I am informing about)                   │
│                              │                                       │
│                              │                                       │
│                              ▼                                       │
│                        ┌──────────┐                                 │
│   SELF-REVELATION      │          │        APPEAL                   │
│   (What I reveal    ──►│ MESSAGE  │◄── (What I want to             │
│    about myself)       │          │     make you do)                │
│                        └──────────┘                                 │
│                              │                                       │
│                              │                                       │
│                              ▼                                       │
│                         RELATIONSHIP                                 │
│                    (What I think of you)                            │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### The Four Sides of a Message

```
┌─────────────────────────────────────────────────────────────────────┐
│           THE FOUR SIDES IN DETAIL                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  1. FACTUAL CONTENT (Factual Ear)                                   │
│     ───────────────────────────────────────────────────────────     │
│     → The pure information                                          │
│     → Facts, data, subject matter                                   │
│     → "What is being discussed?"                                    │
│                                                                      │
│  2. SELF-REVELATION (Self-Revelation Ear)                           │
│     ───────────────────────────────────────────────────────────     │
│     → What the sender reveals about themselves                      │
│     → Feelings, opinions, needs                                     │
│     → "What does this say about the sender?"                        │
│                                                                      │
│  3. RELATIONSHIP (Relationship Ear)                                 │
│     ───────────────────────────────────────────────────────────     │
│     → How the sender relates to the receiver                        │
│     → Appreciation, rejection, respect                              │
│     → "How is this person talking to me?"                           │
│                                                                      │
│  4. APPEAL (Appeal Ear)                                             │
│     ───────────────────────────────────────────────────────────     │
│     → What the sender wants from the receiver                       │
│     → Request, wish, demand                                         │
│     → "What should I do/think/feel?"                                │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Example: "My PC is broken again!"

```
┌─────────────────────────────────────────────────────────────────────┐
│           EXAMPLE ANALYSIS                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  STATEMENT: "My PC is broken again!"                                │
│                                                                      │
│                         FACTUAL CONTENT                              │
│                    "The PC is not working"                           │
│                              │                                       │
│                              │                                       │
│                              ▼                                       │
│                        ┌──────────┐                                 │
│   SELF-REVELATION      │  "My     │        APPEAL                   │
│  "I'm frustrated"   ──│   PC     │──"Fix it!"                      │
│  "This annoys me"      │  is      │  "Help me!"                     │
│  "I'm stressed"        │ broken   │  "Do it right                   │
│                        │ again!"  │   this time!"                   │
│                        └──────────┘                                 │
│                              │                                       │
│                              │                                       │
│                              ▼                                       │
│                         RELATIONSHIP                                 │
│                "IT does poor work"                                   │
│                "This has happened again"                             │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Analyzing Communication in IT Support

| User Statement | Factual Content | Self-Revelation | Relationship | Appeal |
|----------------|-----------------|-----------------|--------------|--------|
| "The internet is slow" | Network has low speed | I'm impatient | Expects quick solution | Make it faster! |
| "It worked yesterday" | Something has changed | I didn't do anything | It's not my fault | Find out what happened |
| "I need to work urgently" | It's time-critical | I'm under pressure | Take me seriously | Handle my problem first |
| "Nobody understands this" | Software is complicated | I'm overwhelmed | Expect better explanation | Explain it more simply |

### Typical Misunderstandings

```
┌─────────────────────────────────────────────────────────────────────┐
│           AVOIDING MISUNDERSTANDINGS                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  PROBLEM: Sender and receiver use different "ears"                  │
│                                                                      │
│  EXAMPLE:                                                           │
│  ──────────────────────────────────────────────────────────────     │
│  User: "This program is totally complicated!"                       │
│                                                                      │
│  Possible interpretations by IT Support:                            │
│                                                                      │
│  ┌────────────────────────────────────────────────────────────┐     │
│  │ FACTUAL EAR:     "The software has many functions"         │     │
│  │                  → Response: "Yes, that's true"            │     │
│  │                                                            │     │
│  │ RELATIONSHIP EAR:"You explained this poorly to me"         │     │
│  │                  → Response (defensive): "I showed         │     │
│  │                    it to you, though!"                     │     │
│  │                                                            │     │
│  │ APPEAL EAR:      "Please help me!"                         │     │
│  │                  → Response: "I'll show you again"        │ ✓   │
│  │                                                            │     │
│  │ SELF-REVELATION EAR: "They're overwhelmed"                 │     │
│  │                  → Response: "I understand that it        │     │
│  │                    can be confusing at first"             │ ✓   │
│  └────────────────────────────────────────────────────────────┘     │
│                                                                      │
│  RECOMMENDATION: Listen with the Appeal or Self-Revelation ear!     │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Recommendations for IT Support

```
┌─────────────────────────────────────────────────────────────────────┐
│           COMMUNICATION TIPS                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  1. LISTEN WITH THE RIGHT EAR                                       │
│     ───────────────────────────────────────────────────────────     │
│     → Don't take everything on the relationship level               │
│     → Recognize the appeal behind the statement                     │
│     → Perceive the self-revelation (frustration)                    │
│                                                                      │
│  2. CONSCIOUSLY SEND ON ALL LEVELS                                  │
│     ───────────────────────────────────────────────────────────     │
│     → Inform clearly on factual level                               │
│     → Show understanding (acknowledge self-revelation)              │
│     → Communicate appreciatively (positive relationship)            │
│     → Give clear options for action (appeal)                        │
│                                                                      │
│  3. EXAMPLE OF A GOOD RESPONSE                                      │
│     ───────────────────────────────────────────────────────────     │
│     User: "This program is totally complicated!"                    │
│                                                                      │
│     Response: "I understand that it's a lot at once               │
│     at first [acknowledge self-revelation]. The program             │
│     does indeed have many functions [factual content].              │
│     I'd be happy to show you the most important steps               │
│     again [respond to appeal]. When would suit you?                 │
│     [positive relationship: user decides]"                          │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Using I-Messages

```
┌─────────────────────────────────────────────────────────────────────┐
│           I-MESSAGES vs. YOU-MESSAGES                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  YOU-MESSAGE (AVOID):                                               │
│  ──────────────────────────────────────────────────────────────     │
│  → "You did that wrong"                                             │
│  → "You forgot that"                                                │
│  → "You need to pay better attention"                               │
│  → Creates defensive reaction                                       │
│                                                                      │
│  I-MESSAGE (RECOMMENDED):                                           │
│  ──────────────────────────────────────────────────────────────     │
│  → "I see that an error has occurred here"                          │
│  → "I noticed that this step is missing"                            │
│  → "I recommend paying attention to this in the future"             │
│  → Avoids blame assignment                                          │
│                                                                      │
│  STRUCTURE OF AN I-MESSAGE:                                         │
│  ──────────────────────────────────────────────────────────────     │
│  1. Observation: "I see that..."                                    │
│  2. Feeling/Reaction: "This leads to..."                            │
│  3. Wish: "I would appreciate if..."                                │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

## Key Terms

| Term | Explanation |
|------|-------------|
| **Factual Content** | The pure factual information of a message |
| **Self-Revelation** | What the sender communicates about themselves |
| **Relationship Level** | How the sender relates to the receiver |
| **Appeal** | What the sender wants from the receiver |
| **I-Message** | Statement without blame assignment |
| **Active Listening** | Conscious, attentive listening |

---

## Exam Tips

1. **Remember the four levels:**
   - **F**actual content
   - **S**elf-revelation
   - **R**elationship
   - **A**ppeal
   - Memory aid: **FSRA** or "Factual-Self-Relationship-Appeal"

2. **Apply in IT context:**
   - Often frustration (self-revelation) and desire for help (appeal) come together
   - Don't take it personally (relationship level)

3. **Typical exam questions:**
   - "Analyze the statement according to the Four-Ears Model"
   - "How should IT support respond?"

## Practice Exercises

### Exercise 1: Analyze Message

Analyze the user's statement according to the Four-Ears Model:

**"I've called three times already and the problem is still not solved!"**

<details>
<summary>Show Solution</summary>

| Level | Interpretation |
|-------|----------------|
| **Factual Content** | The problem persists despite three calls |
| **Self-Revelation** | "I'm frustrated and angry" / "I don't feel taken seriously" |
| **Relationship** | "IT support doesn't work reliably" / "You don't take my problem seriously" |
| **Appeal** | "Finally solve the problem!" / "Treat my case with priority!" |

**Recommended response:**
- Show understanding for the frustration
- Don't become defensive
- Check ticket history
- Offer concrete solution approach
- Make binding commitment

</details>

### Exercise 2: Formulate Response

A user says: "This is so complicated, how am I supposed to work like this?"

Formulate a professional response that considers all four levels.

<details>
<summary>Show Solution</summary>

**Professional response:**

"I understand that the current workflow seems complicated and costs you time **[acknowledge self-revelation]**.

Let's look together at which steps exactly are problematic **[clarify factual level]**.

I'd like to help you find a better solution **[positive relationship]**.

Can you show me where exactly the issue is? Then we'll find a way together **[respond to appeal]**."

</details>

### Exercise 3: Reformulate I-Messages

Reformulate the following you-messages into I-messages:

a) "You entered the password incorrectly."
b) "You should have told me earlier!"
c) "You don't understand this correctly."

<details>
<summary>Show Solution</summary>

a) **"You entered the password incorrectly."**
   → "I see that the login didn't work. This can happen when the password isn't entered correctly. Let's check this together."

b) **"You should have told me earlier!"**
   → "I noticed that I was missing this information until now. With this info, I can solve the problem faster. For the future: The more details you give me right away, the faster I can find the solution."

c) **"You don't understand this correctly."**
   → "I realize that I may not have explained this clearly enough. Let me explain it differently."

</details>
