# Contract Types

## Learning Objectives

After this section, you will be able to:
- Distinguish the most important contract types in the IT sector
- Explain characteristics of purchase, work, and service contracts
- Assign the correct contract type for various IT services
- Name differences in liability and success responsibility

---

### Comparison of Contract Types

| Criterion | Purchase Contract | Work Contract | Service Contract |
|-----------|-------------------|---------------|------------------|
| **Subject** | Item/Right | Work (result) | Activity |
| **Success** | Delivery | Completion | Only effort |
| **Remuneration** | Purchase price | Work fee | Hourly rate |
| **Warranty rights** | Yes (warranty) | Yes (remediation) | Limited |
| **Acceptance** | Handover | Acceptance required | None |
| **Termination** | Withdrawal | Possible at any time | Notice period |
| **German Civil Code** | §§ 433-480 | §§ 631-651 | §§ 611-630 |

### The Purchase Contract

```
┌─────────────────────────────────────────────────────────────────────┐
│                    PURCHASE CONTRACT (§§ 433 ff. BGB)                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  SELLER                                          BUYER              │
│  ┌─────────────┐                           ┌─────────────┐          │
│  │             │  Duty: Deliver            │             │          │
│  │   Seller    │ ────────────────────────► │    Buyer    │          │
│  │             │  the item                 │             │          │
│  │             │                           │             │          │
│  │             │  Duty: Pay purchase       │             │          │
│  │             │ ◄──────────────────────── │             │          │
│  │             │  price                    │             │          │
│  └─────────────┘                           └─────────────┘          │
│                                                                      │
│  Seller's duties:                                                   │
│  → Deliver the item                                                 │
│  → Transfer ownership                                               │
│  → Item free from material and legal defects                        │
│                                                                      │
│  Buyer's duties:                                                    │
│  → Pay purchase price                                               │
│  → Accept the item                                                  │
│                                                                      │
│  IT Examples:                                                       │
│  → Hardware purchase (PC, server, printer)                          │
│  → Software license purchase (one-time payment)                     │
│  → Purchase of consumables                                          │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### The Work Contract

```
┌─────────────────────────────────────────────────────────────────────┐
│                    WORK CONTRACT (§§ 631 ff. BGB)                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  CONTRACTOR                                    CLIENT               │
│  (Entrepreneur)                                (Customer)           │
│  ┌─────────────┐                           ┌─────────────┐          │
│  │             │  Duty: Create             │             │          │
│  │ Contractor  │ ────────────────────────►│   Client    │          │
│  │             │  a work (SUCCESS!)        │             │          │
│  │             │                           │             │          │
│  │             │  Duty: Pay work fee       │             │          │
│  │             │ ◄──────────────────────── │             │          │
│  │             │  after acceptance         │             │          │
│  └─────────────┘                           └─────────────┘          │
│                                                                      │
│  IMPORTANT: Success is owed!                                        │
│  → The work must FUNCTION                                           │
│  → Not just effort, but result                                      │
│                                                                      │
│  Acceptance:                                                        │
│  → Client MUST accept the work (if contractual)                     │
│  → After acceptance: Work fee due, warranty begins                  │
│                                                                      │
│  IT Examples:                                                       │
│  → Custom software development                                      │
│  → Network installation                                             │
│  → Website creation                                                 │
│  → Database migration                                               │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### The Service Contract

```
┌─────────────────────────────────────────────────────────────────────┐
│                    SERVICE CONTRACT (§§ 611 ff. BGB)                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  SERVICE PROVIDER                           SERVICE RECIPIENT       │
│  ┌─────────────┐                           ┌─────────────┐          │
│  │             │  Duty: Perform            │             │          │
│  │  Provider   │ ────────────────────────►│  Recipient  │          │
│  │             │  activity (NO success!)   │             │          │
│  │             │                           │             │          │
│  │             │  Duty: Pay                │             │          │
│  │             │ ◄──────────────────────── │             │          │
│  │             │  remuneration             │             │          │
│  └─────────────┘                           └─────────────┘          │
│                                                                      │
│  IMPORTANT: Only activity, NO success owed!                         │
│  → Consulting can be unsuccessful → still remuneration              │
│  → What matters is the effort, not the result                       │
│                                                                      │
│  Special case: Employment contract                                  │
│  → Is a special service contract                                    │
│  → Employee owes work performance                                   │
│                                                                      │
│  IT Examples:                                                       │
│  → IT consulting by the hour                                        │
│  → Training and courses                                             │
│  → Support hotline (without success guarantee)                      │
│  → Freelancer deployment                                            │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Practical Distinction

```
┌─────────────────────────────────────────────────────────────────────┐
│                    PRACTICAL DISTINCTION                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  "We're buying a server"                                            │
│  → PURCHASE CONTRACT (hardware purchase)                            │
│                                                                      │
│  "We're having software developed"                                  │
│  → WORK CONTRACT (success: functioning software)                    │
│                                                                      │
│  "We're booking IT consulting for 40 hours"                         │
│  → SERVICE CONTRACT (activity, no guaranteed success)               │
│                                                                      │
│  "We're having the network installed"                               │
│  → WORK CONTRACT (success: functioning network)                     │
│                                                                      │
│  "We're buying Microsoft 365 licenses"                              │
│  → PURCHASE CONTRACT (license acquisition)                          │
│                                                                      │
│  "We're renting cloud storage"                                      │
│  → RENTAL CONTRACT (temporary provision)                            │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

| Contract Type | Description | IT Example |
|---------------|-------------|------------|
| **Rental contract** | Temporary provision | Cloud storage, hardware leasing |

## Key Terms

| Term | Explanation |
|------|-------------|
| **Purchase contract** | Transfer of ownership for money |
| **Work contract** | Creation of a work (success owed) |
| **Service contract** | Performance of an activity (no success) |
| **Acceptance** | Confirmation of contract fulfillment in work contract |
| **Work fee** | Remuneration in work contract |
| **Warranty** | Liability for defects |

---

## Exam Tips

1. **Main difference to remember:**
   - **Work** = Success owed
   - **Service** = Only activity owed

2. **Typical exam question:**
   - "Which contract type applies?" → Analyze IT example

3. **Acceptance only for work contracts!**
   - Purchase contract: Delivery
   - Service contract: No acceptance needed

## Practice Exercises

### Exercise 1: Assign Contract Types

Which contract type applies?

a) ConSystem develops inventory management software for a customer
b) ConSystem buys 20 new laptops from a wholesaler
c) ConSystem hires an external consultant for project analysis
d) ConSystem installs a complete network including acceptance
e) ConSystem signs a monthly cloud storage contract

<details>
<summary>Show solution</summary>

a) **Work contract** - Success (functioning software) is owed
b) **Purchase contract** - Purchase of hardware
c) **Service contract** - Activity (consulting) without success guarantee
d) **Work contract** - Success (functioning network) with acceptance
e) **Rental contract** - Temporary provision of storage space

</details>

### Exercise 2: Success or Effort?

A company commissions ConSystem with:
- "Analyze our IT infrastructure and make improvement suggestions"

a) Which contract type applies?
b) What does ConSystem owe?
c) Does ConSystem get paid even if the suggestions are not implemented?

<details>
<summary>Show solution</summary>

a) **Service contract** (or possibly work contract, if a report is agreed as a "work")

b) **ConSystem owes:**
- For service contract: Proper analysis activity, effort
- For work contract: Completed analysis report

c) **Yes!**
- For service contract: The remuneration is for the activity, not for the implementation
- The implementation of the suggestions is up to the customer
- ConSystem has provided its service (analysis and suggestions)

</details>

### Exercise 3: Acceptance

ConSystem GmbH has created a website for a customer. The customer refuses to declare acceptance, although the website works as specified in the contract.

a) What type of contract is this?
b) What does acceptance mean legally?
c) What rights does ConSystem have?

<details>
<summary>Show solution</summary>

a) **Work contract** - Website creation is a work

b) **Meaning of acceptance:**
- Confirmation that the work is contractual
- Work fee becomes due
- Warranty period begins
- Burden of proof shifts (contractor no longer needs to prove)

c) **ConSystem's rights:**
- Right to acceptance if work is contractual (§ 640 BGB)
- In case of unjustified refusal: Set deadline
- After deadline expires: Acceptance is deemed to have occurred (§ 640 para. 2 BGB)
- Work fee can be sued for
- Possibly damages for delay

</details>
