# Service Level (Support Tiers)

## Learning Objectives

After this section, you will be able to:
- Distinguish the three support levels
- Explain tasks and competencies of each tier
- Describe the escalation process
- Explain the significance of escalation in IT support

---

## Core Content

├─────────────────────────────────────────────────────────────────────┤

│                       │    3RD LEVEL        │  Experts              │
│                       │    Specialists      │  Developers           │
│                       │    Vendor           │  Deep Analysis        │
│                       └──────────┬──────────┘                       │
│                                  │ Escalation                       │
│                       ┌──────────┴──────────┐                       │
│                       │    2ND LEVEL        │  Subject Experts      │
│                       │    Experts          │  Complex Problems     │
│                       │                     │  Administration       │
│                       └──────────┬──────────┘                       │
│                                  │ Escalation                       │
│                       ┌──────────┴──────────┐                       │
│                       │    1ST LEVEL        │  First Contact        │

### 1st Level Support (Helpdesk/Service Desk)

```
┌─────────────────────────────────────────────────────────────────────┐
│                    1ST LEVEL SUPPORT                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  TASKS:                                                             │
│  ──────────────────────────────────────────────────────────────     │
│  → First point of contact for all requests                          │
│  → Intake and documentation (create ticket)                         │
│  → Categorization and prioritization                                │
│  → Resolution of standard problems                                  │
│  → Forwarding of complex cases                                      │
│                                                                      │
│  TYPICAL SOLUTIONS:                                                 │
│  ──────────────────────────────────────────────────────────────     │
│  → Reset password                                                   │
│  → Restart PC                                                       │
│  → Standard software problems                                       │
│  → Printer problems (paper, toner)                                  │
│  → Provide instructions (known solutions)                           │
│                                                                      │
│  TOOLS:                                                             │
│  ──────────────────────────────────────────────────────────────     │
│  → Ticket system                                                    │
│  → Knowledge base                                                   │
│  → Remote access (remote support)                                   │
│  → FAQ and instructions                                             │
│                                                                      │
│  GOAL: Resolve as many requests IMMEDIATELY as possible (FCR)       │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

│                                                                      │

│  ──────────────────────────────────────────────────────────────     │

### Comparison of Support Levels

| Criterion | 1st Level | 2nd Level | 3rd Level |
|-----------|-----------|-----------|-----------|
| **Contact** | Direct with user | Ticket | Ticket/Project |
| **Problems** | Standard | Complex | Highly complex |
| **Resolution time** | Minutes | Hours-Days | Days-Weeks |
| **Cost** | Low | Medium | High |
| **Qualification** | Basic knowledge | Subject experts | Specialists |
| **Ticket share** | ~70-80% | ~15-25% | ~5% |

### Escalation Process

```
┌─────────────────────────────────────────────────────────────────────┐
│                    ESCALATION                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  FUNCTIONAL ESCALATION (horizontal)                                 │
│  ──────────────────────────────────────────────────────────────     │
│  → Forwarding to responsible team                                   │
│  → E.g. from PC support to network team                             │
│                                                                      │
│  HIERARCHICAL ESCALATION (vertical)                                 │
│  ──────────────────────────────────────────────────────────────     │
│  → Forwarding to higher level                                       │
│  → E.g. from 1st level to 2nd level                                 │
│  → Or: Involvement of supervisors                                   │
│                                                                      │
│  WHEN TO ESCALATE?                                                  │
│  ──────────────────────────────────────────────────────────────     │
│  → Problem exceeds own competence                                   │
│  → SLA deadline at risk                                             │
│  → Customer unhappy/escalates themselves                            │
│  → Critical system affected                                         │
│                                                                      │
│  IMPORTANT:                                                         │
│  → Documentation in ticket (what was tried?)                        │
│  → Transfer all relevant information                                │
│  → Don't escalate "blindly"                                         │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Single Point of Contact (SPOC)

```
┌─────────────────────────────────────────────────────────────────────┐
│                    SPOC PRINCIPLE                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  The user has ONE point of contact: 1st Level                       │
│                                                                      │
│                    ┌─────────────────┐                              │
│                    │      USER       │                              │
│                    └────────┬────────┘                              │
│                             │                                       │
│                             ▼                                       │
│                    ┌─────────────────┐                              │
│                    │   1ST LEVEL     │ ← Single Point of Contact    │
│                    │   (SPOC)        │                              │
│                    └────────┬────────┘                              │
│                             │                                       │
│            ┌────────────────┼────────────────┐                      │
│            ▼                ▼                ▼                      │
│      ┌──────────┐    ┌──────────┐    ┌──────────┐                   │
│      │ 2nd Level│    │ 2nd Level│    │ 3rd Level│                   │
│      │ Network  │    │ Server   │    │ Dev      │                   │
│      └──────────┘    └──────────┘    └──────────┘                   │
│                                                                      │
│  Advantages:                                                        │
│  → User doesn't need to know who is responsible                     │
│  → No "misdirected" requests                                        │
│  → Central documentation                                            │
│  → Consistent communication                                         │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

## Key Terms

| Term | Explanation |
|------|-------------|
| **1st Level** | First point of contact, standard problems |
| **2nd Level** | Subject experts, complex problems |
| **3rd Level** | Specialists, developers, vendors |
| **Escalation** | Forwarding to another level/team |
| **SPOC** | Single Point of Contact |
| **First Call Resolution** | Resolution on first contact |
| **Knowledge Base** | Database for known solutions |

---

## Exam Tips

1. **Level distinction:**
   - 1st = Standard, fast, cheap
   - 2nd = Complex, experts
   - 3rd = Very complex, developers/vendors

2. **First Call Resolution:**
   - The more resolved at 1st level, the better!
   - Target: >70%

3. **SPOC principle:**
   - User has ONE contact point
   - Internal forwarding is transparent

## Practice Exercises

### Exercise 1: Assign Levels

Which support level is responsible for the following problems?

a) Forgotten password
b) Server hard drive is defective
c) Bug in the self-developed ERP software
d) Printer driver doesn't work after Windows update
e) Microsoft Office won't activate

<details>
<summary>Show solution</summary>

a) **1st Level** - Standard procedure (password reset)
b) **2nd Level** - Hardware replacement on server
c) **3rd Level** - In-house development, developer needed
d) **1st/2nd Level** - Driver reinstallation (1st), for complex problems 2nd
e) **1st Level** - Standard activation, possibly Microsoft support (3rd)

</details>

### Exercise 2: Escalation Decision

An employee reports: "My PC is very slow." The 1st level support has already:
- Restarted PC
- Closed unnecessary programs
- Run virus scanner (no findings)
- Checked hard drive for errors (OK)

The problem persists. What is the next step?

<details>
<summary>Show solution</summary>

**Escalation to 2nd Level recommended**

Reason:
- Standard measures have been performed
- Problem persists
- Deeper analysis required (e.g. resource usage, network, background processes)

**Document for escalation:**
- All measures performed
- Describe symptoms exactly
- Relevant system information (RAM, CPU usage)

**2nd Level could check:**
- Detailed process analysis
- Network performance
- Hardware diagnostics
- Software conflicts

</details>

### Exercise 3: SPOC Principle

A user has a complex problem that has already been escalated to 2nd Level. They call 1st Level again and ask about the status.

How should the 1st level employee respond?

<details>
<summary>Show solution</summary>

**Correct response:**

1. **Check in ticket system:**
   - Check current status
   - Read latest activities
   - Identify assignee

2. **Inform the user:**
   - "Your ticket is currently being processed by the 2nd level team."
   - Explain status (e.g. "A solution is being worked on")
   - State expected duration (if known)

3. **NOT:**
   - Refer the user to 2nd level ("Call them")
   - Claim to know nothing
   - Make promises that cannot be kept

**The SPOC principle means:**
1st Level remains the contact point, even if others are working on the solution. They function as the communication interface to the user.

</details>
