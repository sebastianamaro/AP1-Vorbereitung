# Project Phases

## Learning Objectives

After this chapter, you should be able to:
- Know the four main phases of a project
- Describe the tasks in each phase
- Understand the difference between phase model and process model

---

## Core Content

### The Four Project Phases

Every project fundamentally goes through four main phases:

```
+----------------+   +----------------+   +----------------+   +----------------+
|   1. PROJECT   |   |   2. PROJECT   |   |   3. PROJECT   |   |   4. PROJECT   |
|   DEFINITION   | > |   PLANNING     | > |   EXECUTION    | > |   CLOSURE      |
+----------------+   +----------------+   +----------------+   +----------------+
```

---

### Phase 1: Project Definition

**Goal:** Clarify what should be achieved

**Activities:**
- Formulate project goals (SMART)
- Identify stakeholders
- Conduct feasibility study
- Create project charter
- Create/analyze requirements document

**Results:**
- Project charter
- Requirements document (customer requirements)
- Rough resource estimate

---

### Phase 2: Project Planning

**Goal:** Determine how the goals will be achieved

**Activities:**
- Create detailed plan
- Define work packages
- Create schedule (Gantt chart, network diagram)
- Plan resources
- Conduct risk analysis
- Create functional specification
- Define milestones

**Results:**
- Functional specification
- Project plan
- Schedule
- Resource plan
- Risk analysis

---

### Phase 3: Project Execution

**Goal:** Implementation of planned activities

**Activities:**
- Complete work packages
- Monitor progress
- Conduct quality assurance
- Project control (in case of deviations)
- Coordinate team
- Create documentation

**Results:**
- Project result (product/service)
- Progress reports
- Quality protocols

---

### Phase 4: Project Closure

**Goal:** Properly conclude and document project

**Activities:**
- Present results
- Conduct acceptance
- Document lessons learned
- Complete project documentation
- Dissolve team
- Create final cost calculation

**Results:**
- Acceptance protocol
- Final report
- Lessons learned
- Archiving

---

### Phase Model vs. Process Model

| Phase Model | Process Model |
|-------------|---------------|
| Describes the basic project phases | Describes the specific approach |
| Generally valid for all projects | Specific to project type (e.g., software) |
| 4 phases | Can have more phases |
| Example: The four project phases | Example: Waterfall model, Scrum |

---

## Key Terms

| Term | Explanation |
|------|-------------|
| **Milestone** | Important intermediate step in the project with verifiable result |
| **Work Package** | Smallest unit of project planning |
| **Project Charter** | Document marking the official start of the project |
| **Acceptance** | Formal confirmation that the project result meets requirements |

---

## Exam Tips

- The four phases are often asked in the correct order
- Important: Difference between requirements document (Phase 1) and functional specification (Phase 2)
- Milestones can be set at any phase boundary

---

## Practice Exercises

### Exercise 1
Assign the following activities to the project phases:

a) Creation of Gantt chart
b) Conducting acceptance
c) Formulation of project goals
d) Quality check of work results

**Solution:**
a) Phase 2 - Project Planning
b) Phase 4 - Project Closure
c) Phase 1 - Project Definition
d) Phase 3 - Project Execution

### Exercise 2
In which phase is the functional specification created?

**Solution:** Phase 2 - Project Planning (based on the requirements document from Phase 1)
