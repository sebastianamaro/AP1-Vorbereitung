# Information Security Management System (ISMS)

## Learning Objectives

After this chapter, you should be able to:
- Explain the term ISMS
- Know the components of an ISMS
- Understand the connection to BSI Baseline Protection
- Know about continuous improvement (PDCA)

**Note:** ISO 2700x details are NO LONGER in the 2025 exam catalog, therefore only ISMS in general and BSI Baseline Protection are covered here.

## Core Content

### What is an ISMS?

An **ISMS (Information Security Management System)** is a systematic approach to managing and continuously improving information security in an organization.

```
+------------------------------------------------------------------+
|                            ISMS                                   |
|              Information Security Management System               |
+------------------------------------------------------------------+
|                                                                   |
|  +--------------------------------------------------------------+ |
|  |                      GOAL                                     | |
|  |  Systematic protection of information through                 | |
|  |  * Policies                                                   | |
|  |  * Processes                                                  | |
|  |  * Technical measures                                         | |
|  |  * Continuous improvement                                     | |
|  +--------------------------------------------------------------+ |
|                                                                   |
|                         IMPLEMENTATION                            |
|                            |                                      |
|            +---------------+---------------+                      |
|            v               v               v                      |
|       +-----------+   +-----------+   +-----------+              |
|       |   BSI     |   |   ISO     |   |  Other    |              |
|       | Baseline  |   |  27001    |   | Frame-    |              |
|       | Protection|   |(not in    |   | works     |              |
|       |           |   | AP1)      |   |           |              |
|       +-----------+   +-----------+   +-----------+              |
|                                                                   |
+------------------------------------------------------------------+
```

### Components of an ISMS

| Component | Description |
|-----------|-------------|
| **Security Policy** | Basic orientation and goals |
| **Organization** | Roles and responsibilities |
| **Risk Management** | Identification and treatment of risks |
| **Measures** | Technical and organizational controls |
| **Documentation** | Policies, processes, records |
| **Training** | Awareness and competence |
| **Monitoring** | Control and measurement |
| **Improvement** | Continuous improvement process |

### The PDCA Cycle in ISMS

The ISMS follows the **PDCA cycle** for continuous improvement:

```
+------------------------------------------------------------------+
|                    PDCA CYCLE IN ISMS                             |
+------------------------------------------------------------------+
|                                                                   |
|                      +-----------+                                |
|                      |   PLAN    |                                |
|                      |           |                                |
|                      |  Plan     |                                |
|                      |  ISMS     |                                |
|                      +-----+-----+                                |
|                            |                                      |
|          +-----------------+-----------------+                    |
|          |                 |                 |                    |
|          v                 v                 v                    |
|     +-----------+                       +-----------+             |
|     |    ACT    |                       |    DO     |             |
|     |           |                       |           |             |
|     |  Improve  |<----------------------| Implement |             |
|     |  ISMS     |                       | ISMS      |             |
|     +-----+-----+                       +-----+-----+             |
|           |                                   |                   |
|           |         +-----------+             |                   |
|           |         |   CHECK   |             |                   |
|           |         |           |             |                   |
|           +-------->|  Review   |<------------+                   |
|                     |  ISMS     |                                 |
|                     +-----------+                                 |
|                                                                   |
+------------------------------------------------------------------+
```

| Phase | Description | Activities |
|-------|-------------|------------|
| **Plan** | Plan ISMS | Define goals, analyze risks, plan measures |
| **Do** | Implement ISMS | Implement measures, conduct training |
| **Check** | Review ISMS | Monitor, conduct audits, measure effectiveness |
| **Act** | Improve ISMS | Corrective actions, continuous improvement |

### Roles in ISMS

| Role | Tasks |
|------|-------|
| **Management** | Overall responsibility, provide resources |
| **IT Security Officer (ITSO)** | Coordination of ISMS, contact person |
| **Data Protection Officer (DPO)** | Data protection compliance |
| **IT Department** | Technical implementation |
| **Departments** | Professional requirements, data responsibility |
| **Employees** | Compliance with policies |

---

### Security Policy

The **Security Policy** is the central document of the ISMS:

| Content | Description |
|---------|-------------|
| **Scope** | Which areas/systems are affected? |
| **Goals** | What should be achieved? |
| **Principles** | What principles apply? |
| **Responsibilities** | Who is responsible for what? |
| **Consequences** | What happens in case of violations? |

## Practice Exercises

### Exercise 1
What is an ISMS?

**Solution:** An **ISMS (Information Security Management System)** is a systematic approach to managing and continuously improving information security in an organization. It encompasses policies, processes, technical measures, and organizational structures for protecting information.

### Exercise 2
Explain the PDCA cycle in the context of ISMS.

**Solution:** The PDCA cycle is a model for continuous improvement:
- **Plan:** Plan ISMS - Define goals, analyze risks, plan measures
- **Do:** Implement ISMS - Implement measures, train employees
- **Check:** Review ISMS - Monitor, conduct audits, measure effectiveness
- **Act:** Improve ISMS - Implement corrective actions, optimize processes

The cycle repeats continuously.

### Exercise 3
What should a security policy contain?

**Solution:** A security policy should contain:
1. **Scope** - Which areas are affected?
2. **Goals** - What should be achieved?
3. **Principles** - What security principles apply?
4. **Responsibilities** - Who is responsible for what?
5. **Consequences** - What happens in case of violations?

### Exercise 4
Name three roles in ISMS.

**Solution:**
1. **Management** - Overall responsibility, provide resources
2. **IT Security Officer (ITSO)** - Coordination of ISMS
3. **Data Protection Officer (DPO)** - Data protection compliance

(Others: IT Department, Departments, Employees)
