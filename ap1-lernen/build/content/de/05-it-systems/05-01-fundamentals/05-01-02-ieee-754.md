# IEEE 754 - Gleitkommazahlen

## Lernziele

Nach diesem Kapitel solltest du:
- Den Aufbau von Gleitkommazahlen nach IEEE 754 kennen
- Die Komponenten (Vorzeichen, Exponent, Mantisse) verstehen
- Einfache und doppelte Genauigkeit unterscheiden können
- Typische Rundungsprobleme verstehen

---

## Kerninhalt

### Was sind Gleitkommazahlen?

**Gleitkommazahlen** (Floating Point Numbers) ermöglichen die Darstellung von Dezimalzahlen (Kommazahlen) im Binärsystem. Der Standard **IEEE 754** definiert das Format.

```
┌─────────────────────────────────────────────────────────────────┐
│                    GLEITKOMMAZAHL                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  Wissenschaftliche Notation:  6,022 × 10²³                      │
│                                                                 │
│  Computer-Darstellung:        Vorzeichen × Mantisse × 2^Exponent│
│                                                                 │
│  Beispiel: -3,5 = -1 × 1,75 × 2¹                               │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

---

### Aufbau nach IEEE 754

Eine Gleitkommazahl besteht aus drei Komponenten:

```
┌─────────────────────────────────────────────────────────────────┐
│                 IEEE 754 AUFBAU                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌──────┬────────────────────┬──────────────────────────────┐  │
│  │  V   │      EXPONENT      │          MANTISSE            │  │
│  │ 1Bit │       8 Bit        │           23 Bit             │  │
│  └──────┴────────────────────┴──────────────────────────────┘  │
│                                                                 │
│  V = Vorzeichen:  0 = positiv, 1 = negativ                     │
│  Exponent:        Verschoben um Bias (127 bei Single)          │
│  Mantisse:        Nachkommastellen (implizite 1 davor)         │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

---

### Single vs. Double Precision

| Format | Größe | Vorzeichen | Exponent | Mantisse | Bias |
|--------|-------|------------|----------|----------|------|
| **Single (float)** | 32 Bit | 1 Bit | 8 Bit | 23 Bit | 127 |
| **Double (double)** | 64 Bit | 1 Bit | 11 Bit | 52 Bit | 1023 |

```
SINGLE PRECISION (32 Bit):
┌─┬────────┬───────────────────────┐
│V│EXPONENT│       MANTISSE        │
│1│   8    │          23           │
└─┴────────┴───────────────────────┘

DOUBLE PRECISION (64 Bit):
┌─┬───────────┬────────────────────────────────────────────────┐
│V│ EXPONENT  │                    MANTISSE                     │
│1│    11     │                       52                        │
└─┴───────────┴────────────────────────────────────────────────┘
```

---

### Die Komponenten im Detail

#### 1. Vorzeichen (V)

| Wert | Bedeutung |
|------|-----------|
| 0 | Positive Zahl |
| 1 | Negative Zahl |

#### 2. Exponent

Der Exponent wird mit einem **Bias** gespeichert, um negative Exponenten ohne Vorzeichenbit darzustellen.

| Format | Bias | Speicherung |
|--------|------|-------------|
| Single | 127 | Gespeicherter Exponent = Echter Exponent + 127 |
| Double | 1023 | Gespeicherter Exponent = Echter Exponent + 1023 |

**Beispiel:** Exponent 3 wird als 3 + 127 = **130** gespeichert.

#### 3. Mantisse

Die Mantisse speichert die Nachkommastellen. Da normalisierte Zahlen immer mit 1 beginnen, wird diese 1 nicht gespeichert (**implizite 1**).

**Beispiel:** 1,1011₂ wird als Mantisse **1011...** gespeichert (ohne die führende 1).

---

### Beispiel: Dezimal zu IEEE 754

**Aufgabe:** Wandle **-5,75** in IEEE 754 Single Precision um.

**Schritt 1:** Vorzeichen
- Negativ → V = **1**

**Schritt 2:** Dezimal in Binär
- 5,75₁₀ = 101,11₂
  - 5 = 101
  - 0,75 = 0,5 + 0,25 = 0,11

**Schritt 3:** Normalisieren
- 101,11 = 1,0111 × 2²
- Mantisse: 0111 (ohne die führende 1)
- Exponent: 2

**Schritt 4:** Exponent mit Bias
- Gespeicherter Exponent = 2 + 127 = 129 = 10000001₂

**Ergebnis:**
```
V | Exponent  | Mantisse
1 | 10000001 | 01110000000000000000000
```

---

### Spezielle Werte

IEEE 754 definiert auch spezielle Werte:

| Exponent | Mantisse | Bedeutung |
|----------|----------|-----------|
| Alle 0 | Alle 0 | **±0** (je nach Vorzeichen) |
| Alle 0 | ≠ 0 | **Denormalisierte Zahl** (sehr klein) |
| Alle 1 | Alle 0 | **±∞** (Infinity) |
| Alle 1 | ≠ 0 | **NaN** (Not a Number) |

---

### Genauigkeit und Rundungsfehler

**Problem:** Nicht alle Dezimalzahlen lassen sich exakt binär darstellen.

```
┌────────────────────────────────────────────────────────────────┐
│                 RUNDUNGSFEHLER                                 │
├────────────────────────────────────────────────────────────────┤
│                                                                │
│  0,1₁₀ = 0,0001100110011001100110011...₂  (periodisch!)        │
│                                                                │
│  Daher kann 0,1 + 0,2 ≠ 0,3 sein:                              │
│  In vielen Programmiersprachen ergibt das:                     │
│  0,1 + 0,2 = 0,30000000000000004                               │
│                                                                │
│  ⚠️ Nie Gleitkommazahlen auf Gleichheit prüfen!               │
│                                                                │
└────────────────────────────────────────────────────────────────┘
```

---

### Wertebereich

| Format | Kleinste positive Zahl | Größte Zahl |
|--------|----------------------|-------------|
| **Single** | ca. 1,2 × 10⁻³⁸ | ca. 3,4 × 10³⁸ |
| **Double** | ca. 2,2 × 10⁻³⁰⁸ | ca. 1,8 × 10³⁰⁸ |

## Wichtige Begriffe

| Begriff | Erklärung |
|---------|-----------|
| **Gleitkommazahl** | Binäre Darstellung von Dezimalzahlen |
| **IEEE 754** | Standard für Gleitkommadarstellung |
| **Single Precision** | 32 Bit Gleitkommazahl (float) |
| **Double Precision** | 64 Bit Gleitkommazahl (double) |
| **Mantisse** | Nachkommastellen |
| **Exponent** | Potenz der Basis 2 |
| **Bias** | Verschiebung des Exponenten |
| **NaN** | Not a Number (ungültiger Wert) |

## Prüfungstipps

### Häufige Prüfungsfragen

- Aus welchen Komponenten besteht eine IEEE 754 Gleitkommazahl?
- Was ist der Unterschied zwischen Single und Double Precision?
- Warum können Rundungsfehler bei Gleitkommazahlen auftreten?

### Wichtig zu merken

- **3 Komponenten:** Vorzeichen, Exponent, Mantisse
- **Single:** 32 Bit (1+8+23), Bias 127
- **Double:** 64 Bit (1+11+52), Bias 1023
- **Implizite 1:** Die führende 1 wird nicht gespeichert
- **Rundungsfehler** sind unvermeidlich

---

## Übungsaufgaben

### Aufgabe 1
Aus welchen drei Komponenten besteht eine Gleitkommazahl nach IEEE 754?

**Lösung:**
1. **Vorzeichen** (1 Bit) - 0 für positiv, 1 für negativ
2. **Exponent** (8 oder 11 Bit) - mit Bias verschoben
3. **Mantisse** (23 oder 52 Bit) - Nachkommastellen

### Aufgabe 2
Was ist der Unterschied zwischen Single und Double Precision?

**Lösung:**
| Aspekt | Single | Double |
|--------|--------|--------|
| Größe | 32 Bit | 64 Bit |
| Exponent | 8 Bit | 11 Bit |
| Mantisse | 23 Bit | 52 Bit |
| Genauigkeit | ca. 7 Dezimalstellen | ca. 15-16 Dezimalstellen |
| Wertebereich | ±10³⁸ | ±10³⁰⁸ |

### Aufgabe 3
Warum ergibt 0,1 + 0,2 in vielen Programmiersprachen nicht exakt 0,3?

**Lösung:** Die Dezimalzahl 0,1 kann nicht exakt als binäre Gleitkommazahl dargestellt werden - sie ist im Binärsystem periodisch (wie 1/3 im Dezimalsystem). Daher entstehen bei der Speicherung kleine Rundungsfehler, die sich bei Berechnungen akkumulieren können.

### Aufgabe 4
Was bedeutet "Bias" im Kontext von IEEE 754?

**Lösung:** Der Bias ist ein Offset, der zum echten Exponenten addiert wird, um ihn ohne Vorzeichenbit speichern zu können. Bei Single Precision ist der Bias 127, bei Double 1023. Ein echter Exponent von -5 wird also als -5 + 127 = 122 gespeichert.
