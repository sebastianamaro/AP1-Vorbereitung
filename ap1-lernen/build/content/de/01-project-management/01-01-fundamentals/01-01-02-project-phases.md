# Projektphasen

## Lernziele

Nach diesem Kapitel solltest du:
- Die vier Hauptphasen eines Projekts kennen
- Die Aufgaben in jeder Phase beschreiben können
- Den Unterschied zwischen Phasenmodell und Vorgehensmodell verstehen

---

## Kerninhalt

### Die vier Projektphasen

Jedes Projekt durchläuft grundsätzlich vier Hauptphasen:

```
┌──────────────┐   ┌──────────────┐   ┌──────────────┐   ┌──────────────┐
│   1. PROJEKT-│   │   2. PROJEKT-│   │   3. PROJEKT-│   │   4. PROJEKT-│
│   DEFINITION │ → │   PLANUNG    │ → │   REALISIERUNG│ → │   ABSCHLUSS  │
└──────────────┘   └──────────────┘   └──────────────┘   └──────────────┘
```

---

### Phase 1: Projektdefinition

**Ziel:** Klären, was erreicht werden soll

**Aktivitäten:**
- Projektziele formulieren (SMART)
- Stakeholder identifizieren
- Machbarkeitsstudie durchführen
- Projektauftrag erstellen
- Lastenheft erstellen/analysieren

**Ergebnisse:**
- Projektauftrag
- Lastenheft
- Grobe Ressourcenschätzung

---

### Phase 2: Projektplanung

**Ziel:** Festlegen, wie die Ziele erreicht werden

**Aktivitäten:**
- Detaillierte Planung erstellen
- Arbeitspakete definieren
- Zeitplan erstellen (Gantt-Diagramm, Netzplan)
- Ressourcen planen
- Risikoanalyse durchführen
- Pflichtenheft erstellen
- Meilensteine definieren

**Ergebnisse:**
- Pflichtenheft
- Projektplan
- Zeitplan
- Ressourcenplan
- Risikoanalyse

---

### Phase 3: Projektrealisierung

**Ziel:** Umsetzung der geplanten Aktivitäten

**Aktivitäten:**
- Arbeitspakete abarbeiten
- Fortschritt überwachen
- Qualitätssicherung durchführen
- Projektsteuerung (bei Abweichungen)
- Team koordinieren
- Dokumentation erstellen

**Ergebnisse:**
- Projektergebnis (Produkt/Dienstleistung)
- Fortschrittsberichte
- Qualitätsprotocols

---

### Phase 4: Projektabschluss

**Ziel:** Projekt ordnungsgemäß beenden und dokumentieren

**Aktivitäten:**
- Ergebnisse präsentieren
- Abnahme durchführen
- Lessons Learned dokumentieren
- Projektdocumentation abschließen
- Team auflösen
- Nachkalkulation erstellen

**Ergebnisse:**
- Abnahmeprotokoll
- Abschlussbericht
- Lessons Learned
- Archivierung

---

### Phasenmodell vs. Vorgehensmodell

| Phasenmodell | Vorgehensmodell |
|--------------|-----------------|
| Beschreibt die grundlegenden Projektphasen | Beschreibt die konkrete Vorgehensweise |
| Allgemeingültig für alle Projekte | Spezifisch für Projektart (z.B. Software) |
| 4 Phasen | Kann mehr Phasen haben |
| Beispiel: Die vier Projektphasen | Beispiel: Wasserfallmodell, Scrum |

---

## Wichtige Begriffe

| Begriff | Erklärung |
|---------|-----------|
| **Meilenstein** | Wichtiger Zwischenschritt im Projekt mit prüfbarem Ergebnis |
| **Arbeitspaket** | Kleinste Einheit der Projektplanung |
| **Projektauftrag** | Dokument, das den offiziellen Start des Projekts markiert |
| **Abnahme** | Formale Bestätigung, dass das Projektergebnis den Anforderungen entspricht |

---

## Prüfungstipps

- Die vier Phasen werden oft in der richtigen Reihenfolge abgefragt
- Wichtig: Unterschied zwischen Lastenheft (Phase 1) und Pflichtenheft (Phase 2)
- Meilensteine können an jeder Phasengrenze gesetzt werden

---

## Übungsaufgaben

### Aufgabe 1
Ordnen Sie die folgenden Aktivitäten den Projektphasen zu:

a) Erstellung des Gantt-Diagramms
b) Durchführung der Abnahme
c) Formulierung der Projektziele
d) Qualitätsprüfung der Arbeitsergebnisse

**Lösung:**
a) Phase 2 - Projektplanung
b) Phase 4 - Projektabschluss
c) Phase 1 - Projektdefinition
d) Phase 3 - Projektrealisierung

### Aufgabe 2
In welcher Phase wird das Pflichtenheft erstellt?

**Lösung:** Phase 2 - Projektplanung (auf Basis des Lastenhefts aus Phase 1)
