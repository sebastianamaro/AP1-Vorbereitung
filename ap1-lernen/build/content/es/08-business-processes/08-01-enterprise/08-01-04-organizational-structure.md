# Estructura Organizativa

## Objetivos de Aprendizaje

Despues de esta seccion, podras:
- Distinguir las principales formas organizativas
- Explicar ventajas y desventajas de cada forma organizativa
- Leer y crear organigramas
- Nombrar la forma organizativa apropiada para diferentes situaciones

---

## Contenido Principal

### Que es la Estructura Organizativa?

**La estructura organizativa** describe la estructura jerarquica de una empresa:
- Quien reporta a quien?
- Como estan organizados los departamentos?
- Quien tiene cual responsabilidad?

Se representa graficamente en un **organigrama**.

### Las Tres Formas Organizativas Clasicas

```
┌─────────────────────────────────────────────────────────────────────┐
│                VISION GENERAL DE FORMAS ORGANIZATIVAS               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  1. SISTEMA DE LINEA UNICA                                          │
│     → Cada empleado tiene UN supervisor                             │
│     → Jerarquia clara                                               │
│                                                                      │
│  2. SISTEMA DE LINEA MULTIPLE                                       │
│     → Los empleados pueden tener MULTIPLES supervisores             │
│     → Especializacion funcional                                     │
│                                                                      │
│  3. SISTEMA DE LINEA-STAFF                                          │
│     → Sistema de linea unica CON posiciones de staff                │
│     → El staff asesora, no decide                                   │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### 1. Sistema de Linea Unica

```
┌─────────────────────────────────────────────────────────────────────┐
│                    SISTEMA DE LINEA UNICA                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│                    ┌───────────────────┐                            │
│                    │    Direccion      │                            │
│                    └─────────┬─────────┘                            │
│                              │                                       │
│            ┌─────────────────┼─────────────────┐                    │
│            │                 │                 │                    │
│            ▼                 ▼                 ▼                    │
│    ┌───────────────┐ ┌───────────────┐ ┌───────────────┐           │
│    │    Ventas     │ │    Tecnico    │ │   Admin       │           │
│    └───────┬───────┘ └───────┬───────┘ └───────┬───────┘           │
│            │                 │                 │                    │
│        ┌───┴───┐         ┌───┴───┐         ┌───┴───┐               │
│        │       │         │       │         │       │               │
│        ▼       ▼         ▼       ▼         ▼       ▼               │
│     ┌─────┐ ┌─────┐   ┌─────┐ ┌─────┐   ┌─────┐ ┌─────┐           │
│     │ E1  │ │ E2  │   │ E3  │ │ E4  │   │ E5  │ │ E6  │           │
│     └─────┘ └─────┘   └─────┘ └─────┘   └─────┘ └─────┘           │
│                                                                      │
│  Principio: Un empleado → Un supervisor                             │
│             (Unidad de mando)                                       │
│                                                                      │
│  Cadena de mando: Las instrucciones solo a traves de la linea      │
│  directa                                                            │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

| Ventajas | Desventajas |
|----------|-------------|
| Responsabilidades claras | Caminos de comunicacion largos |
| Estructura simple | Sobrecarga del gerente |
| Responsabilidad inequivoca | Baja flexibilidad |
| Facil de entender | Perdida de informacion a lo largo de la cadena |

**Adecuado para:** Pequenas y medianas empresas, agencias gubernamentales

### 2. Sistema de Linea Multiple (Sistema Funcional)

```
┌─────────────────────────────────────────────────────────────────────┐
│                    SISTEMA DE LINEA MULTIPLE                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│                    ┌───────────────────┐                            │
│                    │    Direccion      │                            │
│                    └─────────┬─────────┘                            │
│                              │                                       │
│            ┌─────────────────┼─────────────────┐                    │
│            │                 │                 │                    │
│            ▼                 ▼                 ▼                    │
│    ┌───────────────┐ ┌───────────────┐ ┌───────────────┐           │
│    │ Area Funcional│ │ Area Funcional│ │ Area Funcional│           │
│    │ 1 (Tecnico)   │ │ 2 (QA)        │ │ 3 (Procesos)  │           │
│    └───────┬───────┘ └───────┬───────┘ └───────┬───────┘           │
│            │                 │                 │                    │
│            └─────────────────┼─────────────────┘                    │
│                              │                                       │
│                              ▼                                       │
│                         ┌─────────┐                                 │
│                         │ Empleado│                                 │
│                         │ recibe  │                                 │
│                         │directivas│                                │
│                         │de LOS 3 │                                 │
│                         └─────────┘                                 │
│                                                                      │
│  Principio: Un empleado → MULTIPLES supervisores (funcionalmente)   │
│             (Mando multiple)                                        │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

| Ventajas | Desventajas |
|----------|-------------|
| Caminos de comunicacion cortos | Responsabilidades poco claras |
| Especializacion funcional | Conflictos de autoridad |
| Alivio de la direccion | Instrucciones contradictorias posibles |
| Decisiones rapidas | Coordinacion dificil |

**Adecuado para:** Proyectos, tareas especializadas

### 3. Sistema de Linea-Staff

```
┌─────────────────────────────────────────────────────────────────────┐
│                    SISTEMA DE LINEA-STAFF                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│                    ┌───────────────────┐                            │
│                    │    Direccion      │◄─── ┌─────────────┐        │
│                    └─────────┬─────────┘     │ UNIDAD STAFF│        │
│                              │               │ (ej., Legal,│        │
│                              │               │ Seg. TI)    │        │
│            ┌─────────────────┼─────────────────┐  └─────────────┘   │
│            │                 │                 │                    │
│            ▼                 ▼                 ▼                    │
│    ┌───────────────┐ ┌───────────────┐ ┌───────────────┐           │
│    │    Ventas     │◄┤   Tecnico     │ │   Admin       │           │
│    └───────┬───────┘ └───────┬───────┘◄───┌──────────┐ │           │
│            │                 │            │UNIDAD    │ │           │
│        ┌───┴───┐         ┌───┴───┐       │STAFF     │  │           │
│        │       │         │       │       │(ej., GC) │  │           │
│        ▼       ▼         ▼       ▼       └──────────┘  │           │
│     ┌─────┐ ┌─────┐   ┌─────┐ ┌─────┐                  │           │
│     │ E1  │ │ E2  │   │ E3  │ │ E4  │                  │           │
│     └─────┘ └─────┘   └─────┘ └─────┘                  │           │
│                                                                      │
│  Unidad de staff:                                                   │
│  → Asesora a la direccion                                           │
│  → NO tiene autoridad para dar directivas                           │
│  → Apoya con tareas especializadas                                  │
│                                                                      │
│  Posiciones de staff tipicas en TI:                                 │
│  → Oficial de Seguridad de TI                                       │
│  → Delegado de Proteccion de Datos                                  │
│  → Gestion de Calidad                                               │
│  → Departamento Legal                                               │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

| Ventajas | Desventajas |
|----------|-------------|
| Jerarquia clara (como linea unica) | Conflictos linea vs. staff posibles |
| Alivio a traves de especialistas | Staff sin responsabilidad |
| Experiencia disponible | Puede llevar a burocracia |
| Asesoramiento sin problemas de autoridad | El staff puede ser ignorado |

**Adecuado para:** Empresas mas grandes donde se necesita experiencia

## Terminos Clave

| Termino | Explicacion |
|---------|-------------|
| **Organigrama** | Representacion grafica de la estructura organizativa |
| **Sistema de linea unica** | Cada empleado tiene un supervisor |
| **Sistema de linea multiple** | Los empleados pueden tener multiples supervisores |
| **Sistema de linea-staff** | Linea unica + posiciones de staff asesoras |
| **Posicion de staff** | Funcion asesora sin autoridad para dar directivas |
| **Cadena de mando** | Camino oficial a traves de la jerarquia |
| **Autoridad** | Derecho a dar instrucciones |
| **Organizacion matricial** | Combinacion de organizacion funcional y divisional |

---

## Consejos para el Examen

1. **Caracteristica distintiva:**
   - Cuantos supervisores tiene un empleado?
   - Linea unica = 1, Linea multiple = multiples, Linea-staff = 1 + asesores

2. **Recordar posicion de staff:**
   - NO tiene autoridad para dar directivas!
   - Solo asesora, no decide

3. **Preguntas tipicas de examen:**
   - Interpretar organigramas
   - Nombrar ventajas/desventajas
   - Elegir forma organizativa apropiada para un escenario

4. **Relevancia para TI:**
   - DPD y OSI son a menudo posiciones de staff
   - Los proyectos de TI a menudo en organizacion matricial

## Ejercicios Practicos

### Ejercicio 1: Identificar Forma Organizativa

Que forma organizativa esta presente cuando:

a) Un empleado de TI recibe instrucciones tanto del jefe del departamento de TI como del gerente de proyecto
b) El director de TI reporta directamente a la direccion, el Oficial de Seguridad de TI asesora a la direccion
c) Cada empleado tiene exactamente un supervisor, no hay posiciones de staff

<details>
<summary>Mostrar Solucion</summary>

a) **Sistema de linea multiple** (u organizacion matricial) - Multiples supervisores
b) **Sistema de linea-staff** - Director de TI en la linea, OSI como staff
c) **Sistema de linea unica** - Un supervisor, sin staff

</details>

### Ejercicio 2: Crear Organigrama

ConSystem GmbH tiene la siguiente estructura:
- Direccion
- Tres departamentos: Ventas, Tecnico, Admin
- Un Oficial de Seguridad de TI asesora a la direccion
- Un Delegado de Proteccion de Datos tambien actua como asesor

Que forma organizativa esta presente y por que?

<details>
<summary>Mostrar Solucion</summary>

**Sistema de linea-staff**

Razonamiento:
- Jerarquia clara: Direccion → Departamentos (linea unica)
- Dos posiciones de staff: OSI y DPD
- El staff asesora a la direccion pero no tiene autoridad para dar directivas

```

          ┌────────►│    Direccion      │◄────────┐
          │         └─────────┬─────────┘         │
     ┌────┴────┐              │              ┌────┴────┐
     │   OSI   │              │              │   DPD   │
     │ (Staff) │              │              │ (Staff) │
     └─────────┘              │              └─────────┘
            ┌─────────────────┼─────────────────┐
            │                 │                 │
            ▼                 ▼                 ▼
    ┌───────────────┐ ┌───────────────┐ ┌───────────────┐

    └───────────────┘ └───────────────┘ └───────────────┘
```

</details>

### Ejercicio 3: Ventajas y Desventajas

Un integrador de sistemas de TI quiere crecer y esta considerando si cambiar de un sistema de linea unica a un sistema de linea multiple.

a) Nombra dos ventajas que apoyen el cambio.
b) Nombra dos riesgos que podrian ocurrir durante la transicion.
c) Que alternativa podria ser un compromiso?

<details>
<summary>Mostrar Solucion</summary>

a) **Ventajas del sistema de linea multiple:**
- Caminos de comunicacion mas cortos (directamente al supervisor funcional)
- Mejor especializacion funcional posible
- Los gerentes se ven aliviados
- Decisiones mas rapidas en asuntos tecnicos

b) **Riesgos de la transicion:**
- Instrucciones contradictorias de diferentes supervisores
- Responsabilidades poco claras pueden llevar a conflictos
- Los empleados no saben a quien priorizar
- El esfuerzo de coordinacion aumenta

c) **Compromiso: Sistema de linea-staff**
- Mantiene la jerarquia clara del sistema de linea unica
- Anade posiciones de staff para temas especializados (ej., GC, Seguridad TI)
- Experiencia disponible sin problemas de autoridad
- Los empleados todavia tienen un supervisor claro

</details>
