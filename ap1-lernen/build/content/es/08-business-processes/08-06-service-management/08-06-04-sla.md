# Service Level Agreement (SLA)

## Objetivos de Aprendizaje

Después de esta sección, serás capaz de:
- Definir y explicar el término SLA
- Nombrar métricas importantes de SLA
- Calcular disponibilidad
- Distinguir entre tiempo de respuesta y tiempo de resolución

---

## Contenido Principal

### ¿Qué es un SLA?

Un **Service Level Agreement (SLA)** es un acuerdo contractual entre el proveedor de servicios y el cliente sobre la calidad y alcance de los servicios de TI.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    SERVICE LEVEL AGREEMENT                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  ┌─────────────────┐                      ┌─────────────────┐       │
│  │PROVEEDOR SERV.  │  ←── Acuerdo ──→     │     CLIENTE     │       │
│  │  (Proveedor TI) │                      │     (cliente)   │       │
│  └─────────────────┘                      └─────────────────┘       │
│                                                                      │
│  EL SLA DEFINE:                                                     │
│  ──────────────────────────────────────────────────────────────     │
│  → ¿QUÉ se proporciona? (Alcance del servicio)                     │
│  → ¿QUÉ TAN BIEN se proporciona? (Criterios de calidad)            │
│  → ¿QUÉ TAN RÁPIDO se responde? (Tiempos de respuesta)             │
│  → ¿QUÉ PASA por incumplimiento? (Consecuencias)                   │
│                                                                      │
│  TIPOS DE SLAs:                                                     │
│  ──────────────────────────────────────────────────────────────     │
│  → SLA Externo: Con clientes/proveedores                           │
│  → SLA Interno: Entre departamentos (OLA)                          │
│  → Underpinning Contract: Con subcontratistas                      │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Componentes de un SLA

```
┌─────────────────────────────────────────────────────────────────────┐
│                    COMPONENTES DEL SLA                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  1. PARTES CONTRATANTES                                             │
│     → ¿Quién es el proveedor de servicios?                         │
│     → ¿Quién es el cliente?                                         │
│     → Personas de contacto en ambos lados                           │
│                                                                      │
│  2. DESCRIPCIÓN DEL SERVICIO                                        │
│     → ¿Qué servicios están cubiertos?                              │
│     → ¿Cuáles NO están? (¡distinción importante!)                  │
│     → Horario de servicio (ej. 24/7, Lun-Vie 8am-6pm)              │
│                                                                      │
│  3. NIVELES DE SERVICIO (Métricas de calidad)                       │
│     → Disponibilidad (ej. 99.9%)                                   │
│     → Tiempo de respuesta (ej. 1 hora)                             │
│     → Tiempo de resolución (ej. 4 horas)                           │
│     → Valores de rendimiento                                        │
│                                                                      │
│  4. MEDICIÓN E INFORMES                                             │
│     → ¿Cómo se mide?                                               │
│     → ¿Qué herramientas se usan?                                   │
│     → ¿Con qué frecuencia se informa?                              │
│                                                                      │
│  5. CONSECUENCIAS                                                   │
│     → Penalizaciones por incumplimiento                             │
│     → Bonificación por superar objetivos                            │
│     → Procesos de escalación                                        │
│                                                                      │
│  6. DURACIÓN Y TERMINACIÓN                                          │
│     → Inicio y fin del contrato                                     │
│     → Períodos de aviso                                             │
│     → Procedimientos de cambio                                      │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Tiempo de Respuesta vs. Tiempo de Resolución

```
┌─────────────────────────────────────────────────────────────────────┐
│           TIEMPO DE RESPUESTA vs. TIEMPO DE RESOLUCIÓN               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  Línea temporal:                                                    │
│  ═══════════════════════════════════════════════════════════════   │
│                                                                      │
│  Incidente     Primera        Problema                              │
│  reportado     respuesta      resuelto                              │
│     │           │              │                                    │
│     ▼           ▼              ▼                                    │
│     ●───────────●──────────────●                                    │
│     │           │              │                                    │
│     │◄─────────►│              │                                    │
│     │ TIEMPO DE │              │                                    │
│     │ RESPUESTA │              │                                    │
│     │                          │                                    │
│     │◄────────────────────────►│                                    │
│     │   TIEMPO DE RESOLUCIÓN   │                                    │
│                                                                      │
│  TIEMPO DE RESPUESTA:                                               │
│  → Tiempo hasta la primera respuesta cualificada                    │
│  → Confirmación de que la solicitud fue recibida                    │
│  → Un empleado ha tomado el problema                                │
│  → ¡NO es el tiempo hasta la resolución!                           │
│                                                                      │
│  TIEMPO DE RESOLUCIÓN:                                              │
│  → Tiempo hasta que el problema realmente está solucionado          │
│  → O: Solución alternativa proporcionada                            │
│  → Depende de la complejidad y prioridad                            │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### SLA por Prioridad (Ejemplo)

| Prioridad | Descripción | Tiempo de Respuesta | Tiempo de Resolución |
|----------|-------------|---------------|-----------------|
| **Crítica** | Falla total, parada de empresa | 15 minutos | 4 horas |
| **Alta** | Función importante interrumpida | 1 hora | 8 horas |
| **Media** | Uso limitado | 4 horas | 24 horas |
| **Baja** | Impacto menor | 8 horas | 5 días laborables |

### Cálculo de Disponibilidad

```
┌─────────────────────────────────────────────────────────────────────┐
│           DISPONIBILIDAD                                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  FÓRMULA:                                                           │
│  ──────────────────────────────────────────────────────────────     │
│                                                                      │
│                    Tiempo de operación - Tiempo de inactividad       │
│  Disponibilidad = ─────────────────────────────────────────── × 100%│
│                        Tiempo de operación                          │
│                                                                      │
│  O:                                                                 │
│                                                                      │
│                    Tiempo total - Tiempo de inactividad              │
│  Disponibilidad = ─────────────────────────────────────────── × 100%│
│                        Tiempo total                                 │
│                                                                      │
│  EJEMPLO DE CÁLCULO:                                                │
│  ──────────────────────────────────────────────────────────────     │
│                                                                      │
│  Tiempo de servicio: 24/7 = 24 × 365 = 8,760 horas/año             │
│  Tiempo de inactividad por año: 8.76 horas                          │
│                                                                      │
│                    8,760 - 8.76                                     │
│  Disponibilidad = ──────────────── × 100% = 99.9%                   │
│                       8,760                                         │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Los "Nueves" de Disponibilidad

```
┌─────────────────────────────────────────────────────────────────────┐
│           CLASES DE DISPONIBILIDAD                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  Disponibilidad │ Inactividad/Año│ Inactividad/Mes │ Caso de Uso   │
│  ══════════════╪════════════════╪═════════════════╪════════════════│
│  99%           │ 3.65 días      │ 7.2 horas       │ Básico         │
│  99.5%         │ 1.83 días      │ 3.6 horas       │ Estándar       │
│  99.9%         │ 8.76 horas     │ 43.8 minutos    │ Elevado        │
│  99.99%        │ 52.6 minutos   │ 4.4 minutos     │ Alto           │
│  99.999%       │ 5.26 minutos   │ 26 segundos     │ Muy Alto       │
│                                                                      │
│  MNEMOTÉCNICO: "Los Nueves"                                         │
│  ──────────────────────────────────────────────────────────────     │
│  99.9%   = "Tres Nueves" = ~8.7 horas inactividad/año              │
│  99.99%  = "Cuatro Nueves" = ~52 minutos inactividad/año           │
│  99.999% = "Cinco Nueves" = ~5 minutos inactividad/año             │
│                                                                      │
│  NOTA: Cada "9" adicional significa ~10× menos tiempo de inactividad│
│        ¡pero también ~10× mayor costo!                              │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Considerar Horario de Servicio

```
┌─────────────────────────────────────────────────────────────────────┐
│           HORARIO DE SERVICIO                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  ¡Los tiempos de SLA solo aplican dentro del horario de servicio!  │
│                                                                      │
│  EJEMPLO 1: Horario de servicio 24/7                                │
│  ──────────────────────────────────────────────────────────────     │
│  → 168 horas/semana (24h × 7 días)                                 │
│  → 1h de tiempo de respuesta aplica SIEMPRE (incluye noches, fines)│
│                                                                      │
│  EJEMPLO 2: Horario de servicio Lun-Vie 8am-6pm                     │
│  ──────────────────────────────────────────────────────────────     │
│  → 50 horas/semana (10h × 5 días)                                  │
│  → 1h de tiempo de respuesta aplica SOLO durante estos horarios    │
│                                                                      │
│  CASO PRÁCTICO:                                                     │
│  ──────────────────────────────────────────────────────────────     │
│  Incidente reportado: Viernes 5:30 PM                               │
│  Horario de servicio: Lun-Vie 8am-6pm                               │
│  Tiempo de respuesta: 1 hora                                        │
│                                                                      │
│  → Tiempo restante viernes: 30 minutos                              │
│  → La respuesta debe ocurrir antes del: Lunes 8:30 AM               │
│    (30 min el viernes + 30 min el lunes = 1 hora)                  │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Penalizaciones (Cláusulas Penales)

```
┌─────────────────────────────────────────────────────────────────────┐
│           PENALIZACIONES POR VIOLACIÓN DE SLA                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  Cuando no se cumple el SLA, pueden aplicarse penalizaciones:       │
│                                                                      │
│  ARREGLOS TÍPICOS:                                                  │
│  ──────────────────────────────────────────────────────────────     │
│  → Crédito en la próxima factura                                    │
│  → Reducción porcentual de la cuota mensual                         │
│  → Servicios adicionales gratuitos                                  │
│  → Derecho de terminación especial sobre cierto umbral              │
│                                                                      │
│  EJEMPLO:                                                           │
│  ──────────────────────────────────────────────────────────────     │
│  Disponibilidad acordada: 99.9%                                     │
│  Disponibilidad real: 99.5%                                         │
│  Cuota mensual: €1,000                                              │
│                                                                      │
│  → Déficit de disponibilidad: 0.4%                                  │
│  → Penalización: 10% crédito por cada 0.1% de déficit               │
│  → 4 × 10% = 40% crédito                                            │
│  → Crédito: €400                                                    │
│                                                                      │
│  IMPORTANTE:                                                        │
│  → ¡Las penalizaciones NO son compensación por daños consecuentes! │
│  → Las penalizaciones usualmente están limitadas (ej. máx. 1 cuota) │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

## Términos Clave

| Término | Explicación |
|------|-------------|
| **SLA** | Service Level Agreement - Acuerdo contractual sobre calidad de servicio |
| **OLA** | Operational Level Agreement - Acuerdo interno entre departamentos |
| **Tiempo de respuesta** | Tiempo hasta la primera respuesta cualificada |
| **Tiempo de resolución** | Tiempo hasta que el problema está solucionado |
| **Disponibilidad** | Porcentaje de tiempo de operación real |
| **Penalización** | Cláusula penal por violación de SLA |
| **Horario de servicio** | Período durante el cual el servicio está disponible |

---

## Consejos para el Examen

1. **Calcular disponibilidad:**
   - Fórmula: (Tiempo total - Inactividad) / Tiempo total × 100%
   - "Tres Nueves" (99.9%) = aprox. 8.7 horas inactividad/año

2. **Respuesta vs. Resolución:**
   - Tiempo de respuesta = Primera respuesta
   - Tiempo de resolución = Problema solucionado
   - El tiempo de respuesta es SIEMPRE menor que el tiempo de resolución

3. **Considerar horario de servicio:**
   - ¡Los tiempos de SLA solo aplican dentro del horario de servicio!
   - Distinguir 24/7 vs. horario laboral

## Ejercicios Prácticos

### Ejercicio 1: Calcular Disponibilidad

Un servidor tiene los siguientes datos:
- Horario de servicio: 24/7
- Mantenimiento planificado por año: 12 horas
- Fallas no planificadas por año: 4.5 horas

Calcula la disponibilidad. El mantenimiento planificado NO cuenta como inactividad.

<details>
<summary>Mostrar solución</summary>

**Cálculo:**

Tiempo total/año: 24 × 365 = 8,760 horas
Tiempo de inactividad (solo no planificado): 4.5 horas

Disponibilidad = (8,760 - 4.5) / 8,760 × 100%
             = 8,755.5 / 8,760 × 100%
             = **99.95%**

Esto corresponde a "casi cuatro nueves" y supera el SLA típico de 99.9%.

</details>

### Ejercicio 2: Verificar Tiempo de Respuesta

Un incidente se reporta el jueves a las 5:00 PM.
- Horario de servicio: Lun-Vie 8am-6pm
- Tiempo de respuesta acordado: 2 horas

¿Para cuándo debe responder el Soporte TI?

<details>
<summary>Mostrar solución</summary>

**Análisis:**

Reporte: Jueves 5:00 PM
El horario de servicio termina: 6:00 PM

Tiempo de servicio restante jueves: 1 hora
Tiempo faltante: 2h - 1h = 1 hora

Esta 1 hora continúa el viernes.
El horario de servicio del viernes comienza: 8:00 AM

**Respuesta: La respuesta debe ocurrir antes del viernes 9:00 AM**

</details>

### Ejercicio 3: Evaluar Violación de SLA

Un proveedor de servicios TI ha acordado el siguiente SLA:
- Disponibilidad mensual: 99.5%
- Penalización: 5% crédito por cada 0.1% de déficit
- Penalización máxima: 25% de la cuota mensual
- Cuota mensual: €2,000

En marzo, la disponibilidad real fue 99.1%.

¿Cuánto es el crédito?

<details>
<summary>Mostrar solución</summary>

**Cálculo:**

Disponibilidad acordada: 99.5%
Disponibilidad real: 99.1%
Déficit: 99.5% - 99.1% = 0.4%

Penalización: 0.4% / 0.1% × 5% = 4 × 5% = 20%

Verificar máximo: 20% < 25% (OK, no está limitado)

Crédito: €2,000 × 20% = **€400**

</details>

### Ejercicio 4: Asignar Parámetros de SLA

Asigna las siguientes declaraciones al parámetro de SLA correcto:

a) "El servidor debe estar accesible el 99.9% del tiempo"
b) "Para incidentes críticos, el soporte responde en 15 minutos"
c) "Los incidentes críticos se resuelven en 4 horas"
d) "El soporte está disponible de lunes a viernes, 7am-8pm"

<details>
<summary>Mostrar solución</summary>

a) **Disponibilidad** - Tiempo de operación del sistema
b) **Tiempo de respuesta** - Tiempo hasta primera respuesta
c) **Tiempo de resolución** - Tiempo hasta resolución del problema
d) **Horario de servicio** - Período de disponibilidad del soporte

</details>
