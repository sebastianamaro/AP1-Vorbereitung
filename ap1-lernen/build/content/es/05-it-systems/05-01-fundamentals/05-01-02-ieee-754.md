# IEEE 754 - Números de Punto Flotante

## Objetivos de Aprendizaje

Después de este capítulo, deberías ser capaz de:
- Conocer la estructura de los números de punto flotante según IEEE 754
- Comprender los componentes (signo, exponente, mantisa)
- Distinguir entre precisión simple y doble
- Entender los problemas típicos de redondeo

---

## Contenido Principal

### ¿Qué son los Números de Punto Flotante?

Los **números de punto flotante** (Floating Point Numbers) permiten la representación de números decimales en el sistema binario. El estándar **IEEE 754** define el formato.

```
┌─────────────────────────────────────────────────────────────────┐
│                    NÚMERO DE PUNTO FLOTANTE                     │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  Notación científica:   6,022 × 10²³                            │
│                                                                 │
│  Representación en computadora: Signo × Mantisa × 2^Exponente   │
│                                                                 │
│  Ejemplo: -3,5 = -1 × 1,75 × 2¹                                 │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

---

### Estructura Según IEEE 754

Un número de punto flotante consiste en tres componentes:

```
┌─────────────────────────────────────────────────────────────────┐
│                 ESTRUCTURA IEEE 754                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌──────┬────────────────────┬──────────────────────────────┐  │
│  │  S   │      EXPONENTE     │          MANTISA             │  │
│  │ 1Bit │       8 Bit        │           23 Bit             │  │
│  └──────┴────────────────────┴──────────────────────────────┘  │
│                                                                 │
│  S = Signo:       0 = positivo, 1 = negativo                    │
│  Exponente:       Desplazado por sesgo (127 para Simple)        │
│  Mantisa:         Dígitos fraccionarios (1 implícito delante)   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

---

### Precisión Simple vs. Doble

| Formato | Tamaño | Signo | Exponente | Mantisa | Sesgo |
|---------|--------|-------|-----------|---------|-------|
| **Simple (float)** | 32 Bit | 1 Bit | 8 Bit | 23 Bit | 127 |
| **Doble (double)** | 64 Bit | 1 Bit | 11 Bit | 52 Bit | 1023 |

```
PRECISIÓN SIMPLE (32 Bit):
┌─┬────────┬───────────────────────┐
│S│EXPONENTE│       MANTISA         │
│1│   8    │          23           │
└─┴────────┴───────────────────────┘

PRECISIÓN DOBLE (64 Bit):
┌─┬───────────┬────────────────────────────────────────────────┐
│S│ EXPONENTE │                    MANTISA                      │
│1│    11     │                       52                        │
└─┴───────────┴────────────────────────────────────────────────┘
```

---

### Los Componentes en Detalle

#### 1. Signo (S)

| Valor | Significado |
|-------|-------------|
| 0 | Número positivo |
| 1 | Número negativo |

#### 2. Exponente

El exponente se almacena con un **sesgo** para representar exponentes negativos sin un bit de signo.

| Formato | Sesgo | Almacenamiento |
|---------|-------|----------------|
| Simple | 127 | Exponente almacenado = Exponente real + 127 |
| Doble | 1023 | Exponente almacenado = Exponente real + 1023 |

**Ejemplo:** El exponente 3 se almacena como 3 + 127 = **130**.

#### 3. Mantisa

La mantisa almacena los dígitos fraccionarios. Como los números normalizados siempre comienzan con 1, este 1 no se almacena (**1 implícito**).

**Ejemplo:** 1,1011₂ se almacena como mantisa **1011...** (sin el 1 inicial).

---

### Ejemplo: Decimal a IEEE 754

**Tarea:** Convertir **-5,75** a IEEE 754 Precisión Simple.

**Paso 1:** Signo
- Negativo → S = **1**

**Paso 2:** Decimal a Binario
- 5,75₁₀ = 101,11₂
  - 5 = 101
  - 0,75 = 0,5 + 0,25 = 0,11

**Paso 3:** Normalizar
- 101,11 = 1,0111 × 2²
- Mantisa: 0111 (sin el 1 inicial)
- Exponente: 2

**Paso 4:** Exponente con Sesgo
- Exponente almacenado = 2 + 127 = 129 = 10000001₂

**Resultado:**
```
S | Exponente | Mantisa
1 | 10000001 | 01110000000000000000000
```

### Valores Especiales

IEEE 754 también define valores especiales:

| Exponente | Mantisa | Significado |
|-----------|---------|-------------|
| Todo 0 | Todo 0 | **±0** (según el signo) |
| Todo 0 | ≠ 0 | **Número desnormalizado** (muy pequeño) |
| Todo 1 | Todo 0 | **±∞** (Infinito) |
| Todo 1 | ≠ 0 | **NaN** (Not a Number) |

---

### Precisión y Errores de Redondeo

**Problema:** No todos los números decimales pueden representarse exactamente en binario.

```
┌────────────────────────────────────────────────────────────────┐
│                 ERRORES DE REDONDEO                            │
├────────────────────────────────────────────────────────────────┤
│                                                                │
│  0,1₁₀ = 0,0001100110011001100110011...₂  (¡periódico!)        │
│                                                                │
│  Por lo tanto 0,1 + 0,2 ≠ 0,3:                                 │
│  En muchos lenguajes de programación esto resulta en:          │
│  0,1 + 0,2 = 0,30000000000000004                               │
│                                                                │
│  ¡Nunca compares números de punto flotante por igualdad!       │
│                                                                │
└────────────────────────────────────────────────────────────────┘
```

---

### Rango de Valores

| Formato | Número positivo más pequeño | Número más grande |
|---------|------------------------------|-------------------|
| **Simple** | aprox. 1,2 × 10⁻³⁸ | aprox. 3,4 × 10³⁸ |
| **Doble** | aprox. 2,2 × 10⁻³⁰⁸ | aprox. 1,8 × 10³⁰⁸ |

## Términos Clave

| Término | Explicación |
|---------|-------------|
| **Número de punto flotante** | Representación binaria de números decimales |
| **IEEE 754** | Estándar para representación de punto flotante |
| **Precisión Simple** | Número de punto flotante de 32 bit (float) |
| **Precisión Doble** | Número de punto flotante de 64 bit (double) |
| **Mantisa** | Dígitos fraccionarios |
| **Exponente** | Potencia de base 2 |
| **Sesgo** | Desplazamiento del exponente |
| **NaN** | Not a Number (valor inválido) |

## Consejos para el Examen

### Preguntas Comunes de Examen

- ¿Qué componentes conforman un número de punto flotante IEEE 754?
- ¿Cuál es la diferencia entre Precisión Simple y Doble?
- ¿Por qué pueden ocurrir errores de redondeo con números de punto flotante?

### Importante Recordar

- **3 Componentes:** Signo, Exponente, Mantisa
- **Simple:** 32 Bit (1+8+23), Sesgo 127
- **Doble:** 64 Bit (1+11+52), Sesgo 1023
- **1 Implícito:** El 1 inicial no se almacena
- Los **errores de redondeo** son inevitables

## Ejercicios Prácticos

### Ejercicio 1
¿Qué tres componentes conforman un número de punto flotante según IEEE 754?

**Solución:**
1. **Signo** (1 Bit) - 0 para positivo, 1 para negativo
2. **Exponente** (8 u 11 Bit) - desplazado con sesgo
3. **Mantisa** (23 o 52 Bit) - dígitos fraccionarios

### Ejercicio 2
¿Cuál es la diferencia entre Precisión Simple y Doble?

**Solución:**
| Aspecto | Simple | Doble |
|---------|--------|-------|
| Tamaño | 32 Bit | 64 Bit |
| Exponente | 8 Bit | 11 Bit |
| Mantisa | 23 Bit | 52 Bit |
| Precisión | aprox. 7 decimales | aprox. 15-16 decimales |
| Rango de valores | ±10³⁸ | ±10³⁰⁸ |

### Ejercicio 3
¿Por qué 0,1 + 0,2 no es exactamente igual a 0,3 en muchos lenguajes de programación?

**Solución:** El número decimal 0,1 no puede representarse exactamente como un número de punto flotante binario - es periódico en el sistema binario (como 1/3 en el sistema decimal). Por lo tanto, pequeños errores de redondeo ocurren durante el almacenamiento, que pueden acumularse durante los cálculos.

### Ejercicio 4
¿Qué significa "Sesgo" en el contexto de IEEE 754?

**Solución:** El sesgo es un desplazamiento que se suma al exponente real para almacenarlo sin un bit de signo. Para Precisión Simple, el sesgo es 127, para Doble es 1023. Un exponente real de -5 se almacena por lo tanto como -5 + 127 = 122.
