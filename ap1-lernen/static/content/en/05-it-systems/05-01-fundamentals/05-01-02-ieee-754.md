# IEEE 754 - Floating-Point Numbers

## Learning Objectives

After this chapter, you should be able to:
- Know the structure of floating-point numbers according to IEEE 754
- Understand the components (sign, exponent, mantissa)
- Distinguish between single and double precision
- Understand typical rounding problems

---

## Core Content

### What are Floating-Point Numbers?

**Floating-point numbers** (Floating Point Numbers) enable the representation of decimal numbers in the binary system. The **IEEE 754** standard defines the format.

```
┌─────────────────────────────────────────────────────────────────┐
│                    FLOATING-POINT NUMBER                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  Scientific notation:   6.022 × 10²³                            │
│                                                                 │
│  Computer representation: Sign × Mantissa × 2^Exponent          │
│                                                                 │
│  Example: -3.5 = -1 × 1.75 × 2¹                                 │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

---

### Structure According to IEEE 754

A floating-point number consists of three components:

```
┌─────────────────────────────────────────────────────────────────┐
│                 IEEE 754 STRUCTURE                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌──────┬────────────────────┬──────────────────────────────┐  │
│  │  S   │      EXPONENT      │          MANTISSA            │  │
│  │ 1Bit │       8 Bit        │           23 Bit             │  │
│  └──────┴────────────────────┴──────────────────────────────┘  │
│                                                                 │
│  S = Sign:      0 = positive, 1 = negative                      │
│  Exponent:      Shifted by bias (127 for Single)                │
│  Mantissa:      Fractional digits (implicit 1 in front)         │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

---

### Single vs. Double Precision

| Format | Size | Sign | Exponent | Mantissa | Bias |
|--------|------|------|----------|----------|------|
| **Single (float)** | 32 Bit | 1 Bit | 8 Bit | 23 Bit | 127 |
| **Double (double)** | 64 Bit | 1 Bit | 11 Bit | 52 Bit | 1023 |

```
SINGLE PRECISION (32 Bit):
┌─┬────────┬───────────────────────┐
│S│EXPONENT│       MANTISSA        │
│1│   8    │          23           │
└─┴────────┴───────────────────────┘

DOUBLE PRECISION (64 Bit):
┌─┬───────────┬────────────────────────────────────────────────┐
│S│ EXPONENT  │                    MANTISSA                     │
│1│    11     │                       52                        │
└─┴───────────┴────────────────────────────────────────────────┘
```

---

### The Components in Detail

#### 1. Sign (S)

| Value | Meaning |
|-------|---------|
| 0 | Positive number |
| 1 | Negative number |

#### 2. Exponent

The exponent is stored with a **bias** to represent negative exponents without a sign bit.

| Format | Bias | Storage |
|--------|------|---------|
| Single | 127 | Stored exponent = True exponent + 127 |
| Double | 1023 | Stored exponent = True exponent + 1023 |

**Example:** Exponent 3 is stored as 3 + 127 = **130**.

#### 3. Mantissa

The mantissa stores the fractional digits. Since normalized numbers always start with 1, this 1 is not stored (**implicit 1**).

**Example:** 1.1011₂ is stored as mantissa **1011...** (without the leading 1).

---

### Example: Decimal to IEEE 754

**Task:** Convert **-5.75** to IEEE 754 Single Precision.

**Step 1:** Sign
- Negative → S = **1**

**Step 2:** Decimal to Binary
- 5.75₁₀ = 101.11₂
  - 5 = 101
  - 0.75 = 0.5 + 0.25 = 0.11

**Step 3:** Normalize
- 101.11 = 1.0111 × 2²
- Mantissa: 0111 (without the leading 1)
- Exponent: 2

**Step 4:** Exponent with Bias
- Stored exponent = 2 + 127 = 129 = 10000001₂

**Result:**
```
S | Exponent  | Mantissa
1 | 10000001 | 01110000000000000000000
```

### Special Values

IEEE 754 also defines special values:

| Exponent | Mantissa | Meaning |
|----------|----------|---------|
| All 0 | All 0 | **±0** (depending on sign) |
| All 0 | ≠ 0 | **Denormalized number** (very small) |
| All 1 | All 0 | **±∞** (Infinity) |
| All 1 | ≠ 0 | **NaN** (Not a Number) |

### Precision and Rounding Errors

**Problem:** Not all decimal numbers can be represented exactly in binary.

```
┌────────────────────────────────────────────────────────────────┐
│                 ROUNDING ERRORS                                │
├────────────────────────────────────────────────────────────────┤
│                                                                │
│  0.1₁₀ = 0.0001100110011001100110011...₂  (repeating!)         │
│                                                                │
│  Therefore 0.1 + 0.2 ≠ 0.3:                                    │
│  In many programming languages this results in:                │
│  0.1 + 0.2 = 0.30000000000000004                               │
│                                                                │
│  ⚠️ Never compare floating-point numbers for equality!        │
│                                                                │
└────────────────────────────────────────────────────────────────┘
```

---

### Value Range

| Format | Smallest positive number | Largest number |
|--------|--------------------------|----------------|
| **Single** | approx. 1.2 × 10⁻³⁸ | approx. 3.4 × 10³⁸ |
| **Double** | approx. 2.2 × 10⁻³⁰⁸ | approx. 1.8 × 10³⁰⁸ |

## Exam Tips

### Common Exam Questions

- What components make up an IEEE 754 floating-point number?
- What is the difference between Single and Double Precision?
- Why can rounding errors occur with floating-point numbers?

### Important to Remember

- **3 Components:** Sign, Exponent, Mantissa
- **Single:** 32 Bit (1+8+23), Bias 127
- **Double:** 64 Bit (1+11+52), Bias 1023
- **Implicit 1:** The leading 1 is not stored
- **Rounding errors** are unavoidable

## Practice Exercises

### Exercise 1
What three components make up a floating-point number according to IEEE 754?

**Solution:**
1. **Sign** (1 Bit) - 0 for positive, 1 for negative
2. **Exponent** (8 or 11 Bit) - shifted with bias
3. **Mantissa** (23 or 52 Bit) - fractional digits

### Exercise 2
What is the difference between Single and Double Precision?

**Solution:**
| Aspect | Single | Double |
|--------|--------|--------|
| Size | 32 Bit | 64 Bit |
| Exponent | 8 Bit | 11 Bit |
| Mantissa | 23 Bit | 52 Bit |
| Precision | approx. 7 decimal places | approx. 15-16 decimal places |
| Value range | ±10³⁸ | ±10³⁰⁸ |

### Exercise 3
Why does 0.1 + 0.2 not equal exactly 0.3 in many programming languages?

**Solution:** The decimal number 0.1 cannot be represented exactly as a binary floating-point number - it is repeating in the binary system (like 1/3 in the decimal system). Therefore, small rounding errors occur during storage, which can accumulate during calculations.

### Exercise 4
What does "Bias" mean in the context of IEEE 754?

**Solution:** The bias is an offset that is added to the true exponent to store it without a sign bit. For Single Precision, the bias is 127, for Double it is 1023. A true exponent of -5 is therefore stored as -5 + 127 = 122.
