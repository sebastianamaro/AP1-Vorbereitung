# Needs Analysis

## Learning Objectives

After this chapter, you should be able to:
- Explain the purpose of a needs analysis
- Know the steps of a needs analysis
- Name methods for needs assessment
- Understand the connection to requirements and functional specifications

---

## Core Content

### What is a Needs Analysis?

A **needs analysis** determines the actual needs and requirements as a basis for decisions and project planning. It answers the question: **"What is really needed?"**

---

### Goals of the Needs Analysis

- **Understand needs:** What does the customer really need?
- **Clarify requirements:** What functions are required?
- **Set priorities:** What is important, what is optional?
- **Create foundation:** Basis for requirements specification and proposal
- **Avoid wrong decisions:** Identify the right solution

---

### Procedure for Needs Analysis

```
+---------------+     +---------------+     +---------------+
|  1. GATHERING |  -> |  2. ANALYSIS  |  -> | 3. DOCUMENTATION
|               |     |               |     |               |
| Collect data  |     | Evaluate and  |     | Record        |
|               |     | prioritize    |     | results       |
|               |     | requirements  |     |               |
+---------------+     +---------------+     +---------------+
```

---

### Phase 1: Gathering

**Goal:** Collect information about needs

**Methods:**

| Method | Description | Suitable for |
|--------|-------------|--------------|
| **Interview** | One-on-one conversations with stakeholders | In-depth information |
| **Questionnaire** | Structured written survey | Many participants |
| **Workshop** | Group work with stakeholders | Creative solutions |
| **Observation** | Analysis of current state | Process understanding |
| **Document Analysis** | Evaluation of existing documents | Background information |

---

### Phase 2: Analysis

**Goal:** Evaluate and prioritize requirements

**Activities:**
- Categorize requirements (functional/non-functional)
- Identify dependencies
- Set priorities (e.g., MoSCoW method)
- Check feasibility
- Resolve contradictions

**MoSCoW Prioritization:**
| Category | Meaning |
|----------|---------|
| **Must have** | Absolutely required |
| **Should have** | Important but not critical |
| **Could have** | Desirable |
| **Won't have** | Not in this project |

---

### Phase 3: Documentation

**Goal:** Record results and align with stakeholders

**Results:**
- Requirements catalog
- Prioritized list of requirements
- Requirements specification (basis)
- Aligned requirements with stakeholders

---

### Identifying Stakeholders

In the needs analysis, all relevant **stakeholders** must be involved:

| Stakeholder | Interest |
|-------------|----------|
| **Client** | Project goals, budget |
| **End users** | Usability, functions |
| **IT Department** | Integration, maintainability |
| **Management** | Economic viability, strategy |
| **External Partners** | Interfaces |

---

### Customer Requirements in Service Delivery

**Customer requirements** are specific demands that a customer places on a product or service. They significantly influence:

- Project planning
- Project execution
- Project control

**Important:** Customer requirements must be clearly documented and understood!

---

## Key Terms

| Term | Explanation |
|------|-------------|
| **Needs Analysis** | Determination of actual needs and requirements |
| **Stakeholder** | All persons/groups with interest in the project |
| **Requirement** | Desired characteristic or function |
| **Functional Requirement** | What the system should do |
| **Non-functional Requirement** | How the system should be (quality) |
| **MoSCoW** | Prioritization method (Must/Should/Could/Won't) |

## Exam Tips

### Frequent Exam Questions
- What is the purpose of a needs analysis?
- What methods are there for needs assessment?
- What are stakeholders?

### Important to Know
- Needs analysis comes BEFORE the requirements specification
- Include all relevant stakeholders
- Requirements must be prioritized

---

## Practice Exercises

### Exercise 1
Name three methods for needs assessment.

**Solution:**
1. Interview
2. Questionnaire
3. Workshop (or: Observation, Document Analysis)

### Exercise 2
What does the abbreviation MoSCoW mean in prioritization?

**Solution:**
- **M**ust have - Absolutely required
- **S**hould have - Important but not critical
- **C**ould have - Desirable
- **W**on't have - Not in this project

### Exercise 3
Why is it important to include all stakeholders in the needs analysis?

**Solution:** To capture all perspectives and requirements. If stakeholders are forgotten, important requirements can be overlooked, which can later lead to expensive changes or project failure.
