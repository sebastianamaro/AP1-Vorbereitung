# Scrum

## Learning Objectives

After this chapter, you should be able to:
- Explain Scrum as an agile framework
- Know and describe the three Scrum roles
- Know the Scrum meetings (events)
- Understand the Scrum artifacts
- Distinguish Scrum from the waterfall model

---

## Core Content

### What is Scrum?

**Scrum** is an **agile framework** for developing and managing products. It enables iterative and incremental development in short cycles (Sprints).

> **Note:** Kanban, XP, and other agile methods were removed from the 2025 exam catalog. Only Scrum is still exam-relevant!

---

### Fundamental Principles of Scrum

- **Iterative:** Repeated cycles (Sprints)
- **Incremental:** Product grows step by step
- **Self-organization:** Team organizes itself
- **Transparency:** Progress visible to everyone
- **Adaptability:** Flexible response to changes

---

### The Three Scrum Roles

```
+----------------------------------------------------------+
|                    SCRUM TEAM                             |
|                                                           |
|  +--------------+  +--------------+  +----------------+  |
|  |   PRODUCT    |  |    SCRUM     |  |  DEVELOPMENT   |  |
|  |    OWNER     |  |   MASTER     |  |     TEAM       |  |
|  +--------------+  +--------------+  +----------------+  |
|                                                           |
+----------------------------------------------------------+
```

| Role | Tasks |
|------|-------|
| **Product Owner** | - Represents stakeholder interests<br>- Maintains and prioritizes the Product Backlog<br>- Defines the product vision<br>- Decides on requirements |
| **Scrum Master** | - Ensures Scrum rules are followed<br>- Removes obstacles (impediments)<br>- Protects the team from disturbances<br>- Facilitates meetings<br>- Coach for the team |
| **Development Team** | - Implements the requirements<br>- Self-organized<br>- Cross-functional (all skills present)<br>- Typical size: 3-9 people |

### The Scrum Events (Meetings)

#### Sprint
- **Duration:** 2-4 weeks (fixed)
- **Goal:** Create a functional product increment
- **Important:** Sprint duration remains constant throughout the project

#### The Four Meetings in the Sprint

| Meeting | When | Duration | Participants | Purpose |
|---------|------|----------|--------------|---------|
| **Sprint Planning** | Sprint start | Max. 8h (for 4-week Sprint) | Entire team | What will be implemented in the Sprint? How? |
| **Daily Stand-up** | Daily | Max. 15 min. | Development team (+ SM) | What did I do? What will I do? What is blocking me? |
| **Sprint Review** | Sprint end | Max. 4h | Team + Stakeholders | Presentation of results, feedback |
| **Sprint Retrospective** | After Review | Max. 3h | Scrum Team | What went well? What can be improved? |

### The Scrum Artifacts

```
+-----------------+     +-----------------+     +-----------------+
|  PRODUCT        | ->  |  SPRINT         | ->  |  PRODUCT        |
|  BACKLOG        |     |  BACKLOG        |     |  INCREMENT      |
+-----------------+     +-----------------+     +-----------------+
   All                     To be done in         Finished
   requirements            current Sprint        partial result
```

| Artifact | Description |
|----------|-------------|
| **Product Backlog** | Prioritized list of all requirements (User Stories), maintained by Product Owner |
| **Sprint Backlog** | Selected items for the current Sprint + plan for implementation |
| **Product Increment** | Potentially shippable product at Sprint end |

### The Scrum Process

```
                    +--------------------------------------+
                    |            SPRINT (2-4 weeks)        |
                    |                                      |
Product ----------> |  Sprint    Daily    Sprint  Sprint   | ---> Product
Backlog             |  Planning  Scrum    Review  Retro    |      Increment
                    |                                      |
                    +--------------------------------------+
                                    |
                                    v
                            Next Sprint...
```

### Scrum vs. Waterfall

| Aspect | Scrum | Waterfall |
|--------|-------|-----------|
| **Process** | Iterative (Sprints) | Sequential (Phases) |
| **Planning** | Short planning cycles | Comprehensive upfront planning |
| **Changes** | Welcome | Difficult to implement |
| **Customer contact** | Continuous (Sprint Review) | Mainly at beginning/end |
| **Documentation** | Minimal, "working software" | Extensive |
| **Results** | After each Sprint | At project end |
| **Risk** | Discovered early through short cycles | Discovered late |

## Key Terms

| Term | Explanation |
|------|-------------|
| **Sprint** | Fixed time period (2-4 weeks) for development |
| **Product Owner** | Responsible for the product and the backlog |
| **Scrum Master** | Guardian of the Scrum process, removes obstacles |
| **User Story** | Requirement from user perspective ("As a... I want... so that...") |
| **Impediment** | Obstacle that hinders progress |
| **Increment** | Functional partial result at Sprint end |

## Practice Exercises

### Exercise 1
Name the three Scrum roles and one main task each.

**Solution:**
1. **Product Owner:** Maintains the Product Backlog
2. **Scrum Master:** Removes obstacles
3. **Development Team:** Implements requirements

### Exercise 2
What is the difference between Sprint Review and Sprint Retrospective?

**Solution:**
- **Sprint Review:** Presentation of work results to stakeholders, feedback on the PRODUCT
- **Sprint Retrospective:** Internal team meeting to improve the PROCESS

### Exercise 3
Who is responsible for maintaining the Product Backlog?

**Solution:** The Product Owner
