# Software Development Life Cycle (SDLC)

## Learning Objectives

After this chapter, you should be able to:
- Know the phases of software development
- Understand and explain the SDLC
- Recognize connections between phases
- Assign important documents to the phases

---

## Core Content

### What is the SDLC?

The **Software Development Life Cycle (SDLC)** describes the entire life cycle of software from idea to decommissioning.

```
┌────────────────────────────────────────────────────────────────────┐
│                  SOFTWARE DEVELOPMENT LIFE CYCLE                    │
├────────────────────────────────────────────────────────────────────┤
│                                                                     │

│                    ┌────│ Analysis│────┐                           │
│                    │    └─────────┘    │                           │
│                    │                   │                           │
│              ┌─────┴────┐        ┌─────┴────┐                      │
│              │Retirement│        │  Design  │                      │
│              └─────┬────┘        └─────┬────┘                      │

### The SDLC Phases in Detail

```
┌────────────────────────────────────────────────────────────────────┐
│                      SDLC PHASES                                   │
├────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   1. ANALYSIS (Requirements)                                        │
│   ┌────────────────────────────────────────────────────────────┐  │
│   │ • Gather requirements                                      │  │
│   │ • Analyze current state                                    │  │
│   │ • Create target concept                                    │  │
│   │ • Check feasibility                                        │  │
│   │                                                             │  │
│   │ Result: Requirements specification, Functional specification│  │
│   └────────────────────────────────────────────────────────────┘  │
│                              ↓                                     │
│   2. DESIGN                                                        │
│   ┌────────────────────────────────────────────────────────────┐  │
│   │ • Define architecture                                      │  │
│   │ • Plan data structures                                     │  │
│   │ • Define interfaces                                        │  │
│   │ • Create UI design                                         │  │
│   │                                                             │  │
│   │ Result: Technical concept, UML diagrams                    │  │
│   └────────────────────────────────────────────────────────────┘  │
│                              ↓                                     │
│   3. IMPLEMENTATION (Coding)                                       │
│   ┌────────────────────────────────────────────────────────────┐  │
│   │ • Write source code                                        │  │
│   │ • Create documentation                                     │  │
│   │ • Perform code reviews                                     │  │
│   │ • Develop unit tests                                       │  │
│   │                                                             │  │
│   │ Result: Executable software                                │  │
│   └────────────────────────────────────────────────────────────┘  │
│                                                                     │
└────────────────────────────────────────────────────────────────────┘
```

```
┌────────────────────────────────────────────────────────────────────┐
│                   SDLC PHASES (Continued)                          │

│   4. TEST                                                          │

│   │ • Acceptance tests                                         │  │
│   │                                                             │  │
│   │ Result: Test protocols, Bug report                         │  │
│   └────────────────────────────────────────────────────────────┘  │
│                              ↓                                     │
│   5. DEPLOYMENT (Go-Live)                                          │
│   ┌────────────────────────────────────────────────────────────┐  │
│   │ • Installation at customer site                            │  │
│   │ • Data migration                                           │  │
│   │ • User training                                            │  │
│   │ • Handover documentation                                   │  │
│   │                                                             │  │
│   │ Result: Production system, Training materials              │  │
│   └────────────────────────────────────────────────────────────┘  │
│                              ↓                                     │
│   6. MAINTENANCE                                                   │
│   ┌────────────────────────────────────────────────────────────┐  │
│   │ • Bug fixes (corrective)                                   │  │
│   │ • Adaptations (adaptive)                                   │  │
│   │ • Enhancements (perfective)                                │  │
│   │ • Preventive measures                                      │  │

### Maintenance Types

```
┌────────────────────────────────────────────────────────────────────┐
│                       MAINTENANCE TYPES                            │
├────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌─────────────────────────────────────────────────────────────┐ │
│   │ CORRECTIVE MAINTENANCE                                      │ │
│   │ • Fixing errors                                             │ │
│   │ • Bug fixes                                                 │ │
│   │ • Example: Login no longer works                            │ │

│   ┌─────────────────────────────────────────────────────────────┐ │

│   │ PERFECTIVE MAINTENANCE                                      │ │
│   │ • Improvements and extensions                               │ │
│   │ • New features                                              │ │
│   │ • Example: Add export function                              │ │
│   └─────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌─────────────────────────────────────────────────────────────┐ │
│   │ PREVENTIVE MAINTENANCE                                      │ │
│   │ • Prevention of future problems                             │ │
│   │ • Code optimization, refactoring                            │ │
│   │ • Example: Close security vulnerabilities                   │ │
│   └─────────────────────────────────────────────────────────────┘ │
│                                                                     │
└────────────────────────────────────────────────────────────────────┘
```

### SDLC in Different Process Models

```
┌────────────────────────────────────────────────────────────────────┐
│           SDLC IN PROCESS MODELS                                   │
├────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   WATERFALL MODEL                                                  │
│   ┌────────────────────────────────────────────────────────────┐  │
│   │ Analysis → Design → Implementation → Test → Deployment     │  │
│   │                                                             │  │
│   │ • Phases strictly sequential                               │  │
│   │ • Each phase passed through once                           │  │
│   │ • Going back only in case of errors                        │  │
│   └────────────────────────────────────────────────────────────┘  │
│                                                                     │
│   SCRUM (Agile)                                                    │
│   ┌────────────────────────────────────────────────────────────┐  │
│   │        ┌───────────────────────────────┐                   │  │
│   │        │    Sprint (2-4 weeks)         │                   │  │
│   │        │  ┌──────────────────────────┐ │                   │  │
│   │   ──→  │  │Analysis→Design→Code→Test │ │  ──→ Increment   │  │
│   │        │  └──────────────────────────┘ │                   │  │
│   │        └───────────────────────────────┘                   │  │
│   │                      ↑                                      │  │
│   │                      └── Repetition ──┘                    │  │
│   │                                                             │  │
│   │ • Short iterations                                         │  │
│   │ • All phases in each sprint                                │  │
│   │ • Regular deliveries                                       │  │
│   └────────────────────────────────────────────────────────────┘  │
│                                                                     │
└────────────────────────────────────────────────────────────────────┘
```

## Key Terms

| Term | Explanation |
|------|-------------|
| **SDLC** | Software Development Life Cycle |
| **Requirements Specification** | Describes WHAT the client wants |
| **Functional Specification** | Describes HOW the contractor implements |
| **Deployment** | Delivery/installation of software |
| **Release** | Published software version |
| **Patch** | Small update for bug fixes |
| **Refactoring** | Code improvement without functionality change |

## Exam Tips

### Common Exam Questions

- Name the phases of software development
- Which documents are created in which phase?
- Distinguish the maintenance types

### Important to Remember

- **6 Phases:** Analysis → Design → Implementation → Test → Deployment → Maintenance
- **Requirements Specification** = WHAT (Client)
- **Functional Specification** = HOW (Contractor)
- **4 Maintenance types:** Corrective, Adaptive, Perfective, Preventive
- Test phase comes BEFORE deployment!

## Practice Exercises

### Exercise 1
Assign the documents to the SDLC phases:
a) User manual
b) Test protocol
c) Functional specification
d) UML class diagram

**Solution:**
- a) User manual → **Deployment**
- b) Test protocol → **Test**
- c) Functional specification → **Analysis**
- d) UML class diagram → **Design**

### Exercise 2
What type of maintenance is this?
- The software must be adapted to a new database version.

**Solution:**
**Adaptive Maintenance** - The software is adapted to a changed environment (new database version) without changing the functionality.

### Exercise 3
Explain the difference between requirements specification and functional specification.

**Solution:**
| Requirements Specification | Functional Specification |
|---------------------------|-------------------------|
| Created by client | Created by contractor |
| Describes WHAT is desired | Describes HOW to implement |
| Business requirements | Technical solution |
| Basis for quotes | Basis for contract |
