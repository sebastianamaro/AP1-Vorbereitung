# Procurement Process

## Learning Objectives

After this section, you will be able to:
- Name the phases of the procurement process
- Distinguish between types of demand
- Describe the process from demand determination to ordering
- Assign relevant documents in the procurement process

---

## Core Content

### The Procurement Process Overview

```
┌─────────────────────────────────────────────────────────────────────┐
│                    PROCUREMENT PROCESS                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  ┌───────────────┐                                                  │
│  │ 1. DETERMINE  │  "What do we need?"                              │
│  │    DEMAND     │  → Determine type of demand                      │
│  └───────┬───────┘  → Define quantity and timing                    │
│          │                                                          │
│          ▼                                                          │
│  ┌───────────────┐                                                  │
│  │ 2. SEARCH FOR │  "Who can deliver?"                              │
│  │    SUPPLIERS  │  → Identify potential vendors                    │
│  └───────┬───────┘  → Send inquiries                                │
│          │                                                          │
│          ▼                                                          │
│  ┌───────────────┐                                                  │
│  │ 3. OBTAIN     │  "What does it cost?"                            │
│  │    QUOTATIONS │  → Request quotations                            │
│  └───────┬───────┘  → Compare conditions                            │
│          │                                                          │
│          ▼                                                          │
│  ┌───────────────┐                                                  │
│  │ 4. COMPARE    │  "Which offer is best?"                          │
│  │    QUOTATIONS │  → Quantitative (landed cost)                    │
│  └───────┬───────┘  → Qualitative (utility analysis)                │
│          │                                                          │
│          ▼                                                          │
│  ┌───────────────┐                                                  │
│  │ 5. MAKE       │  "Who do we choose?"                             │
│  │    DECISION   │  → Supplier selection                            │
│  └───────┬───────┘  → Negotiation if needed                         │
│          │                                                          │
│          ▼                                                          │
│  ┌───────────────┐                                                  │
│  │ 6. PLACE      │  "We're buying!"                                 │
│  │    ORDER      │  → Send purchase order                           │
│  └───────┬───────┘  → Receive order confirmation                    │
│          │                                                          │
│          ▼                                                          │
│  ┌───────────────┐                                                  │
│  │ 7. GOODS      │  "Is everything there?"                          │
│  │    RECEIPT    │  → Check delivery                                │
│  └───────┬───────┘  → Report defects                                │
│          │                                                          │
│          ▼                                                          │
│  ┌───────────────┐                                                  │
│  │ 8. INVOICE    │  "Is the invoice correct?"                       │
│  │    REVIEW     │  → Compare invoice with order                    │
│  └───────┬───────┘  → Initiate payment                              │
│          │                                                          │
│          ▼                                                          │
│  ┌───────────────┐                                                  │
│  │ 9. PAYMENT    │  "We're paying!"                                 │
│  │               │  → Use discount                                   │
│  └───────────────┘  → Booking                                        │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Types of Demand

```
┌─────────────────────────────────────────────────────────────────────┐
│                    TYPES OF DEMAND                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  PRIMARY DEMAND                                                      │
│  → Demand for finished products                                      │
│  → E.g. finished PCs sold to customers                               │
│                                                                      │
│  SECONDARY DEMAND                                                    │
│  → Demand for components/parts for production                        │
│  → E.g. processors, RAM, hard drives for PC assembly                 │
│                                                                      │
│  TERTIARY DEMAND                                                     │
│  → Demand for auxiliary materials and operating supplies             │
│  → E.g. screws, cables, packaging, electricity                       │
│                                                                      │
│  ────────────────────────────────────────────────────────────────   │
│                                                                      │
│  For IT service providers (ConSystem GmbH):                          │
│  → Primary demand: Finished solutions for customers                  │
│  → Secondary demand: Hardware, software, licenses                    │
│  → Tertiary demand: Office supplies, cables, tools                   │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Demand Determination Methods

| Method | Description | Application |
|--------|-------------|-------------|
| **Program-based** | Derived from production program | Known demand |
| **Consumption-based** | Based on historical values | Regular demand |
| **Heuristic** | Expert estimation | New/unknown demand |

### Documents in the Procurement Process

```
┌─────────────────────────────────────────────────────────────────────┐
│                    DOCUMENTS IN THE PROCUREMENT PROCESS              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  Phase                │ Document               │ From → To           │
│  ─────────────────────┼────────────────────────┼───────────────────  │
│  Demand notification  │ Purchase requisition   │ Department → Purchasing │
│  Supplier search      │ Inquiry                │ Purchasing → Supplier   │
│  Obtain quotations    │ Quotation              │ Supplier → Purchasing   │
│  Order                │ Purchase order         │ Purchasing → Supplier   │
│  Confirmation         │ Order confirmation     │ Supplier → Purchasing   │
│  Delivery             │ Delivery note          │ Supplier → Purchasing   │
│  Billing              │ Invoice                │ Supplier → Purchasing   │
│                                                                      │
│  ────────────────────────────────────────────────────────────────   │
│                                                                      │
│  INQUIRY  ≠  QUOTATION  ≠  ORDER                                    │
│                                                                      │
│  Inquiry:   "What does X cost?" (non-binding)                       │
│  Quotation: "X costs Y €" (binds the vendor)                        │
│  Order:     "I'm buying X!" (binds the buyer)                       │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Order vs. Order Confirmation

```
┌─────────────────────────────────────────────────────────────────────┐
│               ORDER AND ORDER CONFIRMATION                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  BUYER                                           SELLER              │
│  (ConSystem GmbH)                                (Supplier)          │
│                                                                      │
│       ┌─────────────────┐                                           │
│       │   ORDER         │ ─────────────────────────────►            │
│       │ • Item          │                                           │
│       │ • Quantity      │                                           │
│       │ • Price         │                                           │
│       │ • Delivery date │                                           │
│       └─────────────────┘                                           │
│                                                                      │
│                         ◄─────────────────────────────              │
│       ┌─────────────────┐                                           │
│       │ORDER CONFIRMA-  │                                           │
│       │TION (OC)        │                                           │
│       │ • Confirms      │                                           │
│       │   the order     │                                           │
│       │ • Or shows      │                                           │
│       │   deviations    │                                           │
│       └─────────────────┘                                           │
│                                                                      │
│  IMPORTANT: ALWAYS check the order confirmation!                     │
│  → Do item, quantity, price, date match?                            │
│  → Report deviations immediately!                                   │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Goods Receipt Inspection

```
┌─────────────────────────────────────────────────────────────────────┐
│                    GOODS RECEIPT INSPECTION                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  1. QUANTITY CHECK (immediately upon delivery)                       │
│     → Does the number of packages match?                            │
│     → Are all ordered items present?                                │
│     → Note on delivery slip!                                        │
│                                                                      │
│  2. VISUAL INSPECTION (immediately upon delivery)                    │
│     → Is packaging undamaged?                                       │
│     → Visible transport damage?                                     │
│     → If damaged: Document and report IMMEDIATELY!                  │
│                                                                      │
│  3. QUALITY CHECK (after unpacking)                                  │
│     → Does the goods match the order?                               │
│     → Does the goods function properly?                             │
│     → Hidden defects?                                               │
│                                                                      │
│  In case of defects:                                                │
│  → Report within the complaint period (usually immediately)         │
│  → Assert warranty rights                                           │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

## Key Terms

| Term | Explanation |
|------|-------------|
| **Demand determination** | Identifying what is needed in which quantity |
| **Inquiry** | Non-binding request for a quotation |
| **Quotation** | Binding statement about price and conditions |
| **Order** | Binding purchase intent of the customer |
| **Order confirmation** | Confirmation of the order by the supplier |
| **Delivery note** | Accompanies the goods during delivery |
| **Complaint period** | Timeframe for reporting defects |

---

## Exam Tips

1. **Sequence of documents:**
   - Inquiry → Quotation → Order → OC → Delivery note → Invoice

2. **Remember the difference:**
   - Inquiry = non-binding
   - Quotation = binds the SELLER
   - Order = binds the BUYER

3. **Goods receipt inspection:**
   - Report defects IMMEDIATELY (duty to complain!)
   - Document on delivery note

### Exercise 3: Goods Receipt Check

Upon delivery of 20 monitors you notice:
- 2 boxes are dented
- According to the delivery note, only 18 monitors are included

What steps must you take?

<details>
<summary>Show solution</summary>

**Immediate measures:**
1. **Document quantity difference:**
   - Note on delivery slip: "Only 18 instead of 20 monitors received"
   - Have the driver sign it

2. **Document damaged boxes:**
   - Take photos
   - Note on delivery slip: "2 boxes damaged"

3. **Contact supplier:**
   - Report immediately by phone/email
   - Report shortage and damage
   - Send written complaint

4. **Check goods:**
   - Open damaged boxes
   - Test monitors for functionality
   - Document results

5. **Internal documentation:**
   - Create goods receipt report
   - For invoice review: Only pay for 18 monitors!

</details>
