# Tipos de Comunicación en Soporte TI

## Objetivos de Aprendizaje

Después de esta sección, podrás:
- Distinguir diferentes canales de comunicación en soporte TI
- Nombrar las ventajas y desventajas de cada canal
- Seleccionar el canal apropiado para diferentes situaciones
- Aplicar reglas básicas de comunicación profesional

---

### Selección de Canal por Situación

| Situación | Canal Recomendado | Justificación |
|-----------|-------------------|---------------|
| **Interrupción total** | Teléfono | Urgente, se necesita respuesta inmediata |
| **Olvidó contraseña** | Autoservicio | Proceso estándar, automatizable |
| **Enviar captura de pantalla** | Correo/Ticket | Posibilidad de adjuntos |
| **Problema complejo** | Soporte remoto + Teléfono | Combinación ideal |
| **Consulta de estado** | Sistema de tickets | El usuario puede verificar por sí mismo |
| **Pregunta de formación** | Base de conocimiento | Respuesta reutilizable |

## Términos Clave

| Término | Explicación |
|---------|-------------|
| **Comunicación Síncrona** | Intercambio simultáneo (p. ej., teléfono) |
| **Comunicación Asíncrona** | Intercambio diferido (p. ej., correo electrónico) |
| **Soporte Remoto** | Acceso remoto al PC del usuario |
| **Base de Conocimiento** | Base de datos con soluciones |
| **FAQ** | Preguntas Frecuentes |
| **Autoservicio** | El usuario resuelve el problema de forma independiente |

---

## Consejos para el Examen

1. **Justificar la selección de canal:**
   - Urgente → síncrono (teléfono)
   - Se necesita documentación → asíncrono (correo/ticket)
   - Complejo → combinación (soporte remoto + teléfono)

2. **Conocer ventajas y desventajas:**
   - Cada canal tiene fortalezas y debilidades específicas
   - Frecuente en exámenes: "¿Qué canal es adecuado y por qué?"

3. **Comportamiento profesional:**
   - Amabilidad incluso con usuarios difíciles
   - Sin asignar culpas

## Ejercicios Prácticos

### Ejercicio 1: Seleccionar Canal

Elige el canal de comunicación apropiado y justifica:

a) Un empleado ya no puede iniciar sesión en su PC
b) Contabilidad necesita un informe sobre costos de TI del último trimestre
c) Un nuevo empleado tiene preguntas sobre configurar su cliente de correo

<details>
<summary>Mostrar Solución</summary>

a) **Teléfono + Soporte Remoto**
   - Urgente (el empleado no puede trabajar)
   - Intervención directa posible
   - Puede tener varias causas (contraseña, cuenta bloqueada, problema de AD)

b) **Correo electrónico**
   - No urgente
   - El informe se puede enviar como adjunto
   - La documentación es importante

c) **Autoservicio/Base de conocimiento** o **Correo con instrucciones**
   - Proceso estándar
   - Las instrucciones se pueden reutilizar
   - No es un problema crítico en tiempo

</details>

### Ejercicio 2: Redactar Correo

Un usuario reportó por correo: "¡¡Mi PC está lento!!"

Formula un correo de respuesta profesional pidiendo más información.

<details>
<summary>Mostrar Solución</summary>

**Ejemplo de respuesta:**

```
Asunto: Re: Su PC está lento - Ticket #2024-0815

Estimado Sr. García,

Gracias por su mensaje.

Para resolver su problema lo más rápido posible, necesito
alguna información adicional:

1. ¿Desde cuándo está lento el PC?
2. ¿Qué programas están particularmente afectados?
3. ¿Ha instalado o cambiado algo recientemente?
4. ¿Cómo se manifiesta exactamente el problema?
   (p. ej., tiempo de inicio largo, programas que no responden)

Mientras tanto, puede probar las siguientes medidas:
- Reiniciar su PC
- Cerrar programas que no necesita

Me pondré en contacto con usted tan pronto tenga más información.

Saludos cordiales,
Carlos López
Soporte TI, ConSystem GmbH
Tel: 0800-1234567
Ticket: #2024-0815
```

</details>

### Ejercicio 3: Identificar Errores de Comunicación

¿Qué está mal con la siguiente declaración de un técnico de soporte TI?

"Usted hizo eso mal. Todo está en el manual. ¿Quién no lee eso?"

<details>
<summary>Mostrar Solución</summary>

**Errores:**

1. **Asignación de culpa** - "Usted hizo eso mal"
   → Mejor: "Este es un error común que podemos solucionar rápidamente"

2. **Acusación** - "Todo está en el manual"
   → Mejor: "Puede encontrar instrucciones para esto en el manual en la página X"

3. **Condescendiente** - "¿Quién no lee eso?"
   → Mejor: Simplemente mostrar la solución sin comentarios

**Alternativa profesional:**
"Entiendo el problema. Déjeme mostrarle rápidamente cómo funciona. Si quiere consultarlo de nuevo más tarde, también puede encontrar instrucciones en el manual en la página 15."

</details>
