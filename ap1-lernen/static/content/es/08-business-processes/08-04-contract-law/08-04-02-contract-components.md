# Componentes del Contrato

## Objetivos de Aprendizaje

Después de esta sección, serás capaz de:
- Explicar cómo se forma un contrato
- Distinguir entre declaración de voluntad, oferta y aceptación
- Nombrar los componentes esenciales del contrato
- Explicar la importancia de los términos y condiciones generales

---

## Contenido Principal

### Formación de un Contrato

```
┌─────────────────────────────────────────────────────────────────────┐
│                    FORMACIÓN DEL CONTRATO                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  Un contrato se forma mediante dos                                  │
│  DECLARACIONES DE VOLUNTAD coincidentes:                            │
│                                                                      │
│           OFERTA                         ACEPTACIÓN                  │
│        (Declaración 1)              (Declaración 2)                  │
│                                                                      │
│        ┌─────────────┐           ┌─────────────┐                    │
│        │             │           │             │                    │
│        │   Vendedor  │ ────────► │   Comprador │                    │
│        │  "Ofrezco   │           │  "¡Acepto!" │                    │
│        │   ..."      │           │             │                    │
│        │             │           │             │                    │
│        └─────────────┘           └─────────────┘                    │
│                                                                      │
│                        CONTRATO                                      │
│                     ¡está formado!                                   │
│                                                                      │
│  Requisitos previos:                                                │
│  → Capacidad jurídica de ambas partes                               │
│  → Capacidad contractual                                            │
│  → Coincidencia de oferta y aceptación                              │
│  → No se requiere forma (usualmente informal)                       │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Solicitud vs. Oferta vs. Pedido

```
┌─────────────────────────────────────────────────────────────────────┐
│                    DISTINCIÓN                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  SOLICITUD                                                          │
│  → "¿Cuánto cuesta un portátil de ustedes?"                        │
│  → NO es una declaración de voluntad                                │
│  → No vincula a nadie                                               │
│                                                                      │
│  OFERTA (Propuesta)                                                 │
│  → "El portátil cuesta €899, entrega en 3 días"                    │
│  → Declaración de voluntad del OFERENTE                             │
│  → Vincula al oferente (vinculación a la oferta)                    │
│                                                                      │
│  PEDIDO (Aceptación)                                                │
│  → "Ordeno el portátil bajo estas condiciones"                     │
│  → Declaración de voluntad del COMPRADOR                            │
│  → ¡El contrato está formado!                                       │
│                                                                      │
│  CASO ESPECIAL: Pedido sin oferta previa                            │
│  → Pedido = Oferta del comprador                                    │
│  → Confirmación de pedido = Aceptación del vendedor                 │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Componentes Esenciales del Contrato

```
┌─────────────────────────────────────────────────────────────────────┐
│                    COMPONENTES DEL CONTRATO                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  ESSENTIALIA NEGOTII (componentes esenciales)                       │
│  ──────────────────────────────────────────────────────────────     │
│  → Deben estar incluidos, ¡sino no hay contrato!                    │
│                                                                      │
│  Para contrato de compraventa:                                      │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │  1. Partes contratantes (¿Quién?)                           │    │
│  │  2. Objeto de la compra (¿Qué?)                             │    │
│  │  3. Precio de compra (¿Cuánto?)                             │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                                                                      │
│  NATURALIA NEGOTII (componentes naturales)                          │
│  ──────────────────────────────────────────────────────────────     │
│  → Se aplican automáticamente si no se acuerda otra cosa           │
│  → Ej. garantía, transferencia de propiedad                         │
│                                                                      │
│  ACCIDENTALIA NEGOTII (componentes accidentales)                    │
│  ──────────────────────────────────────────────────────────────     │
│  → Acuerdos individuales                                            │
│  → Ej. fecha de entrega, condiciones de pago especiales             │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Contenido de un Contrato de TI

| Componente | Descripción | Ejemplo |
|-----------|-------------|---------|
| **Partes contratantes** | ¿Quién firma el contrato? | ConSystem GmbH y Cliente AG |
| **Objeto de la prestación** | ¿Qué se entrega? | 10 portátiles modelo XY |
| **Precio** | ¿Cuánto cuesta? | €8,990 neto |
| **Fecha de entrega** | ¿Cuándo se entrega? | Dentro de 5 días laborables |
| **Condiciones de pago** | ¿Cómo se paga? | 2% descuento en 10 días |
| **Garantía** | Responsabilidad por defectos | 24 meses |
| **Responsabilidad** | Limitaciones de responsabilidad | Máx. valor del pedido |
| **CGC** | Condiciones generales | Anexo al contrato |

### Condiciones Generales de Contratación (CGC)

```
┌─────────────────────────────────────────────────────────────────────┐
│                    CGC - CONDICIONES GENERALES DE CONTRATACIÓN       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  ¿Qué son las CGC?                                                  │
│  → Términos contractuales preformulados                             │
│  → Aplicables a muchos contratos                                    │
│  → "Letra pequeña"                                                  │
│                                                                      │
│  Inclusión de CGC (§ 305 BGB):                                      │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │  1. Referencia expresa                                      │    │
│  │     → "Se aplican nuestras CGC"                             │    │
│  │                                                              │    │
│  │  2. Posibilidad de conocimiento                             │    │
│  │     → Las CGC deben ser accesibles (exhibición, sitio web)  │    │
│  │                                                              │    │
│  │  3. Consentimiento del cliente                              │    │
│  │     → El cliente debe estar de acuerdo                      │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                                                                      │
│  Cláusulas inválidas (§§ 307-309 BGB):                              │
│  → Cláusulas sorpresivas                                            │
│  → Desventaja irrazonable                                           │
│  → Violación de prohibiciones legales                               │
│                                                                      │
│  Ejemplo de cláusula TI inválida:                                   │
│  "Se excluye toda responsabilidad por daños de nuestro software."  │
│  → Inválida para dolo/negligencia grave                             │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Requisitos Formales

```
┌─────────────────────────────────────────────────────────────────────┐
│                    REQUISITOS FORMALES                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  PRINCIPIO: Libertad de contrato = Libertad de forma               │
│  → La mayoría de contratos pueden celebrarse oralmente              │
│                                                                      │
│  EXCEPCIONES (prescritas legalmente):                               │
│  ──────────────────────────────────────────────────────────────     │
│                                                                      │
│  Forma escrita (§ 126 BGB):                                         │
│  → Firma requerida                                                  │
│  → Ej. garantías, referencias laborales                             │
│                                                                      │
│  Forma textual (§ 126b BGB):                                        │
│  → Declaración legible, persona identificable                       │
│  → Ej. email                                                        │
│  → Ej. política de cancelación en comercio online                   │
│                                                                      │
│  Certificación notarial:                                            │
│  → Se requiere notario                                              │
│  → Ej. compra de bienes raíces, constitución de GmbH               │
│                                                                      │
│  CONTRATOS DE TI: ¡Usualmente sin formalidades!                     │
│  → Pero: Forma escrita recomendada para evidencia                   │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

## Términos Clave

| Término | Explicación |
|------|-------------|
| **Declaración de voluntad** | Expresión de una voluntad legalmente relevante |
| **Oferta** | Declaración de voluntad vinculante del oferente |
| **Aceptación** | Conformidad con la oferta |
| **Essentialia negotii** | Componentes esenciales del contrato |
| **CGC** | Condiciones Generales de Contratación |
| **Libertad de forma** | El contrato puede celebrarse en cualquier forma |
| **Forma textual** | Forma escrita sin firma (ej. email) |

---

## Consejos para el Examen

1. **Recuerda la formación del contrato:**
   - Oferta + Aceptación = Contrato

2. **Inclusión de CGC:**
   - Referencia + Posibilidad de conocimiento + Consentimiento

3. **Preguntas típicas del examen:**
   - ¿Cuándo se forma un contrato?
   - ¿Qué pertenece a un contrato de compraventa?

## Ejercicios Prácticos

### Ejercicio 1: Formación del Contrato

ConSystem GmbH envía a un cliente una oferta por 5 servidores por €15,000. El cliente responde por email: "Ordenamos los servidores."

a) ¿Se ha formado un contrato?
b) ¿Qué declaraciones de voluntad están presentes?

<details>
<summary>Mostrar solución</summary>

a) **Sí, se ha formado un contrato.**

b) **Declaraciones de voluntad:**
- Oferta de ConSystem: "5 servidores por €15,000" (vincula a ConSystem)
- Aceptación del cliente: "Ordenamos los servidores" (vincula al cliente)
- Ambas declaraciones de voluntad coinciden → Contrato

</details>

### Ejercicio 2: Inclusión de CGC

Un cliente ordena software online de ConSystem. En el sitio web dice en letra pequeña: "Se aplican nuestras CGC". Las CGC solo son accesibles a través de un pequeño enlace al final de la página.

¿Están las CGC válidamente incluidas?

<details>
<summary>Mostrar solución</summary>

**Examen de requisitos de inclusión:**

1. **Referencia:** Presente ("Se aplican nuestras CGC")
2. **Posibilidad de conocimiento:** Cuestionable - enlace muy pequeño y oculto
3. **Consentimiento:** Poco claro - ¿fue el cliente informado sobre CGC antes de completar el pedido?

**Resultado:** Las CGC pueden estar inválidamente incluidas si:
- El enlace no es claramente visible
- No se requirió casilla de "CGC leídas"
- Las CGC no eran visibles antes de la conclusión del contrato

**Recomendación:** Colocar enlace de CGC prominentemente e incluir casilla de confirmación.

</details>

### Ejercicio 3: Componentes Esenciales

Verifica si la siguiente información es suficiente para un contrato de compraventa válido de servidores:

"ConSystem entrega servidores a Firma Müller"

<details>
<summary>Mostrar solución</summary>

**Análisis de componentes esenciales (Essentialia):**

| Componente | ¿Presente? | Nota |
|-----------|----------|------|
| Partes contratantes | Sí | ConSystem y Firma Müller |
| Objeto de la compra | Cuestionable | "Servidores" - ¿pero cuáles? ¿Cantidad? ¿Tipo? |
| Precio de compra | No | ¡Falta completamente! |

**Resultado:** El contrato **no es válido** porque:
- El objeto de la compra no está especificado con precisión
- Falta el precio de compra

**Correcto sería:**
"ConSystem entrega 3 servidores tipo HP ProLiant DL380 Gen10 a Firma Müller por un total de €12,000 neto."

</details>
