## Contenido Principal

### ¿Qué es DHCP?

```
+--------------------------------------------------------------------+
|                          DHCP                                       |
+--------------------------------------------------------------------+
|                                                                     |
|   DHCP = Dynamic Host Configuration Protocol                        |
|          (Protocolo de Configuración Dinámica de Host)             |
|                                                                     |
|   +-------------------------------------------------------------+ |
|   | DHCP asigna automáticamente configuraciones de red a        | |
|   | dispositivos en la red (dirección IP, gateway, DNS).        | |
|   +-------------------------------------------------------------+ |
|                                                                     |
|   Sin DHCP:                       Con DHCP:                        |
|   +--------------------------+    +--------------------------+     |
|   | Cada dispositivo debe    |    | Los dispositivos reciben |     |
|   | configurarse manualmente:|    | su configuración         |     |
|   | - Dirección IP           |    | automáticamente:         |     |
|   | - Máscara de subred      |    |                          |     |
|   | - Gateway                |    |  PC -> Servidor DHCP -> IP|    |
|   | - Servidor DNS           |    |                          |     |
|   +--------------------------+    | "Plug and Play"          |     |
|                                   +--------------------------+     |
|                                                                     |
|   Ventajas:                                                        |
|   - No se necesita configuración manual                            |
|   - No hay conflictos de direcciones IP                            |
|   - Gestión centralizada                                           |
|   - Cambios en un solo lugar                                       |
|                                                                     |
+--------------------------------------------------------------------+
```

### El Proceso DORA

```
+--------------------------------------------------------------------+
|                       PROCESO DORA                                  |
+--------------------------------------------------------------------+
|                                                                     |
|   DORA = Discover, Offer, Request, Acknowledge                     |
|          (Descubrir, Ofrecer, Solicitar, Confirmar)                |
|                                                                     |
|      Cliente                                    Servidor DHCP      |
|        |                                           |               |
|   1.   |     DHCP DISCOVER (Broadcast)            |               |
|        | ------------------------------------>    |               |
|        |   "¿Hay un servidor DHCP?"               |               |
|        |   (Broadcast a 255.255.255.255)          |               |
|        |                                           |               |
|   2.   |     DHCP OFFER                           |               |
|        | <------------------------------------    |               |
|        |   "Aquí tienes una IP:"                  |               |
|        |   IP: 192.168.1.100                      |               |
|        |   Máscara: 255.255.255.0                 |               |
|        |   Gateway: 192.168.1.1                   |               |
|        |                                           |               |
|   3.   |     DHCP REQUEST (Broadcast)             |               |
|        | ------------------------------------>    |               |
|        |   "¡Acepto esta IP!"                     |               |
|        |   (Broadcast, para que otros serv. sepan)|               |
|        |                                           |               |
|   4.   |     DHCP ACKNOWLEDGE                     |               |
|        | <------------------------------------    |               |
|        |   "¡Confirmado, la IP es tuya!"          |               |
|        |   Tiempo de lease: 24 horas              |               |
|        |                                           |               |
|        |   =====================================   |               |
|        |     El cliente ahora tiene acceso a red  |               |
|        |   =====================================   |               |
|                                                                     |
|   Mnemotécnico: "Dora necesita DHCP"                               |
|   D = Discover, O = Offer, R = Request, A = Acknowledge            |
|                                                                     |
+--------------------------------------------------------------------+
```

### Lease de DHCP

```
+--------------------------------------------------------------------+
|                       LEASE DE DHCP                                 |
+--------------------------------------------------------------------+
|                                                                     |
|   Definición:                                                      |
|   +-------------------------------------------------------------+ |
|   | Un lease de DHCP es el "préstamo" con límite de tiempo de   | |
|   | una dirección IP a un dispositivo.                           | |
|   +-------------------------------------------------------------+ |
|                                                                     |
|   Tiempo de lease (típico: 24 horas a 7 días)                      |
|                                                                     |
|   +-------------------------------------------------------------+ |
|   |                                                              | |
|   |   0%        50%                       100%                  | |
|   |   |----------|--------------------------|                   | |
|   |   Inicio    Renovación     Rebinding    Expiración          | |
|   |              (50%)         (87.5%)                          | |
|   |                                                              | |
|   |   Al 50%: El cliente intenta renovar el lease              | |
|   |   Al 87.5%: El cliente intenta renovar con CUALQUIER serv. | |
|   |   Al 100%: La IP se libera                                 | |
|   |                                                              | |
|   +-------------------------------------------------------------+ |
|                                                                     |
|   ¿Qué pasa cuando expira el lease?                                |
|   - El cliente pierde la dirección IP                              |
|   - El cliente debe iniciar un nuevo proceso DORA                  |
|   - Puede recibir una dirección IP diferente                       |
|                                                                     |
+--------------------------------------------------------------------+
```

### Datos de Configuración DHCP

```
+--------------------------------------------------------------------+
|                DHCP PROPORCIONA ESTOS DATOS                         |
+--------------------------------------------------------------------+
|                                                                     |
|   Requeridos:                                                      |
|   +------------------------------------------------------------+  |
|   | - Dirección IP                                             |  |
|   | - Máscara de subred                                        |  |
|   +------------------------------------------------------------+  |
|                                                                     |
|   Opcionales (pero comunes):                                       |
|   +------------------------------------------------------------+  |
|   | - Gateway predeterminado (dirección del router)            |  |
|   | - Servidor DNS (para resolución de nombres)                |  |
|   | - Tiempo de lease                                          |  |
|   | - Servidor WINS (redes Windows)                            |  |
|   | - Nombre de dominio                                        |  |
|   | - Servidor NTP (servidor de tiempo)                        |  |
|   +------------------------------------------------------------+  |
|                                                                     |
|   Ejemplo de configuración:                                        |
|   +------------------------------------------------------------+  |
|   | Dirección IP:     192.168.1.100                            |  |
|   | Máscara subred:   255.255.255.0                            |  |
|   | Gateway:          192.168.1.1                              |  |
|   | Servidor DNS:     192.168.1.1, 8.8.8.8                     |  |
|   | Tiempo de lease:  86400 segundos (24 horas)                |  |
|   +------------------------------------------------------------+  |
|                                                                     |
+--------------------------------------------------------------------+
```

### Pool de Direcciones DHCP

```
+--------------------------------------------------------------------+
|                     POOL DE DIRECCIONES DHCP                        |
+--------------------------------------------------------------------+
|                                                                     |
|   El administrador define un rango de direcciones IP que           |
|   DHCP puede asignar:                                              |
|                                                                     |
|   +-------------------------------------------------------------+ |
|   |                                                              | |
|   |   Red: 192.168.1.0/24                                       | |
|   |                                                              | |
|   |   |  .1 - .10  |   .11 - .200   |  .201 - .254  |           | |
|   |   |  Estáticas |   Pool DHCP    |   Estáticas   |           | |
|   |   |  (Servers) |   (Clientes)   |   (Impresoras)|           | |
|   |                                                              | |
|   |   .0   = Dirección de red (no utilizable)                   | |
|   |   .255 = Broadcast (no utilizable)                          | |
|   |                                                              | |
|   +-------------------------------------------------------------+ |
|                                                                     |
|   Reservaciones:                                                   |
|   +------------------------------------------------------------+  |
|   | Ciertos dispositivos pueden recibir una IP fija:           |  |
|   | - Basado en dirección MAC                                  |  |
|   | - Servidores, impresoras siempre obtienen la misma IP     |  |
|   | - Aún configurados vía DHCP                                |  |
|   +------------------------------------------------------------+  |
|                                                                     |
+--------------------------------------------------------------------+
```

### DHCP vs. Configuración Estática

```
+--------------------------------------------------------------------+
|               DHCP vs. ESTÁTICA                                     |
+--------------------------------------------------------------------+
|                                                                     |
|                    DHCP               Estática                      |
|   +----------------------------+------------------------------+   |
|   | Asignación automática      | Configuración manual         |   |
|   | La IP puede cambiar        | La IP siempre es la misma    |   |
|   | Menos esfuerzo             | Más esfuerzo                 |   |
|   | Gestión centralizada       | Gestión descentralizada      |   |
|   | Sin conflictos             | Conflictos posibles          |   |
|   | Depende del servidor DHCP  | Independiente                |   |
|   +----------------------------+------------------------------+   |
|                                                                     |
|   ¿Cuándo DHCP?                 ¿Cuándo estática?                  |
|   +----------------------------+------------------------------+   |
|   | - PCs cliente              | - Servidores                 |   |
|   | - Portátiles               | - Routers/Gateways           |   |
|   | - Smartphones              | - Impresoras de red          |   |
|   | - Dispositivos invitados   | - Sistemas NAS               |   |
|   | - Dispositivos con uso     | - Dispositivos que deben ser |   |
|   |   cambiante                |   siempre accesibles         |   |
|   +----------------------------+------------------------------+   |
|                                                                     |
+--------------------------------------------------------------------+
```

## Términos Clave

| Término | Explicación |
|---------|-------------|
| **DHCP** | Dynamic Host Configuration Protocol |
| **DORA** | Discover, Offer, Request, Acknowledge |
| **Lease** | Asignación de IP con límite de tiempo |
| **Pool de direcciones** | Rango de direcciones IP disponibles |
| **Reservación** | IP fija basada en dirección MAC |

## Consejos para el Examen

### Preguntas Comunes del Examen

- Explicar el proceso DORA
- ¿Qué datos proporciona DHCP?
- ¿Cuándo estática, cuándo DHCP?

### Importante Recordar

- **DORA:** Discover -> Offer -> Request -> Acknowledge
- **Puertos:** Servidor 67, Cliente 68 (UDP)
- **DHCP proporciona:** IP, máscara, gateway, DNS
- **Lease** = asignación con límite de tiempo
- **Servidores** = IP estática, **Clientes** = DHCP
- Discover y Request son **broadcasts**

## Ejercicios Prácticos

### Ejercicio 1
Explica el proceso DORA en DHCP.

**Solución:**
1. **Discover:** El cliente envía broadcast ("¿Hay un servidor DHCP?")
2. **Offer:** El servidor responde con oferta de IP
3. **Request:** El cliente acepta la oferta (broadcast)
4. **Acknowledge:** El servidor confirma la asignación

### Ejercicio 2
¿Por qué un servidor debería tener una IP estática y no usar DHCP?

**Solución:**
- Los servidores deben ser **siempre accesibles en la misma IP**
- Otros dispositivos/servicios se refieren a esta IP
- DHCP podría asignar una **IP diferente**
- Si el **servidor DHCP falla**, el servidor sería inaccesible
- Las entradas DNS ya no serían correctas

### Ejercicio 3
¿Qué información recibe típicamente un cliente del servidor DHCP?

**Solución:**
- **Dirección IP** (ej. 192.168.1.100)
- **Máscara de subred** (ej. 255.255.255.0)
- **Gateway predeterminado** (ej. 192.168.1.1)
- **Servidor DNS** (ej. 192.168.1.1, 8.8.8.8)
- **Tiempo de lease** (ej. 24 horas)
