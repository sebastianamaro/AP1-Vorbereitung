# IPv6-Adressierung [NEU 2025]

## Lernziele

Nach diesem Kapitel solltest du:
- Den Aufbau einer IPv6-Adresse verstehen
- Die Unterschiede zu IPv4 kennen
- IPv6-Adressen lesen und kürzen können
- Wichtige IPv6-Adresstypen kennen

## Kerninhalt

### Warum IPv6?

```
┌────────────────────────────────────────────────────────────────────┐
│                    WARUM IPv6?                                     │
├────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   Problem mit IPv4:                                                │
│   ┌─────────────────────────────────────────────────────────────┐ │
│   │ • Nur ca. 4,3 Milliarden Adressen (2^32)                   │ │
│   │ • Weltweite Geräte: Mehr als 20 Milliarden!                │ │
│   │ • IPv4-Adressen sind erschöpft (seit 2011)                 │ │
│   │ • Notlösungen wie NAT sind kompliziert                     │ │
│   └─────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   IPv6 löst das Problem:                                          │
│   ┌─────────────────────────────────────────────────────────────┐ │
│   │ • 128 Bit statt 32 Bit                                     │ │
│   │ • 2^128 = ca. 340 Sextillionen Adressen                   │ │
│   │ • Das reicht für jeden Sandkorn auf der Erde!              │ │
│   └─────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   Vergleich:                                                       │
│   ┌─────────────────────────────────────────────────────────────┐ │
│   │ IPv4:  4.294.967.296 Adressen            (4,3 Milliarden)  │ │
│   │ IPv6:  340.282.366.920.938.463.463.374.607.431.768.211.456 │ │
│   │                                           (340 Sextillionen)│ │
│   └─────────────────────────────────────────────────────────────┘ │
│                                                                     │
└────────────────────────────────────────────────────────────────────┘
```

### Aufbau einer IPv6-Adresse

```
┌────────────────────────────────────────────────────────────────────┐
│                    IPv6-ADRESSFORMAT                               │
├────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   Grundstruktur:                                                   │
│   ┌─────────────────────────────────────────────────────────────┐ │
│   │                                                              │ │
│   │   • 128 Bit lang                                            │ │
│   │   • 8 Gruppen zu je 16 Bit (4 Hexadezimalziffern)          │ │
│   │   • Gruppen getrennt durch Doppelpunkt (:)                  │ │
│   │   • Hexadezimale Darstellung (0-9, a-f)                     │ │
│   │                                                              │ │
│   └─────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   Beispiel:                                                        │
│   ┌─────────────────────────────────────────────────────────────┐ │
│   │                                                              │ │
│   │   2001:0db8:85a3:0000:0000:8a2e:0370:7334                   │ │
│   │   ────  ────  ────  ────  ────  ────  ────  ────            │ │
│   │    │     │     │     │     │     │     │     │              │ │
│   │    └─────┴─────┴─────┴─────┴─────┴─────┴─────┘              │ │
│   │             8 Gruppen à 4 Hex-Ziffern                       │ │
│   │                                                              │ │
│   │   Jede Gruppe = 16 Bit = 2 Byte                             │ │
│   │   8 Gruppen × 16 Bit = 128 Bit                              │ │
│   │                                                              │ │
│   └─────────────────────────────────────────────────────────────┘ │
│                                                                     │
└────────────────────────────────────────────────────────────────────┘
```

### IPv6-Adressen kürzen

```
┌────────────────────────────────────────────────────────────────────┐
│                  IPv6-KURZSCHREIBWEISE                             │
├────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   IPv6-Adressen können gekürzt werden nach zwei Regeln:           │
│                                                                     │
│   REGEL 1: Führende Nullen weglassen                              │
│   ┌────────────────────────────────────────────────────────────┐  │
│   │                                                             │  │
│   │   Vorher: 2001:0db8:0000:0042:0000:8a2e:0370:7334          │  │
│   │                 ↓    ↓    ↓         ↓    ↓                 │  │
│   │   Nachher: 2001:db8:0:42:0:8a2e:370:7334                   │  │
│   │                                                             │  │
│   │   • 0db8 → db8                                             │  │
│   │   • 0000 → 0 (mindestens eine Ziffer bleibt)               │  │
│   │   • 0042 → 42                                              │  │
│   │   • 0370 → 370                                             │  │
│   │                                                             │  │
│   └────────────────────────────────────────────────────────────┘  │
│                                                                     │
│   REGEL 2: Eine zusammenhängende Nullsequenz durch :: ersetzen   │
│   ┌────────────────────────────────────────────────────────────┐  │
│   │                                                             │  │
│   │   Vorher: 2001:db8:0:0:0:0:0:1                             │  │
│   │                  ↓ ↓ ↓ ↓ ↓                                 │  │
│   │   Nachher: 2001:db8::1                                     │  │
│   │                                                             │  │
│   │   ⚠ WICHTIG: :: darf nur EINMAL pro Adresse verwendet      │  │
│   │              werden!                                        │  │
│   │                                                             │  │
│   │   Beispiel Localhost:                                       │  │
│   │   0000:0000:0000:0000:0000:0000:0000:0001                  │  │
│   │   → ::1                                                    │  │
│   │                                                             │  │
│   └────────────────────────────────────────────────────────────┘  │
│                                                                     │
│   Vollständiges Beispiel:                                          │
│   ┌────────────────────────────────────────────────────────────┐  │
│   │                                                             │  │
│   │   Vollständig:  2001:0db8:0000:0000:0000:0000:0000:0001   │  │
│   │   Regel 1:      2001:db8:0:0:0:0:0:1                       │  │
│   │   Regel 1+2:    2001:db8::1                                │  │
│   │                                                             │  │
│   └────────────────────────────────────────────────────────────┘  │
│                                                                     │
└────────────────────────────────────────────────────────────────────┘
```

### IPv6-Adresstypen

```
┌────────────────────────────────────────────────────────────────────┐
│                    IPv6-ADRESSTYPEN                                │
├────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   UNICAST (Einzeladresse)                                         │
│   ┌────────────────────────────────────────────────────────────┐  │
│   │ • Adressiert genau EIN Gerät                               │  │
│   │ • Wie bei IPv4 die normale Adresse                         │  │
│   │                                                             │  │
│   │ Typen:                                                      │  │
│   │ • Global Unicast: Weltweit eindeutig (wie öffentl. IPv4)   │  │
│   │   Beginnt mit: 2000::/3 (2xxx: bis 3xxx:)                  │  │
│   │                                                             │  │
│   │ • Link-Local: Nur im lokalen Netzwerk gültig               │  │
│   │   Beginnt mit: fe80::/10                                   │  │
│   │   (Automatisch auf jedem Gerät vorhanden)                  │  │
│   │                                                             │  │
│   │ • Unique Local: Privat (wie private IPv4)                  │  │
│   │   Beginnt mit: fc00::/7 (fcxx: oder fdxx:)                 │  │
│   └────────────────────────────────────────────────────────────┘  │
│                                                                     │
│   MULTICAST (Gruppenadresse)                                      │
│   ┌────────────────────────────────────────────────────────────┐  │
│   │ • Adressiert eine GRUPPE von Geräten                       │  │
│   │ • Beginnt mit: ff00::/8                                    │  │
│   │ • Ersetzt Broadcast (gibt es bei IPv6 nicht!)              │  │
│   └────────────────────────────────────────────────────────────┘  │
│                                                                     │
│   ANYCAST                                                         │
│   ┌────────────────────────────────────────────────────────────┐  │
│   │ • Eine Adresse, mehrere mögliche Empfänger                 │  │
│   │ • Paket geht zum NÄCHSTEN Empfänger                       │  │
│   │ • Für Lastverteilung und Redundanz                         │  │
│   └────────────────────────────────────────────────────────────┘  │
│                                                                     │
│   ⚠ Kein BROADCAST bei IPv6! Wird durch Multicast ersetzt.      │
│                                                                     │
└────────────────────────────────────────────────────────────────────┘
```

### IPv4 vs. IPv6

```
┌────────────────────────────────────────────────────────────────────┐
│                    IPv4 vs. IPv6                                   │
├────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   Eigenschaft      │ IPv4              │ IPv6                      │
│   ─────────────────┼───────────────────┼──────────────────────────│
│   Adresslänge      │ 32 Bit            │ 128 Bit                  │
│   Notation         │ Dezimal (.)       │ Hexadezimal (:)          │
│   Beispiel         │ 192.168.1.1       │ 2001:db8::1              │
│   Adressanzahl     │ 4,3 Milliarden    │ 340 Sextillionen         │
│   Konfiguration    │ Manuell/DHCP      │ SLAAC/DHCPv6             │
│   Broadcast        │ Ja                │ Nein (Multicast)         │
│   NAT erforderlich │ Oft ja            │ Normalerweise nicht      │
│   Header           │ Variabel          │ Fest (40 Byte)           │
│   IPsec            │ Optional          │ Eingebaut                │
│   Checksum         │ Ja (im Header)    │ Nein                     │
│                                                                     │
│   Notation Beispiele:                                              │
│   ┌────────────────────────────────────────────────────────────┐  │
│   │ IPv4 Localhost: 127.0.0.1                                  │  │
│   │ IPv6 Localhost: ::1                                        │  │
│   │                                                             │  │
│   │ IPv4 Link-Local: 169.254.x.x                               │  │
│   │ IPv6 Link-Local: fe80::...                                 │  │
│   └────────────────────────────────────────────────────────────┘  │
│                                                                     │
└────────────────────────────────────────────────────────────────────┘
```

## Wichtige Begriffe

| Begriff | Erklärung |
|---------|-----------|
| **IPv6** | Internet Protocol Version 6 (128 Bit) |
| **Hexadezimal** | Zahlensystem mit Basis 16 (0-9, a-f) |
| **Unicast** | Adresse für ein einzelnes Gerät |
| **Multicast** | Adresse für eine Gruppe von Geräten |
| **Link-Local** | Nur im lokalen Netz gültig (fe80::) |
| **SLAAC** | Stateless Address Autoconfiguration |
| **::1** | IPv6-Loopback-Adresse |

## Prüfungstipps

### Häufige Prüfungsfragen

- Wie viele Bits hat eine IPv6-Adresse?
- Wie wird eine IPv6-Adresse verkürzt?
- Was ist der Unterschied zwischen IPv4 und IPv6?

### Wichtig zu merken

- **128 Bit** (4× so lang wie IPv4)
- **8 Gruppen** zu je 4 Hex-Ziffern, getrennt durch **:**
- **::1** = Localhost (wie 127.0.0.1)
- **fe80::** = Link-Local (automatisch auf jedem Gerät)
- **Kein Broadcast** bei IPv6! Nur Multicast
- **::** darf nur EINMAL pro Adresse vorkommen
- Führende Nullen weglassen: 0042 → 42

## Übungsaufgaben

### Aufgabe 1
Kürzen Sie die IPv6-Adresse: 2001:0db8:0000:0000:0000:0000:0000:0001

**Lösung:**
Schritt 1 - Führende Nullen entfernen:
2001:db8:0:0:0:0:0:1

Schritt 2 - Nullsequenz durch :: ersetzen:
**2001:db8::1**

### Aufgabe 2
Wie viele Bits hat eine IPv6-Adresse und wie viele Gruppen werden dargestellt?

**Lösung:**
- **128 Bit** Gesamtlänge
- **8 Gruppen** zu je 16 Bit (4 Hexadezimalziffern)

### Aufgabe 3
Nennen Sie zwei wesentliche Unterschiede zwischen IPv4 und IPv6.

**Lösung:**
1. **Adresslänge:** IPv4 = 32 Bit, IPv6 = 128 Bit
2. **Darstellung:** IPv4 = dezimal mit Punkten (192.168.1.1), IPv6 = hexadezimal mit Doppelpunkten (2001:db8::1)

Weitere Unterschiede:
- IPv6 hat kein Broadcast (nur Multicast)
- IPv6 benötigt normalerweise kein NAT
- IPv6 hat mehr verfügbare Adressen
