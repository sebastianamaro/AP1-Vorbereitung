# Software Development Life Cycle (SDLC)

## Lernziele

Nach diesem Kapitel solltest du:
- Die Phasen der Softwareentwicklung kennen
- Den SDLC verstehen und erklären können
- Zusammenhänge zwischen Phasen erkennen
- Wichtige Dokumente den Phasen zuordnen können

---

## Kerninhalt

### Was ist der SDLC?

Der **Software Development Life Cycle (SDLC)** beschreibt den gesamten Lebenszyklus einer Software von der Idee bis zur Außerbetriebnahme.

```
┌────────────────────────────────────────────────────────────────────┐
│                  SOFTWARE DEVELOPMENT LIFE CYCLE                    │
├────────────────────────────────────────────────────────────────────┤
│                                                                     │
│                         ┌─────────┐                                │
│                    ┌────│ Analyse │────┐                           │
│                    │    └─────────┘    │                           │
│                    │                   │                           │
│              ┌─────┴────┐        ┌─────┴────┐                      │
│              │Außerbetr.│        │  Entwurf │                      │
│              └─────┬────┘        └─────┬────┘                      │
│                    │                   │                           │
│              ┌─────┴────┐        ┌─────┴────┐                      │
│              │  Wartung │        │Implemen- │                      │
│              └─────┬────┘        │ tierung  │                      │
│                    │             └─────┬────┘                      │
│                    │    ┌─────────┐    │                           │
│                    └────│  Test   │────┘                           │
│                         └────┬────┘                                │
│                              │                                     │
│                         ┌────┴────┐                                │
│                         │Einführung│                               │
│                         └─────────┘                                │
│                                                                     │
└────────────────────────────────────────────────────────────────────┘
```

### Wartungsarten

```
┌────────────────────────────────────────────────────────────────────┐
│                       WARTUNGSARTEN                                │
├────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌─────────────────────────────────────────────────────────────┐ │
│   │ KORREKTIVE WARTUNG                                          │ │
│   │ • Beseitigung von Fehlern                                   │ │
│   │ • Bugfixes                                                  │ │
│   │ • Beispiel: Login funktioniert nicht mehr                   │ │
│   └─────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌─────────────────────────────────────────────────────────────┐ │
│   │ ADAPTIVE WARTUNG                                            │ │
│   │ • Anpassung an veränderte Umgebung                          │ │
│   │ • Neues Betriebssystem, neue Hardware                       │ │
│   │ • Beispiel: Migration auf Windows 11                        │ │
│   └─────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌─────────────────────────────────────────────────────────────┐ │
│   │ PERFEKTIVE WARTUNG                                          │ │
│   │ • Verbesserungen und Erweiterungen                          │ │
│   │ • Neue Funktionen                                           │ │
│   │ • Beispiel: Export-Funktion hinzufügen                      │ │
│   └─────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌─────────────────────────────────────────────────────────────┐ │
│   │ PRÄVENTIVE WARTUNG                                          │ │
│   │ • Vorbeugung zukünftiger Probleme                           │ │
│   │ • Code-Optimierung, Refactoring                             │ │
│   │ • Beispiel: Sicherheitslücken schließen                     │ │
│   └─────────────────────────────────────────────────────────────┘ │
│                                                                     │
└────────────────────────────────────────────────────────────────────┘
```

│   │                                                             │  │

│   │   ──→  │  │Analyse→Entwurf→Code→Test │ │  ──→ Inkrement   │  │
│   │        │  └──────────────────────────┘ │                   │  │

## Wichtige Begriffe

| Begriff | Erklärung |
|---------|-----------|
| **SDLC** | Software Development Life Cycle |
| **Lastenheft** | Beschreibt WAS der Auftraggeber will |
| **Pflichtenheft** | Beschreibt WIE der Auftragnehmer umsetzt |
| **Deployment** | Auslieferung/Installation der Software |
| **Release** | Veröffentlichte Softwareversion |
| **Patch** | Kleine Aktualisierung zur Fehlerbehebung |
| **Refactoring** | Code-Verbesserung ohne Funktionsänderung |

## Prüfungstipps

### Häufige Prüfungsfragen

- Nennen Sie die Phasen der Softwareentwicklung
- Welche Dokumente entstehen in welcher Phase?
- Unterscheiden Sie die Wartungsarten

### Wichtig zu merken

- **6 Phasen:** Analyse → Entwurf → Implementierung → Test → Einführung → Wartung
- **Lastenheft** = WAS (Auftraggeber)
- **Pflichtenheft** = WIE (Auftragnehmer)
- **4 Wartungsarten:** Korrektiv, Adaptiv, Perfektiv, Präventiv
- Testphase kommt VOR der Einführung!

## Übungsaufgaben

### Aufgabe 1
Ordnen Sie die Dokumente den SDLC-Phasen zu:
a) Benutzerhandbuch
b) Testprotokoll
c) Pflichtenheft
d) UML-Klassendiagramm

**Lösung:**
- a) Benutzerhandbuch → **Einführung**
- b) Testprotokoll → **Test**
- c) Pflichtenheft → **Analyse**
- d) UML-Klassendiagramm → **Entwurf**

### Aufgabe 2
Um welche Wartungsart handelt es sich?
- Die Software muss auf eine neue Datenbankversion angepasst werden.

**Lösung:**
**Adaptive Wartung** - Die Software wird an eine veränderte Umgebung (neue Datenbankversion) angepasst, ohne dass sich die Funktionalität ändert.

### Aufgabe 3
Erklären Sie den Unterschied zwischen Lastenheft und Pflichtenheft.

**Lösung:**
| Lastenheft | Pflichtenheft |
|------------|---------------|
| Vom Auftraggeber erstellt | Vom Auftragnehmer erstellt |
| Beschreibt WAS gewünscht wird | Beschreibt WIE umgesetzt wird |
| Fachliche Anforderungen | Technische Lösung |
| Grundlage für Angebote | Grundlage für Vertrag |
