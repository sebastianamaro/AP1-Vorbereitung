# Kommunikationsarten im IT-Support

## Lernziele

Nach diesem Abschnitt kannst du:
- Verschiedene Kommunikationskanäle im IT-Support unterscheiden
- Vor- und Nachteile jedes Kanals benennen
- Den passenden Kanal für verschiedene Situationen auswählen
- Grundregeln der professionellen Kommunikation anwenden

---

### Kanalwahl nach Situation

| Situation | Empfohlener Kanal | Begründung |
|-----------|-------------------|------------|
| **Totalausfall** | Telefon | Dringend, sofortige Reaktion nötig |
| **Passwort vergessen** | Self-Service | Standardprozess, automatisierbar |
| **Screenshot senden** | E-Mail/Ticket | Anhänge möglich |
| **Komplexes Problem** | Fernwartung + Telefon | Kombination ideal |
| **Statusabfrage** | Ticketsystem | User kann selbst nachsehen |
| **Schulungsfrage** | Knowledge Base | Wiederverwendbare Antwort |

## Wichtige Begriffe

| Begriff | Erklärung |
|---------|-----------|
| **Synchrone Kommunikation** | Gleichzeitiger Austausch (z.B. Telefon) |
| **Asynchrone Kommunikation** | Zeitversetzter Austausch (z.B. E-Mail) |
| **Fernwartung** | Remote-Zugriff auf den PC des Anwenders |
| **Knowledge Base** | Wissensdatenbank mit Lösungen |
| **FAQ** | Frequently Asked Questions (Häufige Fragen) |
| **Self-Service** | User löst Problem selbstständig |

---

## Prüfungstipps

1. **Kanalwahl begründen:**
   - Dringend → synchron (Telefon)
   - Dokumentation nötig → asynchron (E-Mail/Ticket)
   - Komplex → Kombination (Fernwartung + Telefon)

2. **Vor- und Nachteile kennen:**
   - Jeder Kanal hat spezifische Stärken und Schwächen
   - In der Prüfung oft: "Welcher Kanal ist geeignet und warum?"

3. **Professionelles Verhalten:**
   - Freundlichkeit auch bei schwierigen Usern
   - Keine Schuldzuweisungen

## Übungsaufgaben

### Aufgabe 1: Kanal auswählen

Wählen Sie den passenden Kommunikationskanal und begründen Sie:

a) Ein Mitarbeiter kann sich nicht mehr an seinem PC anmelden
b) Die Buchhaltung benötigt einen Bericht über die IT-Kosten des letzten Quartals
c) Ein neuer Mitarbeiter hat Fragen zur Einrichtung seines E-Mail-Clients

<details>
<summary>Lösung anzeigen</summary>

a) **Telefon + Fernwartung**
   - Dringend (Mitarbeiter kann nicht arbeiten)
   - Direktes Eingreifen möglich
   - Kann verschiedene Ursachen haben (Passwort, Konto gesperrt, AD-Problem)

b) **E-Mail**
   - Nicht dringend
   - Bericht als Anhang möglich
   - Dokumentation wichtig

c) **Self-Service/Knowledge Base** oder **E-Mail mit Anleitung**
   - Standardprozess
   - Anleitung kann wiederverwendet werden
   - Kein zeitkritisches Problem

</details>

### Aufgabe 2: E-Mail verfassen

Ein Anwender hat per E-Mail gemeldet: "Mein PC ist langsam!!"

Formulieren Sie eine professionelle Antwort-E-Mail, in der Sie um weitere Informationen bitten.

<details>
<summary>Lösung anzeigen</summary>

**Beispiel-Antwort:**

```
Betreff: Re: Ihr PC ist langsam - Ticket #2024-0815

Sehr geehrter Herr Müller,

vielen Dank für Ihre Nachricht.

Um Ihr Problem schnellstmöglich lösen zu können, benötige ich
noch einige Informationen:

1. Seit wann ist der PC langsam?
2. Bei welchen Programmen tritt das Problem besonders auf?
3. Haben Sie kürzlich etwas installiert oder verändert?
4. Wie äußert sich das Problem genau?
   (z.B. lange Startzeit, Programme reagieren nicht)

In der Zwischenzeit können Sie folgende Maßnahmen versuchen:
- PC neu starten
- Nicht benötigte Programme schließen

Ich melde mich sobald ich mehr Informationen habe.

Mit freundlichen Grüßen
Tim Schmidt
IT-Support, ConSystem GmbH
Tel: 0800-1234567
Ticket: #2024-0815
```

</details>

### Aufgabe 3: Kommunikationsfehler erkennen

Was ist an folgender Aussage eines IT-Supporters falsch?

"Das haben Sie falsch gemacht. Steht doch alles im Handbuch. Wer liest denn sowas nicht?"

<details>
<summary>Lösung anzeigen</summary>

**Fehler:**

1. **Schuldzuweisung** - "Das haben Sie falsch gemacht"
   → Besser: "Das ist ein häufiger Fehler, den wir schnell beheben können"

2. **Vorwurf** - "Steht doch alles im Handbuch"
   → Besser: "Im Handbuch auf Seite X finden Sie eine Anleitung dazu"

3. **Herablassend** - "Wer liest denn sowas nicht?"
   → Besser: Einfach die Lösung zeigen ohne Kommentar

**Professionelle Alternative:**
"Ich verstehe das Problem. Lassen Sie mich Ihnen kurz zeigen, wie es funktioniert. Falls Sie später nochmal nachschauen möchten, finden Sie eine Anleitung auch im Handbuch auf Seite 15."

</details>
