# Das Vier-Ohren-Modell (Schulz von Thun)

## Lernziele

Nach diesem Abschnitt kannst du:
- Die vier Seiten einer Nachricht erklären
- Das Modell auf IT-Support-Situationen anwenden
- Missverständnisse erkennen und vermeiden
- Bewusster kommunizieren

---

## Kerninhalt

### Grundidee des Modells

Jede Nachricht hat **vier Seiten** (oder "Ohren"), auf denen sie gesendet und empfangen werden kann.

```
┌─────────────────────────────────────────────────────────────────────┐
│           DAS VIER-OHREN-MODELL                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│                         SACHINHALT                                   │
│                        (Worüber ich informiere)                      │
│                              │                                       │
│                              │                                       │
│                              ▼                                       │
│                        ┌──────────┐                                 │
│   SELBSTKUNDGABE       │          │        APPELL                   │
│   (Was ich von mir  ──►│ NACHRICHT│◄── (Wozu ich dich              │
│    preisgebe)          │          │     bewegen möchte)             │
│                        └──────────┘                                 │
│                              │                                       │
│                              │                                       │
│                              ▼                                       │
│                         BEZIEHUNG                                    │
│                    (Was ich von dir halte)                          │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Die vier Seiten einer Nachricht

```
┌─────────────────────────────────────────────────────────────────────┐
│           DIE VIER SEITEN IM DETAIL                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  1. SACHINHALT (Sachohr)                                            │
│     ───────────────────────────────────────────────────────────     │
│     → Die reine Information                                         │
│     → Fakten, Daten, Sachverhalte                                   │
│     → "Worüber wird gesprochen?"                                    │
│                                                                      │
│  2. SELBSTKUNDGABE (Selbstkundgabe-Ohr)                            │
│     ───────────────────────────────────────────────────────────     │
│     → Was der Sender über sich selbst verrät                        │
│     → Gefühle, Meinungen, Bedürfnisse                               │
│     → "Was sagt das über den Sender aus?"                           │
│                                                                      │
│  3. BEZIEHUNG (Beziehungsohr)                                       │
│     ───────────────────────────────────────────────────────────     │
│     → Wie der Sender zum Empfänger steht                            │
│     → Wertschätzung, Ablehnung, Respekt                             │
│     → "Wie redet der mit mir?"                                      │
│                                                                      │
│  4. APPELL (Appellohr)                                              │
│     ───────────────────────────────────────────────────────────     │
│     → Was der Sender vom Empfänger möchte                           │
│     → Aufforderung, Wunsch, Bitte                                   │
│     → "Was soll ich tun/denken/fühlen?"                             │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Beispiel: "Mein PC ist wieder kaputt!"

```
┌─────────────────────────────────────────────────────────────────────┐
│           BEISPIEL-ANALYSE                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  AUSSAGE: "Mein PC ist wieder kaputt!"                              │
│                                                                      │
│                         SACHINHALT                                   │
│                    "Der PC funktioniert nicht"                       │
│                              │                                       │
│                              │                                       │
│                              ▼                                       │
│                        ┌──────────┐                                 │
│   SELBSTKUNDGABE       │  "Mein   │        APPELL                   │
│  "Ich bin frustriert"──│   PC     │──"Repariere ihn!"              │
│  "Das nervt mich"      │  ist     │  "Hilf mir!"                    │
│  "Ich bin gestresst"   │ wieder   │  "Mach es diesmal              │
│                        │ kaputt!" │   richtig!"                     │
│                        └──────────┘                                 │
│                              │                                       │
│                              │                                       │
│                              ▼                                       │
│                         BEZIEHUNG                                    │
│                "Die IT macht schlechte Arbeit"                       │
│                "Das ist schon wieder passiert"                       │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Kommunikation im IT-Support analysieren

| Aussage des Users | Sachinhalt | Selbstkundgabe | Beziehung | Appell |
|-------------------|------------|----------------|-----------|--------|
| "Das Internet ist langsam" | Netzwerk hat geringe Geschwindigkeit | Ich bin ungeduldig | Erwartet schnelle Lösung | Mach es schneller! |
| "Gestern ging es noch" | Etwas hat sich verändert | Ich habe nichts gemacht | Es ist nicht meine Schuld | Finde heraus was passiert ist |
| "Ich muss dringend arbeiten" | Es ist zeitkritisch | Ich bin unter Druck | Nimm mich ernst | Bearbeite mein Problem zuerst |
| "Das versteht doch keiner" | Software ist kompliziert | Ich bin überfordert | Erwarte bessere Erklärung | Erkläre es mir einfacher |

### Typische Missverständnisse

```
┌─────────────────────────────────────────────────────────────────────┐
│           MISSVERSTÄNDNISSE VERMEIDEN                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  PROBLEM: Sender und Empfänger nutzen verschiedene "Ohren"         │
│                                                                      │
│  BEISPIEL:                                                          │
│  ──────────────────────────────────────────────────────────────     │
│  User: "Das Programm ist total kompliziert!"                        │
│                                                                      │
│  Mögliche Interpretationen des IT-Supports:                        │
│                                                                      │
│  ┌────────────────────────────────────────────────────────────┐     │
│  │ SACHOHR:      "Die Software hat viele Funktionen"          │     │
│  │               → Antwort: "Ja, das stimmt"                  │     │
│  │                                                            │     │
│  │ BEZIEHUNGSOHR:"Du hast mir das schlecht erklärt"          │     │
│  │               → Antwort (defensiv): "Ich habe es          │     │
│  │                 Ihnen doch gezeigt!"                       │     │
│  │                                                            │     │
│  │ APPELLOHR:    "Hilf mir bitte!"                           │     │
│  │               → Antwort: "Ich zeige es Ihnen nochmal"     │ ✓   │
│  │                                                            │     │
│  │ SELBSTKUNDGABE-OHR: "Er ist überfordert"                  │     │
│  │               → Antwort: "Ich verstehe, dass es           │     │
│  │                 am Anfang verwirrend sein kann"           │ ✓   │
│  └────────────────────────────────────────────────────────────┘     │
│                                                                      │
│  EMPFEHLUNG: Mit dem Appell- oder Selbstkundgabe-Ohr hören!        │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Empfehlungen für den IT-Support

```
┌─────────────────────────────────────────────────────────────────────┐
│           KOMMUNIKATIONSTIPPS                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  1. AUF DEM RICHTIGEN OHR HÖREN                                     │
│     ───────────────────────────────────────────────────────────     │
│     → Nicht alles auf der Beziehungsebene nehmen                   │
│     → Den Appell hinter der Aussage erkennen                       │
│     → Die Selbstkundgabe (Frustration) wahrnehmen                  │
│                                                                      │
│  2. BEWUSST AUF ALLEN EBENEN SENDEN                                 │
│     ───────────────────────────────────────────────────────────     │
│     → Sachlich klar informieren                                     │
│     → Verständnis zeigen (Selbstkundgabe würdigen)                 │
│     → Wertschätzend kommunizieren (Beziehung positiv)              │
│     → Klare Handlungsoptionen geben (Appell)                       │
│                                                                      │
│  3. BEISPIEL FÜR GUTE ANTWORT                                       │
│     ───────────────────────────────────────────────────────────     │
│     User: "Das Programm ist total kompliziert!"                     │
│                                                                      │
│     Antwort: "Ich verstehe, dass es am Anfang viel auf             │
│     einmal ist [Selbstkundgabe würdigen]. Das Programm             │
│     hat tatsächlich viele Funktionen [Sachinhalt].                 │
│     Ich zeige Ihnen gerne die wichtigsten Schritte                 │
│     nochmal [Appell beantworten]. Wann passt es Ihnen?             │
│     [Beziehung positiv: User entscheidet]"                         │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Ich-Botschaften verwenden

```
┌─────────────────────────────────────────────────────────────────────┐
│           ICH-BOTSCHAFTEN vs. DU-BOTSCHAFTEN                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  DU-BOTSCHAFT (VERMEIDEN):                                          │
│  ──────────────────────────────────────────────────────────────     │
│  → "Sie haben das falsch gemacht"                                   │
│  → "Das haben Sie vergessen"                                        │
│  → "Sie müssen besser aufpassen"                                    │
│  → Erzeugt Abwehrreaktion                                          │
│                                                                      │
│  ICH-BOTSCHAFT (EMPFOHLEN):                                         │
│  ──────────────────────────────────────────────────────────────     │
│  → "Ich sehe, dass hier ein Fehler aufgetreten ist"                │
│  → "Mir ist aufgefallen, dass dieser Schritt fehlt"                │
│  → "Ich empfehle, in Zukunft darauf zu achten"                     │
│  → Vermeidet Schuldzuweisung                                       │
│                                                                      │
│  AUFBAU EINER ICH-BOTSCHAFT:                                        │
│  ──────────────────────────────────────────────────────────────     │
│  1. Beobachtung: "Ich sehe, dass..."                               │
│  2. Gefühl/Reaktion: "Das führt dazu, dass..."                     │
│  3. Wunsch: "Ich würde mir wünschen, dass..."                      │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

---

## Wichtige Begriffe

| Begriff | Erklärung |
|---------|-----------|
| **Sachinhalt** | Die reine Sachinformation einer Nachricht |
| **Selbstkundgabe** | Was der Sender über sich selbst mitteilt |
| **Beziehungsebene** | Wie der Sender zum Empfänger steht |
| **Appell** | Was der Sender vom Empfänger möchte |
| **Ich-Botschaft** | Aussage ohne Schuldzuweisung |
| **Aktives Zuhören** | Bewusstes, aufmerksames Zuhören |

---

## Prüfungstipps

1. **Vier Ebenen merken:**
   - **S**achinhalt
   - **S**elbstkundgabe
   - **B**eziehung
   - **A**ppell
   - Merkhilfe: **SSBA** oder "Sach-Selbst-Beziehung-Appell"

2. **Im IT-Kontext anwenden:**
   - Oft kommt Frustration (Selbstkundgabe) und der Wunsch nach Hilfe (Appell) zusammen
   - Nicht persönlich nehmen (Beziehungsebene)

3. **Typische Prüfungsfragen:**
   - "Analysieren Sie die Aussage nach dem Vier-Ohren-Modell"
   - "Wie sollte der IT-Support reagieren?"

## Übungsaufgaben

### Aufgabe 1: Nachricht analysieren

Analysieren Sie die Aussage eines Users nach dem Vier-Ohren-Modell:

**"Ich habe schon dreimal angerufen und das Problem ist immer noch nicht gelöst!"**

<details>
<summary>Lösung anzeigen</summary>

| Ebene | Interpretation |
|-------|----------------|
| **Sachinhalt** | Das Problem besteht trotz dreier Anrufe weiterhin |
| **Selbstkundgabe** | "Ich bin frustriert und verärgert" / "Ich fühle mich nicht ernst genommen" |
| **Beziehung** | "Der IT-Support arbeitet nicht zuverlässig" / "Ihr nehmt mein Problem nicht ernst" |
| **Appell** | "Löst das Problem endlich!" / "Behandelt meinen Fall mit Priorität!" |

**Empfohlene Reaktion:**
- Verständnis zeigen für die Frustration
- Nicht defensiv werden
- Ticket-Historie prüfen
- Konkreten Lösungsansatz anbieten
- Verbindliche Aussage treffen

</details>

### Aufgabe 2: Antwort formulieren

Ein User sagt: "Das ist doch total umständlich, wie soll ich so arbeiten?"

Formulieren Sie eine professionelle Antwort, die alle vier Ebenen berücksichtigt.

<details>
<summary>Lösung anzeigen</summary>

**Professionelle Antwort:**

"Ich verstehe, dass der aktuelle Workflow umständlich erscheint und Sie Zeit kostet **[Selbstkundgabe würdigen]**.

Lassen Sie uns gemeinsam schauen, welche Schritte genau problematisch sind **[Sachebene klären]**.

Ich möchte Ihnen gerne helfen, eine bessere Lösung zu finden **[Beziehung positiv]**.

Können Sie mir zeigen, wo genau es hakt? Dann finden wir zusammen einen Weg **[Appell beantworten]**."

</details>

### Aufgabe 3: Ich-Botschaften umformulieren

Formulieren Sie folgende Du-Botschaften in Ich-Botschaften um:

a) "Sie haben das Passwort falsch eingegeben."
b) "Das hätten Sie mir früher sagen müssen!"
c) "Sie verstehen das nicht richtig."

<details>
<summary>Lösung anzeigen</summary>

a) **"Sie haben das Passwort falsch eingegeben."**
   → "Ich sehe, dass die Anmeldung nicht funktioniert hat. Das kann passieren, wenn das Passwort nicht korrekt eingegeben wird. Lassen Sie uns das gemeinsam prüfen."

b) **"Das hätten Sie mir früher sagen müssen!"**
   → "Mir ist aufgefallen, dass mir diese Information bisher fehlte. Mit dieser Info kann ich das Problem schneller lösen. Für die Zukunft: Je mehr Details Sie mir gleich geben, desto schneller finde ich die Lösung."

c) **"Sie verstehen das nicht richtig."**
   → "Ich merke, dass ich das vielleicht nicht klar genug erklärt habe. Lassen Sie mich das nochmal anders erklären."

</details>
