# Service Level (Support-Stufen)

## Lernziele

Nach diesem Abschnitt kannst du:
- Die drei Support-Level unterscheiden
- Aufgaben und Kompetenzen jeder Stufe erklären
- Den Eskalationsprozess beschreiben
- Die Bedeutung von Eskalation im IT-Support erläutern

---

## Kerninhalt

### Das 3-Level-Support-Modell

```
┌─────────────────────────────────────────────────────────────────────┐
│                    SUPPORT-LEVEL                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│                       ┌─────────────────────┐                       │
│                       │    3RD LEVEL        │  Experten             │
│                       │    Spezialisten     │  Entwickler           │
│                       │    Hersteller       │  Tiefe Analyse        │
│                       └──────────┬──────────┘                       │
│                                  │ Eskalation                       │
│                       ┌──────────┴──────────┐                       │
│                       │    2ND LEVEL        │  Fachspezialisten     │
│                       │    Experten         │  Komplexe Probleme    │
│                       │                     │  Administration       │
│                       └──────────┬──────────┘                       │
│                                  │ Eskalation                       │
│                       ┌──────────┴──────────┐                       │
│                       │    1ST LEVEL        │  Erster Kontakt       │
│                       │    Helpdesk         │  Standardprobleme     │
│                       │    Service Desk     │  Ticket-Erstellung    │
│                       └──────────┬──────────┘                       │
│                                  │                                  │
│                       ┌──────────┴──────────┐                       │
│                       │      ANWENDER       │                       │
│                       └─────────────────────┘                       │
│                                                                      │
│  Lösung: Je niedriger das Level, desto besser (schneller/günstiger) │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### 1st Level Support (Helpdesk/Service Desk)

```
┌─────────────────────────────────────────────────────────────────────┐
│                    1ST LEVEL SUPPORT                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  AUFGABEN:                                                          │
│  ──────────────────────────────────────────────────────────────     │
│  → Erster Ansprechpartner für alle Anfragen                         │
│  → Annahme und Dokumentation (Ticket erstellen)                     │
│  → Kategorisierung und Priorisierung                                │
│  → Lösung von Standardproblemen                                     │
│  → Weiterleitung komplexer Fälle                                    │
│                                                                      │
│  TYPISCHE LÖSUNGEN:                                                 │
│  ──────────────────────────────────────────────────────────────     │
│  → Passwort zurücksetzen                                            │
│  → PC neu starten                                                   │
│  → Standard-Software-Probleme                                       │
│  → Drucker-Probleme (Papier, Toner)                                 │
│  → Anleitungen geben (bekannte Lösungen)                            │
│                                                                      │
│  WERKZEUGE:                                                         │
│  ──────────────────────────────────────────────────────────────     │
│  → Ticketsystem                                                     │
│  → Wissensdatenbank (Knowledge Base)                                │
│  → Remote-Zugriff (Fernwartung)                                     │
│  → FAQ und Anleitungen                                              │
│                                                                      │
│  ZIEL: Möglichst viele Anfragen SOFORT lösen (First Call Resolution)│
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### 2nd Level Support (Fachspezialisten)

```
┌─────────────────────────────────────────────────────────────────────┐
│                    2ND LEVEL SUPPORT                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  AUFGABEN:                                                          │
│  ──────────────────────────────────────────────────────────────     │
│  → Bearbeitung eskalierter Tickets aus 1st Level                    │
│  → Tiefere technische Analyse                                       │
│  → Administration von Systemen                                      │
│  → Erstellung von Workarounds                                       │
│  → Dokumentation von Lösungen für 1st Level                         │
│                                                                      │
│  TYPISCHE PROBLEME:                                                 │
│  ──────────────────────────────────────────────────────────────     │
│  → Komplexe Software-Fehler                                         │
│  → Netzwerk-Probleme                                                │
│  → Server-Administration                                            │
│  → Berechtigungsprobleme                                            │
│  → Performance-Probleme                                             │
│                                                                      │
│  SPEZIALGEBIETE (Teams):                                            │
│  ──────────────────────────────────────────────────────────────     │
│  → Netzwerk-Team                                                    │
│  → Server-Team                                                      │
│  → Datenbank-Team                                                   │
│  → Applikations-Team                                                │
│                                                                      │
│  ZIEL: Komplexe Probleme lösen, die 1st Level nicht kann            │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### 3rd Level Support (Experten/Hersteller)

```
┌─────────────────────────────────────────────────────────────────────┐
│                    3RD LEVEL SUPPORT                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  AUFGABEN:                                                          │
│  ──────────────────────────────────────────────────────────────     │
│  → Lösung von hochkomplexen Problemen                               │
│  → Fehleranalyse auf Code-/System-Ebene                             │
│  → Entwicklung von Patches/Fixes                                    │
│  → Architektur-Entscheidungen                                       │
│  → Kontakt zum Hersteller                                           │
│                                                                      │
│  WER IST 3RD LEVEL?                                                 │
│  ──────────────────────────────────────────────────────────────     │
│  → Interne Entwickler                                               │
│  → Interne Systemarchitekten                                        │
│  → Hersteller-Support (Microsoft, SAP, etc.)                        │
│  → Externe Berater/Spezialisten                                     │
│                                                                      │
│  TYPISCHE PROBLEME:                                                 │
│  ──────────────────────────────────────────────────────────────     │
│  → Software-Bugs                                                    │
│  → Grundlegende Systemfehler                                        │
│  → Kompatibilitätsprobleme                                          │
│  → Sicherheitslücken                                                │
│                                                                      │
│  ZIEL: Probleme lösen, die niemand anderes lösen kann               │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Vergleich der Support-Level

| Kriterium | 1st Level | 2nd Level | 3rd Level |
|-----------|-----------|-----------|-----------|
| **Kontakt** | Direkt mit User | Ticket | Ticket/Projekt |
| **Probleme** | Standard | Komplex | Hochkomplex |
| **Lösungszeit** | Minuten | Stunden-Tage | Tage-Wochen |
| **Kosten** | Niedrig | Mittel | Hoch |
| **Qualifikation** | Grundkenntnisse | Fachexperten | Spezialisten |
| **Anteil Tickets** | ~70-80% | ~15-25% | ~5% |

### Eskalationsprozess

```
┌─────────────────────────────────────────────────────────────────────┐
│                    ESKALATION                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  FUNKTIONALE ESKALATION (horizontal)                                │
│  ──────────────────────────────────────────────────────────────     │
│  → Weiterleitung an zuständiges Team                                │
│  → Z.B. von PC-Support an Netzwerk-Team                             │
│                                                                      │
│  HIERARCHISCHE ESKALATION (vertikal)                                │
│  ──────────────────────────────────────────────────────────────     │
│  → Weiterleitung an höheres Level                                   │
│  → Z.B. von 1st Level an 2nd Level                                  │
│  → Oder: Einschaltung von Vorgesetzten                              │
│                                                                      │
│  WANN ESKALIEREN?                                                   │
│  ──────────────────────────────────────────────────────────────     │
│  → Problem übersteigt eigene Kompetenz                              │
│  → SLA-Frist droht zu verfehlen                                     │
│  → Kunde ist unzufrieden/eskaliert selbst                           │
│  → Kritisches System betroffen                                      │
│                                                                      │
│  WICHTIG:                                                           │
│  → Dokumentation im Ticket (was wurde versucht?)                    │
│  → Übergabe aller relevanten Informationen                          │
│  → Nicht "blind" eskalieren                                         │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Single Point of Contact (SPOC)

```
┌─────────────────────────────────────────────────────────────────────┐
│                    SPOC-PRINZIP                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  Der Anwender hat EINEN Ansprechpartner: 1st Level                  │
│                                                                      │
│                    ┌─────────────────┐                              │
│                    │    ANWENDER     │                              │
│                    └────────┬────────┘                              │
│                             │                                       │
│                             ▼                                       │
│                    ┌─────────────────┐                              │
│                    │   1ST LEVEL     │ ← Single Point of Contact    │
│                    │   (SPOC)        │                              │
│                    └────────┬────────┘                              │
│                             │                                       │
│            ┌────────────────┼────────────────┐                      │
│            ▼                ▼                ▼                      │
│      ┌──────────┐    ┌──────────┐    ┌──────────┐                   │
│      │ 2nd Level│    │ 2nd Level│    │ 3rd Level│                   │
│      │ Netzwerk │    │ Server   │    │ Entwickl.│                   │
│      └──────────┘    └──────────┘    └──────────┘                   │
│                                                                      │
│  Vorteile:                                                          │
│  → Anwender muss nicht wissen, wer zuständig ist                    │
│  → Keine "Irrläufer"                                                │
│  → Zentrale Dokumentation                                           │
│  → Konsistente Kommunikation                                        │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

---

## Wichtige Begriffe

| Begriff | Erklärung |
|---------|-----------|
| **1st Level** | Erster Ansprechpartner, Standardprobleme |
| **2nd Level** | Fachexperten, komplexe Probleme |
| **3rd Level** | Spezialisten, Entwickler, Hersteller |
| **Eskalation** | Weiterleitung an andere Ebene/Team |
| **SPOC** | Single Point of Contact |
| **First Call Resolution** | Lösung beim ersten Kontakt |
| **Knowledge Base** | Wissensdatenbank für bekannte Lösungen |

---

## Prüfungstipps

1. **Level-Unterscheidung:**
   - 1st = Standard, schnell, günstig
   - 2nd = Komplex, Experten
   - 3rd = Sehr komplex, Entwickler/Hersteller

2. **First Call Resolution:**
   - Je mehr im 1st Level gelöst wird, desto besser!
   - Ziel: >70%

3. **SPOC-Prinzip:**
   - User hat EINEN Ansprechpartner
   - Interne Weiterleitung ist transparent

## Übungsaufgaben

### Aufgabe 1: Level zuordnen

Welches Support-Level ist für folgende Probleme zuständig?

a) Passwort vergessen
b) Server-Festplatte ist defekt
c) Bug in der selbst entwickelten ERP-Software
d) Druckertreiber funktioniert nicht nach Windows-Update
e) Microsoft Office lässt sich nicht aktivieren

<details>
<summary>Lösung anzeigen</summary>

a) **1st Level** - Standardvorgang (Passwort zurücksetzen)
b) **2nd Level** - Hardware-Austausch am Server
c) **3rd Level** - Eigenentwicklung, Entwickler nötig
d) **1st/2nd Level** - Treiber-Neuinstallation (1st), bei komplexen Problemen 2nd
e) **1st Level** - Standard-Aktivierung, ggf. Microsoft-Support (3rd)

</details>

### Aufgabe 2: Eskalationsentscheidung

Ein Mitarbeiter meldet: "Mein PC ist sehr langsam." Der 1st-Level-Support hat bereits:
- PC neu gestartet
- Nicht benötigte Programme geschlossen
- Virenscanner laufen lassen (keine Funde)
- Festplatte auf Fehler geprüft (OK)

Das Problem besteht weiterhin. Was ist der nächste Schritt?

<details>
<summary>Lösung anzeigen</summary>

**Eskalation an 2nd Level empfohlen**

Begründung:
- Standard-Maßnahmen wurden durchgeführt
- Problem besteht weiterhin
- Tiefere Analyse erforderlich (z.B. Ressourcenauslastung, Netzwerk, Hintergrundprozesse)

**Bei Eskalation dokumentieren:**
- Alle durchgeführten Maßnahmen
- Symptome genau beschreiben
- Relevante Systeminformationen (RAM, CPU-Auslastung)

**2nd Level könnte prüfen:**
- Detaillierte Prozessanalyse
- Netzwerk-Performance
- Hardware-Diagnose
- Softwarekonflikte

</details>

### Aufgabe 3: SPOC-Prinzip

Ein Anwender hat ein komplexes Problem, das bereits an 2nd Level eskaliert wurde. Er ruft erneut beim 1st Level an und fragt nach dem Status.

Wie sollte der 1st-Level-Mitarbeiter reagieren?

<details>
<summary>Lösung anzeigen</summary>

**Richtige Reaktion:**

1. **Im Ticketsystem nachschauen:**
   - Aktuellen Status prüfen
   - Letzte Aktivitäten lesen
   - Bearbeiter identifizieren

2. **Dem Anwender mitteilen:**
   - "Ihr Ticket wird aktuell vom 2nd-Level-Team bearbeitet."
   - Status erklären (z.B. "Es wird an einer Lösung gearbeitet")
   - Voraussichtliche Dauer nennen (wenn bekannt)

3. **NICHT:**
   - Den Anwender an 2nd Level verweisen ("Rufen Sie dort an")
   - Behaupten, nichts zu wissen
   - Zusagen machen, die nicht gehalten werden können

**Das SPOC-Prinzip bedeutet:**
Der 1st Level bleibt Ansprechpartner, auch wenn andere an der Lösung arbeiten. Er fungiert als Kommunikationsschnittstelle zum Anwender.

</details>
