# Vertragsarten

## Lernziele

Nach diesem Abschnitt kannst du:
- Die wichtigsten Vertragsarten im IT-Bereich unterscheiden
- Merkmale von Kauf-, Werk- und Dienstvertrag erklären
- Die richtige Vertragsart für verschiedene IT-Leistungen zuordnen
- Unterschiede bei Haftung und Erfolgsverantwortung benennen

---

## Kerninhalt

### Wichtigste Vertragsarten

```
┌─────────────────────────────────────────────────────────────────────┐
│                    VERTRAGSARTEN IM ÜBERBLICK                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │                      KAUFVERTRAG                            │    │
│  │  ──────────────────────────────────────────────────────────│    │
│  │  → Übertragung von Eigentum gegen Geld                      │    │
│  │  → Erfolg: Übergabe der Sache                               │    │
│  │                                                              │    │
│  │  IT-Beispiel:                                                │    │
│  │  → Kauf von Hardware (PC, Server)                           │    │
│  │  → Kauf von Softwarelicenses                                │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                                                                      │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │                      WERKVERTRAG                            │    │
│  │  ──────────────────────────────────────────────────────────│    │
│  │  → Herstellung eines WERKS (Erfolg geschuldet)              │    │
│  │  → Erfolg: Funktionierendes Ergebnis                        │    │
│  │                                                              │    │
│  │  IT-Beispiel:                                                │    │
│  │  → Entwicklung einer Individualsoftware                     │    │
│  │  → Installation eines funktionierenden Netzwerks            │    │
│  │  → Erstellung einer Website                                 │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                                                                      │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │                     DIENSTVERTRAG                           │    │
│  │  ──────────────────────────────────────────────────────────│    │
│  │  → Erbringung einer Tätigkeit (KEIN Erfolg geschuldet)     │    │
│  │  → Erfolg: Nur Bemühen geschuldet                           │    │
│  │                                                              │    │
│  │  IT-Beispiel:                                                │    │
│  │  → IT-Beratung (Stundenbasis)                               │    │
│  │  → Arbeitnehmerüberlassung                                  │    │
│  │  → Support-Hotline                                          │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Vergleich der Vertragsarten

| Kriterium | Kaufvertrag | Werkvertrag | Dienstvertrag |
|-----------|-------------|-------------|---------------|
| **Gegenstand** | Sache/Recht | Werk (Ergebnis) | Tätigkeit |
| **Erfolg** | Übergabe | Fertigstellung | Nur Bemühen |
| **Vergütung** | Kaufpreis | Werklohn | Vergütung/Stunde |
| **Mängelrechte** | Ja (Gewährleistung) | Ja (Nachbesserung) | Eingeschränkt |
| **Abnahme** | Übergabe | Abnahmepflicht | Keine |
| **Kündigung** | Rücktritt | Jederzeit möglich | Kündigungsfrist |
| **BGB** | §§ 433-480 | §§ 631-651 | §§ 611-630 |

### Der Kaufvertrag

```
┌─────────────────────────────────────────────────────────────────────┐
│                    KAUFVERTRAG (§§ 433 ff. BGB)                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  VERKÄUFER                                     KÄUFER               │
│  ┌─────────────┐                           ┌─────────────┐          │
│  │             │  Pflicht: Übergabe        │             │          │
│  │  Verkäufer  │ ────────────────────────► │   Käufer    │          │
│  │             │  der Sache                │             │          │
│  │             │                           │             │          │
│  │             │  Pflicht: Kaufpreis       │             │          │
│  │             │ ◄──────────────────────── │             │          │
│  │             │  zahlen                   │             │          │
│  └─────────────┘                           └─────────────┘          │
│                                                                      │
│  Pflichten des Verkäufers:                                          │
│  → Übergabe der Sache                                               │
│  → Eigentumsverschaffung                                            │
│  → Sache frei von Sach- und Rechtsmängeln                           │
│                                                                      │
│  Pflichten des Käufers:                                             │
│  → Kaufpreis zahlen                                                 │
│  → Sache abnehmen                                                   │
│                                                                      │
│  IT-Beispiele:                                                      │
│  → Hardware-Kauf (PC, Server, Drucker)                              │
│  → Software-Lizenzkauf (einmalige Zahlung)                          │
│  → Kauf von Verbrauchsmaterial                                      │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Der Werkvertrag

```
┌─────────────────────────────────────────────────────────────────────┐
│                    WERKVERTRAG (§§ 631 ff. BGB)                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  AUFTRAGNEHMER                                 AUFTRAGGEBER         │
│  (Unternehmer)                                 (Besteller)          │
│  ┌─────────────┐                           ┌─────────────┐          │
│  │             │  Pflicht: Herstellung     │             │          │
│  │ Auftragnehmer│ ────────────────────────►│ Auftraggeber│          │
│  │             │  eines Werks (ERFOLG!)    │             │          │
│  │             │                           │             │          │
│  │             │  Pflicht: Werklohn        │             │          │
│  │             │ ◄──────────────────────── │             │          │
│  │             │  nach Abnahme zahlen      │             │          │
│  └─────────────┘                           └─────────────┘          │
│                                                                      │
│  WICHTIG: Erfolg ist geschuldet!                                    │
│  → Das Werk muss FUNKTIONIEREN                                      │
│  → Nicht nur Bemühen, sondern Ergebnis                              │
│                                                                      │
│  Abnahme:                                                           │
│  → Besteller MUSS das Werk abnehmen (wenn vertragsgemäß)            │
│  → Nach Abnahme: Werklohn fällig, Gewährleistung beginnt            │
│                                                                      │
│  IT-Beispiele:                                                      │
│  → Individuelle Software-Entwicklung                                │
│  → Netzwerk-Installation                                            │
│  → Website-Erstellung                                               │
│  → Datenbank-Migration                                              │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Der Dienstvertrag

```
┌─────────────────────────────────────────────────────────────────────┐
│                    DIENSTVERTRAG (§§ 611 ff. BGB)                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  DIENSTVERPFLICHTETER                      DIENSTBERECHTIGTER       │
│  ┌─────────────┐                           ┌─────────────┐          │
│  │             │  Pflicht: Tätigkeit       │             │          │
│  │ Dienstgeber │ ────────────────────────►│ Dienstnehmer│          │
│  │             │  erbringen (KEIN Erfolg!) │             │          │
│  │             │                           │             │          │
│  │             │  Pflicht: Vergütung       │             │          │
│  │             │ ◄──────────────────────── │             │          │
│  │             │  zahlen                   │             │          │
│  └─────────────┘                           └─────────────┘          │
│                                                                      │
│  WICHTIG: Nur Tätigkeit, KEIN Erfolg geschuldet!                    │
│  → Beratung kann erfolglos sein → trotzdem Vergütung                │
│  → Wichtig ist das Bemühen, nicht das Ergebnis                      │
│                                                                      │
│  Sonderfall: Arbeitsvertrag                                         │
│  → Ist ein besonderer Dienstvertrag                                 │
│  → Arbeitnehmer schuldet Arbeitsleistung                            │
│                                                                      │
│  IT-Beispiele:                                                      │
│  → IT-Beratung nach Stunden                                         │
│  → Schulungen und Trainings                                         │
│  → Support-Hotline (ohne Erfolgsgarantie)                           │
│  → Freelancer-Einsatz                                               │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Abgrenzung in der Praxis

```
┌─────────────────────────────────────────────────────────────────────┐
│                    PRAXIS-ABGRENZUNG                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  "Wir kaufen einen Server"                                          │
│  → KAUFVERTRAG (Hardware-Kauf)                                      │
│                                                                      │
│  "Wir lassen eine Software entwickeln"                              │
│  → WERKVERTRAG (Erfolg: funktionierende Software)                   │
│                                                                      │
│  "Wir buchen IT-Beratung für 40 Stunden"                            │
│  → DIENSTVERTRAG (Tätigkeit, kein garantierter Erfolg)              │
│                                                                      │
│  "Wir lassen das Netzwerk installieren"                             │
│  → WERKVERTRAG (Erfolg: funktionierendes Netzwerk)                  │
│                                                                      │
│  "Wir kaufen Microsoft 365 Lizenzen"                                │
│  → KAUFVERTRAG (Lizenz-Erwerb)                                      │
│                                                                      │
│  "Wir mieten Cloud-Speicher"                                        │
│  → MIETVERTRAG (zeitweise Überlassung)                              │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Weitere relevante Vertragsarten

| Vertragsart | Beschreibung | IT-Beispiel |
|-------------|--------------|-------------|
| **Mietvertrag** | Zeitweise Überlassung | Cloud-Speicher, Hardware-Leasing |
| **Lizenzvertrag** | Nutzungsrecht an Software | SaaS-Verträge, Softwarelicenses |
| **Wartungsvertrag** | Regelmäßige Instandhaltung | IT-Wartung, Updates |
| **Service Level Agreement** | Qualitätsvereinbarung | SLA für IT-Services |

---

## Wichtige Begriffe

| Begriff | Erklärung |
|---------|-----------|
| **Kaufvertrag** | Übertragung von Eigentum gegen Geld |
| **Werkvertrag** | Herstellung eines Werks (Erfolg geschuldet) |
| **Dienstvertrag** | Erbringung einer Tätigkeit (kein Erfolg) |
| **Abnahme** | Bestätigung der Vertragserfüllung beim Werkvertrag |
| **Werklohn** | Vergütung beim Werkvertrag |
| **Gewährleistung** | Haftung für Mängel |

---

## Prüfungstipps

1. **Hauptunterschied merken:**
   - **Werk** = Erfolg geschuldet
   - **Dienst** = nur Tätigkeit geschuldet

2. **Typische Prüfungsfrage:**
   - "Welche Vertragsart liegt vor?" → IT-Beispiel analysieren

3. **Abnahme nur bei Werkvertrag!**
   - Kaufvertrag: Übergabe
   - Dienstvertrag: keine Abnahme nötig

## Übungsaufgaben

### Aufgabe 1: Vertragsarten zuordnen

Welche Vertragsart liegt jeweils vor?

a) ConSystem entwickelt eine Lagerverwaltungssoftware für einen Kunden
b) ConSystem kauft 20 neue Laptops beim Großhändler
c) ConSystem engagiert einen externen Berater für Projektanalyse
d) ConSystem installiert ein komplettes Netzwerk inkl. Abnahme
e) ConSystem schließt einen monatlichen Cloud-Speichervertrag ab

<details>
<summary>Lösung anzeigen</summary>

a) **Werkvertrag** - Erfolg (funktionierende Software) ist geschuldet
b) **Kaufvertrag** - Kauf von Hardware
c) **Dienstvertrag** - Tätigkeit (Beratung) ohne Erfolgsgarantie
d) **Werkvertrag** - Erfolg (funktionierendes Netzwerk) mit Abnahme
e) **Mietvertrag** - Zeitweise Überlassung von Speicherplatz

</details>

### Aufgabe 2: Erfolg oder Bemühen?

Ein Unternehmen beauftragt ConSystem mit:
- "Analysieren Sie unsere IT-Infrastruktur und machen Sie Verbesserungsvorschläge"

a) Welche Vertragsart liegt vor?
b) Was schuldet ConSystem?
c) Bekommt ConSystem Geld, auch wenn die Vorschläge nicht umgesetzt werden?

<details>
<summary>Lösung anzeigen</summary>

a) **Dienstvertrag** (oder ggf. Werkvertrag, wenn ein Bericht als "Werk" vereinbart ist)

b) **ConSystem schuldet:**
- Bei Dienstvertrag: Ordnungsgemäße Analyse-Tätigkeit, Bemühen
- Bei Werkvertrag: Fertigen Analysebericht

c) **Ja!**
- Bei Dienstvertrag: Die Vergütung ist für die Tätigkeit fällig, nicht für die Umsetzung
- Die Umsetzung der Vorschläge liegt beim Kunden
- ConSystem hat seine Leistung erbracht (Analyse und Vorschläge)

</details>

### Aufgabe 3: Abnahme

Die ConSystem GmbH hat eine Website für einen Kunden erstellt. Der Kunde weigert sich, die Abnahme zu erklären, obwohl die Website vertragsgemäß funktioniert.

a) Um welche Vertragsart handelt es sich?
b) Was bedeutet die Abnahme rechtlich?
c) Welche Rechte hat ConSystem?

<details>
<summary>Lösung anzeigen</summary>

a) **Werkvertrag** - Website-Erstellung ist ein Werk

b) **Bedeutung der Abnahme:**
- Bestätigung, dass das Werk vertragsgemäß ist
- Werklohn wird fällig
- Gewährleistungsfrist beginnt
- Beweislast wechselt (Unternehmer muss nicht mehr beweisen)

c) **Rechte von ConSystem:**
- Recht auf Abnahme, wenn Werk vertragsgemäß ist (§ 640 BGB)
- Bei unberechtigter Verweigerung: Fristsetzung
- Nach Fristablauf: Abnahme gilt als erfolgt (§ 640 Abs. 2 BGB)
- Werklohn kann eingeklagt werden
- Ggf. Schadensersatz für Verzögerung

</details>
