# Aufbauorganisation

## Lernziele

Nach diesem Abschnitt kannst du:
- Die wichtigsten Organisationsformen unterscheiden
- Vor- und Nachteile jeder Organisationsform erklären
- Organigramme lesen und erstellen
- Die passende Organisationsform für verschiedene Situationen benennen

---

## Kerninhalt

### Was ist Aufbauorganisation?

Die **Aufbauorganisation** beschreibt die hierarchische Struktur eines Unternehmens:
- Wer ist wem unterstellt?
- Wie sind die Abteilungen gegliedert?
- Wer trägt welche Verantwortung?

Sie wird grafisch in einem **Organigramm** dargestellt.

### Die drei klassischen Organisationsformen

```
┌─────────────────────────────────────────────────────────────────────┐
│                ORGANISATIONSFORMEN IM ÜBERBLICK                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  1. EINLINIENSYSTEM                                                 │
│     → Jeder Mitarbeiter hat EINEN Vorgesetzten                      │
│     → Klare Hierarchie                                              │
│                                                                      │
│  2. MEHRLINIENSYSTEM                                                │
│     → Mitarbeiter können MEHRERE Vorgesetzte haben                  │
│     → Fachliche Spezialisierung                                     │
│                                                                      │
│  3. STABLINIENSYSTEM                                                │
│     → Einliniensystem MIT Stabsstellen                              │
│     → Stäbe beraten, entscheiden nicht                              │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### 1. Einliniensystem

```
┌─────────────────────────────────────────────────────────────────────┐
│                    EINLINIENSYSTEM                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│                    ┌───────────────────┐                            │
│                    │  Geschäftsführung │                            │
│                    └─────────┬─────────┘                            │
│                              │                                       │
│            ┌─────────────────┼─────────────────┐                    │
│            │                 │                 │                    │
│            ▼                 ▼                 ▼                    │
│    ┌───────────────┐ ┌───────────────┐ ┌───────────────┐           │
│    │   Vertrieb    │ │   Technik     │ │  Verwaltung   │           │
│    └───────┬───────┘ └───────┬───────┘ └───────┬───────┘           │
│            │                 │                 │                    │
│        ┌───┴───┐         ┌───┴───┐         ┌───┴───┐               │
│        │       │         │       │         │       │               │
│        ▼       ▼         ▼       ▼         ▼       ▼               │
│     ┌─────┐ ┌─────┐   ┌─────┐ ┌─────┐   ┌─────┐ ┌─────┐           │
│     │ MA1 │ │ MA2 │   │ MA3 │ │ MA4 │   │ MA5 │ │ MA6 │           │
│     └─────┘ └─────┘   └─────┘ └─────┘   └─────┘ └─────┘           │
│                                                                      │
│  Prinzip: Ein Mitarbeiter → Ein Vorgesetzter                        │
│           (Einheit der Auftragserteilung)                           │
│                                                                      │
│  Dienstweg: Anweisungen nur über die direkte Linie                  │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

| Vorteile | Nachteile |
|----------|-----------|
| Klare Zuständigkeiten | Lange Kommunikationswege |
| Einfache Struktur | Überlastung der Führungskräfte |
| Eindeutige Verantwortung | Wenig Flexibilität |
| Übersichtlich | Informationsverlust auf dem Dienstweg |

**Geeignet für:** Kleine und mittlere Unternehmen, Behörden

### 2. Mehrliniensystem (Funktionssystem)

```
┌─────────────────────────────────────────────────────────────────────┐
│                    MEHRLINIENSYSTEM                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│                    ┌───────────────────┐                            │
│                    │  Geschäftsführung │                            │
│                    └─────────┬─────────┘                            │
│                              │                                       │
│            ┌─────────────────┼─────────────────┐                    │
│            │                 │                 │                    │
│            ▼                 ▼                 ▼                    │
│    ┌───────────────┐ ┌───────────────┐ ┌───────────────┐           │
│    │ Fachbereich 1 │ │ Fachbereich 2 │ │ Fachbereich 3 │           │
│    │   (Technik)   │ │   (Qualität)  │ │   (Prozesse)  │           │
│    └───────┬───────┘ └───────┬───────┘ └───────┬───────┘           │
│            │                 │                 │                    │
│            └─────────────────┼─────────────────┘                    │
│                              │                                       │
│                              ▼                                       │
│                         ┌─────────┐                                 │
│                         │Mitarbeiter│                               │
│                         │  erhält   │                               │
│                         │ Weisung   │                               │
│                         │von ALLEN 3│                               │
│                         └─────────┘                                 │
│                                                                      │
│  Prinzip: Ein Mitarbeiter → MEHRERE Vorgesetzte (fachlich)          │
│           (Mehrheit der Auftragserteilung)                          │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

| Vorteile | Nachteile |
|----------|-----------|
| Kurze Kommunikationswege | Unklare Zuständigkeiten |
| Fachliche Spezialisierung | Kompetenzstreitigkeiten |
| Entlastung der Führung | Widersprüchliche Anweisungen möglich |
| Schnelle Entscheidungen | Schwierige Koordination |

**Geeignet für:** Projekte, spezialisierte Aufgaben

### 3. Stabliniensystem

```
┌─────────────────────────────────────────────────────────────────────┐
│                    STABLINIENSYSTEM                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│                    ┌───────────────────┐                            │
│                    │  Geschäftsführung │◄─── ┌─────────────┐        │
│                    └─────────┬─────────┘     │ STABSSTELLE │        │
│                              │               │ (z.B. Recht,│        │
│                              │               │ IT-Sicherheit)       │
│            ┌─────────────────┼─────────────────┐  └─────────────┘   │
│            │                 │                 │                    │
│            ▼                 ▼                 ▼                    │
│    ┌───────────────┐ ┌───────────────┐ ┌───────────────┐           │
│    │   Vertrieb    │◄┤   Technik     │ │  Verwaltung   │           │
│    └───────┬───────┘ └───────┬───────┘◄───┌──────────┐ │           │
│            │                 │            │STABSSTELLE│ │           │
│        ┌───┴───┐         ┌───┴───┐       │(z.B. QM)  │ │           │
│        │       │         │       │       └──────────┘  │           │
│        ▼       ▼         ▼       ▼                     │           │
│     ┌─────┐ ┌─────┐   ┌─────┐ ┌─────┐                  │           │
│     │ MA1 │ │ MA2 │   │ MA3 │ │ MA4 │                  │           │
│     └─────┘ └─────┘   └─────┘ └─────┘                  │           │
│                                                                      │
│  Stabsstelle:                                                       │
│  → Berät die Führungskraft                                          │
│  → Hat KEINE Weisungsbefugnis                                       │
│  → Unterstützt bei Spezialistenaufgaben                             │
│                                                                      │
│  Typische Stabsstellen in IT:                                       │
│  → IT-Sicherheitsbeauftragter                                       │
│  → Datenschutzbeauftragter                                          │
│  → Qualitätsmanagement                                              │
│  → Rechtsabteilung                                                  │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

| Vorteile | Nachteile |
|----------|-----------|
| Klare Hierarchie (wie Einlinie) | Konflikte Linie ↔ Stab möglich |
| Entlastung durch Spezialisten | Stäbe ohne Verantwortung |
| Fachwissen verfügbar | Kann zu Bürokratie führen |
| Beratung ohne Kompetenzprobleme | Stab kann übergangen werden |

**Geeignet für:** Größere Unternehmen, wo Fachwissen gebraucht wird

### Vergleichstabelle

| Kriterium | Einlinie | Mehrlinie | Stablinie |
|-----------|----------|-----------|-----------|
| **Vorgesetzte** | Einer | Mehrere | Einer + Stab |
| **Weisungsweg** | Eindeutig | Mehrfach | Eindeutig |
| **Kommunikation** | Lang | Kurz | Mittel |
| **Flexibilität** | Gering | Hoch | Mittel |
| **Klarheit** | Hoch | Gering | Hoch |
| **Spezialisierung** | Gering | Hoch | Mittel |

### Matrixorganisation (Ergänzung)

```
┌─────────────────────────────────────────────────────────────────────┐
│                    MATRIXORGANISATION                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│                        Fachbereiche (funktional)                    │
│                    Entwicklung    Vertrieb    Support               │
│                         │            │           │                  │
│  Projekte   ───────────┼────────────┼───────────┼────────           │
│  (divisional)          │            │           │                  │
│                        │            │           │                  │
│  Projekt A ────────────┼────────────┼───────────┼────────           │
│                        │     MA     │    MA     │                  │
│                        │            │           │                  │
│  Projekt B ────────────┼────────────┼───────────┼────────           │
│                        │     MA     │           │    MA            │
│                                                                      │
│  Mitarbeiter haben ZWEI Vorgesetzte:                                │
│  → Fachvorgesetzter (vertikal)                                      │
│  → Projektleiter (horizontal)                                       │
│                                                                      │
│  Häufig in: IT-Unternehmen, Beratungen, Automobilindustrie          │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

## Wichtige Begriffe

| Begriff | Erklärung |
|---------|-----------|
| **Organigramm** | Grafische Darstellung der Organisationsstruktur |
| **Einliniensystem** | Jeder Mitarbeiter hat einen Vorgesetzten |
| **Mehrliniensystem** | Mitarbeiter können mehrere Vorgesetzte haben |
| **Stabliniensystem** | Einlinie + beratende Stabsstellen |
| **Stabsstelle** | Beratende Funktion ohne Weisungsbefugnis |
| **Instanzenweg** | Offizieller Dienstweg in der Hierarchie |
| **Weisungsbefugnis** | Recht, Anweisungen zu erteilen |
| **Matrixorganisation** | Kombination aus funktionaler und divisionaler Organisation |

---

## Prüfungstipps

1. **Unterscheidungsmerkmal:**
   - Wie viele Vorgesetzte hat ein Mitarbeiter?
   - Einlinie = 1, Mehrlinie = mehrere, Stablinie = 1 + Berater

2. **Stabsstelle merken:**
   - Hat KEINE Weisungsbefugnis!
   - Berät nur, entscheidet nicht

3. **Typische Prüfungsfragen:**
   - Organigramm interpretieren
   - Vor-/Nachteile nennen
   - Passende Organisationsform für Szenario wählen

4. **IT-Bezug:**
   - DSB und ISB sind oft Stabsstellen
   - IT-Projekte oft in Matrixorganisation

## Übungsaufgaben

### Aufgabe 1: Organisationsform erkennen

Welche Organisationsform liegt vor, wenn:

a) Ein IT-Mitarbeiter erhält Anweisungen sowohl vom Abteilungsleiter IT als auch vom Projektleiter
b) Der IT-Leiter berichtet direkt an die Geschäftsführung, der IT-Sicherheitsbeauftragte berät die Geschäftsführung
c) Jeder Mitarbeiter hat genau einen Vorgesetzten, es gibt keine Stabsstellen

<details>
<summary>Lösung anzeigen</summary>

a) **Mehrliniensystem** (oder Matrixorganisation) - Mehrere Vorgesetzte
b) **Stabliniensystem** - IT-Leiter in der Linie, ISB als Stab
c) **Einliniensystem** - Ein Vorgesetzter, keine Stäbe

</details>

### Aufgabe 2: Organigramm erstellen

Die ConSystem GmbH hat folgende Struktur:
- Geschäftsführung
- Drei Abteilungen: Vertrieb, Technik, Verwaltung
- Ein IT-Sicherheitsbeauftragter berät die Geschäftsführung
- Ein Datenschutzbeauftragter ist ebenfalls als Berater tätig

Welche Organisationsform liegt vor und warum?

<details>
<summary>Lösung anzeigen</summary>

**Stabliniensystem**

Begründung:
- Klare Hierarchie: Geschäftsführung → Abteilungen (Einlinie)
- Zwei Stabsstellen: ISB und DSB
- Stäbe beraten die Geschäftsführung, haben aber keine eigene Weisungsbefugnis

```
                    ┌───────────────────┐
          ┌────────►│  Geschäftsführung │◄────────┐
          │         └─────────┬─────────┘         │
     ┌────┴────┐              │              ┌────┴────┐
     │   ISB   │              │              │   DSB   │
     │ (Stab)  │              │              │ (Stab)  │
     └─────────┘              │              └─────────┘
            ┌─────────────────┼─────────────────┐
            │                 │                 │
            ▼                 ▼                 ▼
    ┌───────────────┐ ┌───────────────┐ ┌───────────────┐
    │   Vertrieb    │ │   Technik     │ │  Verwaltung   │
    └───────────────┘ └───────────────┘ └───────────────┘
```

</details>

### Aufgabe 3: Vor- und Nachteile

Ein IT-Systemhaus möchte wachsen und überlegt, ob es vom Einliniensystem auf ein Mehrliniensystem umstellen soll.

a) Nennen Sie zwei Vorteile, die für den Wechsel sprechen.
b) Nennen Sie zwei Risiken, die bei der Umstellung auftreten können.
c) Welche Alternative könnte ein Kompromiss sein?

<details>
<summary>Lösung anzeigen</summary>

a) **Vorteile des Mehrliniensystems:**
- Kürzere Kommunikationswege (direkt zum Fachvorgesetzten)
- Bessere fachliche Spezialisierung möglich
- Führungskräfte werden entlastet
- Schnellere Entscheidungen bei Fachfragen

b) **Risiken der Umstellung:**
- Widersprüchliche Anweisungen von verschiedenen Vorgesetzten
- Unklare Zuständigkeiten können zu Konflikten führen
- Mitarbeiter wissen nicht, wem sie Priorität geben sollen
- Koordinationsaufwand steigt

c) **Kompromiss: Stabliniensystem**
- Behält die klare Hierarchie des Einliniensystems
- Fügt Stabsstellen für Spezialthemen hinzu (z.B. QM, IT-Sicherheit)
- Fachexpertise verfügbar, ohne Kompetenzprobleme
- Mitarbeiter haben weiterhin einen klaren Vorgesetzten

</details>
