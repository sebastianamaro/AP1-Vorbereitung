## Kerninhalt

### Was ist ein ISMS?

Ein **ISMS (Informationssicherheits-Managementsystem)** ist ein systematischer Ansatz zur Verwaltung und kontinuierlichen Verbesserung der Informationssicherheit in einer Organisation.

```
┌─────────────────────────────────────────────────────────────────┐
│                            ISMS                                 │
│              Informationssicherheits-Managementsystem           │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌──────────────────────────────────────────────────────────┐  │
│  │                      ZIEL                                 │  │
│  │  Systematischer Schutz von Informationen durch            │  │
│  │  • Richtlinien                                            │  │
│  │  • Prozesse                                               │  │
│  │  • Technische Maßnahmen                                   │  │
│  │  • Kontinuierliche Verbesserung                           │  │
│  └──────────────────────────────────────────────────────────┘  │
│                                                                 │
│                         UMSETZUNG                               │
│                            │                                    │
│            ┌───────────────┼───────────────┐                   │
│            ▼               ▼               ▼                   │
│       ┌─────────┐    ┌─────────┐    ┌─────────┐               │
│       │  BSI    │    │   ISO   │    │ Andere  │               │
│       │ Grund-  │    │ 27001   │    │ Frame-  │               │
│       │ schutz  │    │(nicht   │    │ works   │               │
│       │         │    │im AP1)  │    │         │               │
│       └─────────┘    └─────────┘    └─────────┘               │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

---

### Komponenten eines ISMS

| Komponente | Beschreibung |
|------------|--------------|
| **Sicherheitsleitlinie** | Grundsätzliche Ausrichtung und Ziele |
| **Organisation** | Rollen und Verantwortlichkeiten |
| **Risikomanagement** | Identifikation und Behandlung von Risiken |
| **Maßnahmen** | Technische und organisatorische Controls |
| **Dokumentation** | Richtlinien, Prozesse, Aufzeichnungen |
| **Schulung** | Awareness und Kompetenz |
| **Überwachung** | Kontrolle und Messung |
| **Verbesserung** | Kontinuierlicher Verbesserungsprozess |

### Der PDCA-Zyklus im ISMS

Das ISMS folgt dem **PDCA-Zyklus** zur kontinuierlichen Verbesserung:

```
┌─────────────────────────────────────────────────────────────────┐
│                    PDCA-ZYKLUS IM ISMS                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│                      ┌─────────┐                                │
│                      │  PLAN   │                                │
│                      │         │                                │
│                      │ ISMS    │                                │
│                      │ planen  │                                │
│                      └────┬────┘                                │
│                           │                                     │
│          ┌────────────────┼────────────────┐                   │
│          │                │                │                   │
│          ▼                ▼                ▼                   │
│     ┌─────────┐                       ┌─────────┐              │
│     │   ACT   │                       │   DO    │              │
│     │         │                       │         │              │
│     │ ISMS    │◄──────────────────────│ ISMS    │              │
│     │verbess. │                       │ umsetzen│              │
│     └────┬────┘                       └────┬────┘              │
│          │                                 │                   │
│          │         ┌─────────┐             │                   │
│          │         │  CHECK  │             │                   │
│          │         │         │             │                   │
│          └────────►│ ISMS    │◄────────────┘                   │
│                    │ prüfen  │                                 │
│                    └─────────┘                                 │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

| Phase | Beschreibung | Aktivitäten |
|-------|--------------|-------------|
| **Plan** | ISMS planen | Ziele definieren, Risiken analysieren, Maßnahmen planen |
| **Do** | ISMS umsetzen | Maßnahmen implementieren, Schulungen durchführen |
| **Check** | ISMS prüfen | Überwachen, Audits durchführen, Wirksamkeit messen |
| **Act** | ISMS verbessern | Korrekturmaßnahmen, kontinuierliche Verbesserung |

---

### Rollen im ISMS

| Rolle | Aufgaben |
|-------|----------|
| **Geschäftsleitung** | Gesamtverantwortung, Ressourcen bereitstellen |
| **IT-Sicherheitsbeauftragter (ITSB)** | Koordination des ISMS, Ansprechpartner |
| **Datenschutzbeauftragter (DSB)** | Datenschutz-Compliance |
| **IT-Abteilung** | Technische Umsetzung |
| **Fachabteilungen** | Fachliche Anforderungen, Datenverantwortung |
| **Mitarbeiter** | Einhaltung der Richtlinien |

---

### Sicherheitsleitlinie (Security Policy)

Die **Sicherheitsleitlinie** ist das zentrale Dokument des ISMS:

| Inhalt | Beschreibung |
|--------|--------------|
| **Geltungsbereich** | Welche Bereiche/Systeme sind betroffen? |
| **Ziele** | Was soll erreicht werden? |
| **Grundsätze** | Welche Prinzipien gelten? |
| **Verantwortlichkeiten** | Wer ist wofür zuständig? |
| **Konsequenzen** | Was passiert bei Verstößen? |

## Wichtige Begriffe

| Begriff | Erklärung |
|---------|-----------|
| **ISMS** | Informationssicherheits-Managementsystem |
| **Sicherheitsleitlinie** | Zentrale Policy mit Zielen und Grundsätzen |
| **PDCA** | Plan-Do-Check-Act (Verbesserungszyklus) |
| **ITSB** | IT-Sicherheitsbeauftragter |
| **Control** | Sicherheitsmaßnahme |
| **Audit** | Überprüfung der Wirksamkeit |

## Übungsaufgaben

### Aufgabe 1
Was ist ein ISMS?

**Lösung:** Ein **ISMS (Informationssicherheits-Managementsystem)** ist ein systematischer Ansatz zur Verwaltung und kontinuierlichen Verbesserung der Informationssicherheit in einer Organisation. Es umfasst Richtlinien, Prozesse, technische Maßnahmen und organisatorische Strukturen zum Schutz von Informationen.

### Aufgabe 2
Erklären Sie den PDCA-Zyklus im Kontext des ISMS.

**Lösung:** Der PDCA-Zyklus ist ein Modell zur kontinuierlichen Verbesserung:
- **Plan:** ISMS planen - Ziele definieren, Risiken analysieren, Maßnahmen planen
- **Do:** ISMS umsetzen - Maßnahmen implementieren, Mitarbeiter schulen
- **Check:** ISMS prüfen - Überwachen, Audits durchführen, Wirksamkeit messen
- **Act:** ISMS verbessern - Korrekturmaßnahmen umsetzen, Prozesse optimieren

Der Zyklus wiederholt sich kontinuierlich.

### Aufgabe 3
Was sollte eine Sicherheitsleitlinie enthalten?

**Lösung:** Eine Sicherheitsleitlinie sollte enthalten:
1. **Geltungsbereich** - Welche Bereiche sind betroffen?
2. **Ziele** - Was soll erreicht werden?
3. **Grundsätze** - Welche Sicherheitsprinzipien gelten?
4. **Verantwortlichkeiten** - Wer ist wofür zuständig?
5. **Konsequenzen** - Was passiert bei Verstößen?

### Aufgabe 4
Nennen Sie drei Rollen im ISMS.

**Lösung:**
1. **Geschäftsleitung** - Gesamtverantwortung, Ressourcen bereitstellen
2. **IT-Sicherheitsbeauftragter (ITSB)** - Koordination des ISMS
3. **Datenschutzbeauftragter (DSB)** - Datenschutz-Compliance

(Weitere: IT-Abteilung, Fachabteilungen, Mitarbeiter)
