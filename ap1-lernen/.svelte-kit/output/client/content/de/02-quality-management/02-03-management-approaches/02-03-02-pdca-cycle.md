# PDCA-Zyklus

## Lernziele

Nach diesem Kapitel solltest du:
- Den PDCA-Zyklus erklären können
- Die vier Phasen beschreiben können
- Die Anwendung des PDCA-Zyklus verstehen
- Den Zusammenhang mit kontinuierlicher Verbesserung kennen

---

## Kerninhalt

### Was ist der PDCA-Zyklus?

Der **PDCA-Zyklus** (auch Deming-Zyklus oder Deming-Kreis genannt) ist eine iterative Methode zur kontinuierlichen Verbesserung von Prozessen, Produkten und Dienstleistungen.

**PDCA steht für:**
- **P**lan (Planen)
- **D**o (Umsetzen)
- **C**heck (Überprüfen)
- **A**ct (Handeln)

---

### Der Zyklus im Detail

```
                    ┌─────────────────┐
                    │      PLAN       │
                    │    (Planen)     │
                    │                 │
                    │ Problem analysieren
                    │ Maßnahmen planen│
                    └────────┬────────┘
                             │
              ┌──────────────┼──────────────┐
              │              │              │
              │              ▼              │
              │     ┌─────────────────┐     │
              │     │       DO        │     │
              │     │   (Umsetzen)    │     │
              │     │                 │     │
              │     │ Maßnahmen im    │     │
              │     │ kleinen Rahmen  │     │
              │     │ testen          │     │
              │     └────────┬────────┘     │
              │              │              │
    ┌─────────────────┐      │      ┌─────────────────┐
    │      ACT        │      │      │     CHECK       │
    │   (Handeln)     │◄─────┼─────►│   (Überprüfen)  │
    │                 │      │      │                 │
    │ Bei Erfolg:     │      │      │ Ergebnisse      │
    │ standardisieren │      │      │ auswerten       │
    │ Bei Misserfolg: │      │      │ Wirksamkeit     │
    │ korrigieren     │      │      │ prüfen          │
    └────────┬────────┘      │      └────────┬────────┘
             │               │               │
             └───────────────┴───────────────┘
                             │
                             ▼
                    Nächster Zyklus...
```

### Die vier Phasen

| Phase | Englisch | Deutsch | Aktivitäten |
|-------|----------|---------|-------------|
| **P** | Plan | Planen | Problem analysieren, Ursachen ermitteln, Ziele setzen, Maßnahmen planen |
| **D** | Do | Umsetzen | Geplante Maßnahmen in kleinem Rahmen testen/durchführen |
| **C** | Check | Überprüfen | Ergebnisse messen, mit Zielen vergleichen, Wirksamkeit bewerten |
| **A** | Act | Handeln | Bei Erfolg: standardisieren; Bei Misserfolg: korrigieren und neuen Zyklus starten |

---

### Phase 1: Plan (Planen)

**Ziel:** Problem verstehen und Lösung entwickeln

**Aktivitäten:**
1. Problem identifizieren und beschreiben
2. Ist-Zustand analysieren
3. Ursachen ermitteln (z.B. mit Ishikawa-Diagramm)
4. Ziele festlegen (SMART!)
5. Maßnahmen planen
6. Ressourcen und Zeitplan festlegen

---

### Phase 2: Do (Umsetzen)

**Ziel:** Geplante Maßnahmen testen

**Aktivitäten:**
1. Maßnahmen im **kleinen Rahmen** umsetzen
2. Testumgebung oder Pilotprojekt nutzen
3. Daten sammeln
4. Erfahrungen dokumentieren

**Wichtig:** Nicht sofort flächendeckend einführen!

---

### Phase 3: Check (Überprüfen)

**Ziel:** Wirksamkeit der Maßnahmen bewerten

**Aktivitäten:**
1. Ergebnisse messen
2. Mit Zielen vergleichen (Soll-Ist-Vergleich)
3. Abweichungen analysieren
4. Ursachen für Abweichungen ermitteln
5. Schlussfolgerungen ziehen

---

### Phase 4: Act (Handeln)

**Ziel:** Konsequenzen aus der Überprüfung ziehen

**Bei Erfolg:**
- Maßnahmen als Standard festlegen
- Flächendeckend einführen
- Dokumentieren

**Bei Misserfolg:**
- Ursachen für Misserfolg analysieren
- Maßnahmen anpassen
- Neuen PDCA-Zyklus starten

---

### Anwendung des PDCA-Zyklus

| Bereich | Beispiel |
|---------|----------|
| **Softwareentwicklung** | Iterative Verbesserung eines Features |
| **IT-Support** | Optimierung des Ticketprozesses |
| **Projektmanagement** | Lessons Learned umsetzen |
| **Qualitätsmanagement** | Fehlerquote reduzieren |

### PDCA in der ISO 9001

Die **DIN EN ISO 9001** basiert auf dem PDCA-Zyklus:

```
┌───────────────────────────────────────────────────────────────┐
│                     ISO 9001 und PDCA                         │
├───────────────────────────────────────────────────────────────┤
│                                                               │
│   PLAN                          DO                            │
│   • Kontext der Organisation    • Unterstützung               │
│   • Führung                     • Betrieb                     │
│   • Planung                                                   │
│                                                               │
│   ACT                           CHECK                         │
│   • Verbesserung                • Bewertung der Leistung      │
│                                                               │
└───────────────────────────────────────────────────────────────┘
```

## Wichtige Begriffe

| Begriff | Erklärung |
|---------|-----------|
| **PDCA-Zyklus** | Plan-Do-Check-Act, Methode zur kontinuierlichen Verbesserung |
| **Deming-Zyklus** | Anderer Name für PDCA (nach W. Edwards Deming) |
| **KVP** | Kontinuierlicher Verbesserungsprozess |
| **Iteration** | Wiederholter Durchlauf des Zyklus |
| **Standardisierung** | Festlegung erfolgreicher Maßnahmen als verbindlich |

## Übungsaufgaben

### Aufgabe 1
Wofür steht die Abkürzung PDCA?

**Lösung:**
- **P** = Plan (Planen)
- **D** = Do (Umsetzen/Durchführen)
- **C** = Check (Überprüfen/Kontrollieren)
- **A** = Act (Handeln)

### Aufgabe 2
Beschreiben Sie kurz die vier Phasen des PDCA-Zyklus.

**Lösung:**
1. **Plan:** Problem analysieren, Maßnahmen planen
2. **Do:** Maßnahmen in kleinem Rahmen testen
3. **Check:** Ergebnisse auswerten, Wirksamkeit prüfen
4. **Act:** Bei Erfolg standardisieren, bei Misserfolg korrigieren und neuen Zyklus starten

### Aufgabe 3
Ein IT-Unternehmen möchte die Anzahl der Support-Tickets reduzieren. Beschreiben Sie, wie der PDCA-Zyklus angewendet werden könnte.

**Lösung:**
1. **Plan:** Ticket-Statistiken analysieren, häufige Ursachen identifizieren (z.B. fehlende Dokumentation), Maßnahme planen (FAQ erstellen)
2. **Do:** FAQ für ein Pilotthema erstellen und testen
3. **Check:** Ticket-Anzahl für dieses Thema messen, Nutzerfeedback einholen
4. **Act:** Bei Erfolg FAQ erweitern und veröffentlichen, bei Misserfolg Format/Inhalt anpassen und erneut testen
