# Konfliktmanagement im IT-Support

## Lernziele

Nach diesem Abschnitt kannst du:
- Konfliktsituationen im IT-Support erkennen
- Deeskalationstechniken anwenden
- Mit schwierigen Anwendern umgehen
- Beschwerden professionell bearbeiten

---

## Kerninhalt

### Typische Konfliktsituationen im IT-Support

```
┌─────────────────────────────────────────────────────────────────────┐
│           KONFLIKTSITUATIONEN                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  HÄUFIGE AUSLÖSER:                                                  │
│  ──────────────────────────────────────────────────────────────     │
│  → Lange Wartezeiten                                                │
│  → Wiederkehrende Probleme                                          │
│  → Unklare Kommunikation                                            │
│  → Unterschiedliche Erwartungen                                     │
│  → Stress und Zeitdruck beim User                                   │
│  → Gefühl, nicht ernst genommen zu werden                           │
│                                                                      │
│  TYPISCHE AUSSAGEN:                                                 │
│  ──────────────────────────────────────────────────────────────     │
│  → "Das kann doch nicht so schwer sein!"                           │
│  → "Ich rufe jetzt schon zum dritten Mal an!"                      │
│  → "Das ist inakzeptabel!"                                         │
│  → "Ich will mit Ihrem Vorgesetzten sprechen!"                     │
│  → "Ihr seid doch alle inkompetent!"                               │
│                                                                      │
│  ESKALATIONSSTUFEN:                                                 │
│  ──────────────────────────────────────────────────────────────     │
│  1. Unzufriedenheit (leichte Kritik)                               │
│  2. Verärgerung (deutliche Beschwerden)                            │
│  3. Wut (aggressive Kommunikation)                                 │
│  4. Eskalation (Forderung nach Vorgesetztem)                       │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Deeskalationstechniken

```
┌─────────────────────────────────────────────────────────────────────┐
│           DEESKALATION                                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  1. ZUHÖREN UND AUSREDEN LASSEN                                     │
│     ───────────────────────────────────────────────────────────     │
│     → Den User erst einmal "Dampf ablassen" lassen                 │
│     → Nicht unterbrechen                                            │
│     → Aktiv zuhören (Nicken, "Mhm", "Ich verstehe")                │
│     → Notizen machen                                                │
│                                                                      │
│  2. VERSTÄNDNIS ZEIGEN                                              │
│     ───────────────────────────────────────────────────────────     │
│     → "Ich verstehe, dass Sie frustriert sind"                     │
│     → "Das ist wirklich ärgerlich"                                 │
│     → "An Ihrer Stelle wäre ich auch verärgert"                    │
│     → WICHTIG: Verständnis ≠ Schuld eingestehen                    │
│                                                                      │
│  3. PROBLEM ZUSAMMENFASSEN                                          │
│     ───────────────────────────────────────────────────────────     │
│     → "Wenn ich Sie richtig verstehe, ist das Problem..."          │
│     → Zeigt, dass man zugehört hat                                 │
│     → Stellt sicher, dass man richtig verstanden hat               │
│                                                                      │
│  4. LÖSUNG ANBIETEN                                                 │
│     ───────────────────────────────────────────────────────────     │
│     → Konkrete nächste Schritte nennen                             │
│     → Realistische Zeitangaben machen                              │
│     → Verantwortung übernehmen                                      │
│     → "Ich kümmere mich persönlich darum"                          │
│                                                                      │
│  5. VERBINDLICHKEIT SCHAFFEN                                        │
│     ───────────────────────────────────────────────────────────     │
│     → Ticket-Nummer nennen                                          │
│     → Rückruf-Termin vereinbaren                                   │
│     → Namen geben (SPOC-Prinzip)                                   │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Das CALM-Prinzip

```
┌─────────────────────────────────────────────────────────────────────┐
│           DAS CALM-PRINZIP                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  C - CONTROL (Kontrolle behalten)                                   │
│      → Ruhig bleiben, nicht persönlich nehmen                      │
│      → Eigene Emotionen kontrollieren                               │
│      → Stimme ruhig und freundlich halten                          │
│                                                                      │
│  A - ACKNOWLEDGE (Anerkennen)                                       │
│      → Das Problem anerkennen                                       │
│      → Die Gefühle des Users anerkennen                            │
│      → "Ich verstehe Ihre Situation"                               │
│                                                                      │
│  L - LISTEN (Zuhören)                                               │
│      → Aktiv zuhören                                                │
│      → Nicht unterbrechen                                           │
│      → Nachfragen stellen                                           │
│                                                                      │
│  M - MANAGE (Managen)                                               │
│      → Erwartungen managen                                          │
│      → Realistische Lösungen anbieten                              │
│      → Follow-up zusagen                                            │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Umgang mit verschiedenen User-Typen

```
┌─────────────────────────────────────────────────────────────────────┐
│           USER-TYPEN                                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  DER UNGEDULDIGE                                                    │
│  ──────────────────────────────────────────────────────────────     │
│  Verhalten: Will sofortige Lösung, unterbricht, ist ungeduldig     │
│  Strategie:                                                         │
│  → Schnell zur Sache kommen                                        │
│  → Keine langen Erklärungen                                        │
│  → Zeitliche Zusagen machen                                        │
│  → "Ich kümmere mich sofort darum"                                 │
│                                                                      │
│  DER WÜTENDE                                                        │
│  ──────────────────────────────────────────────────────────────     │
│  Verhalten: Aggressiv, vorwurfsvoll, laut                          │
│  Strategie:                                                         │
│  → Ruhig bleiben, nicht dagegen anschreien                         │
│  → Ausreden lassen                                                  │
│  → Verständnis zeigen                                              │
│  → Auf Sachebene zurückführen                                      │
│                                                                      │
│  DER HILFLOSE                                                       │
│  ──────────────────────────────────────────────────────────────     │
│  Verhalten: Überfordert, unsicher, braucht viel Anleitung          │
│  Strategie:                                                         │
│  → Geduldig sein                                                    │
│  → Schritt für Schritt erklären                                    │
│  → Keine Fachbegriffe                                              │
│  → Ermutigen und loben                                             │
│                                                                      │
│  DER BESSERWISSER                                                   │
│  ──────────────────────────────────────────────────────────────     │
│  Verhalten: Glaubt alles besser zu wissen, argumentiert            │
│  Strategie:                                                         │
│  → Ernst nehmen                                                     │
│  → Fachlich korrekt bleiben                                        │
│  → Expertise anerkennen wo angebracht                              │
│  → Nicht in Machtkämpfe einsteigen                                 │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Beschwerdemanagement

```
┌─────────────────────────────────────────────────────────────────────┐
│           BESCHWERDE BEARBEITEN                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  SCHRITT 1: ANNEHMEN                                                │
│  ──────────────────────────────────────────────────────────────     │
│  → Beschwerde ernst nehmen                                         │
│  → Dokumentieren (Ticket)                                          │
│  → Empfangsbestätigung geben                                       │
│                                                                      │
│  SCHRITT 2: PRÜFEN                                                  │
│  ──────────────────────────────────────────────────────────────     │
│  → Sachverhalt klären                                              │
│  → Ticket-Historie lesen                                           │
│  → Mit Kollegen sprechen wenn nötig                                │
│                                                                      │
│  SCHRITT 3: REAGIEREN                                               │
│  ──────────────────────────────────────────────────────────────     │
│  → Zeitnah antworten                                               │
│  → Ergebnis der Prüfung mitteilen                                  │
│  → Bei berechtigter Beschwerde: Entschuldigung + Lösung            │
│  → Bei unberechtigter Beschwerde: Sachlich erklären                │
│                                                                      │
│  SCHRITT 4: NACHFASSEN                                              │
│  ──────────────────────────────────────────────────────────────     │
│  → War der User zufrieden?                                         │
│  → Wurde das Problem gelöst?                                       │
│  → Was können wir verbessern?                                      │
│                                                                      │
│  WICHTIG: Beschwerden sind CHANCEN zur Verbesserung!               │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Grenzen setzen

```
┌─────────────────────────────────────────────────────────────────────┐
│           GRENZEN SETZEN                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  BEI PERSÖNLICHEN ANGRIFFEN:                                        │
│  ──────────────────────────────────────────────────────────────     │
│  → "Ich verstehe, dass Sie verärgert sind, aber ich bitte Sie,    │
│     sachlich zu bleiben."                                          │
│  → "Ich möchte Ihnen gerne helfen, aber bitte in einem            │
│     respektvollen Ton."                                            │
│                                                                      │
│  BEI BELEIDIGUNGEN:                                                 │
│  ──────────────────────────────────────────────────────────────     │
│  → "So kann ich das Gespräch nicht fortführen."                   │
│  → "Wenn Sie sich beruhigt haben, rufe ich Sie gerne zurück."     │
│  → Im Extremfall: Gespräch beenden (nach Ankündigung)              │
│                                                                      │
│  BEI UNREALISTISCHEN FORDERUNGEN:                                   │
│  ──────────────────────────────────────────────────────────────     │
│  → "Das ist leider nicht möglich, weil..."                        │
│  → "Was ich Ihnen anbieten kann, ist..."                          │
│  → Alternative aufzeigen                                            │
│                                                                      │
│  BEI ESKALATION ZUM VORGESETZTEN:                                   │
│  ──────────────────────────────────────────────────────────────     │
│  → Nicht als Versagen verstehen                                    │
│  → Manchmal ist Weitergabe die beste Lösung                        │
│  → Dokumentation übergeben                                         │
│  → Nicht "abwimmeln", sondern aktiv übergeben                      │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Formulierungshilfen

| Situation | Nicht sagen | Besser sagen |
|-----------|-------------|--------------|
| User ist wütend | "Beruhigen Sie sich!" | "Ich verstehe Ihre Frustration" |
| Problem dauert | "Ich weiß nicht, wie lange" | "Ich melde mich in 30 Minuten" |
| Nicht zuständig | "Bin ich nicht zuständig" | "Ich verbinde Sie mit dem Kollegen" |
| Fehler passiert | "Da kann ich nichts dafür" | "Das tut mir leid, ich kümmere mich" |
| Unmögliche Forderung | "Das geht nicht" | "Was ich tun kann, ist..." |

## Wichtige Begriffe

| Begriff | Erklärung |
|---------|-----------|
| **Deeskalation** | Maßnahmen zur Beruhigung einer angespannten Situation |
| **Aktives Zuhören** | Aufmerksames Zuhören mit Rückmeldung |
| **Beschwerdemanagement** | Strukturierter Umgang mit Beschwerden |
| **Empathie** | Einfühlungsvermögen |
| **Sachebene** | Faktische Ebene einer Kommunikation |

---

## Prüfungstipps

1. **Grundprinzip merken:**
   - Erst Verständnis zeigen, dann Lösung anbieten
   - Nie persönlich nehmen
   - Immer sachlich und freundlich bleiben

2. **CALM-Prinzip:**
   - Control, Acknowledge, Listen, Manage
   - Gute Merkhilfe für Prüfung

3. **Typische Prüfungsfragen:**
   - "Wie reagieren Sie auf einen verärgerten Kunden?"
   - "Welche Deeskalationstechniken kennen Sie?"

## Übungsaufgaben

### Aufgabe 1: Reaktion formulieren

Ein User ruft wütend an: "Das ist jetzt das dritte Mal diese Woche, dass ich anrufe! Eure IT ist wirklich unfähig! Ich muss arbeiten und mein PC geht wieder nicht!"

Formulieren Sie eine deeskalierende Antwort.

<details>
<summary>Lösung anzeigen</summary>

**Professionelle Antwort:**

"Ich verstehe total, dass Sie verärgert sind, wenn das Problem wiederholt auftritt - das wäre ich an Ihrer Stelle auch. [Verständnis zeigen]

Lassen Sie mich kurz Ihre Ticket-Historie anschauen, damit ich sehe, was bisher versucht wurde. [Sachebene]

Ich möchte das Problem diesmal nachhaltig lösen und werde mich persönlich darum kümmern. [Verbindlichkeit]

Können Sie mir kurz beschreiben, was genau gerade passiert? [Problem konkret erfassen]

Ich werde danach nicht aufhören, bis wir eine dauerhafte Lösung haben." [Commitment]

</details>

### Aufgabe 2: Grenzen setzen

Ein User sagt: "Sie sind doch alle inkompetent! Ich will sofort mit Ihrem Chef sprechen!"

Wie reagieren Sie?

<details>
<summary>Lösung anzeigen</summary>

**Reaktion in zwei Schritten:**

**1. Grenze setzen (freundlich aber bestimmt):**
"Ich verstehe, dass Sie unzufrieden sind, und ich möchte Ihnen helfen. Allerdings bitte ich Sie, sachlich zu bleiben, damit wir gemeinsam eine Lösung finden können."

**2. Angebot machen:**
"Ich würde gerne zuerst versuchen, Ihr Problem direkt zu lösen. Falls Sie danach immer noch mit meinem Vorgesetzten sprechen möchten, kann ich das gerne arrangieren. Was ist denn konkret das Problem?"

**Falls weiter beleidigend:**
"Ich bin gerne bereit, Ihnen zu helfen, aber nur in einem respektvollen Gespräch. Ich gebe Ihren Fall jetzt an meinen Teamleiter weiter, der sich bei Ihnen meldet. Haben Sie einen Moment?"

**Wichtig:** Eskalation zum Vorgesetzten ist keine Niederlage - manchmal ist es die bessere Lösung für alle Beteiligten.

</details>

### Aufgabe 3: CALM-Prinzip anwenden

Ordnen Sie die folgenden Aussagen dem richtigen CALM-Buchstaben zu:

a) "Ich werde mich in einer Stunde bei Ihnen melden"
b) Ruhig und sachlich bleiben, obwohl der User laut wird
c) "Ich verstehe, dass das ärgerlich für Sie ist"
d) Den User erst ausreden lassen

<details>
<summary>Lösung anzeigen</summary>

a) **M - Manage** - Erwartungen setzen und Follow-up zusagen
b) **C - Control** - Eigene Emotionen kontrollieren
c) **A - Acknowledge** - Problem und Gefühle anerkennen
d) **L - Listen** - Aktiv zuhören

</details>
