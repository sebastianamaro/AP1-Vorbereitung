# DHCP

## Lernziele

Nach diesem Kapitel solltest du:
- Die Funktion von DHCP verstehen
- Den DORA-Prozess erklären können
- Die Vorteile von DHCP kennen
- DHCP-Lease verstehen

---

## Kerninhalt

### Was ist DHCP?

```
┌────────────────────────────────────────────────────────────────────┐
│                          DHCP                                      │
├────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   DHCP = Dynamic Host Configuration Protocol                       │
│                                                                     │
│   ┌─────────────────────────────────────────────────────────────┐ │
│   │ DHCP vergibt automatisch Netzwerkkonfigurationen an        │ │
│   │ Geräte im Netzwerk (IP-Adresse, Gateway, DNS).             │ │
│   └─────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   Ohne DHCP:                          Mit DHCP:                    │
│   ┌──────────────────────────┐       ┌──────────────────────────┐ │
│   │ Jedes Gerät muss         │       │ Geräte bekommen ihre    │ │
│   │ manuell konfiguriert     │       │ Einstellungen           │ │
│   │ werden:                  │       │ automatisch:            │ │
│   │ • IP-Adresse             │       │                         │ │
│   │ • Subnetzmaske           │       │  PC → DHCP-Server → IP  │ │
│   │ • Gateway                │       │                         │ │
│   │ • DNS-Server             │       │ "Plug and Play"         │ │
│   └──────────────────────────┘       └──────────────────────────┘ │
│                                                                     │
│   Vorteile:                                                        │
│   • Keine manuelle Konfiguration nötig                            │
│   • Keine IP-Adresskonflikte                                       │
│   • Zentrale Verwaltung                                            │
│   • Änderungen an einem Ort                                        │
│                                                                     │
└────────────────────────────────────────────────────────────────────┘
```

### Der DORA-Prozess

```
┌────────────────────────────────────────────────────────────────────┐
│                       DORA-PROZESS                                 │
├────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   DORA = Discover, Offer, Request, Acknowledge                    │
│                                                                     │
│      Client                                    DHCP-Server         │
│        │                                           │               │
│   1.   │     DHCP DISCOVER (Broadcast)            │               │
│        │ ────────────────────────────────────→    │               │
│        │   "Gibt es einen DHCP-Server?"           │               │
│        │   (Broadcast an 255.255.255.255)         │               │
│        │                                           │               │
│   2.   │     DHCP OFFER                           │               │
│        │ ←────────────────────────────────────    │               │
│        │   "Hier ist eine IP für dich:"           │               │
│        │   IP: 192.168.1.100                      │               │
│        │   Maske: 255.255.255.0                   │               │
│        │   Gateway: 192.168.1.1                   │               │
│        │                                           │               │
│   3.   │     DHCP REQUEST (Broadcast)             │               │
│        │ ────────────────────────────────────→    │               │
│        │   "Ich nehme diese IP!"                  │               │
│        │   (Broadcast, damit andere Server wissen)│               │
│        │                                           │               │
│   4.   │     DHCP ACKNOWLEDGE                     │               │
│        │ ←────────────────────────────────────    │               │
│        │   "Bestätigt, die IP gehört dir!"        │               │
│        │   Lease-Zeit: 24 Stunden                 │               │
│        │                                           │               │
│        │   ═══════════════════════════════════   │               │
│        │     Client hat jetzt Netzwerkzugang     │               │
│        │   ═══════════════════════════════════   │               │
│                                                                     │
│   Merksatz: "Dora braucht DHCP"                                   │
│   D = Discover, O = Offer, R = Request, A = Acknowledge           │
│                                                                     │
└────────────────────────────────────────────────────────────────────┘
```

### DHCP-Lease

```
┌────────────────────────────────────────────────────────────────────┐
│                       DHCP-LEASE                                   │
├────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   Definition:                                                      │
│   ┌─────────────────────────────────────────────────────────────┐ │
│   │ Eine DHCP-Lease ist die zeitlich begrenzte "Ausleihe"      │ │
│   │ einer IP-Adresse an ein Gerät.                              │ │
│   └─────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   Lease-Zeit (typisch: 24 Stunden bis 7 Tage)                     │
│                                                                     │
│   ┌─────────────────────────────────────────────────────────────┐ │
│   │                                                              │ │
│   │   0%        50%                       100%                  │ │
│   │   │──────────│──────────────────────────│                   │ │
│   │   Start      Renewal       Rebinding    Ablauf              │ │
│   │              (50%)         (87,5%)                          │ │
│   │                                                              │ │
│   │   Bei 50%: Client versucht Lease zu erneuern               │ │
│   │   Bei 87,5%: Client versucht bei JEDEM Server zu erneuern  │ │
│   │   Bei 100%: IP wird freigegeben                            │ │
│   │                                                              │ │
│   └─────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   Was passiert bei Lease-Ablauf?                                  │
│   • Client verliert IP-Adresse                                    │
│   • Client muss neuen DORA-Prozess starten                        │
│   • Erhält möglicherweise neue IP-Adresse                         │
│                                                                     │
└────────────────────────────────────────────────────────────────────┘
```

### DHCP-Adresspool

```
┌────────────────────────────────────────────────────────────────────┐
│                     DHCP-ADRESSPOOL                                │
├────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   Der Administrator definiert einen Bereich von IP-Adressen,      │
│   die DHCP vergeben darf:                                         │
│                                                                     │
│   ┌─────────────────────────────────────────────────────────────┐ │
│   │                                                              │ │

│   │                                                              │ │
│   │   │  .1 - .10  │   .11 - .200   │  .201 - .254  │           │ │
│   │   │  Statisch  │   DHCP-Pool    │   Statisch    │           │ │

│   │                                                              │ │
│   │   .0   = Netzwerkadresse (nicht nutzbar)                    │ │
│   │   .255 = Broadcast (nicht nutzbar)                          │ │
│   │                                                              │ │

│                                                                     │
│   Reservierungen:                                                  │
│   ┌────────────────────────────────────────────────────────────┐  │
│   │ Bestimmte Geräte können eine feste IP erhalten:            │  │
│   │ • Basiert auf MAC-Adresse                                  │  │
│   │ • Server, Drucker bekommen immer dieselbe IP               │  │
│   │ • Trotzdem über DHCP konfiguriert                          │  │
│   └────────────────────────────────────────────────────────────┘  │
│                                                                     │
└────────────────────────────────────────────────────────────────────┘
```

### DHCP vs. Statische Konfiguration

```
┌────────────────────────────────────────────────────────────────────┐
│               DHCP vs. STATISCH                                    │
├────────────────────────────────────────────────────────────────────┤
│                                                                     │
│                    DHCP               Statisch                     │
│   ┌────────────────────────────┬──────────────────────────────┐   │
│   │ Automatische Vergabe       │ Manuelle Konfiguration       │   │
│   │ IP kann sich ändern        │ IP bleibt immer gleich       │   │
│   │ Weniger Aufwand            │ Mehr Aufwand                  │   │
│   │ Zentrale Verwaltung        │ Dezentrale Verwaltung        │   │
│   │ Keine Konflikte            │ Konflikte möglich            │   │
│   │ Abhängig vom DHCP-Server   │ Unabhängig                   │   │
│   └────────────────────────────┴──────────────────────────────┘   │
│                                                                     │
│   Wann DHCP?                    Wann statisch?                    │
│   ┌────────────────────────────┬──────────────────────────────┐   │
│   │ • Client-PCs               │ • Server                     │   │
│   │ • Laptops                  │ • Router/Gateways            │   │
│   │ • Smartphones              │ • Netzwerkdrucker            │   │
│   │ • Gäste-Geräte             │ • NAS-Systeme                │   │
│   │ • Geräte mit wechselnder   │ • Geräte, die immer         │   │
│   │   Nutzung                  │   erreichbar sein müssen    │   │
│   └────────────────────────────┴──────────────────────────────┘   │
│                                                                     │
└────────────────────────────────────────────────────────────────────┘
```

## Wichtige Begriffe

| Begriff | Erklärung |
|---------|-----------|
| **DHCP** | Dynamic Host Configuration Protocol |
| **DORA** | Discover, Offer, Request, Acknowledge |
| **Lease** | Zeitlich begrenzte IP-Zuweisung |
| **Adresspool** | Bereich verfügbarer IP-Adressen |
| **Reservierung** | Feste IP basierend auf MAC-Adresse |
| **Broadcast** | DHCP nutzt Broadcasts (Port 67/68) |

## Prüfungstipps

### Häufige Prüfungsfragen

- Erklären Sie den DORA-Prozess
- Welche Daten liefert DHCP?
- Wann statisch, wann DHCP?

### Wichtig zu merken

- **DORA:** Discover → Offer → Request → Acknowledge
- **Ports:** Server 67, Client 68 (UDP)
- **DHCP liefert:** IP, Maske, Gateway, DNS
- **Lease** = zeitlich begrenzte Zuweisung
- **Server** = statische IP, **Clients** = DHCP
- Discover und Request sind **Broadcasts**

## Übungsaufgaben

### Aufgabe 1
Erklären Sie den DORA-Prozess bei DHCP.

**Lösung:**
1. **Discover:** Client sendet Broadcast ("Gibt es einen DHCP-Server?")
2. **Offer:** Server antwortet mit IP-Angebot
3. **Request:** Client akzeptiert das Angebot (Broadcast)
4. **Acknowledge:** Server bestätigt die Zuweisung

### Aufgabe 2
Warum sollte ein Server eine statische IP haben und kein DHCP nutzen?

**Lösung:**
- Server müssen **immer unter derselben IP erreichbar** sein
- Andere Geräte/Dienste verweisen auf diese IP
- DHCP könnte eine **andere IP** vergeben
- Bei **DHCP-Server-Ausfall** wäre Server nicht erreichbar
- DNS-Einträge würden nicht mehr stimmen

### Aufgabe 3
Welche Informationen erhält ein Client typischerweise vom DHCP-Server?

**Lösung:**
- **IP-Adresse** (z.B. 192.168.1.100)
- **Subnetzmaske** (z.B. 255.255.255.0)
- **Default Gateway** (z.B. 192.168.1.1)
- **DNS-Server** (z.B. 192.168.1.1, 8.8.8.8)
- **Lease-Zeit** (z.B. 24 Stunden)
