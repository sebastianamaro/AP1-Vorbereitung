# Fases del Proyecto

## Objetivos de Aprendizaje

Después de este capítulo, usted debería ser capaz de:
- Conocer las cuatro fases principales de un proyecto
- Describir las tareas en cada fase
- Comprender la diferencia entre modelo de fases y modelo de proceso

---

## Contenido Principal

### Las Cuatro Fases del Proyecto

Todo proyecto pasa fundamentalmente por cuatro fases principales:

```
+----------------+   +----------------+   +----------------+   +----------------+
|  1. DEFINICIÓN |   | 2. PLANIFICACIÓN|   |  3. EJECUCIÓN  |   |   4. CIERRE    |
|  DEL PROYECTO  | > |  DEL PROYECTO  | > |  DEL PROYECTO  | > |  DEL PROYECTO  |
+----------------+   +----------------+   +----------------+   +----------------+
```

---

### Fase 1: Definición del Proyecto

**Objetivo:** Aclarar qué se debe lograr

**Actividades:**
- Formular objetivos del proyecto (SMART)
- Identificar stakeholders
- Realizar estudio de viabilidad
- Crear carta del proyecto
- Crear/analizar documento de requisitos

**Resultados:**
- Carta del proyecto
- Documento de requisitos (requisitos del cliente)
- Estimación aproximada de recursos

---

### Fase 2: Planificación del Proyecto

**Objetivo:** Determinar cómo se lograrán los objetivos

**Actividades:**
- Crear plan detallado
- Definir paquetes de trabajo
- Crear cronograma (diagrama de Gantt, diagrama de red)
- Planificar recursos
- Realizar análisis de riesgos
- Crear especificación funcional
- Definir hitos

**Resultados:**
- Especificación funcional
- Plan del proyecto
- Cronograma
- Plan de recursos
- Análisis de riesgos

---

### Fase 3: Ejecución del Proyecto

**Objetivo:** Implementación de las actividades planificadas

**Actividades:**
- Completar paquetes de trabajo
- Monitorear el progreso
- Realizar aseguramiento de calidad
- Control del proyecto (en caso de desviaciones)
- Coordinar el equipo
- Crear documentación

**Resultados:**
- Resultado del proyecto (producto/servicio)
- Informes de progreso
- Protocolos de calidad

---

### Fase 4: Cierre del Proyecto

**Objetivo:** Concluir y documentar adecuadamente el proyecto

**Actividades:**
- Presentar resultados
- Realizar aceptación
- Documentar lecciones aprendidas
- Completar documentación del proyecto
- Disolver el equipo
- Crear cálculo de costos final

**Resultados:**
- Protocolo de aceptación
- Informe final
- Lecciones aprendidas
- Archivo

---

### Modelo de Fases vs. Modelo de Proceso

| Modelo de Fases | Modelo de Proceso |
|-----------------|-------------------|
| Describe las fases básicas del proyecto | Describe el enfoque específico |
| Generalmente válido para todos los proyectos | Específico del tipo de proyecto (ej., software) |
| 4 fases | Puede tener más fases |
| Ejemplo: Las cuatro fases del proyecto | Ejemplo: Modelo en cascada, Scrum |

---

## Términos Clave

| Término | Explicación |
|---------|-------------|
| **Hito** | Paso intermedio importante en el proyecto con resultado verificable |
| **Paquete de Trabajo** | Unidad más pequeña de la planificación del proyecto |
| **Carta del Proyecto** | Documento que marca el inicio oficial del proyecto |
| **Aceptación** | Confirmación formal de que el resultado del proyecto cumple los requisitos |

---

## Consejos para el Examen

- Las cuatro fases se preguntan frecuentemente en el orden correcto
- Importante: Diferencia entre documento de requisitos (Fase 1) y especificación funcional (Fase 2)
- Los hitos pueden establecerse en cualquier límite de fase

---

## Ejercicios Prácticos

### Ejercicio 1
Asigne las siguientes actividades a las fases del proyecto:

a) Creación del diagrama de Gantt
b) Realización de la aceptación
c) Formulación de objetivos del proyecto
d) Verificación de calidad de los resultados del trabajo

**Solución:**
a) Fase 2 - Planificación del Proyecto
b) Fase 4 - Cierre del Proyecto
c) Fase 1 - Definición del Proyecto
d) Fase 3 - Ejecución del Proyecto

### Ejercicio 2
¿En qué fase se crea la especificación funcional?

**Solución:** Fase 2 - Planificación del Proyecto (basada en el documento de requisitos de la Fase 1)
