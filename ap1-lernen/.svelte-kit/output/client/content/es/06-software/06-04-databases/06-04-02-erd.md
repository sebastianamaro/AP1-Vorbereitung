# Diagrama Entidad-Relación (ERD)

## Objetivos de Aprendizaje

Después de este capítulo, deberías ser capaz de:
- Leer y comprender ERDs
- Identificar entidades, atributos y relaciones
- Interpretar cardinalidades en ERDs
- Crear ERDs simples

---

## Contenido Principal

### ¿Qué es un ERD?

Un **Diagrama Entidad-Relación (ERD)** es un modelo gráfico para representar estructuras de datos y sus relaciones.

```
┌─────────────────────────────────────────────────────────────────┐
│                  DIAGRAMA ENTIDAD-RELACIÓN                      │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   Propósito:                                                    │
│   • Modelado de datos antes de crear la base de datos          │
│   • Visualización de estructuras de datos                      │
│   • Comunicación entre desarrolladores y departamento de negocio│
│   • Base para el diseño de bases de datos                      │
│                                                                 │
│   Componentes:                                                  │
│   • Entidades (objetos/cosas)                                  │
│   • Atributos (propiedades)                                    │
│   • Relaciones (conexiones)                                    │
│   • Cardinalidades (especificaciones de cantidad)              │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### Elementos de un ERD

```
┌────────────────────────────────────────────────────────────────┐
│                    ELEMENTOS DEL ERD                           │
├────────────────────────────────────────────────────────────────┤
│                                                                │
│   1. ENTIDAD                                                   │
│      ┌─────────────┐                                          │
│      │   Cliente   │  Rectángulo con nombre                   │
│      └─────────────┘  = "cosa" sobre la cual se almacenan datos│
│                                                                │
│   2. ATRIBUTO                                                  │
│         ○                                                      │
│        ╱ ╲                                                     │
│      Nombre   Elipse (o lista en la entidad)                  │
│              = propiedad de la entidad                        │
│                                                                │
│   3. ATRIBUTO CLAVE PRIMARIA                                   │
│         ◎         o  ○                                        │
│        ╱ ╲            ╱ ╲                                      │
│      NoCliente      NoCliente (subrayado)                     │
│              = identificador único                            │
│                                                                │
│   4. RELACIÓN                                                  │
│         ◇                                                      │
│        ╱ ╲                                                     │
│      tiene   Rombo con líneas de conexión                     │
│              = cómo se conectan las entidades                 │
│                                                                │
└────────────────────────────────────────────────────────────────┘
```

### Notación Chen (Clásica)

```
┌────────────────────────────────────────────────────────────────┐
│                   NOTACIÓN CHEN                                │
├────────────────────────────────────────────────────────────────┤
│                                                                │
│        ○ Nombre        ○ Descripción                           │
│         \            /                                         │
│      ◎ NoCliente ─┬──┐              ┌──┬─ ◎ NoArtículo        │
│                  │   │              │  │                       │
│              ┌───┴───┴───┐  1   n ┌┴──┴───┐                   │
│              │  Cliente  │───◇───│ Artículo│                  │
│              └───────────┘ compra └────────┘                  │
│                                                                │
│   Símbolos:                                                    │
│   ┌─────────┐ = Entidad                                       │
│   ○         = Atributo                                        │
│   ◎         = Atributo clave (PK)                             │
│   ◇         = Relación                                        │
│                                                                │
│   Cardinalidades:                                              │
│   1  = exactamente uno                                         │
│   n (o m) = muchos                                             │
│                                                                │
└────────────────────────────────────────────────────────────────┘
```

### Notación Moderna/Simplificada

```
┌────────────────────────────────────────────────────────────────┐
│             NOTACIÓN ERD SIMPLIFICADA                          │
├────────────────────────────────────────────────────────────────┤
│                                                                │
│   Común en exámenes: Atributos directamente en la entidad     │
│                                                                │
│   ┌─────────────────┐           ┌─────────────────┐          │
│   │     Cliente     │           │      Pedido     │          │
│   ├─────────────────┤           ├─────────────────┤          │
│   │ *NoCliente      │ 1       n │ *NoPedido       │          │
│   │  Nombre         │◄──────────│  NoCliente (FK) │          │
│   │  PrimerNombre   │           │  Fecha          │          │
│   │  CódigoPostal   │           │  Total          │          │
│   │  Ciudad         │           │                 │          │
│   └─────────────────┘           └─────────────────┘          │
│                                                                │
│   * = Clave Primaria                                          │
│   (FK) = Clave Foránea                                        │
│   1 ◄──── n = Un cliente tiene muchos pedidos                 │
│                                                                │
└────────────────────────────────────────────────────────────────┘
```

### Cardinalidades (Tipos de Relación)

```
┌────────────────────────────────────────────────────────────────┐
│                   CARDINALIDADES                               │
├────────────────────────────────────────────────────────────────┤
│                                                                │
│   1:1 (Uno a Uno)                                              │
│   ┌─────────┐    1        1    ┌─────────┐                   │
│   │ Persona │◄────────────────►│Pasaporte│                   │
│   └─────────┘                  └─────────┘                   │
│   Cada persona tiene exactamente un pasaporte                  │
│   Cada pasaporte pertenece a exactamente una persona           │
│                                                                │
│   ──────────────────────────────────────────────────────────  │
│                                                                │
│   1:n (Uno a Muchos) - LA MÁS COMÚN                           │
│   ┌─────────┐    1        n    ┌──────────────┐              │
│   │ Cliente │◄─────────────────│    Pedido    │              │
│   └─────────┘                  └──────────────┘              │
│   Un cliente tiene muchos pedidos                              │
│   Un pedido pertenece a exactamente un cliente                 │
│                                                                │
│   ──────────────────────────────────────────────────────────  │
│                                                                │
│   m:n (Muchos a Muchos)                                        │
│   ┌─────────┐    n        m    ┌─────────┐                   │
│   │Estudiante◄────────────────►│  Curso  │                   │
│   └─────────┘                  └─────────┘                   │
│   Un estudiante toma muchos cursos                             │
│   Un curso es tomado por muchos estudiantes                    │
│   → ¡Se resuelve mediante tabla intermedia!                    │
│                                                                │
└────────────────────────────────────────────────────────────────┘
```

### Notación Min-Max (Pata de Gallo)

```
┌────────────────────────────────────────────────────────────────┐
│                NOTACIÓN PATA DE GALLO                          │
├────────────────────────────────────────────────────────────────┤
│                                                                │
│   Símbolos en los extremos de la relación:                     │
│                                                                │
│   ───────│       exactamente uno (1)                           │
│   ───────○       cero o uno (0..1)                             │
│   ──────<│       uno o más (1..*)                              │
│   ──────<○       cero o más (0..*)                             │
│                                                                │
│   Ejemplo:                                                     │
│   ┌─────────┐         ┌──────────────┐                       │
│   │ Cliente │──────│──○<──│   Pedido   │                      │
│   └─────────┘         └──────────────┘                       │
│        │    │                   │                             │
│        │    │                   └── Un cliente tiene          │
│        │    │                       0 o más pedidos           │
│        │    └── Un pedido pertenece a exactamente 1 cliente   │
│        └── Cliente                                            │
│                                                                │
│   Dirección de lectura: ¡Leer alejándose de la entidad!       │
│                                                                │
└────────────────────────────────────────────────────────────────┘
```

### Del ERD a las Tablas

```
┌────────────────────────────────────────────────────────────────┐
│               DEL ERD A LA BASE DE DATOS                       │
├────────────────────────────────────────────────────────────────┤
│                                                                │
│   Reglas de conversión:                                        │
│                                                                │
│   1. Cada entidad se convierte en una tabla                   │
│   2. Cada atributo se convierte en una columna                │
│   3. La clave primaria se adopta                              │
│   4. Para 1:n: FK en la tabla "n"                             │
│   5. Para m:n: Nueva tabla intermedia con dos FKs             │
│                                                                │
│   Ejemplo ERD:                                                 │
│   ┌─────────┐  1       n  ┌─────────┐                        │
│   │Departamento◄──────────│Empleado │                        │
│   └─────────┘             └─────────┘                        │
│                                                                │
│   Tablas resultantes:                                          │
│   ┌──────────────────┐    ┌────────────────────┐             │
│   │   Departamento   │    │     Empleado       │             │
│   ├──────────────────┤    ├────────────────────┤             │
│   │ NoDept PK        │◄───│ NoEmpleado PK      │             │
│   │ Descripción      │    │ Nombre             │             │
│   └──────────────────┘    │ NoDept FK ─────────┘             │
│                           └────────────────────┘             │
│                                                                │
└────────────────────────────────────────────────────────────────┘
```

### Consejos para Crear ERDs

| Paso | Acción |
|------|--------|
| 1 | Identificar entidades (sustantivos) |
| 2 | Encontrar atributos para cada entidad |
| 3 | Definir claves primarias |
| 4 | Determinar relaciones entre entidades |
| 5 | Establecer cardinalidades (1:1, 1:n, m:n) |
| 6 | Resolver relaciones m:n en tablas intermedias |

## Términos Clave

| Término | Explicación |
|---------|-------------|
| **ERD** | Diagrama Entidad-Relación |
| **Entidad** | Una cosa/objeto sobre la cual se almacenan datos |
| **Atributo** | Propiedad de una entidad |
| **Relación** | Conexión entre entidades |
| **Cardinalidad** | Proporción de cantidad (1:1, 1:n, m:n) |

## Consejos para el Examen

### Preguntas Comunes del Examen

- Leer el ERD y explicar las relaciones
- ¿Qué cardinalidad existe entre las entidades?
- Crear un ERD para un escenario dado

### Importante Recordar

- **Entidad** = Rectángulo
- **1:n** = Uno a Muchos (relación más común)
- **m:n** = Se resuelve mediante tabla intermedia
- **FK siempre en "n"** (el lado "muchos")
- ERD muestra estructura, no datos

## Ejercicios Prácticos

### Ejercicio 1
¿Qué cardinalidad existe entre "Autor" y "Libro" (un libro tiene múltiples autores, un autor escribe múltiples libros)?

**Solución:**
**m:n (Muchos a Muchos):** Un autor puede escribir múltiples libros, y un libro puede tener múltiples autores.

Esta relación debe resolverse mediante una tabla intermedia:
```
Autor ◄───── Libro_Autor ─────► Libro
      n:1                 1:n
```

La tabla intermedia "Libro_Autor" contiene las claves foráneas de ambas tablas.

### Ejercicio 2
Crear un ERD simple para una clínica médica con: Paciente, Doctor, Cita

**Solución:**
```
┌─────────────────┐           ┌─────────────────┐
│     Paciente    │           │     Doctor      │
├─────────────────┤           ├─────────────────┤
│ *NoPaciente     │           │ *NoDoctor       │
│  Nombre         │           │  Nombre         │
│  FechaNacimiento│           │  Especialidad   │
│  Teléfono       │           │                 │
└────────┬────────┘           └────────┬────────┘
         │ 1                           │ 1
         │                             │
         │ n                           │ n
         └──────────┬──────────────────┘
                    │
           ┌────────┴────────┐
           │      Cita       │
           ├─────────────────┤
           │ *NoCita         │
           │  NoPaciente (FK)│
           │  NoDoctor (FK)  │
           │  Fecha          │
           │  Hora           │
           └─────────────────┘
```

### Ejercicio 3
¿Por qué debe resolverse una relación m:n mediante una tabla intermedia?

**Solución:**
Una relación m:n no puede representarse directamente en una base de datos relacional porque:

1. **Un atributo no puede contener múltiples valores:** Si se quisieran almacenar múltiples claves foráneas en una tabla (p.ej., todos los cursos de un estudiante), violaría la regla de atomicidad.

2. **Redundancia:** Se tendrían que crear muchas columnas o almacenar datos múltiples veces.

La solución es una **tabla intermedia** que contiene dos claves foráneas y así establece el vínculo.
