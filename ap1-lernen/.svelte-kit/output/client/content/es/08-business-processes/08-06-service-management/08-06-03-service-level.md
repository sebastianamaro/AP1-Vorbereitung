# Nivel de Servicio (Niveles de Soporte)

## Objetivos de Aprendizaje

Después de esta sección, serás capaz de:
- Distinguir los tres niveles de soporte
- Explicar las tareas y competencias de cada nivel
- Describir el proceso de escalación
- Explicar la importancia de la escalación en el soporte TI

---

## Contenido Principal

### El Modelo de Soporte de 3 Niveles

```

│                    NIVELES DE SOPORTE                                │

│                                                                      │

│                       │    3ER NIVEL        │  Expertos             │
│                       │    Especialistas    │  Desarrolladores      │
│                       │    Proveedor        │  Análisis profundo    │
│                       └──────────┬──────────┘                       │
│                                  │ Escalación                       │
│                       ┌──────────┴──────────┐                       │
│                       │    2DO NIVEL        │  Expertos en materia  │
│                       │    Expertos         │  Problemas complejos  │
│                       │                     │  Administración       │
│                       └──────────┬──────────┘                       │
│                                  │ Escalación                       │
```

### Soporte de 1er Nivel (Helpdesk/Service Desk)

```
┌─────────────────────────────────────────────────────────────────────┐
│                    SOPORTE DE 1ER NIVEL                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  TAREAS:                                                            │
│  ──────────────────────────────────────────────────────────────     │
│  → Primer punto de contacto para todas las solicitudes              │
│  → Recepción y documentación (crear ticket)                         │
│  → Categorización y priorización                                    │
│  → Resolución de problemas estándar                                 │
│  → Reenvío de casos complejos                                       │
│                                                                      │
│  SOLUCIONES TÍPICAS:                                                │
│  ──────────────────────────────────────────────────────────────     │
│  → Restablecer contraseña                                           │
│  → Reiniciar PC                                                     │
│  → Problemas de software estándar                                   │
│  → Problemas de impresora (papel, tóner)                            │
│  → Proporcionar instrucciones (soluciones conocidas)                │
│                                                                      │
│  HERRAMIENTAS:                                                      │
│  ──────────────────────────────────────────────────────────────     │
│  → Sistema de tickets                                               │
│  → Base de conocimientos                                            │
│  → Acceso remoto (soporte remoto)                                   │
│  → FAQ e instrucciones                                              │
│                                                                      │
│  OBJETIVO: Resolver tantas solicitudes INMEDIATAMENTE como sea     │
│  posible (FCR)                                                      │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

```
│                                                                      │
│  TAREAS:                                                            │
```

### Comparación de Niveles de Soporte

| Criterio | 1er Nivel | 2do Nivel | 3er Nivel |
|-----------|-----------|-----------|-----------|
| **Contacto** | Directo con usuario | Ticket | Ticket/Proyecto |
| **Problemas** | Estándar | Complejos | Altamente complejos |
| **Tiempo de resolución** | Minutos | Horas-Días | Días-Semanas |
| **Costo** | Bajo | Medio | Alto |
| **Calificación** | Conocimiento básico | Expertos en materia | Especialistas |
| **Porcentaje de tickets** | ~70-80% | ~15-25% | ~5% |

### Proceso de Escalación

```
┌─────────────────────────────────────────────────────────────────────┐
│                    ESCALACIÓN                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  ESCALACIÓN FUNCIONAL (horizontal)                                  │
│  ──────────────────────────────────────────────────────────────     │
│  → Reenvío al equipo responsable                                    │
│  → Ej. de soporte PC a equipo de red                                │
│                                                                      │
│  ESCALACIÓN JERÁRQUICA (vertical)                                   │
│  ──────────────────────────────────────────────────────────────     │
│  → Reenvío a nivel superior                                         │
│  → Ej. de 1er nivel a 2do nivel                                     │
│  → O: Involucrar a supervisores                                     │
│                                                                      │
│  ¿CUÁNDO ESCALAR?                                                   │
│  ──────────────────────────────────────────────────────────────     │
│  → El problema excede la competencia propia                         │
│  → El plazo de SLA está en riesgo                                   │
│  → El cliente está insatisfecho/escala él mismo                     │
│  → Sistema crítico afectado                                         │
│                                                                      │
│  IMPORTANTE:                                                        │
│  → Documentación en el ticket (¿qué se intentó?)                    │
│  → Transferir toda la información relevante                         │
│  → No escalar "a ciegas"                                            │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Punto Único de Contacto (SPOC)

```
┌─────────────────────────────────────────────────────────────────────┐
│                    PRINCIPIO SPOC                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  El usuario tiene UN punto de contacto: 1er Nivel                   │
│                                                                      │
│                    ┌─────────────────┐                              │
│                    │     USUARIO     │                              │
│                    └────────┬────────┘                              │
│                             │                                       │
│                             ▼                                       │
│                    ┌─────────────────┐                              │
│                    │   1ER NIVEL     │ ← Punto Único de Contacto    │
│                    │   (SPOC)        │                              │
│                    └────────┬────────┘                              │
│                             │                                       │
│            ┌────────────────┼────────────────┐                      │
│            ▼                ▼                ▼                      │
│      ┌──────────┐    ┌──────────┐    ┌──────────┐                   │
│      │ 2do Nivel│    │ 2do Nivel│    │ 3er Nivel│                   │
│      │ Red      │    │ Servidor │    │ Des      │                   │
│      └──────────┘    └──────────┘    └──────────┘                   │
│                                                                      │
│  Ventajas:                                                          │
│  → El usuario no necesita saber quién es responsable               │
│  → No hay solicitudes "mal dirigidas"                               │
│  → Documentación central                                            │
│  → Comunicación consistente                                         │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

## Términos Clave

| Término | Explicación |
|------|-------------|
| **1er Nivel** | Primer punto de contacto, problemas estándar |
| **2do Nivel** | Expertos en materia, problemas complejos |
| **3er Nivel** | Especialistas, desarrolladores, proveedores |
| **Escalación** | Reenvío a otro nivel/equipo |
| **SPOC** | Punto Único de Contacto |
| **First Call Resolution** | Resolución en primer contacto |
| **Base de conocimientos** | Base de datos para soluciones conocidas |

---

## Consejos para el Examen

1. **Distinción de niveles:**
   - 1ro = Estándar, rápido, barato
   - 2do = Complejo, expertos
   - 3ro = Muy complejo, desarrolladores/proveedores

2. **First Call Resolution:**
   - ¡Cuanto más se resuelva en 1er nivel, mejor!
   - Objetivo: >70%

3. **Principio SPOC:**
   - El usuario tiene UN punto de contacto
   - El reenvío interno es transparente

## Ejercicios Prácticos

### Ejercicio 1: Asignar Niveles

¿Qué nivel de soporte es responsable de los siguientes problemas?

a) Contraseña olvidada
b) El disco duro del servidor está defectuoso
c) Bug en el software ERP desarrollado internamente
d) El driver de impresora no funciona después de actualización de Windows
e) Microsoft Office no se activa

<details>
<summary>Mostrar solución</summary>

a) **1er Nivel** - Procedimiento estándar (restablecimiento de contraseña)
b) **2do Nivel** - Reemplazo de hardware en servidor
c) **3er Nivel** - Desarrollo interno, se necesita desarrollador
d) **1er/2do Nivel** - Reinstalación de driver (1ro), para problemas complejos 2do
e) **1er Nivel** - Activación estándar, posiblemente soporte de Microsoft (3ro)

</details>

### Ejercicio 2: Decisión de Escalación

Un empleado reporta: "Mi PC está muy lento." El soporte de 1er nivel ya ha:
- Reiniciado el PC
- Cerrado programas innecesarios
- Ejecutado antivirus (sin hallazgos)
- Verificado disco duro por errores (OK)

El problema persiste. ¿Cuál es el siguiente paso?

<details>
<summary>Mostrar solución</summary>

**Se recomienda escalación a 2do Nivel**

Razón:
- Se han realizado las medidas estándar
- El problema persiste
- Se requiere análisis más profundo (ej. uso de recursos, red, procesos en segundo plano)

**Documentar para escalación:**
- Todas las medidas realizadas
- Describir síntomas exactamente
- Información del sistema relevante (uso de RAM, CPU)

**El 2do Nivel podría verificar:**
- Análisis detallado de procesos
- Rendimiento de red
- Diagnóstico de hardware
- Conflictos de software

</details>

### Ejercicio 3: Principio SPOC

Un usuario tiene un problema complejo que ya fue escalado a 2do Nivel. Vuelve a llamar a 1er Nivel y pregunta por el estado.

¿Cómo debe responder el empleado de 1er nivel?

<details>
<summary>Mostrar solución</summary>

**Respuesta correcta:**

1. **Verificar en el sistema de tickets:**
   - Revisar estado actual
   - Leer últimas actividades
   - Identificar persona asignada

2. **Informar al usuario:**
   - "Su ticket está siendo procesado actualmente por el equipo de 2do nivel."
   - Explicar estado (ej. "Se está trabajando en una solución")
   - Indicar duración esperada (si se conoce)

3. **NO:**
   - Referir al usuario a 2do nivel ("Llámelos")
   - Afirmar que no sabe nada
   - Hacer promesas que no se pueden cumplir

**El principio SPOC significa:**
El 1er Nivel permanece como punto de contacto, aunque otros estén trabajando en la solución. Funcionan como interfaz de comunicación con el usuario.

</details>
