# Direccionamiento IPv4

## Objetivos de Aprendizaje

Después de este capítulo, deberías ser capaz de:
- Comprender la estructura de una dirección IPv4
- Distinguir entre porción de red y porción de host
- Conocer las clases IP
- Distinguir entre direcciones privadas y públicas
- Comprender las máscaras de subred

---

## Contenido Principal

### Estructura de una Dirección IPv4

```
+--------------------------------------------------------------------+
|                    DIRECCIÓN IPv4                                   |
+--------------------------------------------------------------------+
|                                                                     |
|   IPv4 = Internet Protocol Version 4                                |
|          (Protocolo de Internet Versión 4)                         |
|                                                                     |
|   +-------------------------------------------------------------+  |
|   | Una dirección IPv4 es un número de 32 bits que identifica   |  |
|   | de forma única un dispositivo en la red.                     |  |
|   +-------------------------------------------------------------+  |
|                                                                     |
|   Representación:                                                   |
|   +-------------------------------------------------------------+  |
|   |                                                              |  |
|   |   Decimal (Notación Decimal con Puntos):                    |  |
|   |                                                              |  |
|   |         192  .  168  .  1   .  100                          |  |
|   |          |       |      |      |                            |  |
|   |          +-------+------+------+                            |  |
|   |              4 octetos (8 bits cada uno)                    |  |
|   |                                                              |  |
|   |   Binario (32 bits):                                        |  |
|   |                                                              |  |
|   |   11000000.10101000.00000001.01100100                       |  |
|   |      192      168       1       100                         |  |
|   |                                                              |  |
|   +-------------------------------------------------------------+  |
|                                                                     |
|   Rango de valores por octeto: 0 - 255 (8 bits = 2^8 = 256 valores)|
|                                                                     |
|   Total direcciones posibles: 2^32 = aprox. 4.300 millones         |
|                                                                     |
+--------------------------------------------------------------------+
```

### Porciones de Red y Host

```
+--------------------------------------------------------------------+
|                PORCIONES DE RED Y HOST                              |
+--------------------------------------------------------------------+
|                                                                     |
|   Cada dirección IP consta de dos partes:                          |
|                                                                     |
|   +-------------------------------------------------------------+  |
|   |                                                              |  |
|   |   Dirección IP:  192 . 168 . 1   . 100                      |  |
|   |                  -----------   -----                        |  |
|   |                  PORCIÓN RED   PORCIÓN HOST                 |  |
|   |                                                              |  |
|   |   Porción de red: Identifica la red                         |  |
|   |                   (como el nombre de la calle)              |  |
|   |                                                              |  |
|   |   Porción de host: Identifica el dispositivo en la red     |  |
|   |                    (como el número de casa)                 |  |
|   |                                                              |  |
|   +-------------------------------------------------------------+  |
|                                                                     |
|   La MÁSCARA DE SUBRED determina la división:                       |
|                                                                     |
|   +-------------------------------------------------------------+  |
|   |                                                              |  |
|   |   Dirección IP:   192.168.1.100                             |  |
|   |   Máscara subred: 255.255.255.0                             |  |
|   |                                                              |  |
|   |   Binario:                                                   |  |
|   |   IP:      11000000.10101000.00000001.01100100              |  |
|   |   Máscara: 11111111.11111111.11111111.00000000              |  |
|   |            --------------------------  --------              |  |
|   |            RED (24 bits)               HOST (8 bits)         |  |
|   |                                                              |  |
|   |   Bits en 1 = Porción de red                                |  |
|   |   Bits en 0 = Porción de host                               |  |
|   |                                                              |  |
|   +-------------------------------------------------------------+  |
|                                                                     |
+--------------------------------------------------------------------+
```

### Clases de Direcciones IP (Histórico)

```
+--------------------------------------------------------------------+
|                    CLASES DE DIRECCIONES IP                         |
+--------------------------------------------------------------------+
|                                                                     |
|   (Concepto histórico, ahora reemplazado por CIDR)                 |
|                                                                     |
|   Clase | Rango             | Máscara Defecto| Redes/Hosts         |
|   ------+-------------------+----------------+---------------------+
|   A     | 1.0.0.0 -         | 255.0.0.0      | 126 redes,          |
|         | 126.255.255.255   | /8             | 16 millones hosts   |
|   ------+-------------------+----------------+---------------------+
|   B     | 128.0.0.0 -       | 255.255.0.0    | 16.384 redes,       |
|         | 191.255.255.255   | /16            | 65.534 hosts        |
|   ------+-------------------+----------------+---------------------+
|   C     | 192.0.0.0 -       | 255.255.255.0  | 2 millones redes,   |
|         | 223.255.255.255   | /24            | 254 hosts           |
|   ------+-------------------+----------------+---------------------+
|   D     | 224.0.0.0 -       | -              | Multicast           |
|         | 239.255.255.255   |                |                     |
|   ------+-------------------+----------------+---------------------+
|   E     | 240.0.0.0 -       | -              | Reservado           |
|         | 255.255.255.255   |                |                     |
|                                                                     |
|   Reconocimiento de clase por primer octeto:                       |
|   +-------------------------------------------------------------+  |
|   | Clase A: 1 - 126    (comienza con 0xxx)                     |  |
|   | Clase B: 128 - 191  (comienza con 10xx)                     |  |
|   | Clase C: 192 - 223  (comienza con 110x)                     |  |
|   +-------------------------------------------------------------+  |
|                                                                     |
+--------------------------------------------------------------------+
```

### Direcciones Privadas y Públicas

```
+--------------------------------------------------------------------+
|              DIRECCIONES PRIVADAS vs. PÚBLICAS                      |
+--------------------------------------------------------------------+
|                                                                     |
|   DIRECCIONES IP PÚBLICAS                                           |
|   +------------------------------------------------------------+   |
|   | - Globalmente únicas                                        |   |
|   | - Enrutadas en Internet                                     |   |
|   | - Asignadas por el ISP (Proveedor de Servicios Internet)   |   |
|   | - Cantidad limitada (aprox. 4.300 millones)                |   |
|   +------------------------------------------------------------+   |
|                                                                     |
|   DIRECCIONES IP PRIVADAS                                           |
|   +------------------------------------------------------------+   |
|   | - Solo válidas dentro de redes locales                      |   |
|   | - NO enrutadas en Internet                                  |   |
|   | - Pueden reutilizarse en cualquier red                      |   |
|   | - Conexión a Internet vía NAT (Network Address              |   |
|   |   Translation)                                              |   |
|   |                                                             |   |
|   |   RANGOS DE DIRECCIONES PRIVADAS (RFC 1918):                |   |
|   |   +-----------------------------------------------------+   |   |
|   |   | Clase A:  10.0.0.0    - 10.255.255.255   /8         |   |   |
|   |   | Clase B:  172.16.0.0  - 172.31.255.255   /12        |   |   |
|   |   | Clase C:  192.168.0.0 - 192.168.255.255  /16        |   |   |
|   |   +-----------------------------------------------------+   |   |
|   |                                                             |   |
|   |   ! ¡MEMORIZAR estos rangos!                                |   |
|   +------------------------------------------------------------+   |
|                                                                     |
|   Ejemplo:                                                          |
|   +------------------------------------------------------------+   |
|   |                                                             |   |
|   |   Red doméstica          Internet          Servidor web    |   |
|   |   192.168.1.x      NAT                    203.0.113.50     |   |
|   |                +----------+                                 |   |
|   |   +-----+     | Router   |     +--------------------+      |   |
|   |   | PC  |---->|          |---->|     Internet       |      |   |
|   |   |.100 |     | IP       |     |                    |      |   |
|   |   +-----+     | Pública  |     +--------------------+      |   |
|   |               +----------+                                  |   |
|   |   IP Privada -> se traduce a IP pública                    |   |
|   |                                                             |   |
|   +------------------------------------------------------------+   |
|                                                                     |
+--------------------------------------------------------------------+
```

### Direcciones Especiales

```
+--------------------------------------------------------------------+
|                   DIRECCIONES IP ESPECIALES                         |
+--------------------------------------------------------------------+
|                                                                     |
|   127.0.0.1 (Localhost/Loopback)                                    |
|   +------------------------------------------------------------+   |
|   | - Se refiere al propio ordenador                            |   |
|   | - Para probar servicios locales                             |   |
|   | - Los paquetes de datos nunca salen del dispositivo         |   |
|   +------------------------------------------------------------+   |
|                                                                     |
|   0.0.0.0                                                           |
|   +------------------------------------------------------------+   |
|   | - "Dirección desconocida" o "todas las interfaces"          |   |
|   | - Usado en solicitudes DHCP                                 |   |
|   +------------------------------------------------------------+   |
|                                                                     |
|   255.255.255.255 (Broadcast)                                       |
|   +------------------------------------------------------------+   |
|   | - Envía a TODOS los dispositivos en la red local            |   |
|   | - No se reenvía más allá de los routers                     |   |
|   +------------------------------------------------------------+   |
|                                                                     |
|   Dirección de red (todos los bits de host = 0)                     |
|   +------------------------------------------------------------+   |
|   | - ej. 192.168.1.0 con /24                                   |   |
|   | - Identifica la red en sí                                   |   |
|   | - NO asignable a dispositivos                               |   |
|   +------------------------------------------------------------+   |
|                                                                     |
|   Dirección de broadcast (todos los bits de host = 1)               |
|   +------------------------------------------------------------+   |
|   | - ej. 192.168.1.255 con /24                                 |   |
|   | - Envía a todos los dispositivos en la subred               |   |
|   | - NO asignable a dispositivos                               |   |
|   +------------------------------------------------------------+   |
|                                                                     |
+--------------------------------------------------------------------+
```

### Notación CIDR

```
+--------------------------------------------------------------------+
|                      NOTACIÓN CIDR                                  |
+--------------------------------------------------------------------+
|                                                                     |
|   CIDR = Classless Inter-Domain Routing                             |
|          (Enrutamiento entre Dominios sin Clases)                  |
|                                                                     |
|   Formato: Dirección IP / longitud del prefijo                      |
|                                                                     |
|   +-------------------------------------------------------------+  |
|   |                                                              |  |
|   |   192.168.1.0/24                                            |  |
|   |              ^                                               |  |
|   |              +-- 24 bits para red, 8 bits para host         |  |
|   |                                                              |  |
|   +-------------------------------------------------------------+  |
|                                                                     |
|   Conversión a máscara de subred:                                   |
|                                                                     |
|   Prefijo | Máscara Subred    | Hosts Disponibles                  |
|   --------+-------------------+--------------------------------------+
|   /8      | 255.0.0.0         | 16.777.214                          |
|   /16     | 255.255.0.0       | 65.534                              |
|   /24     | 255.255.255.0     | 254                                 |
|   /25     | 255.255.255.128   | 126                                 |
|   /26     | 255.255.255.192   | 62                                  |
|   /27     | 255.255.255.224   | 30                                  |
|   /28     | 255.255.255.240   | 14                                  |
|   /29     | 255.255.255.248   | 6                                   |
|   /30     | 255.255.255.252   | 2                                   |
|   /32     | 255.255.255.255   | 1 (host único)                      |
|                                                                     |
|   Fórmula: Hosts = 2^(32-prefijo) - 2                               |
|            (menos 2 para direcciones de red y broadcast)           |
|                                                                     |
+--------------------------------------------------------------------+
```

## Términos Clave

| Término | Explicación |
|---------|-------------|
| **IPv4** | Internet Protocol Version 4 (32 bits) |
| **Octeto** | Sección de 8 bits de una dirección IP |
| **Máscara de subred** | Separa porción de red de porción de host |
| **CIDR** | Classless Inter-Domain Routing |
| **NAT** | Network Address Translation (Traducción de Direcciones de Red) |
| **Loopback** | Dirección local 127.0.0.1 |
| **Broadcast** | Dirección para todos en la red |

## Consejos para el Examen

### Preguntas Comunes del Examen

- Nombrar los rangos de direcciones IP privadas
- ¿Qué significa /24?
- ¿Diferencia entre IP pública/privada?

### Importante Recordar

- **32 bits** = 4 octetos de 8 bits cada uno
- **Rangos privados:** 10.x.x.x, 172.16-31.x.x, 192.168.x.x
- **127.0.0.1** = Localhost
- **/24** = 255.255.255.0 = 254 hosts
- Direcciones de red y broadcast NO utilizables
- **Hosts = 2^(bits de host) - 2**

## Ejercicios Prácticos

### Ejercicio 1
¿La dirección IP 192.168.5.100 es una dirección privada o pública?

**Solución:**
**Dirección privada**

Justificación: 192.168.x.x pertenece al rango de direcciones privadas de Clase C (192.168.0.0 - 192.168.255.255). Estas direcciones no se enrutan en Internet.

### Ejercicio 2
Una red tiene la dirección 192.168.10.0/24. ¿Cuántos hosts pueden direccionarse?

**Solución:**
**/24** significa 24 bits para la red, 8 bits para hosts.

Cálculo: 2^8 - 2 = 256 - 2 = **254 hosts**

Los 2 se restan por:
- Dirección de red (192.168.10.0)
- Dirección de broadcast (192.168.10.255)

### Ejercicio 3
Convierte la máscara de subred 255.255.255.192 a notación CIDR.

**Solución:**
255.255.255.192 en binario:
- 255 = 11111111
- 192 = 11000000

Binario: 11111111.11111111.11111111.11000000

Número de bits en 1: 24 + 2 = **26**

Respuesta: **/26**
