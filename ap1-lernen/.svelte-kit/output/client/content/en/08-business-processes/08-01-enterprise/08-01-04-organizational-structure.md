# Organizational Structure

## Learning Objectives

After this section, you will be able to:
- Distinguish the main organizational forms
- Explain advantages and disadvantages of each organizational form
- Read and create organizational charts
- Name the appropriate organizational form for different situations

---

## Core Content

### What is Organizational Structure?

**Organizational structure** describes the hierarchical structure of a company:
- Who reports to whom?
- How are departments organized?
- Who carries which responsibility?

It is graphically represented in an **organizational chart (org chart)**.

### The Three Classic Organizational Forms

```
┌─────────────────────────────────────────────────────────────────────┐
│                ORGANIZATIONAL FORMS OVERVIEW                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  1. SINGLE-LINE SYSTEM                                              │
│     → Each employee has ONE supervisor                              │
│     → Clear hierarchy                                               │
│                                                                      │
│  2. MULTI-LINE SYSTEM                                               │
│     → Employees can have MULTIPLE supervisors                       │
│     → Functional specialization                                     │
│                                                                      │
│  3. STAFF-LINE SYSTEM                                               │
│     → Single-line system WITH staff positions                       │
│     → Staff advises, does not decide                                │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### 1. Single-Line System

```
┌─────────────────────────────────────────────────────────────────────┐
│                    SINGLE-LINE SYSTEM                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│                    ┌───────────────────┐                            │
│                    │    Management     │                            │
│                    └─────────┬─────────┘                            │
│                              │                                       │
│            ┌─────────────────┼─────────────────┐                    │
│            │                 │                 │                    │
│            ▼                 ▼                 ▼                    │
│    ┌───────────────┐ ┌───────────────┐ ┌───────────────┐           │
│    │    Sales      │ │    Tech       │ │   Admin       │           │
│    └───────┬───────┘ └───────┬───────┘ └───────┬───────┘           │
│            │                 │                 │                    │
│        ┌───┴───┐         ┌───┴───┐         ┌───┴───┐               │
│        │       │         │       │         │       │               │
│        ▼       ▼         ▼       ▼         ▼       ▼               │
│     ┌─────┐ ┌─────┐   ┌─────┐ ┌─────┐   ┌─────┐ ┌─────┐           │
│     │ E1  │ │ E2  │   │ E3  │ │ E4  │   │ E5  │ │ E6  │           │
│     └─────┘ └─────┘   └─────┘ └─────┘   └─────┘ └─────┘           │
│                                                                      │
│  Principle: One employee → One supervisor                           │
│             (Unity of command)                                      │
│                                                                      │
│  Chain of command: Instructions only through the direct line        │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

| Advantages | Disadvantages |
|------------|---------------|
| Clear responsibilities | Long communication paths |
| Simple structure | Manager overload |
| Unambiguous accountability | Low flexibility |
| Easy to understand | Information loss along the chain |

**Suitable for:** Small and medium-sized enterprises, government agencies

### 2. Multi-Line System (Functional System)

```
┌─────────────────────────────────────────────────────────────────────┐
│                    MULTI-LINE SYSTEM                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│                    ┌───────────────────┐                            │
│                    │    Management     │                            │
│                    └─────────┬─────────┘                            │
│                              │                                       │
│            ┌─────────────────┼─────────────────┐                    │
│            │                 │                 │                    │
│            ▼                 ▼                 ▼                    │
│    ┌───────────────┐ ┌───────────────┐ ┌───────────────┐           │
│    │ Functional    │ │ Functional    │ │ Functional    │           │
│    │ Area 1 (Tech) │ │ Area 2 (QA)   │ │ Area 3 (Proc.)│           │
│    └───────┬───────┘ └───────┬───────┘ └───────┬───────┘           │
│            │                 │                 │                    │
│            └─────────────────┼─────────────────┘                    │
│                              │                                       │
│                              ▼                                       │
│                         ┌─────────┐                                 │
│                         │ Employee │                                │
│                         │ receives │                                │
│                         │ directives│                               │
│                         │from ALL 3│                                │
│                         └─────────┘                                 │
│                                                                      │
│  Principle: One employee → MULTIPLE supervisors (functionally)      │
│             (Multiple command)                                      │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

| Advantages | Disadvantages |
|------------|---------------|
| Short communication paths | Unclear responsibilities |
| Functional specialization | Authority conflicts |
| Management relief | Contradictory instructions possible |
| Fast decisions | Difficult coordination |

**Suitable for:** Projects, specialized tasks

### 3. Staff-Line System

```
┌─────────────────────────────────────────────────────────────────────┐
│                    STAFF-LINE SYSTEM                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│                    ┌───────────────────┐                            │
│                    │    Management     │◄─── ┌─────────────┐        │
│                    └─────────┬─────────┘     │ STAFF UNIT  │        │
│                              │               │ (e.g., Legal,│       │
│                              │               │ IT Security) │       │
│            ┌─────────────────┼─────────────────┐  └─────────────┘   │
│            │                 │                 │                    │
│            ▼                 ▼                 ▼                    │
│    ┌───────────────┐ ┌───────────────┐ ┌───────────────┐           │
│    │    Sales      │◄┤    Tech       │ │   Admin       │           │
│    └───────┬───────┘ └───────┬───────┘◄───┌──────────┐ │           │
│            │                 │            │STAFF UNIT│ │           │
│        ┌───┴───┐         ┌───┴───┐       │(e.g., QM)│  │           │
│        │       │         │       │       └──────────┘  │           │
│        ▼       ▼         ▼       ▼                     │           │
│     ┌─────┐ ┌─────┐   ┌─────┐ ┌─────┐                  │           │
│     │ E1  │ │ E2  │   │ E3  │ │ E4  │                  │           │
│     └─────┘ └─────┘   └─────┘ └─────┘                  │           │
│                                                                      │
│  Staff unit:                                                        │
│  → Advises management                                               │
│  → Has NO authority to give directives                              │
│  → Supports with specialist tasks                                   │
│                                                                      │
│  Typical staff positions in IT:                                     │
│  → IT Security Officer                                              │
│  → Data Protection Officer                                          │
│  → Quality Management                                               │
│  → Legal Department                                                 │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

| Advantages | Disadvantages |
|------------|---------------|
| Clear hierarchy (like single-line) | Conflicts line vs. staff possible |
| Relief through specialists | Staff without responsibility |
| Expertise available | Can lead to bureaucracy |
| Advice without authority issues | Staff can be bypassed |

**Suitable for:** Larger companies where expertise is needed

### Comparison Table

| Criterion | Single-Line | Multi-Line | Staff-Line |
|-----------|-------------|------------|------------|
| **Supervisors** | One | Multiple | One + Staff |
| **Command path** | Clear | Multiple | Clear |
| **Communication** | Long | Short | Medium |
| **Flexibility** | Low | High | Medium |
| **Clarity** | High | Low | High |
| **Specialization** | Low | High | Medium |

## Key Terms

| Term | Explanation |
|------|-------------|
| **Organizational chart** | Graphical representation of organizational structure |
| **Single-line system** | Each employee has one supervisor |
| **Multi-line system** | Employees can have multiple supervisors |
| **Staff-line system** | Single-line + advisory staff positions |
| **Staff position** | Advisory function without authority to give directives |
| **Chain of command** | Official path through the hierarchy |
| **Authority** | Right to give instructions |
| **Matrix organization** | Combination of functional and divisional organization |

---

## Exam Tips

1. **Distinguishing feature:**
   - How many supervisors does an employee have?
   - Single-line = 1, Multi-line = multiple, Staff-line = 1 + advisors

2. **Remember staff position:**
   - Has NO authority to give directives!
   - Only advises, does not decide

3. **Typical exam questions:**
   - Interpret organizational charts
   - Name advantages/disadvantages
   - Choose appropriate organizational form for a scenario

4. **IT relevance:**
   - DPO and ISO are often staff positions
   - IT projects often in matrix organization

## Practice Exercises

### Exercise 1: Identifying Organizational Form

Which organizational form is present when:

a) An IT employee receives instructions from both the IT department head and the project manager
b) The IT director reports directly to management, the IT Security Officer advises management
c) Each employee has exactly one supervisor, there are no staff positions

<details>
<summary>Show Solution</summary>

a) **Multi-line system** (or matrix organization) - Multiple supervisors
b) **Staff-line system** - IT director in the line, ISO as staff
c) **Single-line system** - One supervisor, no staff

</details>

### Exercise 2: Create Organizational Chart

ConSystem GmbH has the following structure:
- Management
- Three departments: Sales, Tech, Admin
- An IT Security Officer advises management
- A Data Protection Officer also acts as an advisor

Which organizational form is present and why?

<details>
<summary>Show Solution</summary>

**Staff-line system**

Reasoning:
- Clear hierarchy: Management → Departments (single-line)
- Two staff positions: ISO and DPO
- Staff advises management but has no authority to give directives

### Exercise 3: Advantages and Disadvantages

An IT system integrator wants to grow and is considering whether to switch from a single-line system to a multi-line system.

a) Name two advantages that support the change.
b) Name two risks that could occur during the transition.
c) What alternative could be a compromise?

<details>
<summary>Show Solution</summary>

a) **Advantages of multi-line system:**
- Shorter communication paths (directly to functional supervisor)
- Better functional specialization possible
- Managers are relieved
- Faster decisions on technical matters

b) **Risks of the transition:**
- Contradictory instructions from different supervisors
- Unclear responsibilities can lead to conflicts
- Employees don't know whom to prioritize
- Coordination effort increases

c) **Compromise: Staff-line system**
- Maintains the clear hierarchy of the single-line system
- Adds staff positions for specialized topics (e.g., QM, IT Security)
- Expertise available without authority issues
- Employees still have one clear supervisor

</details>
