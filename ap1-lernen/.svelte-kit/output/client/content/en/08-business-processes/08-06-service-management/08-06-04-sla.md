# Service Level Agreement (SLA)

## Learning Objectives

After this section, you will be able to:
- Define and explain the term SLA
- Name important SLA metrics
- Calculate availability
- Distinguish between response time and resolution time

---

## Core Content

### What is an SLA?

A **Service Level Agreement (SLA)** is a contractual agreement between service provider and customer about the quality and scope of IT services.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    SERVICE LEVEL AGREEMENT                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  ┌─────────────────┐                      ┌─────────────────┐       │
│  │ SERVICE PROVIDER│  ←── Agreement ──→   │     CUSTOMER    │       │
│  │   (IT vendor)   │                      │     (client)    │       │
│  └─────────────────┘                      └─────────────────┘       │
│                                                                      │
│  SLA DEFINES:                                                       │
│  ──────────────────────────────────────────────────────────────     │
│  → WHAT is provided? (Scope of service)                            │
│  → HOW WELL is it provided? (Quality criteria)                     │
│  → HOW FAST is response? (Response times)                          │
│  → WHAT HAPPENS on non-compliance? (Consequences)                  │
│                                                                      │
│  TYPES OF SLAs:                                                     │
│  ──────────────────────────────────────────────────────────────     │
│  → External SLA: With customers/suppliers                          │
│  → Internal SLA: Between departments (OLA)                         │
│  → Underpinning Contract: With sub-contractors                     │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Components of an SLA

```
┌─────────────────────────────────────────────────────────────────────┐
│                    SLA COMPONENTS                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  1. CONTRACTING PARTIES                                             │
│     → Who is the service provider?                                  │
│     → Who is the customer?                                          │
│     → Contact persons on both sides                                 │
│                                                                      │
│  2. SERVICE DESCRIPTION                                             │
│     → Which services are covered?                                   │
│     → Which are NOT (important distinction!)                       │
│     → Service hours (e.g. 24/7, Mon-Fri 8am-6pm)                   │
│                                                                      │
│  3. SERVICE LEVELS (Quality metrics)                                │
│     → Availability (e.g. 99.9%)                                    │
│     → Response time (e.g. 1 hour)                                  │
│     → Resolution time (e.g. 4 hours)                               │
│     → Performance values                                            │
│                                                                      │
│  4. MEASUREMENT & REPORTING                                         │
│     → How is it measured?                                           │
│     → Which tools are used?                                         │
│     → How often is it reported?                                     │
│                                                                      │
│  5. CONSEQUENCES                                                    │
│     → Penalties for non-compliance                                  │
│     → Bonus for exceeding targets                                   │
│     → Escalation processes                                          │
│                                                                      │
│  6. DURATION & TERMINATION                                          │
│     → Contract start and end                                        │
│     → Notice periods                                                │
│     → Change procedures                                             │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Response Time vs. Resolution Time

```
┌─────────────────────────────────────────────────────────────────────┐
│           RESPONSE TIME vs. RESOLUTION TIME                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  Timeline:                                                          │
│  ═══════════════════════════════════════════════════════════════   │
│                                                                      │
│  Incident     First          Problem                                │
│  reported     response       resolved                               │
│     │           │              │                                    │
│     ▼           ▼              ▼                                    │
│     ●───────────●──────────────●                                    │
│     │           │              │                                    │
│     │◄─────────►│              │                                    │
│     │ RESPONSE  │              │                                    │
│     │   TIME    │              │                                    │
│     │                          │                                    │
│     │◄────────────────────────►│                                    │
│     │      RESOLUTION TIME     │                                    │
│                                                                      │
│  RESPONSE TIME:                                                     │
│  → Time until first qualified feedback                              │
│  → Confirmation that request was received                           │
│  → Employee has taken on the problem                                │
│  → NOT the time until resolution!                                   │
│                                                                      │
│  RESOLUTION TIME:                                                   │
│  → Time until the problem is actually fixed                         │
│  → Or: Workaround provided                                          │
│  → Depends on complexity and priority                               │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### SLA by Priority (Example)

| Priority | Description | Response Time | Resolution Time |
|----------|-------------|---------------|-----------------|
| **Critical** | Total outage, company standstill | 15 minutes | 4 hours |
| **High** | Important function disrupted | 1 hour | 8 hours |
| **Medium** | Limited use | 4 hours | 24 hours |
| **Low** | Minor impact | 8 hours | 5 business days |

### Calculating Availability

```
┌─────────────────────────────────────────────────────────────────────┐
│           AVAILABILITY                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  FORMULA:                                                           │
│  ──────────────────────────────────────────────────────────────     │
│                                                                      │
│                    Operating time - Downtime                        │
│  Availability = ────────────────────────────── × 100%               │
│                        Operating time                               │
│                                                                      │
│  OR:                                                                │
│                                                                      │
│                    Total time - Downtime                            │
│  Availability = ────────────────────────────── × 100%               │
│                        Total time                                   │
│                                                                      │
│  EXAMPLE CALCULATION:                                               │
│  ──────────────────────────────────────────────────────────────     │
│                                                                      │
│  Service time: 24/7 = 24 × 365 = 8,760 hours/year                  │
│  Downtime per year: 8.76 hours                                     │
│                                                                      │
│                    8,760 - 8.76                                     │
│  Availability = ──────────────── × 100% = 99.9%                    │
│                       8,760                                         │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### The "Nines" of Availability

```
┌─────────────────────────────────────────────────────────────────────┐
│           AVAILABILITY CLASSES                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  Availability │ Downtime/Year  │ Downtime/Month  │ Use Case        │
│  ══════════════╪════════════════╪═════════════════╪════════════════│
│  99%           │ 3.65 days      │ 7.2 hours       │ Basic           │
│  99.5%         │ 1.83 days      │ 3.6 hours       │ Standard        │
│  99.9%         │ 8.76 hours     │ 43.8 minutes    │ Elevated        │
│  99.99%        │ 52.6 minutes   │ 4.4 minutes     │ High            │
│  99.999%       │ 5.26 minutes   │ 26 seconds      │ Very High       │
│                                                                      │
│  MNEMONIC: "The Nines"                                              │
│  ──────────────────────────────────────────────────────────────     │
│  99.9%   = "Three Nines" = ~8.7 hours downtime/year                │
│  99.99%  = "Four Nines" = ~52 minutes downtime/year                │
│  99.999% = "Five Nines" = ~5 minutes downtime/year                 │
│                                                                      │
│  NOTE: Each additional "9" means ~10× less downtime                │
│        but also ~10× higher cost!                                   │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Consider Service Hours

```
┌─────────────────────────────────────────────────────────────────────┐
│           SERVICE HOURS                                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  SLA times only apply within the agreed service hours!             │
│                                                                      │
│  EXAMPLE 1: Service hours 24/7                                      │
│  ──────────────────────────────────────────────────────────────     │
│  → 168 hours/week (24h × 7 days)                                   │
│  → 1h response time applies ALWAYS (including nights, weekends)    │
│                                                                      │
│  EXAMPLE 2: Service hours Mon-Fri 8am-6pm                           │
│  ──────────────────────────────────────────────────────────────     │
│  → 50 hours/week (10h × 5 days)                                    │
│  → 1h response time applies ONLY during these times                │
│                                                                      │
│  PRACTICAL CASE:                                                    │
│  ──────────────────────────────────────────────────────────────     │
│  Incident reported: Friday 5:30 PM                                  │
│  Service hours: Mon-Fri 8am-6pm                                     │
│  Response time: 1 hour                                              │
│                                                                      │
│  → Remaining time Friday: 30 minutes                                │
│  → Response must occur by: Monday 8:30 AM                           │
│    (30 min on Fri + 30 min on Mon = 1 hour)                        │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Penalties (Contract Penalties)

```
┌─────────────────────────────────────────────────────────────────────┐
│           PENALTIES FOR SLA VIOLATION                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  When SLA is not met, contract penalties may be due:               │
│                                                                      │
│  TYPICAL ARRANGEMENTS:                                              │
│  ──────────────────────────────────────────────────────────────     │
│  → Credit on next invoice                                           │
│  → Percentage reduction of monthly fee                              │
│  → Free additional services                                         │
│  → Special termination right above certain threshold                │
│                                                                      │
│  EXAMPLE:                                                           │
│  ──────────────────────────────────────────────────────────────     │
│  Agreed availability: 99.9%                                        │
│  Actual availability: 99.5%                                        │
│  Monthly fee: €1,000                                               │
│                                                                      │
│  → Availability shortfall: 0.4%                                    │
│  → Penalty: 10% credit per 0.1% shortfall                          │
│  → 4 × 10% = 40% credit                                            │
│  → Credit: €400                                                    │
│                                                                      │
│  IMPORTANT:                                                         │
│  → Penalties are NOT compensation for consequential damages!       │
│  → Penalties are usually capped (e.g. max. 1 monthly fee)          │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

## Key Terms

| Term | Explanation |
|------|-------------|
| **SLA** | Service Level Agreement - Contractual agreement on service quality |
| **OLA** | Operational Level Agreement - Internal agreement between departments |
| **Response time** | Time until first qualified feedback |
| **Resolution time** | Time until problem is fixed |
| **Availability** | Percentage of actual operating time |
| **Penalty** | Contract penalty for SLA violation |
| **Service hours** | Time period during which service is available |

---

## Exam Tips

1. **Calculate availability:**
   - Formula: (Total time - Downtime) / Total time × 100%
   - "Three Nines" (99.9%) = approx. 8.7 hours downtime/year

2. **Response vs. Resolution time:**
   - Response time = First feedback
   - Resolution time = Problem fixed
   - Response time is ALWAYS shorter than resolution time

3. **Consider service hours:**
   - SLA times only apply within service hours!
   - Distinguish 24/7 vs. business hours

## Practice Exercises

### Exercise 1: Calculate Availability

A server has the following data:
- Service hours: 24/7
- Planned maintenance per year: 12 hours
- Unplanned outages per year: 4.5 hours

Calculate the availability. Planned maintenance does NOT count as downtime.

<details>
<summary>Show solution</summary>

**Calculation:**

Total time/year: 24 × 365 = 8,760 hours
Downtime (unplanned only): 4.5 hours

Availability = (8,760 - 4.5) / 8,760 × 100%
             = 8,755.5 / 8,760 × 100%
             = **99.95%**

This corresponds to "almost four nines" and exceeds the typical SLA of 99.9%.

</details>

### Exercise 2: Check Response Time

An incident is reported on Thursday at 5:00 PM.
- Service hours: Mon-Fri 8am-6pm
- Agreed response time: 2 hours

By when must IT Support respond?

<details>
<summary>Show solution</summary>

**Analysis:**

Report: Thursday 5:00 PM
Service hours end: 6:00 PM

Remaining service time Thursday: 1 hour
Missing time: 2h - 1h = 1 hour

This 1 hour continues on Friday.
Friday service hours start: 8:00 AM

**Answer: Response must occur by Friday 9:00 AM**

</details>

### Exercise 3: Evaluate SLA Violation

An IT service provider has agreed the following SLA:
- Monthly availability: 99.5%
- Penalty: 5% credit per 0.1% shortfall
- Maximum penalty: 25% of monthly fee
- Monthly fee: €2,000

In March, actual availability was 99.1%.

How much is the credit?

<details>
<summary>Show solution</summary>

**Calculation:**

Agreed availability: 99.5%
Actual availability: 99.1%
Shortfall: 99.5% - 99.1% = 0.4%

Penalty: 0.4% / 0.1% × 5% = 4 × 5% = 20%

Check maximum: 20% < 25% (OK, not capped)

Credit: €2,000 × 20% = **€400**

</details>

### Exercise 4: Assign SLA Parameters

Assign the following statements to the correct SLA parameter:

a) "The server must be reachable 99.9% of the time"
b) "For critical incidents, support responds within 15 minutes"
c) "Critical incidents are resolved within 4 hours"
d) "Support is available Monday to Friday, 7am-8pm"

<details>
<summary>Show solution</summary>

a) **Availability** - System operating time
b) **Response time** - Time until first feedback
c) **Resolution time** - Time until problem resolution
d) **Service hours** - Period of support availability

</details>
