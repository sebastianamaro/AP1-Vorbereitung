# Contract Components

## Learning Objectives

After this section, you will be able to:
- Explain how a contract is formed
- Distinguish between declaration of intent, offer, and acceptance
- Name essential contract components
- Explain the significance of general terms and conditions

---

## Core Content

### Formation of a Contract

```
┌─────────────────────────────────────────────────────────────────────┐
│                    CONTRACT FORMATION                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  A contract is formed by two matching                                │
│  DECLARATIONS OF INTENT:                                            │
│                                                                      │
│           OFFER                         ACCEPTANCE                   │
│        (Declaration 1)              (Declaration 2)                  │
│                                                                      │
│        ┌─────────────┐           ┌─────────────┐                    │
│        │             │           │             │                    │
│        │   Seller    │ ────────► │    Buyer    │                    │
│        │  "I offer   │           │  "I accept!"│                    │
│        │   ..."      │           │             │                    │
│        │             │           │             │                    │
│        └─────────────┘           └─────────────┘                    │
│                                                                      │
│                        CONTRACT                                      │
│                     is formed!                                       │
│                                                                      │
│  Prerequisites:                                                     │
│  → Legal capacity of both parties                                   │
│  → Contractual capacity                                             │
│  → Agreement of offer and acceptance                                │
│  → No form required (usually informal)                              │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Inquiry vs. Offer vs. Order

```
┌─────────────────────────────────────────────────────────────────────┐
│                    DISTINCTION                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  INQUIRY                                                            │
│  → "What does a laptop cost from you?"                              │
│  → NOT a declaration of intent                                      │
│  → Binds no one                                                     │
│                                                                      │
│  OFFER (Proposal)                                                   │
│  → "The laptop costs €899, delivery in 3 days"                      │
│  → Declaration of intent of the OFFEROR                             │
│  → Binds the offeror (binding to the offer)                         │
│                                                                      │
│  ORDER (Acceptance)                                                 │
│  → "I order the laptop under these conditions"                      │
│  → Declaration of intent of the BUYER                               │
│  → Contract is formed!                                              │
│                                                                      │
│  SPECIAL CASE: Order without prior offer                            │
│  → Order = Offer by the buyer                                       │
│  → Order confirmation = Acceptance by the seller                    │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Essential Contract Components

```
┌─────────────────────────────────────────────────────────────────────┐
│                    CONTRACT COMPONENTS                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  ESSENTIALIA NEGOTII (essential components)                         │
│  ──────────────────────────────────────────────────────────────     │
│  → Must be included, otherwise no contract!                         │
│                                                                      │
│  For purchase contract:                                             │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │  1. Contracting parties (Who?)                              │    │
│  │  2. Subject of purchase (What?)                             │    │
│  │  3. Purchase price (How much?)                              │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                                                                      │
│  NATURALIA NEGOTII (natural components)                             │
│  ──────────────────────────────────────────────────────────────     │
│  → Apply automatically if nothing else agreed                       │
│  → E.g. warranty, transfer of ownership                             │
│                                                                      │
│  ACCIDENTALIA NEGOTII (incidental components)                       │
│  ──────────────────────────────────────────────────────────────     │
│  → Individual agreements                                            │
│  → E.g. delivery date, special payment terms                        │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Contents of an IT Contract

| Component | Description | Example |
|-----------|-------------|---------|
| **Contracting parties** | Who signs the contract? | ConSystem GmbH and Customer AG |
| **Subject of performance** | What is delivered? | 10 laptops model XY |
| **Price** | How much does it cost? | €8,990 net |
| **Delivery date** | When is it delivered? | Within 5 working days |
| **Payment terms** | How is payment made? | 2% discount within 10 days |
| **Warranty** | Liability for defects | 24 months |
| **Liability** | Liability limitations | Max. order value |
| **T&C** | General conditions | Annex to contract |

### General Terms and Conditions (T&C)

```
┌─────────────────────────────────────────────────────────────────────┐
│                    T&C - GENERAL TERMS AND CONDITIONS               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  What are T&C?                                                      │
│  → Pre-formulated contract terms                                    │
│  → Apply to many contracts                                          │
│  → "Fine print"                                                     │
│                                                                      │
│  Inclusion of T&C (§ 305 BGB):                                      │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │  1. Express reference                                       │    │
│  │     → "Our T&C apply"                                       │    │
│  │                                                              │    │
│  │  2. Possibility to take notice                              │    │
│  │     → T&C must be accessible (display, website)             │    │
│  │                                                              │    │
│  │  3. Customer consent                                        │    │
│  │     → Customer must agree                                   │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                                                                      │
│  Invalid clauses (§§ 307-309 BGB):                                  │
│  → Surprising clauses                                               │
│  → Unreasonable disadvantage                                        │
│  → Violation of statutory prohibitions                              │
│                                                                      │
│  Example of invalid IT clause:                                      │
│  "All liability for damages from our software is excluded."        │
│  → Invalid for intent/gross negligence                              │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Formal Requirements

```
┌─────────────────────────────────────────────────────────────────────┐
│                    FORMAL REQUIREMENTS                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  PRINCIPLE: Freedom of contract = Freedom of form                   │
│  → Most contracts can be concluded orally                           │
│                                                                      │
│  EXCEPTIONS (legally prescribed):                                   │
│  ──────────────────────────────────────────────────────────────     │
│                                                                      │
│  Written form (§ 126 BGB):                                          │
│  → Signature required                                               │
│  → E.g. guarantees, employment references                           │
│                                                                      │
│  Text form (§ 126b BGB):                                            │
│  → Readable statement, person identifiable                          │
│  → E.g. email                                                       │
│  → E.g. cancellation policy in online retail                        │
│                                                                      │
│  Notarial certification:                                            │
│  → Notary required                                                  │
│  → E.g. real estate purchase, GmbH formation                        │
│                                                                      │
│  IT CONTRACTS: Usually informal!                                    │
│  → But: Written form recommended for evidence                       │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

## Key Terms

| Term | Explanation |
|------|-------------|
| **Declaration of intent** | Expression of a legally relevant will |
| **Offer** | Binding declaration of intent by the offeror |
| **Acceptance** | Agreement to the offer |
| **Essentialia negotii** | Essential contract components |
| **T&C** | General Terms and Conditions |
| **Freedom of form** | Contract can be concluded in any form |
| **Text form** | Written form without signature (e.g. email) |

## Exam Tips

1. **Remember contract formation:**
   - Offer + Acceptance = Contract

2. **T&C inclusion:**
   - Reference + Possibility to take notice + Consent

3. **Typical exam questions:**
   - When is a contract formed?
   - What belongs in a purchase contract?

## Practice Exercises

### Exercise 1: Contract Formation

ConSystem GmbH sends a customer an offer for 5 servers for €15,000. The customer replies by email: "We order the servers."

a) Has a contract been formed?
b) Which declarations of intent are present?

<details>
<summary>Show solution</summary>

a) **Yes, a contract has been formed.**

b) **Declarations of intent:**
- Offer from ConSystem: "5 servers for €15,000" (binds ConSystem)
- Acceptance by customer: "We order the servers" (binds the customer)
- Both declarations of intent match → Contract

</details>

### Exercise 2: T&C Inclusion

A customer orders software online from ConSystem. On the website it says in small print: "Our T&C apply". The T&C are only accessible via a small link at the bottom of the page.

Are the T&C validly included?

<details>
<summary>Show solution</summary>

**Examination of inclusion requirements:**

1. **Reference:** Present ("Our T&C apply")
2. **Possibility to take notice:** Questionable - link very small and hidden
3. **Consent:** Unclear - was the customer informed about T&C before order completion?

**Result:** The T&C may be invalidly included if:
- The link is not clearly visible
- No checkbox for "T&C read" was required
- The T&C were not viewable before contract conclusion

**Recommendation:** Place T&C link prominently and include checkbox for confirmation.

</details>

### Exercise 3: Essential Components

Check whether the following information is sufficient for a valid purchase contract for servers:

"ConSystem delivers servers to Firma Müller"

<details>
<summary>Show solution</summary>

**Analysis of essential components (Essentialia):**

| Component | Present? | Note |
|-----------|----------|------|
| Contracting parties | Yes | ConSystem and Firma Müller |
| Subject of purchase | Questionable | "Servers" - but which? Quantity? Type? |
| Purchase price | No | Missing completely! |

**Result:** The contract is **not valid** because:
- The subject of purchase is not precisely specified
- The purchase price is missing

**Correct would be:**
"ConSystem delivers 3 servers of type HP ProLiant DL380 Gen10 to Firma Müller for a total of €12,000 net."

</details>
