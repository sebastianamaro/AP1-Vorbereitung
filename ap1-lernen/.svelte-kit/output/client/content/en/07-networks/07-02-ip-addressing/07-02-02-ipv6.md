# IPv6 Addressing [NEW 2025]

## Learning Objectives

After this chapter, you should be able to:
- Understand the structure of an IPv6 address
- Know the differences from IPv4
- Read and abbreviate IPv6 addresses
- Know important IPv6 address types

---

## Core Content

### Why IPv6?

```
+--------------------------------------------------------------------+
|                    WHY IPv6?                                        |
+--------------------------------------------------------------------+
|                                                                     |
|   Problem with IPv4:                                                |
|   +-------------------------------------------------------------+  |
|   | - Only approx. 4.3 billion addresses (2^32)                 |  |
|   | - Worldwide devices: More than 20 billion!                  |  |
|   | - IPv4 addresses are exhausted (since 2011)                 |  |
|   | - Workarounds like NAT are complicated                      |  |
|   +-------------------------------------------------------------+  |
|                                                                     |
|   IPv6 solves the problem:                                          |
|   +-------------------------------------------------------------+  |
|   | - 128 bits instead of 32 bits                               |  |
|   | - 2^128 = approx. 340 sextillion addresses                  |  |
|   | - That's enough for every grain of sand on Earth!           |  |
|   +-------------------------------------------------------------+  |
|                                                                     |
|   Comparison:                                                       |
|   +-------------------------------------------------------------+  |
|   | IPv4:  4,294,967,296 addresses            (4.3 billion)     |  |
|   | IPv6:  340,282,366,920,938,463,463,374,607,431,768,211,456   |  |
|   |                                           (340 sextillion)  |  |
|   +-------------------------------------------------------------+  |
|                                                                     |
+--------------------------------------------------------------------+
```

### Structure of an IPv6 Address

```
+--------------------------------------------------------------------+
|                    IPv6 ADDRESS FORMAT                              |
+--------------------------------------------------------------------+
|                                                                     |
|   Basic structure:                                                  |
|   +-------------------------------------------------------------+  |
|   |                                                              |  |
|   |   - 128 bits long                                           |  |
|   |   - 8 groups of 16 bits each (4 hexadecimal digits)         |  |
|   |   - Groups separated by colon (:)                           |  |
|   |   - Hexadecimal representation (0-9, a-f)                   |  |
|   |                                                              |  |
|   +-------------------------------------------------------------+  |
|                                                                     |
|   Example:                                                          |
|   +-------------------------------------------------------------+  |
|   |                                                              |  |
|   |   2001:0db8:85a3:0000:0000:8a2e:0370:7334                   |  |
|   |   ----  ----  ----  ----  ----  ----  ----  ----            |  |
|   |    |     |     |     |     |     |     |     |              |  |
|   |    +-----+-----+-----+-----+-----+-----+-----+              |  |
|   |             8 groups of 4 hex digits each                   |  |
|   |                                                              |  |
|   |   Each group = 16 bits = 2 bytes                            |  |
|   |   8 groups x 16 bits = 128 bits                             |  |
|   |                                                              |  |
|   +-------------------------------------------------------------+  |
|                                                                     |
+--------------------------------------------------------------------+
```

### Abbreviating IPv6 Addresses

```
+--------------------------------------------------------------------+
|                  IPv6 SHORT NOTATION                                |
+--------------------------------------------------------------------+
|                                                                     |
|   IPv6 addresses can be abbreviated according to two rules:         |
|                                                                     |
|   RULE 1: Omit leading zeros                                        |
|   +------------------------------------------------------------+   |
|   |                                                             |   |
|   |   Before: 2001:0db8:0000:0042:0000:8a2e:0370:7334          |   |
|   |                 |    |    |         |    |                 |   |
|   |   After:  2001:db8:0:42:0:8a2e:370:7334                    |   |
|   |                                                             |   |
|   |   - 0db8 -> db8                                            |   |
|   |   - 0000 -> 0 (at least one digit remains)                 |   |
|   |   - 0042 -> 42                                             |   |
|   |   - 0370 -> 370                                            |   |
|   |                                                             |   |
|   +------------------------------------------------------------+   |
|                                                                     |
|   RULE 2: Replace one contiguous zero sequence with ::              |
|   +------------------------------------------------------------+   |
|   |                                                             |   |
|   |   Before: 2001:db8:0:0:0:0:0:1                             |   |
|   |                  | | | | |                                 |   |
|   |   After:  2001:db8::1                                      |   |
|   |                                                             |   |
|   |   ! IMPORTANT: :: may only be used ONCE per address!        |   |
|   |                                                             |   |
|   |   Example Localhost:                                        |   |
|   |   0000:0000:0000:0000:0000:0000:0000:0001                   |   |
|   |   -> ::1                                                   |   |
|   |                                                             |   |
|   +------------------------------------------------------------+   |
|                                                                     |
|   Complete example:                                                 |
|   +------------------------------------------------------------+   |
|   |                                                             |   |
|   |   Full:     2001:0db8:0000:0000:0000:0000:0000:0001        |   |
|   |   Rule 1:   2001:db8:0:0:0:0:0:1                           |   |
|   |   Rule 1+2: 2001:db8::1                                    |   |
|   |                                                             |   |
|   +------------------------------------------------------------+   |
|                                                                     |
+--------------------------------------------------------------------+
```

### IPv6 Address Types

```
+--------------------------------------------------------------------+
|                    IPv6 ADDRESS TYPES                               |
+--------------------------------------------------------------------+
|                                                                     |
|   UNICAST (Single address)                                          |
|   +------------------------------------------------------------+   |
|   | - Addresses exactly ONE device                              |   |
|   | - Like the normal address in IPv4                           |   |
|   |                                                             |   |
|   | Types:                                                      |   |
|   | - Global Unicast: Globally unique (like public IPv4)        |   |
|   |   Starts with: 2000::/3 (2xxx: to 3xxx:)                    |   |
|   |                                                             |   |
|   | - Link-Local: Only valid in local network                   |   |
|   |   Starts with: fe80::/10                                    |   |
|   |   (Automatically present on every device)                   |   |
|   |                                                             |   |
|   | - Unique Local: Private (like private IPv4)                 |   |
|   |   Starts with: fc00::/7 (fcxx: or fdxx:)                    |   |
|   +------------------------------------------------------------+   |
|                                                                     |
|   MULTICAST (Group address)                                         |
|   +------------------------------------------------------------+   |
|   | - Addresses a GROUP of devices                              |   |
|   | - Starts with: ff00::/8                                     |   |
|   | - Replaces broadcast (doesn't exist in IPv6!)               |   |
|   +------------------------------------------------------------+   |
|                                                                     |
|   ANYCAST                                                           |
|   +------------------------------------------------------------+   |
|   | - One address, multiple possible recipients                 |   |
|   | - Packet goes to the NEAREST recipient                      |   |
|   | - For load balancing and redundancy                         |   |
|   +------------------------------------------------------------+   |
|                                                                     |
|   ! No BROADCAST in IPv6! Replaced by multicast.                    |
|                                                                     |
+--------------------------------------------------------------------+
```

### Important IPv6 Addresses

```
+--------------------------------------------------------------------+
|                 IMPORTANT IPv6 ADDRESSES                            |
+--------------------------------------------------------------------+
|                                                                     |
|   ::1                                                               |
|   +------------------------------------------------------------+   |
|   | Loopback address (like 127.0.0.1 in IPv4)                   |   |
|   | Refers to the own computer                                  |   |
|   +------------------------------------------------------------+   |
|                                                                     |
|   ::                                                                |
|   +------------------------------------------------------------+   |
|   | Unspecified address (like 0.0.0.0 in IPv4)                  |   |
|   | "No address" or "all addresses"                             |   |
|   +------------------------------------------------------------+   |
|                                                                     |
|   fe80::/10                                                         |
|   +------------------------------------------------------------+   |
|   | Link-local addresses                                        |   |
|   | Automatically generated, only valid in local network        |   |
|   | Example: fe80::1a2b:3c4d:5e6f:7890                          |   |
|   +------------------------------------------------------------+   |
|                                                                     |
|   2001:db8::/32                                                     |
|   +------------------------------------------------------------+   |
|   | Documentation range (like 192.0.2.0 in IPv4)                |   |
|   | Reserved for examples and guides                            |   |
|   | Not used on the real Internet                               |   |
|   +------------------------------------------------------------+   |
|                                                                     |
+--------------------------------------------------------------------+
```

### IPv4 vs. IPv6

```
+--------------------------------------------------------------------+
|                    IPv4 vs. IPv6                                    |
+--------------------------------------------------------------------+
|                                                                     |
|   Property       | IPv4              | IPv6                         |
|   ---------------+-------------------+-----------------------------+
|   Address length | 32 bits           | 128 bits                     |
|   Notation       | Decimal (.)       | Hexadecimal (:)              |
|   Example        | 192.168.1.1       | 2001:db8::1                  |
|   Address count  | 4.3 billion       | 340 sextillion               |
|   Configuration  | Manual/DHCP       | SLAAC/DHCPv6                 |
|   Broadcast      | Yes               | No (Multicast)               |
|   NAT required   | Often yes         | Usually not                  |
|   Header         | Variable          | Fixed (40 bytes)             |
|   IPsec          | Optional          | Built-in                     |
|   Checksum       | Yes (in header)   | No                           |
|                                                                     |
|   Notation examples:                                                |
|   +------------------------------------------------------------+   |
|   | IPv4 Localhost: 127.0.0.1                                   |   |
|   | IPv6 Localhost: ::1                                         |   |
|   |                                                             |   |
|   | IPv4 Link-Local: 169.254.x.x                                |   |
|   | IPv6 Link-Local: fe80::...                                  |   |
|   +------------------------------------------------------------+   |
|                                                                     |
+--------------------------------------------------------------------+
```

## Key Terms

| Term | Explanation |
|------|-------------|
| **IPv6** | Internet Protocol Version 6 (128 bits) |
| **Hexadecimal** | Number system with base 16 (0-9, a-f) |
| **Unicast** | Address for a single device |
| **Multicast** | Address for a group of devices |
| **Link-Local** | Only valid in local network (fe80::) |
| **SLAAC** | Stateless Address Autoconfiguration |
| **::1** | IPv6 loopback address |

## Exam Tips

### Common Exam Questions

- How many bits does an IPv6 address have?
- How is an IPv6 address abbreviated?
- What is the difference between IPv4 and IPv6?

### Important to Remember

- **128 bits** (4x as long as IPv4)
- **8 groups** of 4 hex digits each, separated by **:**
- **::1** = Localhost (like 127.0.0.1)
- **fe80::** = Link-Local (automatically on every device)
- **No broadcast** in IPv6! Only multicast
- **::** may only appear ONCE per address
- Omit leading zeros: 0042 -> 42

## Practice Exercises

### Exercise 1
Abbreviate the IPv6 address: 2001:0db8:0000:0000:0000:0000:0000:0001

**Solution:**
Step 1 - Remove leading zeros:
2001:db8:0:0:0:0:0:1

Step 2 - Replace zero sequence with :::
**2001:db8::1**

### Exercise 2
How many bits does an IPv6 address have and how many groups are represented?

**Solution:**
- **128 bits** total length
- **8 groups** of 16 bits each (4 hexadecimal digits)

### Exercise 3
Name two essential differences between IPv4 and IPv6.

**Solution:**
1. **Address length:** IPv4 = 32 bits, IPv6 = 128 bits
2. **Representation:** IPv4 = decimal with dots (192.168.1.1), IPv6 = hexadecimal with colons (2001:db8::1)

Additional differences:
- IPv6 has no broadcast (only multicast)
- IPv6 usually doesn't require NAT
- IPv6 has more available addresses
