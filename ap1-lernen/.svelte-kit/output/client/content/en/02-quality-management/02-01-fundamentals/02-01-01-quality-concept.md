# Quality Concept

## Learning Objectives

After this chapter you should be able to:
- Define the term quality
- Distinguish between product and process quality
- Explain the Kano model
- Understand inherent characteristics

---

## Core Content

### What is Quality?

**Quality** refers to the degree to which a set of inherent characteristics of an object fulfills requirements (according to DIN EN ISO 9000).

Put more simply:
> Quality = Degree of fulfillment of requirements

**Inherent characteristic:** A characteristic that is inherent to a product or service (e.g., the color of a car, the response time of software).

---

### Product vs. Process Quality

| Aspect | Product Quality | Process Quality |
|--------|-----------------|-----------------|
| **Definition** | Quality of the end product | Quality of the manufacturing processes |
| **Focus** | Properties, functions | Efficiency, error-free operation |
| **IT Example** | Error-free software | Efficient development process |
| **Measurement** | Customer satisfaction, error rate | Throughput time, reject rate |

**Important:** Good process quality leads to good product quality!

---

### The Kano Model

The **Kano Model** (Professor Noriaki Kano, Tokyo 1978) shows the relationship between realized quality characteristics and customer satisfaction.

```
Customer Satisfaction
        ▲
  fully │                    ✱ Excitement Features
satisfied│                 ✱
        │              ✱
        │           ✱      ╱ Performance Features
        │        ✱       ╱
        │     ✱        ╱
        │──────────────────────────────► Realized
        │            ╱              Quality Characteristics
        │          ╱
        │        ╱ ─────── Basic Features
        │      ╱
completely│    ╱
dissatisfied
```

#### The Three Feature Categories

| Feature | Description | Effect |
|---------|-------------|--------|
| **Basic Features** | Self-evident properties | Absence leads to strong dissatisfaction, presence is not positively noticed |
| **Performance Features** | Consciously demanded by the customer | The more, the more satisfied |
| **Excitement Features** | Surprising extras | Generate high satisfaction, but are not expected |

#### Practical Example: Smartphone

| Feature | Type | Explanation |
|---------|------|-------------|
| Phone function | Basic | Expected, cannot do without |
| Battery life | Performance | More = better |
| Innovative feature | Excitement | Surprisingly positive |

---

### Quality Dimensions

Quality can be viewed in different dimensions:

| Dimension | Description | IT Example |
|-----------|-------------|------------|
| **Functionality** | Fulfillment of required functions | Software does what it should |
| **Reliability** | Consistent performance over time | Uptime of 99.9% |
| **Usability** | Ease of use | Intuitive user interface |
| **Efficiency** | Resource consumption | Low CPU load |
| **Maintainability** | Ease of modification/repair | Clean, documented code |
| **Portability** | Transferability | Runs on different systems |

## Key Terms

| Term | Explanation |
|------|-------------|
| **Quality** | Degree of fulfillment of requirements |
| **Inherent characteristic** | Property inherent to a product |
| **Product quality** | Quality of the end product |
| **Process quality** | Quality of the manufacturing processes |
| **Basic features** | Self-evident, expected properties |
| **Performance features** | Consciously demanded properties |
| **Excitement features** | Surprising, unexpected extras |

### Memory Aid for the Kano Model
- **Basic** = "Must be" (like a foundation)
- **Performance** = "The more, the better"
- **Excitement** = "Wow effect"

## Practice Exercises

### Exercise 1
Define the term quality.

**Solution:** Quality is the degree to which a set of inherent characteristics of an object fulfills requirements.

### Exercise 2
Assign the following features of an online shop to the Kano categories:
a) The shop is accessible
b) Fast loading times
c) Personal product recommendations based on AI

**Solution:**
- a) Basic feature (expected as self-evident)
- b) Performance feature (the faster, the more satisfied)
- c) Excitement feature (surprising, not expected)

### Exercise 3
What is the difference between product and process quality?

**Solution:**
- **Product quality** refers to the characteristics of the finished product
- **Process quality** refers to the quality of the manufacturing and development processes
- Good process quality is a prerequisite for good product quality
