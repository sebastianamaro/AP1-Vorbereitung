# Technical and Organizational Measures (TOM)

## Learning Objectives

After this chapter, you should be able to:
- Explain the term TOM
- Know the categories of TOMs
- Provide examples of technical and organizational measures
- Understand the connection with GDPR

---

## Core Content

### What are TOMs?

**Technical and Organizational Measures (TOM)** are protective measures required under Art. 32 GDPR to ensure the security of personal data.

> **Art. 32 GDPR:** The controller and processor shall implement appropriate technical and organizational measures to ensure a level of security appropriate to the risk.

### The 8 Protection Goals under GDPR/BDSG

| Protection Goal | Description | Examples |
|-----------------|-------------|----------|
| **Physical Access Control** | Prevent unauthorized physical entry | Locks, badges, alarm systems |
| **System Access Control** | Prevent unauthorized use of systems | Passwords, 2FA, user accounts |
| **Data Access Control** | Make only authorized data accessible | Authorization concept, roles |
| **Transfer Control** | Protect data during transport | Encryption, VPN |
| **Input Control** | Traceability of changes | Logging, audit logs |
| **Job Control** | Control commissioned processing | Contracts, audits |
| **Availability Control** | Protection against loss | Backup, UPS, redundancy |
| **Separation Control** | Separate data for different purposes | Tenant separation, database separation |

---

### Technical Measures in Detail

| Category | Measures |
|----------|----------|
| **Encryption** | AES, TLS/SSL, end-to-end encryption |
| **Access Controls** | Passwords, two-factor authentication, biometric methods |
| **Network Security** | Firewalls, VPN, intrusion detection |
| **Data Backup** | Regular backups, mirrored systems |
| **Logging** | Audit logs, access logs |
| **Physical Security** | Access control, video surveillance, fire protection |

---

### Organizational Measures in Detail

| Category | Measures |
|----------|----------|
| **Policies** | Data protection policy, IT security policy |
| **Training** | Data protection awareness, security training |
| **Roles and Rights** | Authorization concept, need-to-know principle |
| **Processes** | Incident management, data deletion |
| **Documentation** | Processing records, TOM documentation |
| **Contracts** | Data processing agreements, confidentiality agreements |

---

### Selection of Appropriate TOMs

The selection must consider the following factors (Art. 32 GDPR):

| Factor | Explanation |
|--------|-------------|
| **State of the Art** | Current technical possibilities |
| **Implementation Costs** | Economically justifiable |
| **Nature of Processing** | What data, how processed |
| **Scope and Purposes** | Amount of data, processing purpose |
| **Risk** | Likelihood and severity |

---

### TOM Documentation

TOMs must be documented:

| Content | Details |
|---------|---------|
| **What?** | All implemented measures |
| **When?** | In the contract for commissioned processing |
| **Why?** | Evidence of accountability |
| **Review** | Regular review and update |

## Practice Exercises

### Exercise 1
What are technical and organizational measures (TOM)?

**Solution:** TOMs are protective measures under Art. 32 GDPR that ensure the security of personal data. They include technical measures (e.g., encryption) and organizational measures (e.g., training).

### Exercise 2
Distinguish between physical access control, system access control, and data access control.

**Solution:**
- **Physical Access Control:** Prevents unauthorized physical entry to rooms/buildings (e.g., locks, badges)
- **System Access Control:** Prevents unauthorized use of IT systems (e.g., passwords, 2FA)
- **Data Access Control:** Ensures only authorized persons can access specific data (e.g., authorization concept)

### Exercise 3
Name two technical and two organizational measures each.

**Solution:**
Technical measures:
1. Encryption
2. Firewall

Organizational measures:
1. Data protection training
2. Authorization concept
