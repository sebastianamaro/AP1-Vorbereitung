# Derechos de Autor en TI

## Objetivos de Aprendizaje

Después de este capítulo, deberías ser capaz de:
- Comprender los derechos de autor en el contexto de TI
- Conocer los derechos de los desarrolladores de software
- Distinguir derechos de uso de derechos de autor
- Conocer las consecuencias de la infracción de derechos de autor

---

## Contenido Principal

### ¿Qué son los Derechos de Autor?

```
┌────────────────────────────────────────────────────────────────────┐
│                       DERECHOS DE AUTOR                            │
├────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   Definición:                                                      │
│   ┌─────────────────────────────────────────────────────────────┐ │
│   │ Los derechos de autor protegen las creaciones intelectuales │ │
│   │ (obras) y otorgan al creador derechos exclusivos.           │ │
│   └─────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   En Alemania regulado por:                                        │
│   • Ley de Derechos de Autor (UrhG - Urheberrechtsgesetz)         │
│   • Específicamente para software: §§ 69a-69g UrhG                │
│                                                                     │
│   Principio básico:                                                │
│   ┌─────────────────────────────────────────────────────────────┐ │
│   │              Autor = Creador de la obra                     │ │
│   │                                                              │ │
│   │         ┌───────────────┐                                   │ │
│   │         │    AUTOR      │                                   │ │
│   │         │   (persona    │                                   │ │
│   │         │   natural)    │                                   │ │
│   │         └───────┬───────┘                                   │ │
│   │                 │                                           │ │
│   │                 ↓ crea                                      │ │
│   │         ┌───────────────┐                                   │ │
│   │         │     OBRA      │                                   │ │
│   │         │  (Software,   │                                   │ │
│   │         │   Texto, ...) │                                   │ │
│   │         └───────────────┘                                   │ │
│   │                                                              │ │
│   │   ⚠ ¡Solo las personas naturales pueden ser autores!       │ │
│   │     (No empresas, no IA)                                    │ │
│   └─────────────────────────────────────────────────────────────┘ │
│                                                                     │
└────────────────────────────────────────────────────────────────────┘
```

### Infracción de Derechos de Autor

```
┌────────────────────────────────────────────────────────────────────┐
│                 INFRACCIÓN DE DERECHOS DE AUTOR                    │
├────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   Violaciones típicas:                                              │
│   ┌─────────────────────────────────────────────────────────────┐ │
│   │ ✗ Usar software sin licencia (piratería)                   │ │
│   │ ✗ No cumplir con los términos de licencia                  │ │
│   │ ✗ Más instalaciones de las licenciadas                     │ │
│   │ ✗ Distribuir software sin permiso                          │ │
│   │ ✗ Copiar/modificar código fuente sin permiso               │ │
│   │ ✗ Ignorar condiciones de licencia de código abierto        │ │
│   └─────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   Posibles consecuencias:                                           │
│   ┌─────────────────────────────────────────────────────────────┐ │
│   │ DERECHO CIVIL                     DERECHO PENAL             │ │
│   │ • Cesación y desistimiento       • Multa                    │ │
│   │ • Indemnización por daños        • Prisión                  │ │
│   │ • Divulgación del alcance           (hasta 3 años)          │ │
│   │ • Destrucción de copias          • Para actividad           │ │
│   │ • Costos legales                   comercial: hasta 5 años  │ │
│   └─────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   Particularmente crítico para empresas:                           │
│   • Auditorías de software por proveedores (p.ej., BSA)           │
│   • Altos pagos retroactivos + cargos por penalización            │
│   • Daño a la reputación                                           │
│                                                                     │
└────────────────────────────────────────────────────────────────────┘
```

## Términos Clave

| Término | Explicación |
|---------|-------------|
| **Autor** | Creador de la obra (persona natural) |
| **Obra** | Creación intelectual (p.ej., software, texto) |
| **Umbral de originalidad** | Nivel mínimo de creatividad |
| **Derecho de uso** | Derecho transferible para usar |
| **Derecho patrimonial** | Derecho al uso comercial |
| **Dominio público** | Sin derechos de autor, uso libre |
| **Descompilación** | Traducción inversa del código máquina |

## Consejos para el Examen

### Preguntas Comunes del Examen

- ¿Quién es el autor del software creado en empleo?
- ¿Qué puedes hacer con software sin licencia?
- ¿Cuánto tiempo está protegido el software por derechos de autor?

### Importante Recordar

- **Autor** = siempre la persona natural (¡nunca una empresa!)
- **En empleo:** Los derechos de uso van automáticamente al empleador
- **Duración:** 70 años después de la muerte del autor
- **Permitido sin licencia:** Uso previsto, backup, corrección de errores
- **Ideas y conceptos** NO están protegidos
- **Código fuente** está protegido

---

## Ejercicios Prácticos

### Ejercicio 1
Un desarrollador de software programa una nueva aplicación para su empleador. ¿Quién es el autor? ¿Quién tiene los derechos de uso?

**Solución:**
- **Autor:** El desarrollador de software (empleado) SIEMPRE permanece como el autor, ya que solo las personas naturales pueden ser autores.
- **Derechos de uso:** El empleador recibe automáticamente todos los derechos de uso según § 69b UrhG cuando el software se crea como parte de los deberes laborales.

### Ejercicio 2
¿Puedes crear una copia de seguridad de software comprado?

**Solución:**
**Sí**, una copia de seguridad está permitida según § 69d UrhG si:
- Es necesaria para respaldar el programa
- Es solo UNA copia
- No se distribuye a terceros

### Ejercicio 3
¿Cuánto tiempo está protegido un programa de computadora por derechos de autor?

**Solución:**
Un programa de computadora está protegido durante **70 años después de la muerte del autor**. Para múltiples autores: 70 años después de la muerte del último autor sobreviviente. Después de eso, el software pasa a dominio público.
