# Ciclo de Vida del Desarrollo de Software (SDLC)

## Objetivos de Aprendizaje

Después de este capítulo, deberías ser capaz de:
- Conocer las fases del desarrollo de software
- Comprender y explicar el SDLC
- Reconocer conexiones entre fases
- Asignar documentos importantes a las fases

---

## Contenido Principal

### ¿Qué es el SDLC?

El **Ciclo de Vida del Desarrollo de Software (SDLC)** describe todo el ciclo de vida del software desde la idea hasta la baja del sistema.

```
┌────────────────────────────────────────────────────────────────────┐
│              CICLO DE VIDA DEL DESARROLLO DE SOFTWARE              │
├────────────────────────────────────────────────────────────────────┤
│                                                                     │
│                         ┌─────────┐                                │
│                    ┌────│ Análisis│────┐                           │
│                    │    └─────────┘    │                           │
│                    │                   │                           │
│              ┌─────┴────┐        ┌─────┴────┐                      │
│              │  Retiro  │        │  Diseño  │                      │
│              └─────┬────┘        └─────┬────┘                      │
│                    │                   │                           │
│              ┌─────┴────┐        ┌─────┴────┐                      │
│              │Manteni-  │        │Implemen- │                      │
│              │ miento   │        │ tación   │                      │
│              └─────┬────┘        └─────┬────┘                      │
│                    │    ┌─────────┐    │                           │
│                    └────│ Pruebas │────┘                           │
│                         └────┬────┘                                │
│                              │                                     │
│                         ┌────┴────┐                                │
│                         │Despliegue│                               │
│                         └─────────┘                                │
│                                                                     │
└────────────────────────────────────────────────────────────────────┘
```

### Las Fases del SDLC en Detalle

```
┌────────────────────────────────────────────────────────────────────┐
│                      FASES DEL SDLC                                │
├────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   1. ANÁLISIS (Requisitos)                                         │
│   ┌────────────────────────────────────────────────────────────┐  │
│   │ • Recopilar requisitos                                     │  │
│   │ • Analizar estado actual                                   │  │
│   │ • Crear concepto objetivo                                  │  │
│   │ • Verificar factibilidad                                   │  │
│   │                                                             │  │
│   │ Resultado: Especificación de requisitos,                   │  │
│   │            Especificación funcional                        │  │
│   └────────────────────────────────────────────────────────────┘  │
│                              ↓                                     │
│   2. DISEÑO                                                        │
│   ┌────────────────────────────────────────────────────────────┐  │
│   │ • Definir arquitectura                                     │  │
│   │ • Planificar estructuras de datos                          │  │
│   │ • Definir interfaces                                       │  │
│   │ • Crear diseño de UI                                       │  │
│   │                                                             │  │
│   │ Resultado: Concepto técnico, diagramas UML                 │  │
│   └────────────────────────────────────────────────────────────┘  │
│                              ↓                                     │
│   3. IMPLEMENTACIÓN (Codificación)                                 │
│   ┌────────────────────────────────────────────────────────────┐  │
│   │ • Escribir código fuente                                   │  │
│   │ • Crear documentación                                      │  │
│   │ • Realizar revisiones de código                            │  │
│   │ • Desarrollar pruebas unitarias                            │  │
│   │                                                             │  │
│   │ Resultado: Software ejecutable                             │  │
│   └────────────────────────────────────────────────────────────┘  │
│                                                                     │
└────────────────────────────────────────────────────────────────────┘
```

```
┌────────────────────────────────────────────────────────────────────┐
│                   FASES DEL SDLC (Continuación)                    │
├────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   4. PRUEBAS                                                       │
│   ┌────────────────────────────────────────────────────────────┐  │
│   │ • Pruebas de módulo (Unit Tests)                           │  │
│   │ • Pruebas de integración                                   │  │
│   │ • Pruebas de sistema                                       │  │
│   │ • Pruebas de aceptación                                    │  │
│   │                                                             │  │
│   │ Resultado: Protocolos de prueba, Informe de errores        │  │
│   └────────────────────────────────────────────────────────────┘  │
│                              ↓                                     │
│   5. DESPLIEGUE (Puesta en Marcha)                                 │
│   ┌────────────────────────────────────────────────────────────┐  │
│   │ • Instalación en sitio del cliente                         │  │
│   │ • Migración de datos                                       │  │
│   │ • Capacitación de usuarios                                 │  │
│   │ • Documentación de entrega                                 │  │
│   │                                                             │  │
│   │ Resultado: Sistema de producción, Materiales de capacitación│  │
│   └────────────────────────────────────────────────────────────┘  │
│                              ↓                                     │
│   6. MANTENIMIENTO                                                 │
│   ┌────────────────────────────────────────────────────────────┐  │
│   │ • Corrección de errores (correctivo)                       │  │
│   │ • Adaptaciones (adaptativo)                                │  │
│   │ • Mejoras (perfectivo)                                     │  │
│   │ • Medidas preventivas                                      │  │
│   │                                                             │  │
│   │ Resultado: Actualizaciones, Parches, Nuevas versiones      │  │
│   └────────────────────────────────────────────────────────────┘  │
│                                                                     │
└────────────────────────────────────────────────────────────────────┘
```

### Tipos de Mantenimiento

```
┌────────────────────────────────────────────────────────────────────┐
│                       TIPOS DE MANTENIMIENTO                       │
├────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌─────────────────────────────────────────────────────────────┐ │
│   │ MANTENIMIENTO CORRECTIVO                                    │ │
│   │ • Corrección de errores                                     │ │
│   │ • Corrección de bugs                                        │ │
│   │ • Ejemplo: El login ya no funciona                          │ │
│   └─────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌─────────────────────────────────────────────────────────────┐ │
│   │ MANTENIMIENTO ADAPTATIVO                                    │ │
│   │ • Adaptación a entorno cambiado                             │ │
│   │ • Nuevo sistema operativo, nuevo hardware                   │ │
│   │ • Ejemplo: Migración a Windows 11                           │ │
│   └─────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌─────────────────────────────────────────────────────────────┐ │
│   │ MANTENIMIENTO PERFECTIVO                                    │ │
│   │ • Mejoras y extensiones                                     │ │
│   │ • Nuevas funcionalidades                                    │ │
│   │ • Ejemplo: Agregar función de exportación                   │ │
│   └─────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌─────────────────────────────────────────────────────────────┐ │
│   │ MANTENIMIENTO PREVENTIVO                                    │ │
│   │ • Prevención de problemas futuros                           │ │
│   │ • Optimización de código, refactorización                   │ │
│   │ • Ejemplo: Cerrar vulnerabilidades de seguridad             │ │
│   └─────────────────────────────────────────────────────────────┘ │
│                                                                     │
└────────────────────────────────────────────────────────────────────┘
```

### SDLC en Diferentes Modelos de Proceso

```
┌────────────────────────────────────────────────────────────────────┐
│           SDLC EN MODELOS DE PROCESO                               │
├────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   MODELO CASCADA                                                   │
│   ┌────────────────────────────────────────────────────────────┐  │
│   │ Análisis → Diseño → Implementación → Pruebas → Despliegue  │  │
│   │                                                             │  │
│   │ • Fases estrictamente secuenciales                         │  │
│   │ • Cada fase se recorre una vez                             │  │
│   │ • Retroceso solo en caso de errores                        │  │
│   └────────────────────────────────────────────────────────────┘  │
│                                                                     │
│   SCRUM (Ágil)                                                     │
│   ┌────────────────────────────────────────────────────────────┐  │
│   │        ┌───────────────────────────────┐                   │  │
│   │        │    Sprint (2-4 semanas)       │                   │  │
│   │        │  ┌──────────────────────────┐ │                   │  │
│   │   ──→  │  │Análisis→Diseño→Código→Test│ │  ──→ Incremento  │  │
│   │        │  └──────────────────────────┘ │                   │  │
│   │        └───────────────────────────────┘                   │  │
│   │                      ↑                                      │  │
│   │                      └── Repetición ──┘                    │  │
│   │                                                             │  │
│   │ • Iteraciones cortas                                       │  │
│   │ • Todas las fases en cada sprint                           │  │
│   │ • Entregas regulares                                       │  │
│   └────────────────────────────────────────────────────────────┘  │
│                                                                     │
└────────────────────────────────────────────────────────────────────┘
```

## Términos Clave

| Término | Explicación |
|---------|-------------|
| **SDLC** | Ciclo de Vida del Desarrollo de Software |
| **Especificación de requisitos** | Describe QUÉ quiere el cliente |
| **Especificación funcional** | Describe CÓMO implementa el contratista |
| **Despliegue** | Entrega/instalación del software |
| **Release** | Versión publicada del software |
| **Patch** | Pequeña actualización para corrección de errores |
| **Refactorización** | Mejora del código sin cambio de funcionalidad |

## Consejos para el Examen

### Preguntas Comunes del Examen

- Nombrar las fases del desarrollo de software
- ¿Qué documentos se crean en qué fase?
- Distinguir los tipos de mantenimiento

### Importante Recordar

- **6 Fases:** Análisis → Diseño → Implementación → Pruebas → Despliegue → Mantenimiento
- **Especificación de requisitos** = QUÉ (Cliente)
- **Especificación funcional** = CÓMO (Contratista)
- **4 tipos de mantenimiento:** Correctivo, Adaptativo, Perfectivo, Preventivo
- ¡La fase de pruebas viene ANTES del despliegue!

## Ejercicios Prácticos

### Ejercicio 1
Asignar los documentos a las fases del SDLC:
a) Manual de usuario
b) Protocolo de pruebas
c) Especificación funcional
d) Diagrama de clases UML

**Solución:**
- a) Manual de usuario → **Despliegue**
- b) Protocolo de pruebas → **Pruebas**
- c) Especificación funcional → **Análisis**
- d) Diagrama de clases UML → **Diseño**

### Ejercicio 2
¿Qué tipo de mantenimiento es este?
- El software debe adaptarse a una nueva versión de base de datos.

**Solución:**
**Mantenimiento Adaptativo** - El software se adapta a un entorno cambiado (nueva versión de base de datos) sin cambiar la funcionalidad.

### Ejercicio 3
Explicar la diferencia entre especificación de requisitos y especificación funcional.

**Solución:**
| Especificación de requisitos | Especificación funcional |
|------------------------------|-------------------------|
| Creada por el cliente | Creada por el contratista |
| Describe QUÉ se desea | Describe CÓMO implementar |
| Requisitos de negocio | Solución técnica |
| Base para presupuestos | Base para el contrato |
