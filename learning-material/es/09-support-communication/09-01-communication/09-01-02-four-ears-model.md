# El Modelo de las Cuatro Orejas (Schulz von Thun)

## Objetivos de Aprendizaje

Después de esta sección, podrás:
- Explicar los cuatro lados de un mensaje
- Aplicar el modelo a situaciones de soporte TI
- Reconocer y evitar malentendidos
- Comunicarte de manera más consciente

---

## Contenido Principal

### Idea Básica del Modelo

Cada mensaje tiene **cuatro lados** (u "orejas") por los cuales puede ser enviado y recibido.

```
┌─────────────────────────────────────────────────────────────────────┐
│           EL MODELO DE LAS CUATRO OREJAS                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│                         CONTENIDO FACTUAL                            │
│                        (De qué estoy informando)                     │
│                              │                                       │
│                              │                                       │
│                              ▼                                       │
│                        ┌──────────┐                                 │
│   AUTORREVELACIÓN      │          │        APELACIÓN                │
│   (Lo que revelo   ──► │ MENSAJE  │◄── (Lo que quiero               │
│    sobre mí mismo)     │          │     que hagas)                  │
│                        └──────────┘                                 │
│                              │                                       │
│                              │                                       │
│                              ▼                                       │
│                          RELACIÓN                                    │
│                    (Lo que pienso de ti)                            │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Los Cuatro Lados de un Mensaje

```
┌─────────────────────────────────────────────────────────────────────┐
│           LOS CUATRO LADOS EN DETALLE                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  1. CONTENIDO FACTUAL (Oreja Factual)                               │
│     ───────────────────────────────────────────────────────────     │
│     → La información pura                                           │
│     → Hechos, datos, tema                                           │
│     → "¿De qué se está hablando?"                                   │
│                                                                      │
│  2. AUTORREVELACIÓN (Oreja de Autorrevelación)                      │
│     ───────────────────────────────────────────────────────────     │
│     → Lo que el emisor revela sobre sí mismo                        │
│     → Sentimientos, opiniones, necesidades                          │
│     → "¿Qué dice esto sobre el emisor?"                             │
│                                                                      │
│  3. RELACIÓN (Oreja de Relación)                                    │
│     ───────────────────────────────────────────────────────────     │
│     → Cómo el emisor se relaciona con el receptor                   │
│     → Apreciación, rechazo, respeto                                 │
│     → "¿Cómo me está hablando esta persona?"                        │
│                                                                      │
│  4. APELACIÓN (Oreja de Apelación)                                  │
│     ───────────────────────────────────────────────────────────     │
│     → Lo que el emisor quiere del receptor                          │
│     → Solicitud, deseo, demanda                                     │
│     → "¿Qué debo hacer/pensar/sentir?"                              │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Ejemplo: "¡Mi PC se averió de nuevo!"

```
┌─────────────────────────────────────────────────────────────────────┐
│           ANÁLISIS DE EJEMPLO                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  DECLARACIÓN: "¡Mi PC se averió de nuevo!"                          │
│                                                                      │
│                         CONTENIDO FACTUAL                            │
│                    "El PC no funciona"                               │
│                              │                                       │
│                              │                                       │
│                              ▼                                       │
│                        ┌──────────┐                                 │
│   AUTORREVELACIÓN      │  "¡Mi    │        APELACIÓN                │
│  "Estoy frustrado"  ──│   PC     │──"¡Arréglalo!"                   │
│  "Esto me molesta"     │  se      │  "¡Ayúdame!"                    │
│  "Estoy estresado"     │ averió   │  "¡Hazlo bien                   │
│                        │ de nuevo!│   esta vez!"                    │
│                        └──────────┘                                 │
│                              │                                       │
│                              │                                       │
│                              ▼                                       │
│                          RELACIÓN                                    │
│                "TI hace un trabajo pobre"                            │
│                "Esto ha pasado de nuevo"                             │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Análisis de Comunicación en Soporte TI

| Declaración del Usuario | Contenido Factual | Autorrevelación | Relación | Apelación |
|-------------------------|-------------------|-----------------|----------|-----------|
| "Internet está lento" | La red tiene baja velocidad | Estoy impaciente | Espera solución rápida | ¡Hazlo más rápido! |
| "Ayer funcionaba" | Algo ha cambiado | No hice nada | No es mi culpa | Averigua qué pasó |
| "Necesito trabajar urgentemente" | Es crítico en tiempo | Estoy bajo presión | Tómame en serio | ¡Maneja mi problema primero! |
| "Nadie entiende esto" | El software es complicado | Estoy abrumado | Espero mejor explicación | Explícalo más simple |

### Malentendidos Típicos

```
┌─────────────────────────────────────────────────────────────────────┐
│           EVITAR MALENTENDIDOS                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  PROBLEMA: Emisor y receptor usan diferentes "orejas"               │
│                                                                      │
│  EJEMPLO:                                                           │
│  ──────────────────────────────────────────────────────────────     │
│  Usuario: "¡Este programa es totalmente complicado!"                │
│                                                                      │
│  Posibles interpretaciones por Soporte TI:                          │
│                                                                      │
│  ┌────────────────────────────────────────────────────────────┐     │
│  │ OREJA FACTUAL: "El software tiene muchas funciones"        │     │
│  │                → Respuesta: "Sí, es cierto"                │     │
│  │                                                            │     │
│  │ OREJA DE RELACIÓN: "Me lo explicaste mal"                  │     │
│  │                → Respuesta (defensiva): "¡Pero te          │     │
│  │                  lo mostré!"                               │     │
│  │                                                            │     │
│  │ OREJA DE APELACIÓN: "¡Por favor ayúdame!"                  │     │
│  │                → Respuesta: "Te lo muestro de nuevo"     ✓ │     │
│  │                                                            │     │
│  │ OREJA DE AUTORREVELACIÓN: "Está abrumado"                  │     │
│  │                → Respuesta: "Entiendo que puede ser        │     │
│  │                  confuso al principio"                   ✓ │     │
│  └────────────────────────────────────────────────────────────┘     │
│                                                                      │
│  RECOMENDACIÓN: ¡Escucha con la oreja de Apelación o                │
│                 Autorrevelación!                                    │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Recomendaciones para Soporte TI

```
┌─────────────────────────────────────────────────────────────────────┐
│           CONSEJOS DE COMUNICACIÓN                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  1. ESCUCHAR CON LA OREJA CORRECTA                                  │
│     ───────────────────────────────────────────────────────────     │
│     → No tomar todo a nivel de relación                             │
│     → Reconocer la apelación detrás de la declaración               │
│     → Percibir la autorrevelación (frustración)                     │
│                                                                      │
│  2. ENVIAR CONSCIENTEMENTE EN TODOS LOS NIVELES                     │
│     ───────────────────────────────────────────────────────────     │
│     → Informar claramente a nivel factual                           │
│     → Mostrar comprensión (reconocer autorrevelación)               │
│     → Comunicar con aprecio (relación positiva)                     │
│     → Dar opciones de acción claras (apelación)                     │
│                                                                      │
│  3. EJEMPLO DE UNA BUENA RESPUESTA                                  │
│     ───────────────────────────────────────────────────────────     │
│     Usuario: "¡Este programa es totalmente complicado!"             │
│                                                                      │
│     Respuesta: "Entiendo que es mucho al principio                  │
│     [reconocer autorrevelación]. El programa efectivamente          │
│     tiene muchas funciones [contenido factual].                     │
│     Con gusto te muestro los pasos más importantes                  │
│     de nuevo [responder a apelación]. ¿Cuándo te vendría           │
│     bien? [relación positiva: el usuario decide]"                   │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Uso de Mensajes-Yo

```
┌─────────────────────────────────────────────────────────────────────┐
│           MENSAJES-YO vs. MENSAJES-TÚ                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  MENSAJE-TÚ (EVITAR):                                               │
│  ──────────────────────────────────────────────────────────────     │
│  → "Tú hiciste eso mal"                                             │
│  → "Tú olvidaste eso"                                               │
│  → "Tú necesitas prestar mejor atención"                            │
│  → Crea reacción defensiva                                          │
│                                                                      │
│  MENSAJE-YO (RECOMENDADO):                                          │
│  ──────────────────────────────────────────────────────────────     │
│  → "Veo que ha ocurrido un error aquí"                              │
│  → "Noté que falta este paso"                                       │
│  → "Recomiendo prestar atención a esto en el futuro"                │
│  → Evita asignación de culpa                                        │
│                                                                      │
│  ESTRUCTURA DE UN MENSAJE-YO:                                       │
│  ──────────────────────────────────────────────────────────────     │
│  1. Observación: "Veo que..."                                       │
│  2. Sentimiento/Reacción: "Esto lleva a..."                         │
│  3. Deseo: "Agradecería si..."                                      │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

## Términos Clave

| Término | Explicación |
|---------|-------------|
| **Contenido Factual** | La información puramente factual de un mensaje |
| **Autorrevelación** | Lo que el emisor comunica sobre sí mismo |
| **Nivel de Relación** | Cómo el emisor se relaciona con el receptor |
| **Apelación** | Lo que el emisor quiere del receptor |
| **Mensaje-Yo** | Declaración sin asignación de culpa |
| **Escucha Activa** | Escucha consciente y atenta |

---

## Consejos para el Examen

1. **Recordar los cuatro niveles:**
   - Contenido **F**actual
   - Autorr**e**velación
   - **R**elación
   - **A**pelación
   - Ayuda de memoria: **FERA** o "Factual-Emoción-Relación-Apelación"

2. **Aplicar en contexto TI:**
   - Frecuentemente frustración (autorrevelación) y deseo de ayuda (apelación) vienen juntos
   - No tomarlo personalmente (nivel de relación)

3. **Preguntas típicas de examen:**
   - "Analiza la declaración según el Modelo de las Cuatro Orejas"
   - "¿Cómo debería responder el soporte TI?"

## Ejercicios Prácticos

### Ejercicio 1: Analizar Mensaje

Analiza la declaración del usuario según el Modelo de las Cuatro Orejas:

**"¡Ya he llamado tres veces y el problema todavía no está resuelto!"**

<details>
<summary>Mostrar Solución</summary>

| Nivel | Interpretación |
|-------|----------------|
| **Contenido Factual** | El problema persiste a pesar de tres llamadas |
| **Autorrevelación** | "Estoy frustrado y enojado" / "No me siento tomado en serio" |
| **Relación** | "El soporte TI no funciona de manera confiable" / "No toman mi problema en serio" |
| **Apelación** | "¡Finalmente resuelvan el problema!" / "¡Traten mi caso con prioridad!" |

**Respuesta recomendada:**
- Mostrar comprensión por la frustración
- No ponerse a la defensiva
- Revisar historial de tickets
- Ofrecer enfoque de solución concreto
- Hacer compromiso vinculante

</details>

### Ejercicio 2: Formular Respuesta

Un usuario dice: "Esto es tan complicado, ¿cómo se supone que debo trabajar así?"

Formula una respuesta profesional que considere los cuatro niveles.

<details>
<summary>Mostrar Solución</summary>

**Respuesta profesional:**

"Entiendo que el flujo de trabajo actual parece complicado y te cuesta tiempo **[reconocer autorrevelación]**.

Miremos juntos qué pasos exactamente son problemáticos **[aclarar nivel factual]**.

Me gustaría ayudarte a encontrar una mejor solución **[relación positiva]**.

¿Puedes mostrarme dónde exactamente está el problema? Entonces encontraremos una solución juntos **[responder a apelación]**."

</details>

### Ejercicio 3: Reformular Mensajes-Yo

Reformula los siguientes mensajes-tú en mensajes-yo:

a) "Ingresaste la contraseña incorrectamente."
b) "¡Deberías habérmelo dicho antes!"
c) "No entiendes esto correctamente."

<details>
<summary>Mostrar Solución</summary>

a) **"Ingresaste la contraseña incorrectamente."**
   → "Veo que el inicio de sesión no funcionó. Esto puede pasar cuando la contraseña no se ingresa correctamente. Vamos a verificar esto juntos."

b) **"¡Deberías habérmelo dicho antes!"**
   → "Noté que me faltaba esta información hasta ahora. Con esta información, puedo resolver el problema más rápido. Para el futuro: Mientras más detalles me des de una vez, más rápido puedo encontrar la solución."

c) **"No entiendes esto correctamente."**
   → "Me doy cuenta de que quizás no expliqué esto con suficiente claridad. Déjame explicarlo de otra manera."

</details>
