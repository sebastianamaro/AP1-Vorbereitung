Después de este capítulo, deberías ser capaz de:
- Comprender la estructura de una dirección IPv6

## Contenido Principal

### ¿Por qué IPv6?

```
+--------------------------------------------------------------------+
|                    ¿POR QUÉ IPv6?                                   |
+--------------------------------------------------------------------+
|                                                                     |
|   Problema con IPv4:                                                |
|   +-------------------------------------------------------------+  |
|   | - Solo aprox. 4.300 millones de direcciones (2^32)          |  |
|   | - Dispositivos mundiales: ¡Más de 20.000 millones!          |  |
|   | - Las direcciones IPv4 están agotadas (desde 2011)          |  |
|   | - Soluciones temporales como NAT son complicadas            |  |
|   +-------------------------------------------------------------+  |
|                                                                     |
|   IPv6 resuelve el problema:                                        |
|   +-------------------------------------------------------------+  |
|   | - 128 bits en lugar de 32 bits                              |  |
|   | - 2^128 = aprox. 340 sextillones de direcciones             |  |
|   | - ¡Eso es suficiente para cada grano de arena en la Tierra! |  |
|   +-------------------------------------------------------------+  |
|                                                                     |
|   Comparación:                                                      |
|   +-------------------------------------------------------------+  |
|   | IPv4:  4.294.967.296 direcciones           (4.300 millones) |  |
|   | IPv6:  340.282.366.920.938.463.463.374.607.431.768.211.456   |  |
|   |                                           (340 sextillones) |  |
|   +-------------------------------------------------------------+  |
|                                                                     |
+--------------------------------------------------------------------+
```

### Estructura de una Dirección IPv6

```
+--------------------------------------------------------------------+
|                    FORMATO DE DIRECCIÓN IPv6                        |
+--------------------------------------------------------------------+
|                                                                     |
|   Estructura básica:                                                |
|   +-------------------------------------------------------------+  |
|   |                                                              |  |
|   |   - 128 bits de longitud                                    |  |
|   |   - 8 grupos de 16 bits cada uno (4 dígitos hexadecimales)  |  |
|   |   - Grupos separados por dos puntos (:)                     |  |
|   |   - Representación hexadecimal (0-9, a-f)                   |  |
|   |                                                              |  |
|   +-------------------------------------------------------------+  |
|                                                                     |
|   Ejemplo:                                                          |
|   +-------------------------------------------------------------+  |
|   |                                                              |  |
|   |   2001:0db8:85a3:0000:0000:8a2e:0370:7334                   |  |
|   |   ----  ----  ----  ----  ----  ----  ----  ----            |  |
|   |    |     |     |     |     |     |     |     |              |  |
|   |    +-----+-----+-----+-----+-----+-----+-----+              |  |
|   |             8 grupos de 4 dígitos hex cada uno              |  |
|   |                                                              |  |
|   |   Cada grupo = 16 bits = 2 bytes                            |  |
|   |   8 grupos x 16 bits = 128 bits                             |  |
|   |                                                              |  |
|   +-------------------------------------------------------------+  |
|                                                                     |
+--------------------------------------------------------------------+
```

### Abreviando Direcciones IPv6

```
+--------------------------------------------------------------------+
|                  NOTACIÓN CORTA IPv6                                |
+--------------------------------------------------------------------+
|                                                                     |
|   Las direcciones IPv6 pueden abreviarse según dos reglas:         |
|                                                                     |
|   REGLA 1: Omitir ceros iniciales                                   |
|   +------------------------------------------------------------+   |
|   |                                                             |   |
|   |   Antes: 2001:0db8:0000:0042:0000:8a2e:0370:7334           |   |
|   |                |    |    |         |    |                  |   |
|   |   Después: 2001:db8:0:42:0:8a2e:370:7334                   |   |
|   |                                                             |   |
|   |   - 0db8 -> db8                                            |   |
|   |   - 0000 -> 0 (al menos un dígito permanece)               |   |
|   |   - 0042 -> 42                                             |   |
|   |   - 0370 -> 370                                            |   |
|   |                                                             |   |
|   +------------------------------------------------------------+   |
|                                                                     |
|   REGLA 2: Reemplazar una secuencia contigua de ceros con ::       |
|   +------------------------------------------------------------+   |
|   |                                                             |   |
|   |   Antes: 2001:db8:0:0:0:0:0:1                              |   |
|   |               | | | | |                                    |   |
|   |   Después: 2001:db8::1                                     |   |
|   |                                                             |   |
|   |   ! IMPORTANTE: :: solo puede usarse UNA VEZ por dirección!|   |
|   |                                                             |   |
|   |   Ejemplo Localhost:                                        |   |
|   |   0000:0000:0000:0000:0000:0000:0000:0001                   |   |
|   |   -> ::1                                                    |   |
|   |                                                             |   |
|   +------------------------------------------------------------+   |
|                                                                     |
|   Ejemplo completo:                                                 |
|   +------------------------------------------------------------+   |
|   |                                                             |   |
|   |   Completa:  2001:0db8:0000:0000:0000:0000:0000:0001       |   |
|   |   Regla 1:   2001:db8:0:0:0:0:0:1                          |   |
|   |   Regla 1+2: 2001:db8::1                                   |   |
|   |                                                             |   |
|   +------------------------------------------------------------+   |
|                                                                     |
+--------------------------------------------------------------------+
```

### Tipos de Direcciones IPv6

```
+--------------------------------------------------------------------+
|                    TIPOS DE DIRECCIONES IPv6                        |
+--------------------------------------------------------------------+
|                                                                     |
|   UNICAST (Dirección única)                                         |
|   +------------------------------------------------------------+   |
|   | - Direcciona exactamente UN dispositivo                    |   |
|   | - Como la dirección normal en IPv4                         |   |
|   |                                                             |   |
|   | Tipos:                                                      |   |
|   | - Global Unicast: Globalmente única (como IPv4 pública)    |   |
|   |   Comienza con: 2000::/3 (2xxx: a 3xxx:)                   |   |
|   |                                                             |   |
|   | - Link-Local: Solo válida en red local                     |   |
|   |   Comienza con: fe80::/10                                  |   |
|   |   (Automáticamente presente en cada dispositivo)           |   |
|   |                                                             |   |
|   | - Unique Local: Privada (como IPv4 privada)                |   |
|   |   Comienza con: fc00::/7 (fcxx: o fdxx:)                   |   |
|   +------------------------------------------------------------+   |
|                                                                     |
|   MULTICAST (Dirección de grupo)                                    |
|   +------------------------------------------------------------+   |
|   | - Direcciona un GRUPO de dispositivos                      |   |
|   | - Comienza con: ff00::/8                                   |   |
|   | - Reemplaza al broadcast (¡no existe en IPv6!)             |   |
|   +------------------------------------------------------------+   |
|                                                                     |
|   ANYCAST                                                           |
|   +------------------------------------------------------------+   |
|   | - Una dirección, múltiples posibles destinatarios          |   |
|   | - El paquete va al destinatario MÁS CERCANO                |   |
|   | - Para balanceo de carga y redundancia                     |   |
|   +------------------------------------------------------------+   |
|                                                                     |
|   ! ¡No hay BROADCAST en IPv6! Reemplazado por multicast.          |
|                                                                     |
+--------------------------------------------------------------------+
```

### Direcciones IPv6 Importantes

```
+--------------------------------------------------------------------+
|                 DIRECCIONES IPv6 IMPORTANTES                        |
+--------------------------------------------------------------------+
|                                                                     |
|   ::1                                                               |
|   +------------------------------------------------------------+   |
|   | Dirección loopback (como 127.0.0.1 en IPv4)                |   |
|   | Se refiere al propio ordenador                              |   |
|   +------------------------------------------------------------+   |
|                                                                     |
|   ::                                                                |
|   +------------------------------------------------------------+   |
|   | Dirección no especificada (como 0.0.0.0 en IPv4)           |   |
|   | "Sin dirección" o "todas las direcciones"                   |   |
|   +------------------------------------------------------------+   |
|                                                                     |
|   fe80::/10                                                         |
|   +------------------------------------------------------------+   |
|   | Direcciones link-local                                      |   |
|   | Generadas automáticamente, solo válidas en red local       |   |
|   | Ejemplo: fe80::1a2b:3c4d:5e6f:7890                          |   |
|   +------------------------------------------------------------+   |
|                                                                     |
|   2001:db8::/32                                                     |
|   +------------------------------------------------------------+   |
|   | Rango de documentación (como 192.0.2.0 en IPv4)            |   |
|   | Reservado para ejemplos y guías                             |   |
|   | No se usa en Internet real                                  |   |
|   +------------------------------------------------------------+   |
|                                                                     |
+--------------------------------------------------------------------+
```

### IPv4 vs. IPv6

```
+--------------------------------------------------------------------+
|                    IPv4 vs. IPv6                                    |
+--------------------------------------------------------------------+
|                                                                     |
|   Propiedad      | IPv4              | IPv6                         |
|   ---------------+-------------------+-----------------------------+
|   Long. dirección| 32 bits           | 128 bits                     |
|   Notación       | Decimal (.)       | Hexadecimal (:)              |
|   Ejemplo        | 192.168.1.1       | 2001:db8::1                  |
|   Núm. direcciones| 4.300 millones   | 340 sextillones              |
|   Configuración  | Manual/DHCP       | SLAAC/DHCPv6                 |
|   Broadcast      | Sí                | No (Multicast)               |
|   NAT requerido  | A menudo sí       | Normalmente no               |
|   Cabecera       | Variable          | Fija (40 bytes)              |
|   IPsec          | Opcional          | Integrado                    |
|   Checksum       | Sí (en cabecera)  | No                           |
|                                                                     |
|   Ejemplos de notación:                                             |
|   +------------------------------------------------------------+   |
|   | IPv4 Localhost: 127.0.0.1                                   |   |
|   | IPv6 Localhost: ::1                                         |   |
|   |                                                             |   |
|   | IPv4 Link-Local: 169.254.x.x                                |   |
|   | IPv6 Link-Local: fe80::...                                  |   |
|   +------------------------------------------------------------+   |
|                                                                     |
+--------------------------------------------------------------------+
```

## Términos Clave

| Término | Explicación |
|---------|-------------|
| **IPv6** | Internet Protocol Version 6 (128 bits) |
| **Hexadecimal** | Sistema numérico con base 16 (0-9, a-f) |
| **Unicast** | Dirección para un solo dispositivo |
| **Multicast** | Dirección para un grupo de dispositivos |
| **Link-Local** | Solo válida en red local (fe80::) |
| **SLAAC** | Stateless Address Autoconfiguration (Autoconfiguración sin estado) |
| **::1** | Dirección loopback IPv6 |

## Consejos para el Examen

### Preguntas Comunes del Examen

- ¿Cuántos bits tiene una dirección IPv6?
- ¿Cómo se abrevia una dirección IPv6?
- ¿Cuál es la diferencia entre IPv4 e IPv6?

### Importante Recordar

- **128 bits** (4x más larga que IPv4)
- **8 grupos** de 4 dígitos hex cada uno, separados por **:**
- **::1** = Localhost (como 127.0.0.1)
- **fe80::** = Link-Local (automáticamente en cada dispositivo)
- **No hay broadcast** en IPv6! Solo multicast
- **::** solo puede aparecer UNA VEZ por dirección
- Omitir ceros iniciales: 0042 -> 42

## Ejercicios Prácticos

### Ejercicio 1
Abrevia la dirección IPv6: 2001:0db8:0000:0000:0000:0000:0000:0001

**Solución:**
Paso 1 - Eliminar ceros iniciales:
2001:db8:0:0:0:0:0:1

Paso 2 - Reemplazar secuencia de ceros con :::
**2001:db8::1**

### Ejercicio 2
¿Cuántos bits tiene una dirección IPv6 y cuántos grupos se representan?

**Solución:**
- **128 bits** de longitud total
- **8 grupos** de 16 bits cada uno (4 dígitos hexadecimales)

### Ejercicio 3
Nombra dos diferencias esenciales entre IPv4 e IPv6.

**Solución:**
1. **Longitud de dirección:** IPv4 = 32 bits, IPv6 = 128 bits
2. **Representación:** IPv4 = decimal con puntos (192.168.1.1), IPv6 = hexadecimal con dos puntos (2001:db8::1)

Diferencias adicionales:
- IPv6 no tiene broadcast (solo multicast)
- IPv6 normalmente no requiere NAT
- IPv6 tiene más direcciones disponibles
