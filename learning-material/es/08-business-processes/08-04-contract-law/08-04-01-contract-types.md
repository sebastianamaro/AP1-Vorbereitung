# Tipos de Contrato

## Objetivos de Aprendizaje

Después de esta sección, serás capaz de:
- Distinguir los tipos de contrato más importantes en el sector de TI
- Explicar las características de los contratos de compraventa, obra y servicios
- Asignar el tipo de contrato correcto para diversos servicios de TI
- Nombrar diferencias en responsabilidad y obligación de éxito

---

### Comparación de Tipos de Contrato

| Criterio | Contrato de Compraventa | Contrato de Obra | Contrato de Servicios |
|-----------|-------------------|---------------|------------------|
| **Objeto** | Artículo/Derecho | Obra (resultado) | Actividad |
| **Éxito** | Entrega | Terminación | Solo esfuerzo |
| **Remuneración** | Precio de compra | Honorarios de obra | Tarifa por hora |
| **Derechos de garantía** | Sí (garantía legal) | Sí (subsanación) | Limitados |
| **Aceptación** | Entrega | Aceptación requerida | Ninguna |
| **Terminación** | Desistimiento | Posible en cualquier momento | Plazo de preaviso |
| **Código Civil Alemán** | §§ 433-480 | §§ 631-651 | §§ 611-630 |

### El Contrato de Compraventa

```
┌─────────────────────────────────────────────────────────────────────┐
│                    CONTRATO DE COMPRAVENTA (§§ 433 y ss. BGB)        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  VENDEDOR                                          COMPRADOR         │
│  ┌─────────────┐                           ┌─────────────┐          │
│  │             │  Obligación: Entregar     │             │          │
│  │   Vendedor  │ ────────────────────────► │   Comprador │          │
│  │             │  el artículo              │             │          │
│  │             │                           │             │          │
│  │             │  Obligación: Pagar precio │             │          │
│  │             │ ◄──────────────────────── │             │          │
│  │             │  de compra                │             │          │
│  └─────────────┘                           └─────────────┘          │
│                                                                      │
│  Obligaciones del vendedor:                                         │
│  → Entregar el artículo                                             │
│  → Transferir la propiedad                                          │
│  → Artículo libre de defectos materiales y legales                  │
│                                                                      │
│  Obligaciones del comprador:                                        │
│  → Pagar el precio de compra                                        │
│  → Aceptar el artículo                                              │
│                                                                      │
│  Ejemplos TI:                                                       │
│  → Compra de hardware (PC, servidor, impresora)                     │
│  → Compra de licencia de software (pago único)                      │
│  → Compra de consumibles                                            │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### El Contrato de Obra

```
┌─────────────────────────────────────────────────────────────────────┐
│                    CONTRATO DE OBRA (§§ 631 y ss. BGB)               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  CONTRATISTA                                    CLIENTE              │
│  (Empresario)                                   (Cliente)            │
│  ┌─────────────┐                           ┌─────────────┐          │
│  │             │  Obligación: Crear        │             │          │
│  │ Contratista │ ────────────────────────►│   Cliente   │          │
│  │             │  una obra (¡ÉXITO!)       │             │          │
│  │             │                           │             │          │
│  │             │  Obligación: Pagar        │             │          │
│  │             │ ◄──────────────────────── │             │          │
│  │             │  honorarios tras aceptar  │             │          │
│  └─────────────┘                           └─────────────┘          │
│                                                                      │
│  IMPORTANTE: ¡Se debe el éxito!                                     │
│  → La obra debe FUNCIONAR                                           │
│  → No solo esfuerzo, sino resultado                                 │
│                                                                      │
│  Aceptación:                                                        │
│  → El cliente DEBE aceptar la obra (si es contractual)              │
│  → Tras la aceptación: Honorarios debidos, comienza garantía        │
│                                                                      │
│  Ejemplos TI:                                                       │
│  → Desarrollo de software personalizado                             │
│  → Instalación de red                                               │
│  → Creación de sitio web                                            │
│  → Migración de base de datos                                       │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### El Contrato de Servicios

```
┌─────────────────────────────────────────────────────────────────────┐
│                    CONTRATO DE SERVICIOS (§§ 611 y ss. BGB)          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  PROVEEDOR DE SERVICIOS                    RECEPTOR DE SERVICIOS    │
│  ┌─────────────┐                           ┌─────────────┐          │
│  │             │  Obligación: Realizar     │             │          │
│  │  Proveedor  │ ────────────────────────►│  Receptor   │          │
│  │             │  actividad (¡SIN éxito!)  │             │          │
│  │             │                           │             │          │
│  │             │  Obligación: Pagar        │             │          │
│  │             │ ◄──────────────────────── │             │          │
│  │             │  remuneración             │             │          │
│  └─────────────┘                           └─────────────┘          │
│                                                                      │
│  IMPORTANTE: ¡Solo actividad, NO se debe éxito!                     │
│  → La consultoría puede no tener éxito → aún así remuneración      │
│  → Lo que importa es el esfuerzo, no el resultado                   │
│                                                                      │
│  Caso especial: Contrato de trabajo                                 │
│  → Es un contrato de servicios especial                             │
│  → El empleado debe la prestación laboral                           │
│                                                                      │
│  Ejemplos TI:                                                       │
│  → Consultoría TI por horas                                         │
│  → Formación y cursos                                               │
│  → Línea de soporte (sin garantía de éxito)                         │
│  → Despliegue de freelancer                                         │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

```
│  → CONTRATO DE COMPRAVENTA (compra de hardware)                     │
│                                                                      │
│  "Nos desarrollan un software"                                      │
│  → CONTRATO DE OBRA (éxito: software funcional)                     │
│                                                                      │
│  "Contratamos consultoría TI por 40 horas"                          │
│  → CONTRATO DE SERVICIOS (actividad, sin garantía de éxito)         │
│                                                                      │
│  "Nos instalan la red"                                              │
│  → CONTRATO DE OBRA (éxito: red funcional)                          │
│                                                                      │
│  "Compramos licencias de Microsoft 365"                             │
│  → CONTRATO DE COMPRAVENTA (adquisición de licencia)                │
│                                                                      │
│  "Alquilamos almacenamiento en la nube"                             │
│  → CONTRATO DE ARRENDAMIENTO (provisión temporal)                   │
│                                                                      │

```

### Otros Tipos de Contrato Relevantes

| Tipo de Contrato | Descripción | Ejemplo TI |
|---------------|-------------|------------|
| **Contrato de arrendamiento** | Provisión temporal | Almacenamiento en la nube, leasing de hardware |
| **Contrato de licencia** | Derecho a usar software | Contratos SaaS, licencias de software |
| **Contrato de mantenimiento** | Mantenimiento regular | Mantenimiento TI, actualizaciones |
| **Service Level Agreement** | Acuerdo de calidad | SLA para servicios TI |

## Términos Clave

| Término | Explicación |
|------|-------------|
| **Contrato de compraventa** | Transferencia de propiedad por dinero |
| **Contrato de obra** | Creación de una obra (se debe el éxito) |
| **Contrato de servicios** | Realización de una actividad (sin éxito) |
| **Aceptación** | Confirmación del cumplimiento del contrato en contrato de obra |
| **Honorarios de obra** | Remuneración en contrato de obra |
| **Garantía** | Responsabilidad por defectos |

---

## Consejos para el Examen

1. **Diferencia principal a recordar:**
   - **Obra** = Se debe el éxito
   - **Servicios** = Solo se debe la actividad

2. **Pregunta típica del examen:**
   - "¿Qué tipo de contrato aplica?" → Analizar ejemplo de TI

3. **¡Aceptación solo para contratos de obra!**
   - Contrato de compraventa: Entrega
   - Contrato de servicios: No se necesita aceptación

## Ejercicios Prácticos

### Ejercicio 1: Asignar Tipos de Contrato

¿Qué tipo de contrato aplica?

a) ConSystem desarrolla software de gestión de inventario para un cliente
b) ConSystem compra 20 portátiles nuevos de un mayorista
c) ConSystem contrata a un consultor externo para análisis de proyecto
d) ConSystem instala una red completa incluyendo aceptación
e) ConSystem firma un contrato mensual de almacenamiento en la nube

<details>
<summary>Mostrar solución</summary>

a) **Contrato de obra** - Se debe el éxito (software funcional)
b) **Contrato de compraventa** - Compra de hardware
c) **Contrato de servicios** - Actividad (consultoría) sin garantía de éxito
d) **Contrato de obra** - Éxito (red funcional) con aceptación
e) **Contrato de arrendamiento** - Provisión temporal de espacio de almacenamiento

</details>

### Ejercicio 2: ¿Éxito o Esfuerzo?

Una empresa encarga a ConSystem:
- "Analizar nuestra infraestructura TI y hacer sugerencias de mejora"

a) ¿Qué tipo de contrato aplica?
b) ¿Qué debe ConSystem?
c) ¿ConSystem recibe pago aunque las sugerencias no se implementen?

<details>
<summary>Mostrar solución</summary>

a) **Contrato de servicios** (o posiblemente contrato de obra, si se acuerda un informe como "obra")

b) **ConSystem debe:**
- Para contrato de servicios: Actividad de análisis adecuada, esfuerzo
- Para contrato de obra: Informe de análisis completado

c) **¡Sí!**
- Para contrato de servicios: La remuneración es por la actividad, no por la implementación
- La implementación de las sugerencias depende del cliente
- ConSystem ha proporcionado su servicio (análisis y sugerencias)

</details>

### Ejercicio 3: Aceptación

ConSystem GmbH ha creado un sitio web para un cliente. El cliente se niega a declarar la aceptación, aunque el sitio web funciona según lo especificado en el contrato.

a) ¿Qué tipo de contrato es este?
b) ¿Qué significa la aceptación legalmente?
c) ¿Qué derechos tiene ConSystem?

<details>
<summary>Mostrar solución</summary>

a) **Contrato de obra** - La creación de sitio web es una obra

b) **Significado de la aceptación:**
- Confirmación de que la obra es contractual
- Los honorarios de obra se hacen exigibles
- Comienza el período de garantía
- La carga de la prueba se invierte (el contratista ya no necesita probar)

c) **Derechos de ConSystem:**
- Derecho a la aceptación si la obra es contractual (§ 640 BGB)
- En caso de rechazo injustificado: Establecer plazo
- Tras expirar el plazo: Se considera que la aceptación ha ocurrido (§ 640 párr. 2 BGB)
- Se pueden reclamar los honorarios de obra judicialmente
- Posiblemente daños por retraso

</details>
