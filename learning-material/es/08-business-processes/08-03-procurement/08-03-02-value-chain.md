# Creación de Valor y Cadena de Valor

## Objetivos de Aprendizaje

Después de esta sección, serás capaz de:
- Explicar el concepto de creación de valor
- Describir la cadena de valor de Porter
- Distinguir entre actividades primarias y de apoyo
- Identificar la creación de valor en empresas de TI

---

## Contenido Principal

### ¿Qué es la Creación de Valor?

La **creación de valor** es el valor añadido generado al transformar insumos en productos.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    CREACIÓN DE VALOR                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  Creación de Valor = Valor de Productos/Servicios - Valor de Insumos│
│                                                                      │
│  ┌───────────────┐     ┌───────────────┐     ┌───────────────┐      │
│  │               │     │               │     │               │      │
│  │    INSUMO     │ ──► │ PROCESAMIENTO │ ──► │   PRODUCTO    │      │
│  │               │     │               │     │               │      │
│  │ Materias      │     │ Producción    │     │ Productos     │      │
│  │ primas        │     │ Servicios     │     │ Terminados    │      │
│  │               │     │               │     │               │      │
│  │  Valor: €100  │     │ + Valor añad. │     │  Valor: €300  │      │
│  │               │     │   + €200      │     │               │      │
│  └───────────────┘     └───────────────┘     └───────────────┘      │
│                                                                      │
│  Creación de Valor = €300 - €100 = €200                             │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### La Cadena de Valor de Porter

La **cadena de valor** muestra todas las actividades de una empresa que contribuyen a la creación de valor.

```
┌─────────────────────────────────────────────────────────────────────┐
│           CADENA DE VALOR DE PORTER                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  ACTIVIDADES DE APOYO:                                              │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │  Infraestructura Empresarial (Gestión, Planificación, Finanzas)│  │
│  ├─────────────────────────────────────────────────────────────┤    │
│  │  Gestión de Recursos Humanos (Contratación, Formación, Salario)│  │
│  ├─────────────────────────────────────────────────────────────┤    │
│  │  Desarrollo Tecnológico (I+D, Mejora de Procesos)           │    │
│  ├─────────────────────────────────────────────────────────────┤    │
│  │  Aprovisionamiento (Compra de Materiales y Servicios)       │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                                                                      │
│  ACTIVIDADES PRIMARIAS:                                             │
│  ┌──────────┬──────────┬──────────┬──────────┬──────────┐          │
│  │ Logística│          │ Logística│Marketing │          │          │
│  │ de       │Operaciones│ de      │ y Ventas │ Servicio │  ──► M   │
│  │ Entrada  │          │ Salida   │          │          │      A   │
│  │ Recep.   │Producción│ Almacén  │Publicidad│ Atención │      R   │
│  │ Mercancía│          │ Envío    │ Ventas   │ al Cliente│     G   │
│  │ Almacén  │          │          │          │ Soporte  │      E   │
│  └──────────┴──────────┴──────────┴──────────┴──────────┘      N   │
│                                                                      │
│  ¡Cada actividad contribuye al MARGEN DE BENEFICIO!                 │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Actividades Primarias

| Actividad | Descripción | Ejemplo TI |
|----------|-------------|------------|
| **Logística de Entrada** | Recepción de mercancías, almacenamiento | Recepción de hardware |
| **Operaciones** | Producción, procesamiento | Desarrollo de software, servicio TI |
| **Logística de Salida** | Almacenamiento, distribución | Entrega, despliegue en la nube |
| **Marketing y Ventas** | Publicidad, ventas | Consultoría TI, preparación de ofertas |
| **Servicio** | Atención al cliente, soporte | Soporte TI, mantenimiento |

### Actividades de Apoyo

| Actividad | Descripción | Ejemplo TI |
|----------|-------------|------------|
| **Infraestructura Empresarial** | Gestión, planificación, control | Estrategia TI, presupuesto |
| **Gestión de Recursos Humanos** | Contratación, formación | Especialistas TI, formación continua |
| **Desarrollo Tecnológico** | I+D, innovación | Nuevas tecnologías, desarrollo interno |
| **Aprovisionamiento** | Compras | Hardware, licencias, servicios en la nube |

### Cadena de Valor en TI

```
┌─────────────────────────────────────────────────────────────────────┐
│          CADENA DE VALOR - EMPRESA DE SISTEMAS TI                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  Apoyo:                                                             │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │  Infraestructura TI (servidores propios, herramientas, licencias)││
│  ├─────────────────────────────────────────────────────────────┤    │
│  │  Personal (consultores TI, desarrolladores, personal de soporte)││
│  ├─────────────────────────────────────────────────────────────┤    │
│  │  Tecnología (desarrollo continuo, nuevas herramientas)      │    │
│  ├─────────────────────────────────────────────────────────────┤    │
│  │  Aprovisionamiento (hardware, software, servicios en la nube)│   │
│  └─────────────────────────────────────────────────────────────┘    │
│                                                                      │
│  Primarias (Ejemplo: Proyecto de servidor para cliente):            │
│  ┌──────────┬──────────┬──────────┬──────────┬──────────┐          │
│  │          │          │          │          │          │          │
│  │ Compra   │  Confi-  │  Insta-  │Consultoría│ Soporte │ ──► €   │
│  │ Servidor │guración  │ lación   │ Ventas   │ Manten.  │          │
│  │  +€500   │ +€800    │ +€400    │ +€300    │ +€200/año│          │
│  │          │          │          │          │          │          │
│  └──────────┴──────────┴──────────┴──────────┴──────────┘          │
│                                                                      │
│  Creación de Valor = Precio de Venta - Precio de Compra = Margen   │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Profundidad de Creación de Valor

La **profundidad de creación de valor** describe cuánto hace una empresa por sí misma vs. lo que compra.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    PROFUNDIDAD DE CREACIÓN DE VALOR                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  ALTA profundidad de creación de valor:                             │
│  → Hacer mucho por uno mismo                                        │
│  → Comprar poco                                                     │
│  → Ejemplo: Desarrollo de software propio                           │
│                                                                      │
│  BAJA profundidad de creación de valor:                             │
│  → Hacer poco por uno mismo                                         │
│  → Comprar mucho (externalización)                                  │
│  → Ejemplo: Solo ventas, sin producción propia                      │
│                                                                      │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │                                                              │    │
│  │  Materia prima → Componente → Producción → Ventas → Servicio│    │
│  │                                                              │    │
│  │  ◄──────────────────────────────────────────────────────────►   │
│  │                                                              │    │
│  │  Alta profundidad: Todos los pasos hechos internamente       │    │
│  │  Baja profundidad: Solo ventas y servicio                    │    │
│  │                                                              │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                                                                      │
│  Ejemplo TI:                                                        │
│  → Alta profundidad: Desarrollo de sistema ERP propio              │
│  → Baja profundidad: Reventa de software estándar                  │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Decisión de Hacer o Comprar

| Criterio | Hacer (Interno) | Comprar (Externo) |
|-----------|-----------------|----------------|
| **Costos** | Altos costos fijos | Costos variables |
| **Calidad** | Control total | Dependiente del proveedor |
| **Flexibilidad** | Menos flexible | Más adaptable |
| **Know-how** | Construir internamente | Usar experiencia externa |
| **Capacidad** | Recursos propios necesarios | Escalable |

## Términos Clave

| Término | Explicación |
|------|-------------|
| **Creación de valor** | Valor añadido a través del procesamiento |
| **Cadena de valor** | Todas las actividades que añaden valor |
| **Actividades primarias** | Directamente sobre el producto (producción, ventas) |
| **Actividades de apoyo** | Indirectas (personal, TI, compras) |
| **Profundidad de creación de valor** | Proporción de trabajo propio |
| **Hacer o comprar** | Decisión: Hacer uno mismo o comprar |
| **Externalización** | Subcontratación de actividades |

## Consejos para el Examen

1. **Conoce el modelo de Porter:**
   - 5 actividades primarias (mnemotécnico: "L O L M S" - Logística entrada, Operaciones, Logística salida, Marketing, Servicio)
   - 4 actividades de apoyo

2. **Distinción:**
   - Primarias = directamente sobre producto/cliente
   - Apoyo = permiten las actividades primarias

3. **Referencia a TI:**
   - TI a menudo es una actividad de apoyo (desarrollo tecnológico)
   - ¡Para empresas de TI: los servicios de TI son actividades primarias!

## Ejercicios Prácticos

### Ejercicio 1: Asignar Actividades

Asigna las actividades de ConSystem GmbH:

a) Desarrollo de una nueva solución de software
b) Contratación de un nuevo consultor de TI
c) Instalación de servidores en el sitio del cliente
d) Compra de hardware a un mayorista
e) Línea de ayuda de soporte TI

<details>
<summary>Mostrar solución</summary>

a) **Primaria: Operaciones** (para empresas de TI, el desarrollo es el negocio central)
   o **Apoyo: Desarrollo Tecnológico** (si son herramientas internas)

b) **Apoyo: Gestión de Recursos Humanos**

c) **Primaria: Operaciones** o **Logística de Salida** (entrega al cliente)

d) **Apoyo: Aprovisionamiento**

e) **Primaria: Servicio**

</details>

### Ejercicio 2: Calcular Creación de Valor

ConSystem GmbH compra un servidor por €2,000 y lo vende configurado e instalado al cliente por €3,800.

a) ¿Cuál es la creación de valor?
b) ¿Qué actividades contribuyen a la creación de valor?

<details>
<summary>Mostrar solución</summary>

a) **Creación de valor:**
€3,800 - €2,000 = **€1,800**

b) **Actividades que añaden valor:**
- Consultoría al cliente (Marketing y Ventas)
- Configuración del servidor (Operaciones)
- Instalación en el sitio del cliente (Operaciones/Logística de Salida)
- Documentación
- Formación si aplica

Estas actividades justifican el precio adicional de €1,800.

</details>

### Ejercicio 3: Hacer o Comprar

ConSystem GmbH está considerando desarrollar su propio software de contabilidad (Hacer) o comprar uno (Comprar).

Nombra dos argumentos para Hacer y para Comprar.

<details>
<summary>Mostrar solución</summary>

**Argumentos para HACER (Desarrollo interno):**
1. Adaptado exactamente a los procesos de la empresa
2. Independencia a largo plazo de costos de licencia
3. Control total sobre características y actualizaciones
4. El know-how permanece en la empresa

**Argumentos para COMPRAR (Adquisición):**
1. Disponible más rápido, sin tiempo de desarrollo
2. Menor inversión inicial
3. Soporte profesional del proveedor
4. Actualizaciones regulares del fabricante
5. La contabilidad no es la competencia central

**Recomendación para ConSystem:** COMPRAR - La contabilidad no es el negocio central, el software estándar está maduro.

</details>
