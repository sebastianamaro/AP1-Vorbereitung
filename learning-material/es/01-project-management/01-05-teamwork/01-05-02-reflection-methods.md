# Métodos de Reflexión

## Objetivos de Aprendizaje

Después de este capítulo, usted debería ser capaz de:
- Comprender el propósito de los métodos de reflexión
- Explicar las Lecciones Aprendidas
- Conocer la importancia de una cultura de retroalimentación
- Saber cuándo tiene lugar la reflexión en un proyecto

---

## Contenido Principal

### ¿Por Qué Reflexionar?

La reflexión sirve para aprender de las experiencias y mejorar continuamente. En la gestión de proyectos y métodos ágiles, la reflexión regular es un componente importante.

**Objetivos de la Reflexión:**
- Aprender de los errores
- Identificar y repetir éxitos
- Mejorar procesos
- Fortalecer la cohesión del equipo

---

### Lecciones Aprendidas

#### Definición

**Lecciones Aprendidas** se refiere al análisis post-proyecto para aprender sistemáticamente de experiencias y eventos y hacer que estos conocimientos sean útiles para proyectos futuros.

#### Preguntas Típicas

| Categoría | Preguntas |
|-----------|-----------|
| **Éxitos** | ¿Qué funcionó bien? ¿Qué debemos seguir haciendo? |
| **Problemas** | ¿Qué no funcionó? ¿Cuáles fueron las causas? |
| **Sorpresas** | ¿Qué fue inesperado? ¿Cómo reaccionamos? |
| **Mejoras** | ¿Qué haríamos diferente la próxima vez? |

#### Proceso de una Sesión de Lecciones Aprendidas

```
+-------------+    +-------------+    +-------------+    +-------------+
| 1. RECOPILAR| -> | 2. AGRUPAR  | -> | 3. EVALUAR  | -> |4. DOCUMENTAR|
|             |    |             |    |             |    |             |
| Reunir      |    | Agrupar     |    | Priorizar   |    | Registrar   |
| experiencias|    | temas       |    | ¿Qué es     |    | para el     |
|             |    |             |    | importante? |    | futuro      |
+-------------+    +-------------+    +-------------+    +-------------+
```

#### Momento
- **Cierre del proyecto:** Análisis completo
- **Después de hitos:** Revisión intermedia
- **Con Scrum:** Sprint Retrospective

### Cultura de Retroalimentación

#### Definición

Una **cultura de retroalimentación** significa que se da retroalimentación regularmente y de manera abierta. Es el fundamento para la mejora continua.

#### Características de una Buena Retroalimentación

| Característica | Descripción |
|----------------|-------------|
| **Honesta** | Veraz, sin endulzar |
| **Respetuosa** | Apreciativa, no hiriente |
| **Constructiva** | Con sugerencias de mejora |
| **Oportuna** | Lo antes posible después del evento |
| **Específica** | Relacionada con situaciones específicas |
| **Útil** | Útil para el receptor |

#### Reglas de Retroalimentación

**Dar Retroalimentación:**
- Usar declaraciones en primera persona ("Observé que...")
- Describir comportamiento, no juzgar a la persona
- Dar ejemplos concretos
- Hacer sugerencias de mejora

**Recibir Retroalimentación:**
- Escuchar sin interrumpir
- Hacer preguntas si no está claro
- No justificarse
- Agradecer la retroalimentación

### Reflexión en Scrum: Sprint Retrospective

La **Sprint Retrospective** es la reunión de Scrum para la reflexión del equipo:

| Aspecto | Detalles |
|---------|----------|
| **Cuándo** | Al final de cada Sprint, después del Sprint Review |
| **Duración** | Máximo 3 horas (para Sprint de 4 semanas) |
| **Participantes** | Equipo Scrum (PO, SM, Equipo de Desarrollo) |
| **Enfoque** | El proceso, no el producto |

**Preguntas Típicas:**
1. ¿Qué salió bien en el Sprint?
2. ¿Qué podemos mejorar?
3. ¿Qué acciones concretas tomaremos?

---

### Métodos para la Reflexión

#### Start-Stop-Continue

| Categoría | Pregunta |
|-----------|----------|
| **Start** | ¿Qué deberíamos empezar a hacer? |
| **Stop** | ¿Qué deberíamos dejar de hacer? |
| **Continue** | ¿Qué deberíamos seguir haciendo? |

#### Retrospectiva 4-L

| L | Significado |
|---|-------------|
| **Liked** | ¿Qué me gustó? |
| **Learned** | ¿Qué aprendí? |
| **Lacked** | ¿Qué me faltó? |
| **Longed for** | ¿Qué deseaba? |

## Términos Clave

| Término | Explicación |
|---------|-------------|
| **Lecciones Aprendidas** | Análisis sistemático de experiencias del proyecto |
| **Cultura de Retroalimentación** | Ambiente donde se da retroalimentación abierta |
| **Sprint Retrospective** | Reunión de Scrum para mejora del proceso |
| **Retroalimentación Constructiva** | Retroalimentación con sugerencias de mejora |
| **Mejora Continua** | Esfuerzo constante por la optimización |

## Consejos para el Examen

### Preguntas Frecuentes del Examen
- ¿Qué se entiende por Lecciones Aprendidas?
- ¿Cuándo tiene lugar la Sprint Retrospective?
- ¿Cuál es la diferencia entre Sprint Review y Retrospective?

### Recordar
- **Sprint Review** = Retroalimentación del producto (¿Qué construimos?)
- **Sprint Retrospective** = Retroalimentación del proceso (¿Cómo trabajamos?)

---

## Ejercicios Prácticos

### Ejercicio 1
¿Qué se entiende por Lecciones Aprendidas?

**Solución:** El análisis post-proyecto para aprender sistemáticamente de las experiencias y hacer que estos conocimientos sean útiles para proyectos futuros.

### Ejercicio 2
Nombre tres características de una buena retroalimentación.

**Solución:**
1. Honesta
2. Respetuosa
3. Constructiva (o: oportuna, específica, útil)

### Ejercicio 3
¿Cuál es la diferencia entre Sprint Review y Sprint Retrospective?

**Solución:**
- **Sprint Review:** Presentación de resultados del trabajo, retroalimentación sobre el PRODUCTO
- **Sprint Retrospective:** Análisis de la colaboración, mejora del PROCESO
