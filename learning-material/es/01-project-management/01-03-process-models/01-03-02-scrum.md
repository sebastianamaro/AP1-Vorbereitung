# Scrum

Después de este capítulo, usted debería ser capaz de:
- Explicar Scrum como un marco ágil

## Contenido Principal

### ¿Qué es Scrum?

**Scrum** es un **marco ágil** para desarrollar y gestionar productos. Permite el desarrollo iterativo e incremental en ciclos cortos (Sprints).

> **Nota:** Kanban, XP y otros métodos ágiles fueron eliminados del catálogo de examen de 2025. ¡Solo Scrum sigue siendo relevante para el examen!

---

### Principios Fundamentales de Scrum

- **Iterativo:** Ciclos repetidos (Sprints)
- **Incremental:** El producto crece paso a paso
- **Auto-organización:** El equipo se organiza a sí mismo
- **Transparencia:** El progreso es visible para todos
- **Adaptabilidad:** Respuesta flexible a los cambios

---

### Los Tres Roles de Scrum

```
+----------------------------------------------------------+
|                    EQUIPO SCRUM                           |
|                                                           |
|  +--------------+  +--------------+  +----------------+  |
|  |   PRODUCT    |  |    SCRUM     |  |   EQUIPO DE    |  |
|  |    OWNER     |  |   MASTER     |  |  DESARROLLO    |  |
|  +--------------+  +--------------+  +----------------+  |
|                                                           |
+----------------------------------------------------------+
```

| Rol | Tareas |
|-----|--------|
| **Product Owner** | - Representa los intereses de los stakeholders<br>- Mantiene y prioriza el Product Backlog<br>- Define la visión del producto<br>- Decide sobre los requisitos |
| **Scrum Master** | - Asegura que se sigan las reglas de Scrum<br>- Elimina obstáculos (impedimentos)<br>- Protege al equipo de perturbaciones<br>- Facilita las reuniones<br>- Coach del equipo |
| **Equipo de Desarrollo** | - Implementa los requisitos<br>- Auto-organizado<br>- Multifuncional (todas las habilidades presentes)<br>- Tamaño típico: 3-9 personas |

---

### Los Eventos de Scrum (Reuniones)

#### Sprint
- **Duración:** 2-4 semanas (fija)
- **Objetivo:** Crear un incremento de producto funcional
- **Importante:** La duración del Sprint permanece constante durante todo el proyecto

#### Las Cuatro Reuniones en el Sprint

| Reunión | Cuándo | Duración | Participantes | Propósito |
|---------|--------|----------|---------------|-----------|
| **Sprint Planning** | Inicio del Sprint | Máx. 8h (para Sprint de 4 semanas) | Todo el equipo | ¿Qué se implementará en el Sprint? ¿Cómo? |
| **Daily Stand-up** | Diariamente | Máx. 15 min. | Equipo de desarrollo (+ SM) | ¿Qué hice? ¿Qué haré? ¿Qué me bloquea? |
| **Sprint Review** | Fin del Sprint | Máx. 4h | Equipo + Stakeholders | Presentación de resultados, retroalimentación |
| **Sprint Retrospective** | Después del Review | Máx. 3h | Equipo Scrum | ¿Qué salió bien? ¿Qué se puede mejorar? |

### Los Artefactos de Scrum

```
+-----------------+     +-----------------+     +-----------------+
|  PRODUCT        | ->  |  SPRINT         | ->  |  INCREMENTO DE  |
|  BACKLOG        |     |  BACKLOG        |     |  PRODUCTO       |
+-----------------+     +-----------------+     +-----------------+
   Todos los              Por hacer en          Resultado
   requisitos             el Sprint actual      parcial terminado
```

| Artefacto | Descripción |
|-----------|-------------|
| **Product Backlog** | Lista priorizada de todos los requisitos (User Stories), mantenida por el Product Owner |
| **Sprint Backlog** | Elementos seleccionados para el Sprint actual + plan de implementación |
| **Incremento de Producto** | Producto potencialmente entregable al final del Sprint |

---

### El Proceso Scrum

```
                    +--------------------------------------+
                    |            SPRINT (2-4 semanas)      |
                    |                                      |
Product ----------> |  Sprint    Daily    Sprint  Sprint   | ---> Incremento
Backlog             |  Planning  Scrum    Review  Retro    |      de Producto
                    |                                      |
                    +--------------------------------------+
                                    |
                                    v
                            Siguiente Sprint...
```

### Scrum vs. Cascada

| Aspecto | Scrum | Cascada |
|---------|-------|---------|
| **Proceso** | Iterativo (Sprints) | Secuencial (Fases) |
| **Planificación** | Ciclos de planificación cortos | Planificación integral al inicio |
| **Cambios** | Bienvenidos | Difíciles de implementar |
| **Contacto con cliente** | Continuo (Sprint Review) | Principalmente al principio/fin |
| **Documentación** | Mínima, "software funcionando" | Extensa |
| **Resultados** | Después de cada Sprint | Al final del proyecto |
| **Riesgo** | Descubierto temprano por ciclos cortos | Descubierto tarde |

## Términos Clave

| Término | Explicación |
|---------|-------------|
| **Sprint** | Período de tiempo fijo (2-4 semanas) para desarrollo |
| **Product Owner** | Responsable del producto y del backlog |
| **Scrum Master** | Guardián del proceso Scrum, elimina obstáculos |
| **User Story** | Requisito desde la perspectiva del usuario ("Como... quiero... para que...") |
| **Impedimento** | Obstáculo que dificulta el progreso |
| **Incremento** | Resultado parcial funcional al final del Sprint |

## Consejos para el Examen

### Preguntas Frecuentes del Examen
1. ¿Cuáles son los tres roles en Scrum?
2. ¿Cuáles son las tareas del Scrum Master?
3. ¿Qué reuniones hay?
4. ¿Cuál es la diferencia entre Product Backlog y Sprint Backlog?

### Ayuda de Memoria para los Roles
- **P**roduct **O**wner = **QUÉ** (¿Qué se está construyendo?)
- **S**crum **M**aster = **CÓMO** (¿Cómo funciona el proceso?)
- **E**quipo de **D**esarrollo = **HACER** (Hace el trabajo)

## Ejercicios Prácticos

### Ejercicio 1
Nombre los tres roles de Scrum y una tarea principal de cada uno.

**Solución:**
1. **Product Owner:** Mantiene el Product Backlog
2. **Scrum Master:** Elimina obstáculos
3. **Equipo de Desarrollo:** Implementa requisitos

### Ejercicio 2
¿Cuál es la diferencia entre Sprint Review y Sprint Retrospective?

**Solución:**
- **Sprint Review:** Presentación de resultados del trabajo a los stakeholders, retroalimentación sobre el PRODUCTO
- **Sprint Retrospective:** Reunión interna del equipo para mejorar el PROCESO

### Ejercicio 3
¿Quién es responsable de mantener el Product Backlog?

**Solución:** El Product Owner
