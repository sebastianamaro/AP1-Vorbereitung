# Communication Types in IT Support

## Learning Objectives

After this section, you will be able to:
- Distinguish different communication channels in IT support
- Name the advantages and disadvantages of each channel
- Select the appropriate channel for different situations
- Apply basic rules of professional communication

---

### Phone Support

```
┌─────────────────────────────────────────────────────────────────────┐
│                    PHONE SUPPORT                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  ADVANTAGES:                                                        │
│  ──────────────────────────────────────────────────────────────     │
│  → Quick problem resolution                                         │
│  → Direct follow-up questions possible                              │
│  → Emotions recognizable (tone of voice)                            │
│  → Personal contact                                                 │
│                                                                      │
│  DISADVANTAGES:                                                     │
│  ──────────────────────────────────────────────────────────────     │
│  → No visual support                                                │
│  → Difficult to document                                            │
│  → Both must be available simultaneously                            │
│  → Technical descriptions difficult                                 │
│                                                                      │
│  BEST PRACTICES:                                                    │
│  ──────────────────────────────────────────────────────────────     │
│  → Friendly greeting with name                                      │
│  → Active listening                                                 │
│  → Summarizing the problem                                          │
│  → Assign and communicate ticket number                             │
│  → Follow up for longer processing times                            │
│                                                                      │
│  SUITABLE FOR:                                                      │
│  → Initial intake of incidents                                      │
│  → Urgent problems                                                  │
│  → Complex matters requiring explanation                            │
│  → Escalations                                                      │
│                                                                      │
```

### Channel Selection by Situation

| Situation | Recommended Channel | Rationale |
|-----------|---------------------|-----------|
| **Total outage** | Phone | Urgent, immediate response needed |
| **Forgot password** | Self-Service | Standard process, automatable |
| **Send screenshot** | Email/Ticket | Attachments possible |
| **Complex problem** | Remote Support + Phone | Combination ideal |
| **Status inquiry** | Ticket System | User can check themselves |
| **Training question** | Knowledge Base | Reusable answer |

### Professional Communication

```
┌─────────────────────────────────────────────────────────────────────┐
│           RULES FOR PROFESSIONAL COMMUNICATION                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  DO's:                                                              │
│  ──────────────────────────────────────────────────────────────     │
│  ✓ Stay friendly and respectful                                     │
│  ✓ Avoid technical jargon or explain it                             │
│  ✓ Listen actively and ask follow-up questions                      │
│  ✓ Summarize the problem for confirmation                           │
│  ✓ Provide realistic time estimates                                 │
│  ✓ Always mention ticket/case number                                │
│  ✓ Proactively inform for longer processing times                   │
│                                                                      │
│  DON'Ts:                                                            │
│  ──────────────────────────────────────────────────────────────     │
│  ✗ Blame user for errors                                            │
│  ✗ Show impatience or appear annoyed                                │
│  ✗ Use too much technical jargon                                    │
│  ✗ Make unrealistic promises                                        │
│  ✗ "Forget" user on hold                                            │
│  ✗ Disparaging comments about colleagues/systems                    │
│                                                                      │
│  KEY PRINCIPLE:                                                     │
│  "The user is not the problem - they HAVE a problem!"               │
│                                                                      │

```

## Key Terms

| Term | Explanation |
|------|-------------|
| **Synchronous Communication** | Simultaneous exchange (e.g., phone) |
| **Asynchronous Communication** | Time-delayed exchange (e.g., email) |
| **Remote Support** | Remote access to the user's PC |
| **Knowledge Base** | Database with solutions |
| **FAQ** | Frequently Asked Questions |
| **Self-Service** | User solves problem independently |

---

## Exam Tips

1. **Justify channel selection:**
   - Urgent → synchronous (phone)
   - Documentation needed → asynchronous (email/ticket)
   - Complex → combination (remote support + phone)

2. **Know advantages and disadvantages:**
   - Each channel has specific strengths and weaknesses
   - Often in exams: "Which channel is suitable and why?"

3. **Professional behavior:**
   - Friendliness even with difficult users
   - No blame assignment

## Practice Exercises

### Exercise 1: Select Channel

Choose the appropriate communication channel and justify:

a) An employee can no longer log in to their PC
b) Accounting needs a report on IT costs from the last quarter
c) A new employee has questions about setting up their email client

<details>
<summary>Show Solution</summary>

a) **Phone + Remote Support**
   - Urgent (employee cannot work)
   - Direct intervention possible
   - Can have various causes (password, account locked, AD problem)

b) **Email**
   - Not urgent
   - Report can be sent as attachment
   - Documentation important

c) **Self-Service/Knowledge Base** or **Email with instructions**
   - Standard process
   - Instructions can be reused
   - Not a time-critical problem

</details>

### Exercise 2: Compose Email

A user reported via email: "My PC is slow!!"

Formulate a professional reply email asking for more information.

<details>
<summary>Show Solution</summary>

**Example response:**

```
Subject: Re: Your PC is slow - Ticket #2024-0815

Dear Mr. Mueller,

Thank you for your message.

To resolve your problem as quickly as possible, I need
some additional information:

1. Since when has the PC been slow?
2. Which programs are particularly affected?
3. Have you recently installed or changed anything?
4. How exactly does the problem manifest?
   (e.g., long startup time, programs not responding)

In the meantime, you can try the following measures:
- Restart your PC
- Close programs you don't need

I will get back to you as soon as I have more information.

Best regards,
Tim Schmidt
IT Support, ConSystem GmbH
Tel: 0800-1234567
Ticket: #2024-0815
```

</details>

### Exercise 3: Identify Communication Errors

What is wrong with the following statement from an IT supporter?

"You did that wrong. It's all in the manual. Who doesn't read that?"

<details>
<summary>Show Solution</summary>

**Errors:**

1. **Blame assignment** - "You did that wrong"
   → Better: "This is a common error that we can quickly fix"

2. **Accusation** - "It's all in the manual"
   → Better: "You can find instructions for this in the manual on page X"

3. **Condescending** - "Who doesn't read that?"
   → Better: Simply show the solution without comment

**Professional alternative:**
"I understand the problem. Let me quickly show you how it works. If you want to look it up again later, you can also find instructions in the manual on page 15."

</details>
