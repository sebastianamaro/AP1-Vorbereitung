# Software Testing

## Learning Objectives

After this chapter, you should be able to:
- Know and distinguish different test types
- Understand the test levels in the SDLC
- Explain black-box and white-box tests
- Create test cases

## Key Terms

| Term | Explanation |
|------|-------------|
| **Unit Test** | Test of individual functions/modules |
| **Integration Test** | Test of module interaction |
| **System Test** | Test of complete system |
| **Acceptance Test** | Customer test before production |
| **Black-Box Test** | Test without knowledge of code |
| **White-Box Test** | Test with knowledge of code |
| **Regression Test** | Test after changes |
| **Test Case** | Defined test condition with result |

---

## Exam Tips

### Common Exam Questions

- Distinguish the test levels
- What is the difference between black-box and white-box?
- Create test cases for a given scenario

### Important to Remember

- **Test levels:** Unit вҶ’ Integration вҶ’ System вҶ’ Acceptance
- **Black-Box:** Tester does NOT see code (input/output)
- **White-Box:** Tester sees code (structure)
- **Regression test:** After every change!
- **Boundary values** are error-prone вҶ’ always test!
- Acceptance test = Customer checks against **Requirements Specification**
- System test = Team checks against **Functional Specification**

## Practice Exercises

### Exercise 1
A function accepts values from 1 to 100. Which test cases would you create for boundary value analysis?

**Solution:**
Test cases for boundary value analysis:
- **0** (invalid, below boundary)
- **1** (valid, lower boundary)
- **2** (valid, just above lower boundary)
- **99** (valid, just below upper boundary)
- **100** (valid, upper boundary)
- **101** (invalid, above boundary)

### Exercise 2
Who typically performs the acceptance test?

**Solution:**
The **client/customer** performs the acceptance test. They check whether the software meets their requirements from the requirements specification and decide on acceptance.

### Exercise 3
Match: Which test level checks against which document?
- a) Acceptance test
- b) System test

**Solution:**
- a) Acceptance test вҶ’ **Requirements Specification** (customer requirements)
- b) System test вҶ’ **Functional Specification** (technical specification)
