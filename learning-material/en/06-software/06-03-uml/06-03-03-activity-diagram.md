# UML Activity Diagram

## Learning Objectives

After this chapter, you should be able to:
- Read and understand activity diagrams
- Know the most important symbols and their meaning
- Represent branches and parallelism
- Model simple processes as activity diagrams

| Term | Explanation |

| **Join** | Merge of parallel paths |

## Exam Tips

### Common Exam Questions

- Read the activity diagram and explain the flow
- Which activities run in parallel?
- Draw an activity diagram for a given process

### Important to Remember

- **в—Ҹ** = Start, **в—ү** = End
- **в—Ү** = Decision (branch)
- **в•җв•җв•җ** = Fork (start parallel) / Join (wait for all)
- [In brackets] = Condition
- Activities = rounded rectangles

## Practice Exercises

### Exercise 1
What is the difference between Fork/Join and Decision?

**Solution:**
- **Fork/Join:** Activities are executed **in parallel**. All paths are traversed, the join waits until all are complete.
- **Decision:** Only **one** path is selected based on a condition. The other paths are not traversed.

### Exercise 3
Explain the following flow:

```
    в—Ҹ
    в”Ӯ
    в–ј
в”Ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”җ

в””в”Җв”Җв”Җв”Җв”¬в”Җв”Җв”Җв”ҳ
   в•җв•җв•Әв•җв•җ
   в”Ӯ   в”Ӯ
   в–ј   в–ј
в”Ңв”Җв”Җв”Җв”Җв”җв”Ңв”Җв”Җв”Җв”Җв”җ
в”Ӯ A  в”Ӯв”Ӯ B  в”Ӯ
в””в”Җв”Җв”¬в”Җв”ҳв””в”Җв”Җв”¬в”Җв”ҳ

     в”Ӯ
```

**Solution:**
1. The process starts (в—Ҹ)
2. The activity "Start" is executed
3. At the fork (в•җв•җв•җ) the flow splits into two parallel paths
4. Activities A and B run **simultaneously** (in parallel)
5. At the join (в•җв•җв•җ) the system waits until **both** activities are complete
6. The process ends (в—ү)
