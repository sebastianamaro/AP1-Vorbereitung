# DHCP

## Core Content

### What is DHCP?

```
+--------------------------------------------------------------------+
|                          DHCP                                      |
+--------------------------------------------------------------------+
|                                                                     |
|   DHCP = Dynamic Host Configuration Protocol                       |
|                                                                     |
|   +-------------------------------------------------------------+ |
|   | DHCP automatically assigns network configurations to        | |
|   | devices in the network (IP address, gateway, DNS).          | |
|   +-------------------------------------------------------------+ |
|                                                                     |
|   Without DHCP:                      With DHCP:                    |
|   +--------------------------+       +--------------------------+ |
|   | Each device must be      |       | Devices receive their   | |
|   | manually configured:     |       | settings                | |
|   | - IP address             |       | automatically:          | |
|   | - Subnet mask            |       |                         | |
|   | - Gateway                |       |  PC -> DHCP Server -> IP| |
|   | - DNS server             |       |                         | |
|   +--------------------------+       | "Plug and Play"         | |
|                                      +--------------------------+ |
|                                                                     |
|   Advantages:                                                      |
|   - No manual configuration needed                                 |
|   - No IP address conflicts                                        |
|   - Centralized management                                         |
|   - Changes in one place                                           |
|                                                                     |
+--------------------------------------------------------------------+
```

### The DORA Process

```
+--------------------------------------------------------------------+
|                       DORA PROCESS                                 |
+--------------------------------------------------------------------+
|                                                                     |
|   DORA = Discover, Offer, Request, Acknowledge                    |
|                                                                     |
|      Client                                    DHCP Server         |
|        |                                           |               |
|   1.   |     DHCP DISCOVER (Broadcast)            |               |
|        | ------------------------------------>    |               |
|        |   "Is there a DHCP server?"              |               |
|        |   (Broadcast to 255.255.255.255)         |               |
|        |                                           |               |
|   2.   |     DHCP OFFER                           |               |
|        | <------------------------------------    |               |
|        |   "Here is an IP for you:"               |               |
|        |   IP: 192.168.1.100                      |               |
|        |   Mask: 255.255.255.0                    |               |
|        |   Gateway: 192.168.1.1                   |               |
|        |                                           |               |
|   3.   |     DHCP REQUEST (Broadcast)             |               |
|        | ------------------------------------>    |               |
|        |   "I'll take this IP!"                   |               |
|        |   (Broadcast, so other servers know)     |               |
|        |                                           |               |
|   4.   |     DHCP ACKNOWLEDGE                     |               |
|        | <------------------------------------    |               |
|        |   "Confirmed, the IP is yours!"          |               |
|        |   Lease time: 24 hours                   |               |
|        |                                           |               |
|        |   =======================================   |               |
|        |     Client now has network access        |               |
|        |   =======================================   |               |
|                                                                     |
|   Mnemonic: "Dora needs DHCP"                                     |
|   D = Discover, O = Offer, R = Request, A = Acknowledge           |
|                                                                     |
+--------------------------------------------------------------------+
```

### DHCP Lease

```
+--------------------------------------------------------------------+
|                       DHCP LEASE                                   |
+--------------------------------------------------------------------+
|                                                                     |
|   Definition:                                                      |
|   +-------------------------------------------------------------+ |
|   | A DHCP lease is the time-limited "lending" of an IP         | |
|   | address to a device.                                         | |
|   +-------------------------------------------------------------+ |
|                                                                     |
|   Lease time (typical: 24 hours to 7 days)                        |
|                                                                     |
|   +-------------------------------------------------------------+ |
|   |                                                              | |
|   |   0%        50%                       100%                  | |
|   |   |----------|--------------------------|                   | |
|   |   Start      Renewal       Rebinding    Expiry              | |
|   |              (50%)         (87.5%)                          | |
|   |                                                              | |
|   |   At 50%: Client attempts to renew lease                   | |
|   |   At 87.5%: Client attempts to renew with ANY server       | |
|   |   At 100%: IP is released                                  | |
|   |                                                              | |
|   +-------------------------------------------------------------+ |
|                                                                     |
|   What happens when lease expires?                                 |
|   - Client loses IP address                                        |
|   - Client must start new DORA process                             |
|   - May receive different IP address                               |
|                                                                     |
+--------------------------------------------------------------------+
```

### DHCP Configuration Data

```
+--------------------------------------------------------------------+
|                DHCP PROVIDES THIS DATA                             |
+--------------------------------------------------------------------+
|                                                                     |
|   Required:                                                        |
|   +------------------------------------------------------------+  |
|   | - IP address                                               |  |
|   | - Subnet mask                                              |  |
|   +------------------------------------------------------------+  |
|                                                                     |
|   Optional (but common):                                           |
|   +------------------------------------------------------------+  |
|   | - Default gateway (router address)                         |  |
|   | - DNS server (for name resolution)                         |  |
|   | - Lease time                                               |  |
|   | - WINS server (Windows networks)                           |  |
|   | - Domain name                                              |  |
|   | - NTP server (time server)                                 |  |
|   +------------------------------------------------------------+  |
|                                                                     |
|   Example configuration:                                           |
|   +------------------------------------------------------------+  |
|   | IP address:     192.168.1.100                              |  |
|   | Subnet mask:    255.255.255.0                              |  |
|   | Gateway:        192.168.1.1                                |  |
|   | DNS server:     192.168.1.1, 8.8.8.8                       |  |
|   | Lease time:     86400 seconds (24 hours)                   |  |
|   +------------------------------------------------------------+  |
|                                                                     |
+--------------------------------------------------------------------+
```

### DHCP Address Pool

```
+--------------------------------------------------------------------+
|                     DHCP ADDRESS POOL                              |
+--------------------------------------------------------------------+
|                                                                     |
|   The administrator defines a range of IP addresses that          |
|   DHCP may assign:                                                 |
|                                                                     |
|   +-------------------------------------------------------------+ |
|   |                                                              | |
|   |   Network: 192.168.1.0/24                                   | |
|   |                                                              | |
|   |   |  .1 - .10  |   .11 - .200   |  .201 - .254  |           | |
|   |   |  Static    |   DHCP Pool    |   Static      |           | |
|   |   |  (Servers) |   (Clients)    |   (Printers)  |           | |
|   |                                                              | |
|   |   .0   = Network address (not usable)                       | |
|   |   .255 = Broadcast (not usable)                             | |
|   |                                                              | |
|   +-------------------------------------------------------------+ |
|                                                                     |
|   Reservations:                                                    |
|   +------------------------------------------------------------+  |
|   | Certain devices can receive a fixed IP:                    |  |
|   | - Based on MAC address                                     |  |
|   | - Servers, printers always get the same IP                 |  |
|   | - Still configured via DHCP                                |  |
|   +------------------------------------------------------------+  |
|                                                                     |
+--------------------------------------------------------------------+
```

### DHCP vs. Static Configuration

```
+--------------------------------------------------------------------+
|               DHCP vs. STATIC                                      |
+--------------------------------------------------------------------+
|                                                                     |
|                    DHCP               Static                       |
|   +----------------------------+------------------------------+   |
|   | Automatic assignment       | Manual configuration         |   |
|   | IP can change              | IP always stays the same     |   |
|   | Less effort                | More effort                  |   |
|   | Centralized management     | Decentralized management     |   |
|   | No conflicts               | Conflicts possible           |   |
|   | Dependent on DHCP server   | Independent                  |   |
|   +----------------------------+------------------------------+   |
|                                                                     |
|   When DHCP?                    When static?                       |
|   +----------------------------+------------------------------+   |
|   | - Client PCs               | - Servers                    |   |
|   | - Laptops                  | - Routers/Gateways           |   |
|   | - Smartphones              | - Network printers           |   |
|   | - Guest devices            | - NAS systems                |   |
|   | - Devices with changing    | - Devices that must always  |   |
|   |   usage                    |   be reachable              |   |
|   +----------------------------+------------------------------+   |
|                                                                     |
+--------------------------------------------------------------------+
```

## Key Terms

| Term | Explanation |
|------|-------------|
| **DHCP** | Dynamic Host Configuration Protocol |
| **DORA** | Discover, Offer, Request, Acknowledge |
| **Lease** | Time-limited IP assignment |
| **Address pool** | Range of available IP addresses |
| **Reservation** | Fixed IP based on MAC address |
| **Broadcast** | DHCP uses broadcasts (Port 67/68) |

## Exam Tips

### Common Exam Questions

- Explain the DORA process
- What data does DHCP provide?
- When static, when DHCP?

### Important to Remember

- **DORA:** Discover -> Offer -> Request -> Acknowledge
- **Ports:** Server 67, Client 68 (UDP)
- **DHCP provides:** IP, mask, gateway, DNS
- **Lease** = time-limited assignment
- **Servers** = static IP, **Clients** = DHCP
- Discover and Request are **broadcasts**

## Practice Exercises

### Exercise 1
Explain the DORA process in DHCP.

**Solution:**
1. **Discover:** Client sends broadcast ("Is there a DHCP server?")
2. **Offer:** Server responds with IP offer
3. **Request:** Client accepts the offer (broadcast)
4. **Acknowledge:** Server confirms the assignment

### Exercise 2
Why should a server have a static IP and not use DHCP?

**Solution:**
- Servers must **always be reachable at the same IP**
- Other devices/services refer to this IP
- DHCP could assign a **different IP**
- If **DHCP server fails**, server would be unreachable
- DNS entries would no longer be correct

### Exercise 3
What information does a client typically receive from the DHCP server?

**Solution:**
- **IP address** (e.g., 192.168.1.100)
- **Subnet mask** (e.g., 255.255.255.0)
- **Default gateway** (e.g., 192.168.1.1)
- **DNS server** (e.g., 192.168.1.1, 8.8.8.8)
- **Lease time** (e.g., 24 hours)
