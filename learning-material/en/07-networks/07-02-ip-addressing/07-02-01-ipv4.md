# IPv4 Addressing

## Learning Objectives

After this chapter, you should be able to:
- Understand the structure of an IPv4 address
- Distinguish between network and host portions
- Know the IP classes
- Distinguish between private and public addresses
- Understand subnet masks

---

## Core Content

### Structure of an IPv4 Address

```
+--------------------------------------------------------------------+
|                    IPv4 ADDRESS                                     |
+--------------------------------------------------------------------+
|                                                                     |
|   IPv4 = Internet Protocol Version 4                                |
|                                                                     |
|   +-------------------------------------------------------------+  |
|   | An IPv4 address is a 32-bit number that uniquely identifies |  |
|   | a device on the network.                                     |  |
|   +-------------------------------------------------------------+  |
|                                                                     |
|   Representation:                                                   |
|   +-------------------------------------------------------------+  |
|   |                                                              |  |
|   |   Decimal (Dotted Decimal Notation):                        |  |
|   |                                                              |  |
|   |         192  .  168  .  1   .  100                          |  |
|   |          |       |      |      |                            |  |
|   |          +-------+------+------+                            |  |
|   |              4 octets (8 bits each)                         |  |
|   |                                                              |  |
|   |   Binary (32 bits):                                         |  |
|   |                                                              |  |
|   |   11000000.10101000.00000001.01100100                       |  |
|   |      192      168       1       100                         |  |
|   |                                                              |  |
|   +-------------------------------------------------------------+  |
|                                                                     |
|   Value range per octet: 0 - 255 (8 bits = 2^8 = 256 values)       |
|                                                                     |
|   Total possible addresses: 2^32 = approx. 4.3 billion              |
|                                                                     |
+--------------------------------------------------------------------+
```

---

### Network and Host Portions

```
+--------------------------------------------------------------------+
|                NETWORK AND HOST PORTIONS                            |
+--------------------------------------------------------------------+
|                                                                     |
|   Every IP address consists of two parts:                           |
|                                                                     |
|   +-------------------------------------------------------------+  |
|   |                                                              |  |
|   |   IP Address:    192 . 168 . 1   . 100                      |  |
|   |                  -----------   -----                        |  |
|   |                  NETWORK PORTION  HOST PORTION              |  |
|   |                                                              |  |
|   |   Network portion: Identifies the network                   |  |
|   |                    (like street name)                       |  |
|   |                                                              |  |
|   |   Host portion:    Identifies the device in the network     |  |
|   |                    (like house number)                      |  |
|   |                                                              |  |
|   +-------------------------------------------------------------+  |
|                                                                     |
|   The SUBNET MASK determines the division:                          |
|                                                                     |
|   +-------------------------------------------------------------+  |
|   |                                                              |  |
|   |   IP Address:     192.168.1.100                             |  |
|   |   Subnet mask:    255.255.255.0                             |  |
|   |                                                              |  |
|   |   Binary:                                                    |  |
|   |   IP:     11000000.10101000.00000001.01100100               |  |
|   |   Mask:   11111111.11111111.11111111.00000000               |  |
|   |           --------------------------  --------              |  |
|   |           NETWORK (24 bits)           HOST (8 bits)         |  |
|   |                                                              |  |
|   |   1-bits = Network portion                                  |  |
|   |   0-bits = Host portion                                     |  |
|   |                                                              |  |
|   +-------------------------------------------------------------+  |
|                                                                     |
+--------------------------------------------------------------------+
```

### IP Address Classes (Historical)

```
+--------------------------------------------------------------------+
|                    IP ADDRESS CLASSES                               |
+--------------------------------------------------------------------+
|                                                                     |
|   (Historical concept, now replaced by CIDR)                        |
|                                                                     |
|   Class | Range             | Default Mask   | Networks/Hosts      |
|   ------+-------------------+----------------+---------------------+
|   A     | 1.0.0.0 -         | 255.0.0.0      | 126 networks,       |
|         | 126.255.255.255   | /8             | 16 million hosts    |
|   ------+-------------------+----------------+---------------------+
|   B     | 128.0.0.0 -       | 255.255.0.0    | 16,384 networks,    |
|         | 191.255.255.255   | /16            | 65,534 hosts        |
|   ------+-------------------+----------------+---------------------+
|   C     | 192.0.0.0 -       | 255.255.255.0  | 2 million networks, |
|         | 223.255.255.255   | /24            | 254 hosts           |
|   ------+-------------------+----------------+---------------------+
|   D     | 224.0.0.0 -       | -              | Multicast           |
|         | 239.255.255.255   |                |                     |
|   ------+-------------------+----------------+---------------------+
|   E     | 240.0.0.0 -       | -              | Reserved            |
|         | 255.255.255.255   |                |                     |
|                                                                     |
|   Recognition of class by first octet:                              |
|   +-------------------------------------------------------------+  |
|   | Class A: 1 - 126    (starts with 0xxx)                      |  |
|   | Class B: 128 - 191  (starts with 10xx)                      |  |
|   | Class C: 192 - 223  (starts with 110x)                      |  |
|   +-------------------------------------------------------------+  |
|                                                                     |
+--------------------------------------------------------------------+
```

### Private and Public Addresses

```
+--------------------------------------------------------------------+
|              PRIVATE vs. PUBLIC ADDRESSES                           |
+--------------------------------------------------------------------+
|                                                                     |
|   PUBLIC IP ADDRESSES                                               |
|   +------------------------------------------------------------+   |
|   | - Globally unique                                           |   |
|   | - Routed on the Internet                                    |   |
|   | - Assigned by ISP (Internet Service Provider)               |   |
|   | - Limited quantity (approx. 4.3 billion)                    |   |
|   +------------------------------------------------------------+   |
|                                                                     |
|   PRIVATE IP ADDRESSES                                              |
|   +------------------------------------------------------------+   |
|   | - Only valid within local networks                          |   |
|   | - NOT routed on the Internet                                |   |
|   | - Can be reused in any network                              |   |
|   | - Internet connection via NAT (Network Address              |   |
|   |   Translation)                                              |   |
|   |                                                             |   |
|   |   PRIVATE ADDRESS RANGES (RFC 1918):                        |   |
|   |   +-----------------------------------------------------+   |   |
|   |   | Class A:  10.0.0.0    - 10.255.255.255   /8         |   |   |
|   |   | Class B:  172.16.0.0  - 172.31.255.255   /12        |   |   |
|   |   | Class C:  192.168.0.0 - 192.168.255.255  /16        |   |   |
|   |   +-----------------------------------------------------+   |   |
|   |                                                             |   |
|   |   ! MEMORIZE these ranges!                                  |   |
|   +------------------------------------------------------------+   |
|                                                                     |
|   Example:                                                          |
|   +------------------------------------------------------------+   |
|   |                                                             |   |
|   |   Home network            Internet           Web server    |   |
|   |   192.168.1.x      NAT                      203.0.113.50  |   |
|   |                +----------+                                |   |
|   |   +-----+     | Router   |     +--------------------+     |   |
|   |   | PC  |---->|          |---->|     Internet       |     |   |
|   |   |.100 |     | Public   |     |                    |     |   |
|   |   +-----+     |   IP     |     +--------------------+     |   |
|   |               +----------+                                |   |
|   |   Private IP -> is translated to public IP                |   |
|   |                                                             |   |
|   +------------------------------------------------------------+   |
|                                                                     |
+--------------------------------------------------------------------+
```

### Special Addresses

```
+--------------------------------------------------------------------+
|                   SPECIAL IP ADDRESSES                              |
+--------------------------------------------------------------------+
|                                                                     |
|   127.0.0.1 (Localhost/Loopback)                                    |
|   +------------------------------------------------------------+   |
|   | - Refers to the own computer                                |   |
|   | - For testing local services                                |   |
|   | - Data packets never leave the device                       |   |
|   +------------------------------------------------------------+   |
|                                                                     |
|   0.0.0.0                                                           |
|   +------------------------------------------------------------+   |
|   | - "Unknown address" or "all interfaces"                     |   |
|   | - Used in DHCP requests                                     |   |
|   +------------------------------------------------------------+   |
|                                                                     |
|   255.255.255.255 (Broadcast)                                       |
|   +------------------------------------------------------------+   |
|   | - Sends to ALL devices in the local network                 |   |
|   | - Not forwarded beyond routers                              |   |
|   +------------------------------------------------------------+   |
|                                                                     |
|   Network address (all host bits = 0)                               |
|   +------------------------------------------------------------+   |
|   | - e.g. 192.168.1.0 with /24                                 |   |
|   | - Identifies the network itself                             |   |
|   | - NOT assignable to devices                                 |   |
|   +------------------------------------------------------------+   |
|                                                                     |
|   Broadcast address (all host bits = 1)                             |
|   +------------------------------------------------------------+   |
|   | - e.g. 192.168.1.255 with /24                               |   |
|   | - Sends to all devices in the subnet                        |   |
|   | - NOT assignable to devices                                 |   |
|   +------------------------------------------------------------+   |
|                                                                     |
+--------------------------------------------------------------------+
```

### CIDR Notation

```
+--------------------------------------------------------------------+
|                      CIDR NOTATION                                  |
+--------------------------------------------------------------------+
|                                                                     |
|   CIDR = Classless Inter-Domain Routing                             |
|                                                                     |
|   Format: IP address / prefix length                                |
|                                                                     |
|   +-------------------------------------------------------------+  |
|   |                                                              |  |
|   |   192.168.1.0/24                                            |  |
|   |              ^                                               |  |
|   |              +-- 24 bits for network, 8 bits for host       |  |
|   |                                                              |  |
|   +-------------------------------------------------------------+  |
|                                                                     |
|   Conversion to subnet mask:                                        |
|                                                                     |
|   Prefix | Subnet Mask       | Available Hosts                      |
|   -------+-------------------+--------------------------------------+
|   /8     | 255.0.0.0         | 16,777,214                           |
|   /16    | 255.255.0.0       | 65,534                               |
|   /24    | 255.255.255.0     | 254                                  |
|   /25    | 255.255.255.128   | 126                                  |
|   /26    | 255.255.255.192   | 62                                   |
|   /27    | 255.255.255.224   | 30                                   |
|   /28    | 255.255.255.240   | 14                                   |
|   /29    | 255.255.255.248   | 6                                    |
|   /30    | 255.255.255.252   | 2                                    |
|   /32    | 255.255.255.255   | 1 (single host)                      |
|                                                                     |
|   Formula: Hosts = 2^(32-prefix) - 2                                |
|            (minus 2 for network and broadcast addresses)            |
|                                                                     |
+--------------------------------------------------------------------+
```

## Key Terms

| Term | Explanation |
|------|-------------|
| **IPv4** | Internet Protocol Version 4 (32 bits) |
| **Octet** | 8-bit section of an IP address |
| **Subnet mask** | Separates network from host portion |
| **CIDR** | Classless Inter-Domain Routing |
| **NAT** | Network Address Translation |
| **Loopback** | Local address 127.0.0.1 |
| **Broadcast** | Address for all in the network |

## Exam Tips

### Common Exam Questions

- Name the private IP address ranges
- What does /24 mean?
- Difference between public/private IP?

### Important to Remember

- **32 bits** = 4 octets of 8 bits each
- **Private ranges:** 10.x.x.x, 172.16-31.x.x, 192.168.x.x
- **127.0.0.1** = Localhost
- **/24** = 255.255.255.0 = 254 hosts
- Network and broadcast addresses NOT usable
- **Hosts = 2^(host bits) - 2**

## Practice Exercises

### Exercise 1
Is the IP address 192.168.5.100 a private or public address?

**Solution:**
**Private address**

Justification: 192.168.x.x belongs to the private address range of Class C (192.168.0.0 - 192.168.255.255). These addresses are not routed on the Internet.

### Exercise 2
A network has the address 192.168.10.0/24. How many hosts can be addressed?

**Solution:**
**/24** means 24 bits for the network, 8 bits for hosts.

Calculation: 2^8 - 2 = 256 - 2 = **254 hosts**

The 2 are subtracted for:
- Network address (192.168.10.0)
- Broadcast address (192.168.10.255)

### Exercise 3
Convert the subnet mask 255.255.255.192 to CIDR notation.

**Solution:**
255.255.255.192 in binary:
- 255 = 11111111
- 192 = 11000000

Binary: 11111111.11111111.11111111.11000000

Number of 1-bits: 24 + 2 = **26**

Answer: **/26**
