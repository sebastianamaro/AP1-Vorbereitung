# Subnetting

## Learning Objectives

After this chapter, you should be able to:
- Understand the purpose of subnetting
- Calculate subnets
- Determine the number of available hosts
- Determine network and broadcast addresses

---

## Core Content

### What is Subnetting?

```
+--------------------------------------------------------------------+
|                       SUBNETTING                                    |
+--------------------------------------------------------------------+
|                                                                     |
|   Definition:                                                       |
|   +-------------------------------------------------------------+  |
|   | Subnetting is the division of a larger network              |  |
|   | into multiple smaller sub-networks (subnets).               |  |
|   +-------------------------------------------------------------+  |
|                                                                     |
|   Before: One large network                                         |
|   +-------------------------------------------------------------+  |
|   | 192.168.0.0/24 (254 hosts)                                  |  |
|   | +-------------------------------------------------------+   |  |
|   | |  PC  PC  PC  PC  PC  PC  PC  PC  PC  PC  PC  PC ...   |   |  |
|   | +-------------------------------------------------------+   |  |
|   +-------------------------------------------------------------+  |
|                                                                     |
|   After: Multiple small subnets                                     |
|   +-------------------------------------------------------------+  |
|   | 192.168.0.0/26 (62 hosts)  192.168.0.64/26 (62 hosts)      |  |
|   | +---------------------+    +---------------------+          |  |
|   | |  PC  PC  PC  PC ... |    |  PC  PC  PC  PC ... |          |  |
|   | +---------------------+    +---------------------+          |  |
|   |                                                              |  |
|   | 192.168.0.128/26 (62 hosts) 192.168.0.192/26 (62 hosts)    |  |
|   | +---------------------+    +---------------------+          |  |
|   | |  PC  PC  PC  PC ... |    |  PC  PC  PC  PC ... |          |  |
|   | +---------------------+    +---------------------+          |  |
|   +-------------------------------------------------------------+  |
|                                                                     |
+--------------------------------------------------------------------+
```

### Reasons for Subnetting

```
+--------------------------------------------------------------------+
|                  WHY SUBNETTING?                                    |
+--------------------------------------------------------------------+
|                                                                     |
|   1. SECURITY                                                       |
|   +------------------------------------------------------------+   |
|   | - Separation of different areas                             |   |
|   | - Accounting separated from production                      |   |
|   | - Separate guest WLAN from internal network                 |   |
|   +------------------------------------------------------------+   |
|                                                                     |
|   2. PERFORMANCE                                                    |
|   +------------------------------------------------------------+   |
|   | - Smaller broadcast domains                                 |   |
|   | - Less unnecessary network traffic                          |   |
|   | - Broadcasts stay within the subnet                         |   |
|   +------------------------------------------------------------+   |
|                                                                     |
|   3. ORGANIZATION                                                   |
|   +------------------------------------------------------------+   |
|   | - Logical structure (departments, floors)                   |   |
|   | - Better overview                                           |   |
|   | - Easier administration                                     |   |
|   +------------------------------------------------------------+   |
|                                                                     |
|   4. EFFICIENT UTILIZATION                                          |
|   +------------------------------------------------------------+   |
|   | - Optimally distribute IP addresses                         |   |
|   | - No waste of addresses                                     |   |
|   +------------------------------------------------------------+   |
|                                                                     |
+--------------------------------------------------------------------+
```

### Understanding the Subnet Mask

```
+--------------------------------------------------------------------+
|                    SUBNET MASK                                      |
+--------------------------------------------------------------------+
|                                                                     |
|   The subnet mask separates the network and host portions:          |
|                                                                     |
|   +-------------------------------------------------------------+  |
|   |                                                              |  |
|   |   IP Address:    192.168.1.100                              |  |
|   |   Subnet mask:   255.255.255.0                              |  |
|   |                                                              |  |
|   |   Binary:                                                    |  |
|   |   IP:    11000000.10101000.00000001.01100100                |  |
|   |   Mask:  11111111.11111111.11111111.00000000                |  |
|   |          --------------------------  --------               |  |
|   |              Network (24 bits)       Host (8 bits)          |  |
|   |                                                              |  |
|   |   AND operation yields network address:                     |  |
|   |   IP:    11000000.10101000.00000001.01100100                |  |
|   |   Mask:  11111111.11111111.11111111.00000000                |  |
|   |   ------------------------------------------                |  |
|   |   Net:   11000000.10101000.00000001.00000000                |  |
|   |          = 192.168.1.0 (Network address)                    |  |
|   |                                                              |  |
|   +-------------------------------------------------------------+  |
|                                                                     |
|   Important masks:                                                  |
|   +-----------------------------------------------------------+   |
|   | /24 = 255.255.255.0   -> 254 hosts                        |   |
|   | /25 = 255.255.255.128 -> 126 hosts                        |   |
|   | /26 = 255.255.255.192 ->  62 hosts                        |   |
|   | /27 = 255.255.255.224 ->  30 hosts                        |   |
|   | /28 = 255.255.255.240 ->  14 hosts                        |   |
|   +-----------------------------------------------------------+   |
|                                                                     |
+--------------------------------------------------------------------+
```

### Calculation Formulas

```
+--------------------------------------------------------------------+
|                    CALCULATIONS                                     |
+--------------------------------------------------------------------+
|                                                                     |
|   NUMBER OF HOSTS                                                   |
|   +------------------------------------------------------------+   |
|   |                                                             |   |
|   |   Hosts = 2^(host bits) - 2                                 |   |
|   |                                                             |   |
|   |   The 2 are subtracted for:                                 |   |
|   |   - Network address (all host bits = 0)                     |   |
|   |   - Broadcast address (all host bits = 1)                   |   |
|   |                                                             |   |
|   |   Example /26:                                              |   |
|   |   Host bits = 32 - 26 = 6                                   |   |
|   |   Hosts = 2^6 - 2 = 64 - 2 = 62                            |   |
|   |                                                             |   |
|   +------------------------------------------------------------+   |
|                                                                     |
|   NUMBER OF SUBNETS                                                 |
|   +------------------------------------------------------------+   |
|   |                                                             |   |
|   |   Subnets = 2^(additional network bits)                     |   |
|   |                                                             |   |
|   |   Example: /24 becomes /26                                  |   |
|   |   Additional bits = 26 - 24 = 2                             |   |
|   |   Subnets = 2^2 = 4                                         |   |
|   |                                                             |   |
|   +------------------------------------------------------------+   |
|                                                                     |
|   BLOCK SIZE (Increment)                                            |
|   +------------------------------------------------------------+   |
|   |                                                             |   |
|   |   Block size = 256 - value of last mask octet               |   |
|   |                                                             |   |
|   |   Example /26: Mask = 255.255.255.192                       |   |
|   |   Block size = 256 - 192 = 64                               |   |
|   |                                                             |   |
|   |   Subnets: .0, .64, .128, .192                              |   |
|   |                                                             |   |
|   +------------------------------------------------------------+   |
|                                                                     |
+--------------------------------------------------------------------+
```

### Example Calculation

```
+--------------------------------------------------------------------+
|                 EXAMPLE: 192.168.1.0/26                             |
+--------------------------------------------------------------------+
|                                                                     |
|   Given: 192.168.1.0/26                                             |
|   Subnet mask: 255.255.255.192                                      |
|                                                                     |
|   Step 1: Calculate host bits                                       |
|   +------------------------------------------------------------+   |
|   | Host bits = 32 - 26 = 6 bits                                |   |
|   +------------------------------------------------------------+   |
|                                                                     |
|   Step 2: Calculate number of hosts                                 |
|   +------------------------------------------------------------+   |
|   | Hosts = 2^6 - 2 = 64 - 2 = 62 usable hosts                  |   |
|   +------------------------------------------------------------+   |
|                                                                     |
|   Step 3: Calculate block size                                      |
|   +------------------------------------------------------------+   |
|   | Block size = 256 - 192 = 64                                 |   |
|   +------------------------------------------------------------+   |
|                                                                     |
|   Step 4: Determine subnets                                         |
|   +------------------------------------------------------------+   |
|   |                                                             |   |
|   | Subnet 1: 192.168.1.0/26                                    |   |
|   |   Network address:  192.168.1.0                             |   |
|   |   First host:       192.168.1.1                             |   |
|   |   Last host:        192.168.1.62                            |   |
|   |   Broadcast:        192.168.1.63                            |   |
|   |                                                             |   |
|   | Subnet 2: 192.168.1.64/26                                   |   |
|   |   Network address:  192.168.1.64                            |   |
|   |   First host:       192.168.1.65                            |   |
|   |   Last host:        192.168.1.126                           |   |
|   |   Broadcast:        192.168.1.127                           |   |
|   |                                                             |   |
|   | Subnet 3: 192.168.1.128/26                                  |   |
|   |   Network address:  192.168.1.128                           |   |
|   |   First host:       192.168.1.129                           |   |
|   |   Last host:        192.168.1.190                           |   |
|   |   Broadcast:        192.168.1.191                           |   |
|   |                                                             |   |
|   | Subnet 4: 192.168.1.192/26                                  |   |
|   |   Network address:  192.168.1.192                           |   |
|   |   First host:       192.168.1.193                           |   |
|   |   Last host:        192.168.1.254                           |   |
|   |   Broadcast:        192.168.1.255                           |   |
|   |                                                             |   |
|   +------------------------------------------------------------+   |
|                                                                     |
+--------------------------------------------------------------------+
```

### Quick Reference: CIDR Table

```
+--------------------------------------------------------------------+
|                   CIDR QUICK REFERENCE                              |
+--------------------------------------------------------------------+
|                                                                     |
|   CIDR | Mask            | Hosts | Block | Subnets from /24        |
|   -----+-----------------+-------+-------+------------------------+
|   /24  | 255.255.255.0   |  254  |  256  |  1                      |
|   /25  | 255.255.255.128 |  126  |  128  |  2                      |
|   /26  | 255.255.255.192 |   62  |   64  |  4                      |
|   /27  | 255.255.255.224 |   30  |   32  |  8                      |
|   /28  | 255.255.255.240 |   14  |   16  | 16                      |
|   /29  | 255.255.255.248 |    6  |    8  | 32                      |
|   /30  | 255.255.255.252 |    2  |    4  | 64                      |
|   /31  | 255.255.255.254 |    0* |    2  | 128 (Point-to-Point)    |
|   /32  | 255.255.255.255 |    1  |    1  | 256 (single host)       |
|                                                                     |
|   * /31 is used for point-to-point connections                      |
|                                                                     |
|   To remember (last octet of mask):                                 |
|   +-----------------------------------------------------------+   |
|   | 128 -> /25                                                 |   |
|   | 192 -> /26 (128 + 64)                                      |   |
|   | 224 -> /27 (128 + 64 + 32)                                 |   |
|   | 240 -> /28 (128 + 64 + 32 + 16)                            |   |
|   | 248 -> /29 (128 + 64 + 32 + 16 + 8)                        |   |
|   | 252 -> /30 (128 + 64 + 32 + 16 + 8 + 4)                    |   |
|   +-----------------------------------------------------------+   |
|                                                                     |
+--------------------------------------------------------------------+
```

## Key Terms

| Term | Explanation |
|------|-------------|
| **Subnet** | Sub-network of a larger network |
| **Subnet mask** | Determines network/host portion |
| **CIDR** | Classless Inter-Domain Routing (/24) |
| **Network address** | All host bits = 0 |
| **Broadcast address** | All host bits = 1 |
| **Block size** | Distance between subnets |

## Exam Tips

### Common Exam Questions

- How many hosts with /26?
- Which subnet does an IP belong to?
- How many subnets from /24 with /27?

### Important to Remember

- **Hosts = 2^host bits - 2**
- **/24** = 254 hosts, **/25** = 126, **/26** = 62
- **Block size** = 256 - mask octet
- Network and broadcast addresses **NOT usable**
- /30 = 2 hosts (for router connections)

---

## Practice Exercises

### Exercise 1
How many usable hosts does a /27 network have?

**Solution:**
Host bits = 32 - 27 = 5
Hosts = 2^5 - 2 = 32 - 2 = **30 usable hosts**

### Exercise 2
Which subnet does IP 192.168.10.200 belong to with /26?

**Solution:**
Block size with /26: 256 - 192 = 64
Subnets: .0, .64, .128, .192

200 lies between 192 and 255 -> **Subnet 192.168.10.192/26**

- Network address: 192.168.10.192
- Broadcast: 192.168.10.255

### Exercise 3
A company needs 50 hosts per department. Which subnet mask is suitable?

**Solution:**
50 hosts needed -> at least 50 + 2 = 52 addresses

- /26 = 62 hosts (suitable)
- /27 = 30 hosts (too few)

Answer: **/26** (255.255.255.192)
