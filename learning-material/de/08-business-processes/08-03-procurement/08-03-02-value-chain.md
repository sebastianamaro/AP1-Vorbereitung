# Wertschöpfung und Wertschöpfungskette

## Lernziele

Nach diesem Abschnitt kannst du:
- Den Begriff Wertschöpfung erklären
- Die Wertschöpfungskette nach Porter beschreiben
- Primäre und unterstützende Aktivitäten unterscheiden
- Wertschöpfung in IT-Unternehmen identifizieren

---

## Kerninhalt

### Was ist Wertschöpfung?

**Wertschöpfung** ist der Mehrwert, der durch die Verarbeitung von Inputs zu Outputs entsteht.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    WERTSCHÖPFUNG                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  Wertschöpfung = Wert der Produkte/Leistungen - Wert der Vorleistungen │
│                                                                      │
│  ┌───────────────┐     ┌───────────────┐     ┌───────────────┐      │
│  │               │     │               │     │               │      │
│  │    INPUT      │ ──► │  VERARBEITUNG │ ──► │    OUTPUT     │      │
│  │               │     │               │     │               │      │
│  │ Rohstoffe     │     │ Produktion    │     │ Fertige       │      │
│  │ Vorleistungen │     │ Dienstleistung│     │ Produkte      │      │
│  │               │     │               │     │               │      │
│  │  Wert: 100 €  │     │ + Mehrwert    │     │  Wert: 300 €  │      │
│  │               │     │   + 200 €     │     │               │      │
│  └───────────────┘     └───────────────┘     └───────────────┘      │
│                                                                      │
│  Wertschöpfung = 300 € - 100 € = 200 €                              │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Wertschöpfungskette nach Porter

Die **Wertschöpfungskette** (Value Chain) zeigt alle Aktivitäten eines Unternehmens, die zur Wertschöpfung beitragen.

```
┌─────────────────────────────────────────────────────────────────────┐
│           WERTSCHÖPFUNGSKETTE NACH PORTER                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  UNTERSTÜTZENDE AKTIVITÄTEN:                                        │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │  Unternehmensinfrastruktur (Führung, Planung, Finanzen)     │    │
│  ├─────────────────────────────────────────────────────────────┤    │
│  │  Personalwirtschaft (Rekrutierung, Schulung, Gehalt)        │    │
│  ├─────────────────────────────────────────────────────────────┤    │
│  │  Technologieentwicklung (F&E, Prozessverbesserung)          │    │
│  ├─────────────────────────────────────────────────────────────┤    │
│  │  Beschaffung (Einkauf von Materialien und Dienstleistungen) │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                                                                      │
│  PRIMÄRE AKTIVITÄTEN:                                               │
│  ┌──────────┬──────────┬──────────┬──────────┬──────────┐          │
│  │ Eingangs-│          │ Ausgangs-│Marketing │          │          │
│  │ logistik │Operations│ logistik │ & Vertrieb│ Service │  ──► M   │
│  │          │          │          │          │          │      A   │
│  │ Waren-   │Produktion│ Lagerung │ Werbung  │ Kunden-  │      R   │
│  │ eingang  │          │ Versand  │ Verkauf  │ dienst   │      G   │
│  │ Lagerung │          │          │          │ Support  │      E   │
│  └──────────┴──────────┴──────────┴──────────┴──────────┘          │
│                                                                      │
│  Jede Aktivität trägt zur GEWINNMARGE bei!                          │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Primäre Aktivitäten

| Aktivität | Beschreibung | IT-Beispiel |
|-----------|--------------|-------------|
| **Eingangslogistik** | Warenannahme, Lagerhaltung | Hardware-Warenannahme |
| **Operations** | Produktion, Verarbeitung | Software-Entwicklung, IT-Service |
| **Ausgangslogistik** | Lagerung, Distribution | Auslieferung, Cloud-Deployment |
| **Marketing & Vertrieb** | Werbung, Verkauf | IT-Beratung, Angebotserstellung |
| **Service** | Kundendienst, Support | IT-Support, Wartung |

### Unterstützende Aktivitäten

| Aktivität | Beschreibung | IT-Beispiel |
|-----------|--------------|-------------|
| **Unternehmensinfrastruktur** | Führung, Planung, Controlling | IT-Strategie, Budgetierung |
| **Personalwirtschaft** | Rekrutierung, Schulung | IT-Fachkräfte, Weiterbildung |
| **Technologieentwicklung** | F&E, Innovation | Neue Technologien, Eigenentwicklung |
| **Beschaffung** | Einkauf | Hardware, Lizenzen, Cloud-Services |

### Wertschöpfungstiefe

Die **Wertschöpfungstiefe** beschreibt, wie viel ein Unternehmen selbst macht vs. zukauft.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    WERTSCHÖPFUNGSTIEFE                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  HOHE Wertschöpfungstiefe:                                          │
│  → Viel selbst machen                                               │
│  → Wenig zukaufen                                                   │
│  → Beispiel: Eigenentwicklung von Software                          │
│                                                                      │
│  NIEDRIGE Wertschöpfungstiefe:                                      │
│  → Wenig selbst machen                                              │
│  → Viel zukaufen (Outsourcing)                                      │
│  → Beispiel: Nur Vertrieb, keine eigene Produktion                  │
│                                                                      │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │                                                              │    │
│  │  Rohstoff → Komponente → Produktion → Vertrieb → Service    │    │
│  │                                                              │    │
│  │  ◄──────────────────────────────────────────────────────────►   │
│  │                                                              │    │
│  │  Hohe Tiefe: Alle Schritte selbst                           │    │
│  │  Niedrige Tiefe: Nur Vertrieb und Service                   │    │
│  │                                                              │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                                                                      │
│  IT-Beispiel:                                                       │
│  → Hohe Tiefe: Eigenentwicklung von ERP-System                      │
│  → Niedrige Tiefe: Wiederverkauf von Standard-Software              │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Make-or-Buy-Entscheidung

| Kriterium | Make (Selbst) | Buy (Zukaufen) |
|-----------|---------------|----------------|
| **Kosten** | Hohe Fixkosten | Variable Kosten |
| **Qualität** | Volle Kontrolle | Abhängig vom Lieferanten |
| **Flexibilität** | Weniger flexibel | Schneller anpassbar |
| **Know-how** | Aufbau intern | Nutzung externer Expertise |
| **Kapazität** | Eigene Ressourcen nötig | Skalierbar |

## Wichtige Begriffe

| Begriff | Erklärung |
|---------|-----------|
| **Wertschöpfung** | Mehrwert durch Verarbeitung |
| **Wertschöpfungskette** | Alle wertsteigernden Aktivitäten |
| **Primäre Aktivitäten** | Direkt am Produkt (Produktion, Vertrieb) |
| **Unterstützende Aktivitäten** | Indirekt (Personal, IT, Einkauf) |
| **Wertschöpfungstiefe** | Anteil der Eigenleistung |
| **Make-or-Buy** | Entscheidung: Selbst machen oder kaufen |
| **Outsourcing** | Auslagerung von Aktivitäten |

---

## Prüfungstipps

1. **Porter-Modell kennen:**
   - 5 primäre Aktivitäten (Merksatz: "Ein Opfer ahnt Marketing-Service")
   - 4 unterstützende Aktivitäten

2. **Unterscheidung:**
   - Primär = direkt am Produkt/Kunden
   - Unterstützend = ermöglichen die primären Aktivitäten

3. **IT-Bezug:**
   - IT ist oft unterstützende Aktivität (Technologieentwicklung)
   - Bei IT-Unternehmen: IT-Services sind primäre Aktivität!

## Übungsaufgaben

### Aufgabe 1: Aktivitäten zuordnen

Ordnen Sie die Aktivitäten der ConSystem GmbH zu:

a) Entwicklung einer neuen Software-Lösung
b) Rekrutierung eines neuen IT-Beraters
c) Installation von Servern beim Kunden
d) Einkauf von Hardware beim Großhändler
e) IT-Support-Hotline

<details>
<summary>Lösung anzeigen</summary>

a) **Primär: Operations** (bei IT-Unternehmen ist Entwicklung Kerngeschäft)
   oder **Unterstützend: Technologieentwicklung** (wenn es interne Tools sind)

b) **Unterstützend: Personalwirtschaft**

c) **Primär: Operations** oder **Ausgangslogistik** (Lieferung an Kunden)

d) **Unterstützend: Beschaffung**

e) **Primär: Service**

</details>

### Aufgabe 2: Wertschöpfung berechnen

Die ConSystem GmbH kauft einen Server für 2.000 € ein und verkauft ihn konfiguriert und installiert für 3.800 € an den Kunden.

a) Wie hoch ist die Wertschöpfung?
b) Welche Aktivitäten tragen zur Wertschöpfung bei?

<details>
<summary>Lösung anzeigen</summary>

a) **Wertschöpfung:**
3.800 € - 2.000 € = **1.800 €**

b) **Wertschöpfende Aktivitäten:**
- Beratung des Kunden (Marketing & Vertrieb)
- Konfiguration des Servers (Operations)
- Installation beim Kunden (Operations/Ausgangslogistik)
- Dokumentation
- Ggf. Schulung

Diese Tätigkeiten rechtfertigen den Mehrpreis von 1.800 €.

</details>

### Aufgabe 3: Make-or-Buy

Die ConSystem GmbH überlegt, ob sie eine eigene Buchhaltungssoftware entwickeln (Make) oder eine kaufen soll (Buy).

Nennen Sie je zwei Argumente für Make und für Buy.

<details>
<summary>Lösung anzeigen</summary>

**Argumente für MAKE (Eigenentwicklung):**
1. Genau auf Unternehmensprozesse zugeschnitten
2. Langfristig unabhängig von Lizenzkosten
3. Volle Kontrolle über Funktionen und Updates
4. Know-how bleibt im Unternehmen

**Argumente für BUY (Zukauf):**
1. Schneller verfügbar, keine Entwicklungszeit
2. Geringere Anfangsinvestition
3. Professioneller Support vom Anbieter
4. Regelmäßige Updates durch den Hersteller
5. Buchhaltung ist nicht Kernkompetenz

**Empfehlung für ConSystem:** BUY - Buchhaltung ist nicht das Kerngeschäft, Standard-Software ist ausgereift.

</details>
