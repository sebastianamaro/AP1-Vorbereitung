# Calidad del Software (ISO/IEC 25000)

## Objetivos de Aprendizaje

Después de este capítulo deberías ser capaz de:
- Conocer el propósito y el origen de la norma ISO/IEC 25000
- Nombrar y explicar las características de calidad esenciales
- Distinguir requisitos funcionales de no funcionales

---

## Contenido Principal

### ISO/IEC 25000 (SQuaRE)

La **ISO/IEC 25000** (*Software Quality Requirements and Evaluation*, "SQuaRE") es la norma internacional para la **calidad de los productos de software**. Surgió de dos normas anteriores:

- **ISO/IEC 9126** – definía un **modelo de calidad** para la evaluación de productos de software
- **ISO/IEC 14598** – definía el **proceso** de evaluación de productos de software

> *"ISO/IEC 25000 is the result of the evolution of several other standards; specifically from ISO/IEC 9126 (quality model) and ISO/IEC 14598 (evaluation process)."*

### Características de calidad (según ISO/IEC 25010)

| Característica | Significado |
|---------|-----------|
| **Adecuación funcional** | ¿El software cumple correctamente las funciones requeridas? |
| **Fiabilidad** | Disponibilidad, tolerancia a fallos, capacidad de recuperación |
| **Usabilidad** | Facilidad de aprendizaje, operabilidad, accesibilidad |
| **Eficiencia (rendimiento)** | Comportamiento temporal, consumo de recursos |
| **Compatibilidad** | Coexistencia e interoperabilidad con otros sistemas |
| **Seguridad** | Confidencialidad, integridad, no repudio |
| **Mantenibilidad** | Modularidad, analizabilidad, capacidad de modificación, capacidad de prueba |
| **Portabilidad** | Adaptabilidad, capacidad de instalación, capacidad de sustitución |

### Funcional vs. no funcional

- **Requisitos funcionales:** *qué* debe hacer el sistema (funciones).
- **Requisitos no funcionales:** *cómo de bien* (rendimiento, seguridad, usabilidad …) – a menudo exactamente las características de calidad anteriores.

## Términos Clave

| Término | Explicación |
|---------|-----------|
| **ISO/IEC 25000 (SQuaRE)** | Familia de normas para la calidad del software y su evaluación |
| **ISO/IEC 9126** | Norma predecesora (modelo de calidad) |
| **Requisito no funcional** | Requisito de calidad (rendimiento, seguridad …) |

## Consejos para el Examen

- El **origen** (9126 + 14598 → 25000) es una pregunta corta típica.
- Tener listas algunas **características de calidad**, cada una con una breve explicación.
- Distinguir con seguridad **funcional vs. no funcional** – también aparece en GA1 (análisis de requisitos).
- Saber traducir textos de normas en inglés (el libro incluye una tarea de traducción).
