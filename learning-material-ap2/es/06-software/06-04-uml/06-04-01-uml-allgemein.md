# UML en general

## Objetivos de Aprendizaje

Después de este capítulo deberías:
- Conocer el propósito de UML
- Saber distinguir y clasificar los diagramas de estructura y de comportamiento

---

## Contenido Principal

**UML** (Unified Modeling Language) es un **lenguaje de modelado gráfico** estandarizado para software. Los diagramas se dividen en dos grupos:

### Diagramas de estructura (estructura estática)

- **Diagrama de clases**
- **Diagrama de objetos**
- **Diagrama de paquetes**
- **Diagrama de componentes**
- **Diagrama de estructura compuesta**
- **Diagrama de despliegue**
- **Diagrama de perfiles**

### Diagramas de comportamiento (comportamiento dinámico)

- **Diagrama de casos de uso (Anwendungsfalldiagramm)**
- **Diagrama de actividad**
- **Diagrama de estados (Zustandsdiagramm)**
- **Diagrama de secuencia** (interacción)
- **Diagrama de comunicación** (interacción)
- **Diagrama de tiempos (Zeitverlaufsdiagramm)** (interacción)
- **Diagrama de resumen de interacciones**

```
UML
├── Strukturdiagramme  → ¿CÓMO está construido el sistema? (clase, paquete, …)
└── Verhaltensdiagramme → ¿QUÉ hace el sistema / cómo se ejecuta? (caso de uso, secuencia, …)
```

---

## Términos Clave

| Término | Explicación |
|---------|-----------|
| **Strukturdiagramm (diagrama de estructura)** | Muestra la estructura estática (p. ej. diagrama de clases) |
| **Verhaltensdiagramm (diagrama de comportamiento)** | Muestra el comportamiento/procesos dinámicos |
| **Interaktionsdiagramm (diagrama de interacción)** | Subgrupo de los diagramas de comportamiento (secuencia, comunicación, tiempos) |

---

## Consejos para el Examen

- **Tarea de asignación** (como en el libro): clasificar los tipos de diagrama en estructura vs. comportamiento.
- Diagrama de clases = **estructura**; caso de uso/secuencia/actividad/estados = **comportamiento**.
