# MVC (Model-View-Controller)

## Objetivos de Aprendizaje

Después de este capítulo deberías:
- Explicar los tres componentes de MVC y sus tareas
- Describir la interacción (accesos/notificaciones)
- Nombrar las ventajas del patrón

---

## Contenido Principal

**MVC** es un **patrón de arquitectura** que separa una aplicación en tres responsabilidades:

| Componente | Tarea |
|------------|---------|
| **Model** | Datos y lógica de negocio (no conoce ni a la View ni al Controller) |
| **View** | Presentación/interfaz (muestra los datos del Model) |
| **Controller** | Control: recibe las entradas del usuario, modifica el Model, elige la View |

### Interacción

```
   Usuario
      │  entrada
      ▼
 ┌────────────┐  actualiza      ┌────────┐
 │ Controller │────────────────►│ Model  │
 └─────┬──────┘                 └───┬────┘
       │ selecciona/actualiza       │ notifica / entrega datos
       ▼                            ▼
    ┌──────┐   ◄───────────────  ┌──────┐
    │ View │      lee datos       │ View │
    └──────┘                      └──────┘
```

- El usuario maneja la **View**, el **Controller** procesa la entrada y modifica el **Model**, y el Model informa a la **View** de los cambios (a menudo mediante **Observer**).

### Ventajas

- **Separación de responsabilidades** (Separation of Concerns)
- **Mantenibilidad** y **testabilidad**
- **Reutilización** (varias Views sobre un Model)
- desarrollo en paralelo (UI vs. lógica)

---

## Términos Clave

| Término | Explicación |
|---------|-----------|
| **Model** | Datos + lógica de negocio |
| **View** | Presentación/UI |
| **Controller** | Mediador, procesa las entradas |
| **Separation of Concerns** | Separación de responsabilidades |

---

## Consejos para el Examen

- Nombrar con seguridad los **tres componentes y sus tareas**.
- Describir la **interacción** (el Controller modifica el Model, el Model notifica a la View).
- MVC se puede ilustrar con un **diagrama de secuencia/clases** (tarea del libro).
