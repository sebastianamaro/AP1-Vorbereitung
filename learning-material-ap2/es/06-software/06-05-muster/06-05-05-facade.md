# Facade Pattern

## Objetivos de Aprendizaje

Después de este capítulo deberías:
- Explicar el propósito del patrón Facade
- Describir la interacción entre la fachada y los subsistemas
- Nombrar las ventajas (interfaz sencilla, acoplamiento débil)

---

## Contenido Principal

El **patrón Facade** (patrón estructural) oculta un **subsistema complejo detrás de una interfaz sencilla**. El cliente solo llama a la fachada; esta coordina las llamadas a los componentes individuales.

### Ventajas

| Ventaja | Significado |
|---------|-----------|
| **Interfaz simplificada** | un método en lugar de muchas llamadas individuales |
| **Acoplamiento débil** | el cliente no conoce los subsistemas |
| **Mantenibilidad** | los cambios en el subsistema quedan encapsulados |
| **Claridad** | punto de entrada claro |

### Ejemplo (Java) – Smart Home

```java
class Licht     { void aus() { } }
class Heizung   { void absenken() { } }
class Alarm     { void aktivieren() { } }

class SmartHomeFassade {
    private Licht licht = new Licht();
    private Heizung heizung = new Heizung();
    private Alarm alarm = new Alarm();

    void verlasseHaus() {       // eine einfache Aktion ...
        licht.aus();            // ... koordiniert das ganze Subsystem
        heizung.absenken();
        alarm.aktivieren();
    }
}

// Client:
new SmartHomeFassade().verlasseHaus();
```

> **Analogía:** Un **botón de smart home** «salir de casa» apaga las luces, baja la calefacción y activa la alarma con un solo clic.

---

## Términos Clave

| Término | Explicación |
|---------|-----------|
| **Facade (Fassade / fachada)** | Interfaz sencilla delante de un subsistema complejo |
| **Subsistema** | Los componentes individuales encapsulados |
| **Acoplamiento débil** | El cliente depende solo de la fachada |

---

## Consejos para el Examen

- Clasificación: **patrón estructural**.
- Núcleo: ocultar un **subsistema complejo detrás de una interfaz sencilla**.
- Diferencia: Facade *simplifica* (sin comportamiento nuevo); Factory *crea* objetos.
