# SOAP

## Objetivos de Aprendizaje

Después de este capítulo deberías:
- Clasificar SOAP como protocolo
- Conocer la estructura de los mensajes
- Comparar SOAP y REST

---

## Contenido Principal

**SOAP** (Simple Object Access Protocol) es un **protocolo basado en XML** para el intercambio de mensajes entre sistemas distribuidos. Estrictamente especificado, independiente del transporte (por lo general HTTP), con sus propios estándares.

### Estructura del mensaje

```
<Envelope>
  <Header>   … optional (z. B. Sicherheit) …
  <Body>     … Nutzdaten / Methodenaufruf …
</Envelope>
```

- **Envelope** – elemento raíz del mensaje
- **Header** – metadatos opcionales (p. ej. WS-Security)
- **Body** – los datos propiamente dichos

### Estándares

- **WSDL** – describe el servicio (métodos, parámetros, valores de retorno)
- **XSD** – valida la estructura de datos (XML Schema)
- **WS-Security** – cifrado, autenticación, firmas digitales a nivel de mensaje

### SOAP vs. REST

| | SOAP | REST |
|--|------|------|
| Tipo | Protocolo | Estilo arquitectónico |
| Formato | **XML** | en su mayoría **JSON** |
| Contrato | estricto (WSDL/XSD) | flexible |
| Seguridad | WS-Security (nivel de mensaje) | TLS (nivel de transporte) |
| Uso | Enterprise, bancos, administración pública, salud | APIs web/móviles |

---

## Términos Clave

| Término | Explicación |
|---------|-----------|
| **SOAP** | Protocolo de mensajería basado en XML |
| **Envelope/Header/Body** | Estructura de un mensaje SOAP |
| **WSDL/XSD** | Descripción del servicio / validación de esquema |
| **WS-Security** | Seguridad a nivel de mensaje |

---

## Consejos para el Examen

- **SOAP = protocolo/XML/estricto** vs. **REST = estilo arquitectónico/JSON/flexible** – saber la tabla comparativa.
- Estructura del mensaje **Envelope → Header → Body**.
- SOAP es típico de escenarios enterprise formales y de alta seguridad.
