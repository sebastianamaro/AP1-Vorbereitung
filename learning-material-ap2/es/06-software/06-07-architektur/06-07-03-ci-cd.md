# CI/CD

## Objetivos de Aprendizaje

Después de este capítulo deberías:
- Distinguir Continuous Integration y Continuous Delivery/Deployment
- Explicar el beneficio de una pipeline de CI/CD
- Clasificar las herramientas típicas

---

## Contenido Principal

**CI/CD** automatiza el camino desde el código hasta la entrega – de forma **rápida, fiable y reproducible**.

| Término | Significado |
|---------|-----------|
| **CI – Continuous Integration** | Fusión frecuente de cambios de código con **build + test automáticos** |
| **CD – Continuous Delivery** | Artefactos **entregables** automáticamente; publicación con solo pulsar un botón |
| **CD – Continuous Deployment** | **Despliegue automático a producción** (sin aprobación manual) |

### Pipeline típica

```
Commit → Build → automatische Tests → (Staging) → Deployment → Monitoring
```

### Herramientas

| Fase | Ejemplos |
|-------|-----------|
| Build/CI | Jenkins, GitLab CI, GitHub Actions |
| Deployment | Argo CD, Spinnaker |
| Monitoring | Prometheus, Grafana |

**Requisito** para Continuous Deployment: **alta cobertura de pruebas** e infraestructura estable (de lo contrario, los errores llegan a producción sin revisar).

---

## Términos Clave

| Término | Explicación |
|---------|-----------|
| **CI** | Integración + build + test automáticos |
| **Continuous Delivery** | Entregable en cualquier momento (publicación manual) |
| **Continuous Deployment** | Automático hasta producción |
| **Pipeline** | Secuencia automatizada build→test→deploy |

---

## Consejos para el Examen

- Separar claramente **CI vs. CD**; **Delivery (publicación manual) vs. Deployment (automático)**.
- Beneficio: rápido, fiable, reproducible; requisito = alta **cobertura de pruebas**.
- Saber nombrar una herramienta por cada fase.

---

## Ejercicio Práctico

**Tarea:** Esboza una pipeline de CI/CD desde el commit hasta el deployment y explica la diferencia entre Continuous Delivery y Continuous Deployment.
