# REST

## Objetivos de Aprendizaje

Después de este capítulo deberías:
- Explicar REST como estilo arquitectónico
- Conocer los métodos HTTP y su semántica
- Nombrar los principios más importantes de REST

---

## Contenido Principal

**REST** (Representational State Transfer) **no es un protocolo, sino un estilo arquitectónico** para APIs web sobre HTTP. Una API que sigue los principios se denomina **RESTful**. El formato de datos suele ser **JSON**.

### Principios (selección)

- **Cliente-servidor** – separación de la UI y el almacenamiento de datos
- **Sin estado (stateless)** – cada petición contiene **toda** la información necesaria
- **Cacheable** – las respuestas pueden almacenarse en caché
- **Uniform Interface** – recursos mediante **URIs**, métodos HTTP uniformes
- **Layered System**, (opcional) Code-on-Demand

### Métodos HTTP

| Método | Significado |
|---------|-----------|
| **GET** | Leer |
| **POST** | Crear nuevo |
| **PUT** | Reemplazar por completo |
| **PATCH** | Modificar parcialmente |
| **DELETE** | Eliminar |

```http
GET /api/produkte/42   →   { "id": 42, "name": "Gaming-Maus", "preis": 59.99 }
```

**Códigos de estado** importantes: 200 OK, 201 Created, 400 Bad Request, 401 Unauthorized, 404 Not Found, 500 Server Error.

---

## Términos Clave

| Término | Explicación |
|---------|-----------|
| **REST** | Estilo arquitectónico para APIs web sobre HTTP |
| **Stateless** | Sin estado de sesión en el servidor |
| **Recurso/URI** | Objeto direccionable de la API |
| **Métodos HTTP** | GET/POST/PUT/PATCH/DELETE |

---

## Consejos para el Examen

- **REST = estilo arquitectónico, no un protocolo** – pregunta frecuente.
- Métodos HTTP ↔ CRUD (GET=Read, POST=Create, PUT/PATCH=Update, DELETE=Delete).
- Saber explicar **stateless**. REST usa en su mayoría **JSON** (cf. SOAP = XML).
