# Orientación a Objetos (OOP)

## Objetivos de Aprendizaje

Después de este capítulo deberías poder:
- Explicar los conceptos básicos de la OOP (clase, objeto, atributo, método)
- Aplicar los cuatro principios fundamentales de la OOP
- Reconocer el polimorfismo y la herencia en un ejemplo

---

## Contenido Principal

### Conceptos Básicos

| Término | Significado |
|---------|-------------|
| **Clase** | Plano/plantilla para objetos |
| **Objeto (instancia)** | Manifestación concreta de una clase |
| **Atributo** | Propiedad/dato de un objeto |
| **Método** | Comportamiento/función de un objeto |
| **Constructor** | Crea e inicializa un objeto |

### Los Cuatro Principios Fundamentales

1. **Abstracción** – modelar solo las propiedades esenciales
2. **Encapsulamiento** (alemán: *Kapselung*) – ocultar los atributos (`private`) y hacerlos accesibles mediante métodos (`get`/`set`) → protección/consistencia de los datos
3. **Herencia** (alemán: *Vererbung*) – una subclase hereda atributos/métodos de la superclase (`Kreis` hereda de `Basisform`) → reutilización. *(Eliminado de AP1, ahora es materia de AP2.)*
4. **Polimorfismo** – "múltiples formas": el mismo método se comporta de manera diferente según el objeto concreto (p. ej. `flaeche()` en `Kreis` vs. `Dreieck`); implementado mediante **sobrescritura (overriding)** y **sobrecarga (overloading)**

### Ejemplo: Jerarquía de Clases de Formas Geométricas

```
        Basisform (abstrakt)
        + flaeche(): double   ◄── überall überschrieben (Polymorphie)
          ▲            ▲
       Kreis        Dreieck
   + flaeche()    + flaeche()
```

Un **error de polimorfismo** frecuente: el método `flaeche()` **no se sobrescribe correctamente** en una subclase (firma incorrecta / falta `@Override`), de modo que se aplica la implementación de la base y produce resultados erróneos.

### Relaciones (para Diagramas de Clases)

- **Asociación** – "conoce a"
- **Agregación** – "tiene un" (la parte puede existir por sí sola)
- **Composición** – "se compone de" (la parte muere con el todo)
- **Herencia/generalización** – "es un"

---

## Términos Clave

| Término | Explicación |
|---------|-------------|
| **Encapsulamiento** (*Kapselung*) | Ocultar atributos, acceso mediante métodos |
| **Herencia** (*Vererbung*) | La subclase hereda de la superclase ("es un") |
| **Polimorfismo** | Mismo método, distinto comportamiento por objeto |
| **Sobrescritura (Override)** | Reimplementación de un método heredado |

---

## Consejos para el Examen

- Explicar con seguridad los **cuatro principios** (abstracción, encapsulamiento, herencia, polimorfismo).
- Explicar el **polimorfismo** mediante la sobrescritura – una tarea frecuente de GA2 (revisar código en busca de un error de polimorfismo).
- Conocer las relaciones (asociación/agregación/composición/herencia) para el **diagrama de clases**.
