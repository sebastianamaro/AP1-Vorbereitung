# MVC (Model-View-Controller)

## Learning Objectives

After this chapter you should:
- Explain the three MVC components and their tasks
- Describe the interplay (accesses/notifications)
- Name the advantages of the pattern

---

## Core Content

**MVC** is an **architecture pattern** that separates an application into three responsibilities:

| Component | Task |
|------------|---------|
| **Model** | Data and business logic (knows neither View nor Controller) |
| **View** | Presentation/interface (shows the Model's data) |
| **Controller** | Control: accepts user input, changes the Model, selects the View |

### Interplay

```
   User
      │  input
      ▼
 ┌────────────┐  updates        ┌────────┐
 │ Controller │────────────────►│ Model  │
 └─────┬──────┘                 └───┬────┘
       │ selects/updates            │ notifies / provides data
       ▼                            ▼
    ┌──────┐   ◄───────────────  ┌──────┐
    │ View │      reads data      │ View │
    └──────┘                      └──────┘
```

- The user operates the **View**, the **Controller** processes the input and changes the **Model**, the Model informs the **View** about changes (often via **Observer**).

### Advantages

- **Separation of Concerns** (Trennung der Zuständigkeiten)
- **Maintainability** and **testability**
- **Reusability** (several Views on one Model)
- parallel development (UI vs. logic)

---

## Key Terms

| Term | Explanation |
|---------|-----------|
| **Model** | Data + business logic |
| **View** | Presentation/UI |
| **Controller** | Mediator, processes input |
| **Separation of Concerns** | Separation of responsibilities |
