# Facade Pattern

## Learning Objectives

After this chapter you should:
- Explain the purpose of the Facade pattern
- Describe the interplay of facade and subsystems
- Name the advantages (simple interface, loose coupling)

---

## Core Content

The **Facade pattern** (structural pattern) hides a **complex subsystem behind a simple interface**. The client only calls the facade; it coordinates the calls to the individual components.

### Advantages

| Advantage | Meaning |
|---------|-----------|
| **Simplified interface** | one method instead of many individual calls |
| **Loose coupling** | client does not know the subsystems |
| **Maintainability** | changes in the subsystem stay encapsulated |
| **Clarity** | clear entry point |

### Example (Java) – Smart Home

```java
class Licht     { void aus() { } }
class Heizung   { void absenken() { } }
class Alarm     { void aktivieren() { } }

class SmartHomeFassade {
    private Licht licht = new Licht();
    private Heizung heizung = new Heizung();
    private Alarm alarm = new Alarm();

    void verlasseHaus() {       // eine einfache Aktion ...
        licht.aus();            // ... koordiniert das ganze Subsystem
        heizung.absenken();
        alarm.aktivieren();
    }
}

// Client:
new SmartHomeFassade().verlasseHaus();
```

> **Analogy:** A **smart-home button** "leave house" turns off the lights, lowers the heating and activates the alarm with a single click.

---

## Key Terms

| Term | Explanation |
|---------|-----------|
| **Facade (Fassade)** | Simple interface in front of a complex subsystem |
| **Subsystem** | The encapsulated individual components |
| **Loose coupling** | Client depends only on the facade |

---

## Exam Tips

- Classification: **structural pattern**.
- Core: hide a **complex subsystem behind a simple interface**.
- Distinction: Facade *simplifies* (no new behavior); Factory *creates* objects.
