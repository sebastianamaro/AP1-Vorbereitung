# UML in general

## Learning Objectives

After this chapter you should:
- Know the purpose of UML
- Be able to distinguish and classify structure and behavior diagrams

---

## Core Content

**UML** (Unified Modeling Language) is a standardized **graphical modeling language** for software. The diagrams fall into two groups:

### Structure diagrams (static build-up)

- **Class diagram**
- **Object diagram**
- **Package diagram**
- **Component diagram**
- **Composite structure diagram**
- **Deployment diagram**
- **Profile diagram**

### Behavior diagrams (dynamic behavior)

- **Use-Case diagram (Anwendungsfalldiagramm)**
- **Activity diagram**
- **State diagram (Zustandsdiagramm)**
- **Sequence diagram** (interaction)
- **Communication diagram** (interaction)
- **Timing diagram (Zeitverlaufsdiagramm)** (interaction)
- **Interaction overview diagram**

```
UML
├── Strukturdiagramme  → HOW is the system built? (class, package, …)
└── Verhaltensdiagramme → WHAT does the system do / how does it run? (use-case, sequence, …)
```

---

## Key Terms

| Term | Explanation |
|---------|-----------|
| **Strukturdiagramm (structure diagram)** | Shows the static build-up (e.g. class diagram) |
| **Verhaltensdiagramm (behavior diagram)** | Shows dynamic behavior/processes |
| **Interaktionsdiagramm (interaction diagram)** | Subgroup of behavior diagrams (sequence, communication, timing) |

---

## Exam Tips

- **Matching task** (as in the book): sort diagram types into structure vs. behavior.
- Class diagram = **structure**; use-case/sequence/activity/state = **behavior**.
