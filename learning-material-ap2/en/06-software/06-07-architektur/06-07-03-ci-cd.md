# CI/CD

## Learning Objectives

After this chapter you should:
- Distinguish Continuous Integration and Continuous Delivery/Deployment
- Explain the benefit of a CI/CD pipeline
- Classify typical tools

---

## Core Content

**CI/CD** automates the path from code to delivery – **fast, reliable and reproducible**.

| Term | Meaning |
|---------|-----------|
| **CI – Continuous Integration** | Frequent merging of code changes with **automatic build + test** |
| **CD – Continuous Delivery** | Automatically **deliverable** artifacts; release at the push of a button |
| **CD – Continuous Deployment** | Automatic **deployment to production** (without manual release) |

### Typical pipeline

```
Commit → Build → automatische Tests → (Staging) → Deployment → Monitoring
```

### Tools

| Phase | Examples |
|-------|-----------|
| Build/CI | Jenkins, GitLab CI, GitHub Actions |
| Deployment | Argo CD, Spinnaker |
| Monitoring | Prometheus, Grafana |

**Prerequisite** for Continuous Deployment: **high test coverage** and stable infrastructure (otherwise errors reach production unchecked).

---

## Key Terms

| Term | Explanation |
|---------|-----------|
| **CI** | Automatic integration + build + test |
| **Continuous Delivery** | Deliverable at any time (release manual) |
| **Continuous Deployment** | Automatic all the way to production |
| **Pipeline** | Automated sequence build→test→deploy |

---

## Exam Tips

- Clearly separate **CI vs. CD**; **Delivery (release manual) vs. Deployment (automatic)**.
- Benefit: fast, reliable, reproducible; prerequisite = high **test coverage**.
- Be able to name one tool per phase.
