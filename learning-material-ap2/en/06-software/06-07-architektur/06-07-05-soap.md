# SOAP

## Learning Objectives

After this chapter you should:
- Classify SOAP as a protocol
- Know the message structure
- Compare SOAP and REST

---

## Core Content

**SOAP** (Simple Object Access Protocol) is an **XML-based protocol** for exchanging messages between distributed systems. Strictly specified, transport-independent (mostly HTTP), with its own standards.

### Message structure

```
<Envelope>
  <Header>   … optional (z. B. Sicherheit) …
  <Body>     … Nutzdaten / Methodenaufruf …
</Envelope>
```

- **Envelope** – root element of the message
- **Header** – optional metadata (e.g. WS-Security)
- **Body** – the actual data

### Standards

- **WSDL** – describes the service (methods, parameters, return values)
- **XSD** – validates the data structure (XML Schema)
- **WS-Security** – encryption, authentication, digital signatures at the message level

### SOAP vs. REST

| | SOAP | REST |
|--|------|------|
| Type | Protocol | Architectural style |
| Format | **XML** | mostly **JSON** |
| Contract | strict (WSDL/XSD) | flexible |
| Security | WS-Security (message-level) | TLS (transport-level) |
| Use | Enterprise, banks, authorities, health | web/mobile APIs |

---

## Key Terms

| Term | Explanation |
|---------|-----------|
| **SOAP** | XML-based messaging protocol |
| **Envelope/Header/Body** | Structure of a SOAP message |
| **WSDL/XSD** | Service description / schema validation |
| **WS-Security** | Security at the message level |

---

## Exam Tips

- **SOAP = protocol/XML/strict** vs. **REST = architectural style/JSON/flexible** – know the comparison table.
- Message structure **Envelope → Header → Body**.
- SOAP is typical for highly secure/formal enterprise scenarios.
