# Object Orientation (OOP)

## Learning Objectives

After this chapter you should be able to:
- Explain the basic OOP terms (class, object, attribute, method)
- Apply the four fundamental OOP principles
- Recognise polymorphism and inheritance in an example

---

## Core Content

### Basic Terms

| Term | Meaning |
|------|---------|
| **Class** | Blueprint/template for objects |
| **Object (instance)** | Concrete manifestation of a class |
| **Attribute** | Property/data of an object |
| **Method** | Behaviour/function of an object |
| **Constructor** | Creates and initialises an object |

### The Four Fundamental Principles

1. **Abstraction** – model only the essential properties
2. **Encapsulation** (German: *Kapselung*) – hide attributes (`private`) and make them accessible via methods (`get`/`set`) → data protection/consistency
3. **Inheritance** (German: *Vererbung*) – a subclass inherits attributes/methods from the superclass (`Kreis` inherits from `Basisform`) → reuse. *(Removed from AP1, now AP2 material.)*
4. **Polymorphism** – "many forms": the same method behaves differently depending on the concrete object (e.g. `flaeche()` on `Kreis` vs. `Dreieck`); implemented through **overriding** and **overloading**

### Example: Class Hierarchy of Geometric Shapes

```
        Basisform (abstrakt)
        + flaeche(): double   ◄── überall überschrieben (Polymorphie)
          ▲            ▲
       Kreis        Dreieck
   + flaeche()    + flaeche()
```

A common **polymorphism error**: the method `flaeche()` is **not correctly overridden** in a subclass (wrong signature / missing `@Override`), so the base implementation takes effect and produces wrong results.

### Relationships (for Class Diagrams)

- **Association** – "knows a"
- **Aggregation** – "has a" (the part can exist on its own)
- **Composition** – "consists of" (the part dies with the whole)
- **Inheritance/generalisation** – "is a"

---

## Key Terms

| Term | Explanation |
|------|-------------|
| **Encapsulation** (*Kapselung*) | Hiding attributes, access via methods |
| **Inheritance** (*Vererbung*) | Subclass inherits from superclass ("is a") |
| **Polymorphism** | Same method, different behaviour per object |
| **Overriding** | Reimplementation of an inherited method |

---

## Exam Tips

- Confidently explain the **four principles** (abstraction, encapsulation, inheritance, polymorphism).
- Explain **polymorphism** via overriding – a common GA2 task (check code for a polymorphism error).
- Know the relationships (association/aggregation/composition/inheritance) for the **class diagram**.
