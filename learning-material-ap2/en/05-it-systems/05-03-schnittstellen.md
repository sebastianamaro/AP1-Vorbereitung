# Interfaces & Software Ergonomics

## Learning Objectives

After this chapter you should be able to:
- Distinguish types of interfaces
- Define software ergonomics
- Name the requirements of the ArbStättV (German Workplace Ordinance) for computer workstations

---

## Core Content

### Types of Interfaces

| Type | Description | Examples |
|------|-------------|----------|
| **Hardware interface** | physical connection of devices | USB, HDMI, Ethernet |
| **Software interface** | communication between programs | API, REST, SOAP |
| **User interface** | human ↔ system | GUI, CLI, voice control |

### Software Ergonomics

**Software ergonomics** deals with designing software so that it can be used **effectively, efficiently and satisfactorily** (usability). Its basis includes, among others, the standard **ISO 9241** (principles of dialogue design: suitability for the task, self-descriptiveness, conformity with user expectations, error tolerance, controllability, suitability for individualization, suitability for learning).

### Workplace Ordinance (ArbStättV, Arbeitsstättenverordnung / German Workplace Ordinance) – Computer Workstation

Requirements for a computer workstation (selection):

- **Monitor:** flicker-free, glare-free, rotatable/tiltable, sufficiently large
- **Lighting:** sufficient, glare-free; windows with glare protection
- **Desk/chair:** sufficiently large, height-adjustable, ergonomic
- **Space:** enough room to move, sufficient viewing distance
- **Noise/climate:** low noise exposure, pleasant room climate
- **Software:** ergonomically designed (see ISO 9241)

## Key Terms

| Term | Explanation |
|------|-------------|
| **API** | Programming interface between software components |
| **Software ergonomics** | Usability of the software (ISO 9241) |
| **ArbStättV** | Arbeitsstättenverordnung (German Workplace Ordinance; incl. computer workstations) |
