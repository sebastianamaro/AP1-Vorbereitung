# Virtual Desktops (VDI)

## Learning Objectives

After this chapter you should be able to:
- Explain the basic concept of a VDI
- Distinguish the self-hosted and cloud variants
- Name the advantages and disadvantages of VDI

---

## Core Content

### Basic Concept

With a **VDI (Virtual Desktop Infrastructure)** the workstation desktops run as **virtual machines centrally in the data center** (or in the cloud). Users access them from an end device (often a **thin client**) via a remote protocol; the actual computing power sits centrally.

### Variants

| Variant | Description |
|---------|-------------|
| **Self-hosted (on-premises)** | own servers/infrastructure in the company; full control, high initial investment |
| **Cloud (DaaS – Desktop as a Service)** | provided by the vendor; low initial cost, ongoing fees, dependence on vendor/network |

### Advantages and Disadvantages

**Advantages:** central management/updates, fast provisioning, longer usability of weak end devices (thin clients), data security (data stays central), easy scaling.

**Disadvantages:** dependence on network/server (no work during an outage), initial investment or ongoing costs, performance with graphics-intensive applications.

---

## Key Terms

| Term | Explanation |
|------|-------------|
| **VDI** | Centrally hosted, virtual desktops |
| **Thin client** | Lightweight end device that accesses central desktops |
| **DaaS** | Desktop as a Service (VDI from the cloud) |

---

## Exam Tips

- VDI core idea: **the desktop runs centrally, access via a thin client** – computing power is central.
- Distinguish **on-premises vs. cloud (DaaS)** with one advantage/disadvantage each.
- Justify VDI with data security/central management as advantages.
