# SQL Joins

## Lernziele

Nach diesem Kapitel solltest du:
- Die vier Join-Typen unterscheiden
- Tabellen über Primär-/Fremdschlüssel verknüpfen
- Den passenden Join für eine Fragestellung wählen

---

## Kerninhalt

Ein **JOIN** verknüpft Zeilen mehrerer Tabellen über eine Bedingung (meist **PK = FK**).

| Join | Ergebnis |
|------|----------|
| **INNER JOIN** | nur Zeilen mit **Treffer in beiden** Tabellen |
| **LEFT (OUTER) JOIN** | alle Zeilen **links** + passende rechts (sonst NULL) |
| **RIGHT (OUTER) JOIN** | alle Zeilen **rechts** + passende links (sonst NULL) |
| **FULL OUTER JOIN** | **alle** Zeilen beider Tabellen (fehlende Seite NULL) |

```sql
-- INNER: nur Kunden mit Bestellungen
SELECT k.name, b.artikel
FROM kunden k
INNER JOIN bestellungen b ON k.kunden_id = b.kunden_id;

-- LEFT: alle Kunden, auch ohne Bestellung
SELECT k.name, b.artikel
FROM kunden k
LEFT JOIN bestellungen b ON k.kunden_id = b.kunden_id;
```

- **INNER** = Schnittmenge; **LEFT/RIGHT** = eine Seite vollständig; **FULL** = Vereinigung.
- **FULL OUTER JOIN** wird nicht von allen DBMS unterstützt (z. B. MySQL → mit `UNION` aus LEFT+RIGHT nachbilden).

---

## Wichtige Begriffe

| Begriff | Erklärung |
|---------|-----------|
| **INNER JOIN** | Nur übereinstimmende Zeilen |
| **LEFT/RIGHT JOIN** | Eine Tabelle vollständig, andere mit NULL |
| **FULL OUTER JOIN** | Alle Zeilen beider Tabellen |
| **ON-Bedingung** | Verknüpfungskriterium (meist PK = FK) |

---

## Prüfungstipps

- **INNER vs. LEFT** ist der Klassiker: „auch Kunden **ohne** Bestellung" → LEFT JOIN.
- Bei Handschrift **Tabellennamen abkürzen** (`k`, `b`) und mit Alias arbeiten – spart Zeit.
- Immer die **ON-Bedingung** angeben (sonst Kreuzprodukt!).

---

## Übungsaufgabe

**Aufgabe:** Gib alle Kunden mit ihren Bestellungen aus – (a) nur Kunden, die bestellt haben, (b) alle Kunden inkl. solcher ohne Bestellung. Welcher Join jeweils?

<details>
<summary>Lösung</summary>

(a) INNER JOIN, (b) LEFT JOIN (kunden links).

</details>
