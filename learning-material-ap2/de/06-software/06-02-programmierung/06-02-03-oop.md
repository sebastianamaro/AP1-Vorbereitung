# Objektorientierung (OOP)

## Lernziele

Nach diesem Kapitel solltest du:
- Die Grundbegriffe der OOP (Klasse, Objekt, Attribut, Methode) erklären
- Die vier OOP-Grundprinzipien anwenden
- Polymorphie und Vererbung an einem Beispiel erkennen

---

## Kerninhalt

### Grundbegriffe

| Begriff | Bedeutung |
|---------|-----------|
| **Klasse** | Bauplan/Schablone für Objekte |
| **Objekt (Instanz)** | Konkrete Ausprägung einer Klasse |
| **Attribut** | Eigenschaft/Datum eines Objekts |
| **Methode** | Verhalten/Funktion eines Objekts |
| **Konstruktor** | Erzeugt und initialisiert ein Objekt |

### Die vier Grundprinzipien

1. **Abstraktion** – nur wesentliche Eigenschaften modellieren
2. **Kapselung (Encapsulation)** – Attribute verbergen (`private`) und über Methoden (`get`/`set`) zugänglich machen → Datenschutz/Konsistenz
3. **Vererbung (Inheritance)** – eine Unterklasse erbt Attribute/Methoden der Oberklasse (`Kreis` erbt von `Basisform`) → Wiederverwendung. *(In AP1 gestrichen, jetzt AP2-Stoff.)*
4. **Polymorphie (Polymorphism)** – „Vielgestaltigkeit“: dieselbe Methode verhält sich je nach konkretem Objekt unterschiedlich (z. B. `flaeche()` bei `Kreis` vs. `Dreieck`); umgesetzt durch **Überschreiben (Overriding)** und **Überladen (Overloading)**

### Beispiel: Klassenhierarchie geometrischer Formen

```
        Basisform (abstrakt)
        + flaeche(): double   ◄── überall überschrieben (Polymorphie)
          ▲            ▲
       Kreis        Dreieck
   + flaeche()    + flaeche()
```

Ein häufiger **Polymorphie-Fehler**: Die Methode `flaeche()` wird in einer Unterklasse **nicht korrekt überschrieben** (falsche Signatur / fehlendes `@Override`), sodass die Basis-Implementierung greift und falsche Ergebnisse liefert.

### Beziehungen (für Klassendiagramme)

- **Assoziation** – „kennt ein“
- **Aggregation** – „hat ein“ (Teil kann allein existieren)
- **Komposition** – „besteht aus“ (Teil stirbt mit dem Ganzen)
- **Vererbung/Generalisierung** – „ist ein“

---

## Wichtige Begriffe

| Begriff | Erklärung |
|---------|-----------|
| **Kapselung** | Verbergen von Attributen, Zugriff über Methoden |
| **Vererbung** | Unterklasse erbt von Oberklasse („ist ein“) |
| **Polymorphie** | Gleiche Methode, unterschiedliches Verhalten je Objekt |
| **Überschreiben (Override)** | Neuimplementierung einer geerbten Methode |

---

## Prüfungstipps

- Die **vier Prinzipien** (Abstraktion, Kapselung, Vererbung, Polymorphie) sicher erklären.
- **Polymorphie** über Überschreiben erklären – häufige GA2-Aufgabe (Code auf Polymorphie-Fehler prüfen).
- Beziehungen (Assoziation/Aggregation/Komposition/Vererbung) fürs **Klassendiagramm** kennen.
