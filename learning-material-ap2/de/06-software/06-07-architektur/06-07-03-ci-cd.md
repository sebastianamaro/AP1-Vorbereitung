# CI/CD

## Lernziele

Nach diesem Kapitel solltest du:
- Continuous Integration und Continuous Delivery/Deployment unterscheiden
- Den Nutzen einer CI/CD-Pipeline erklären
- Typische Werkzeuge einordnen

---

## Kerninhalt

**CI/CD** automatisiert den Weg vom Code zur Auslieferung – **schnell, zuverlässig und reproduzierbar**.

| Begriff | Bedeutung |
|---------|-----------|
| **CI – Continuous Integration** | Häufiges Zusammenführen von Codeänderungen mit **automatischem Build + Test** |
| **CD – Continuous Delivery** | Automatisch **auslieferbare** Artefakte; Freigabe per Knopfdruck |
| **CD – Continuous Deployment** | Automatisches **Deployment in Produktion** (ohne manuelle Freigabe) |

### Typische Pipeline

```
Commit → Build → automatische Tests → (Staging) → Deployment → Monitoring
```

### Werkzeuge

| Phase | Beispiele |
|-------|-----------|
| Build/CI | Jenkins, GitLab CI, GitHub Actions |
| Deployment | Argo CD, Spinnaker |
| Monitoring | Prometheus, Grafana |

**Voraussetzung** für Continuous Deployment: **hohe Testabdeckung** und stabile Infrastruktur (sonst gelangen Fehler ungeprüft in Produktion).

---

## Wichtige Begriffe

| Begriff | Erklärung |
|---------|-----------|
| **CI** | Automatisches Integrieren + Build + Test |
| **Continuous Delivery** | Jederzeit auslieferbar (Freigabe manuell) |
| **Continuous Deployment** | Automatisch bis in Produktion |
| **Pipeline** | Automatisierte Abfolge Build→Test→Deploy |

---

## Prüfungstipps

- **CI vs. CD** klar trennen; **Delivery (Freigabe manuell) vs. Deployment (automatisch)**.
- Nutzen: schnell, zuverlässig, reproduzierbar; Voraussetzung = hohe **Testabdeckung**.
- Ein Werkzeug je Phase nennen können.

---

## Übungsaufgabe

**Aufgabe:** Skizziere eine CI/CD-Pipeline vom Commit bis zum Deployment und erkläre den Unterschied zwischen Continuous Delivery und Continuous Deployment.
