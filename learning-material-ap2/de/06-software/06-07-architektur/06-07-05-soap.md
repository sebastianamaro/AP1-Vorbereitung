# SOAP

## Lernziele

Nach diesem Kapitel solltest du:
- SOAP als Protokoll einordnen
- Den Nachrichtenaufbau kennen
- SOAP und REST vergleichen

---

## Kerninhalt

**SOAP** (Simple Object Access Protocol) ist ein **XML-basiertes Protokoll** für den Nachrichtenaustausch zwischen verteilten Systemen. Streng spezifiziert, transport-unabhängig (meist HTTP), mit eigenen Standards.

### Nachrichtenaufbau

```
<Envelope>
  <Header>   … optional (z. B. Sicherheit) …
  <Body>     … Nutzdaten / Methodenaufruf …
</Envelope>
```

- **Envelope** – Wurzelelement der Nachricht
- **Header** – optionale Metadaten (z. B. WS-Security)
- **Body** – die eigentlichen Daten

### Standards

- **WSDL** – beschreibt den Service (Methoden, Parameter, Rückgaben)
- **XSD** – validiert die Datenstruktur (XML Schema)
- **WS-Security** – Verschlüsselung, Authentifizierung, digitale Signaturen auf Nachrichtenebene

### SOAP vs. REST

| | SOAP | REST |
|--|------|------|
| Art | Protokoll | Architekturstil |
| Format | **XML** | meist **JSON** |
| Vertrag | streng (WSDL/XSD) | flexibel |
| Sicherheit | WS-Security (Message-Level) | TLS (Transport-Level) |
| Einsatz | Enterprise, Banken, Behörden, Health | Web-/Mobile-APIs |

---

## Wichtige Begriffe

| Begriff | Erklärung |
|---------|-----------|
| **SOAP** | XML-basiertes Nachrichtenprotokoll |
| **Envelope/Header/Body** | Aufbau einer SOAP-Nachricht |
| **WSDL/XSD** | Servicebeschreibung / Schema-Validierung |
| **WS-Security** | Sicherheit auf Nachrichtenebene |

---

## Prüfungstipps

- **SOAP = Protokoll/XML/streng** vs. **REST = Architekturstil/JSON/flexibel** – die Vergleichstabelle können.
- Nachrichtenaufbau **Envelope → Header → Body**.
- SOAP typisch für hochsichere/formale Enterprise-Szenarien.

---

## Übungsaufgabe

**Aufgabe:** Nenne drei Unterschiede zwischen SOAP und REST und je ein passendes Einsatzszenario.
