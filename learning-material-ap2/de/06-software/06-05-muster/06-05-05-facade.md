# Facade Pattern

## Lernziele

Nach diesem Kapitel solltest du:
- Zweck des Facade-Musters erklären
- Das Zusammenspiel von Fassade und Subsystemen beschreiben
- Die Vorteile (einfache Schnittstelle, lose Kopplung) nennen

---

## Kerninhalt

Das **Facade Pattern** (Strukturmuster) versteckt ein **komplexes Subsystem hinter einer einfachen Schnittstelle**. Der Client ruft nur die Fassade auf; diese koordiniert die Aufrufe an die Einzelkomponenten.

### Vorteile

| Vorteil | Bedeutung |
|---------|-----------|
| **Vereinfachte Schnittstelle** | eine Methode statt vieler Einzelaufrufe |
| **Lose Kopplung** | Client kennt die Subsysteme nicht |
| **Wartbarkeit** | Änderungen im Subsystem bleiben gekapselt |
| **Übersichtlichkeit** | klarer Einstiegspunkt |

### Beispiel (Java) – Smart Home

```java
class Licht     { void aus() { } }
class Heizung   { void absenken() { } }
class Alarm     { void aktivieren() { } }

class SmartHomeFassade {
    private Licht licht = new Licht();
    private Heizung heizung = new Heizung();
    private Alarm alarm = new Alarm();

    void verlasseHaus() {       // eine einfache Aktion ...
        licht.aus();            // ... koordiniert das ganze Subsystem
        heizung.absenken();
        alarm.aktivieren();
    }
}

// Client:
new SmartHomeFassade().verlasseHaus();
```

> **Analogie:** Ein **Smart-Home-Knopf** „Haus verlassen" schaltet mit einem Klick Licht aus, Heizung runter und Alarm an.

---

## Wichtige Begriffe

| Begriff | Erklärung |
|---------|-----------|
| **Facade (Fassade)** | Einfache Schnittstelle vor einem komplexen Subsystem |
| **Subsystem** | Die gekapselten Einzelkomponenten |
| **Lose Kopplung** | Client hängt nur an der Fassade |

---

## Prüfungstipps

- Einordnung: **Strukturmuster**.
- Kern: **komplexes Subsystem hinter einer einfachen Schnittstelle** verbergen.
- Abgrenzung: Facade *vereinfacht* (kein neues Verhalten); Factory *erzeugt* Objekte.

---

## Übungsaufgabe

**Aufgabe:** Entwirf eine `BestellFassade`, die `Lagerpruefung`, `Zahlung` und `Versand` hinter einer Methode `bestelle(...)` koordiniert. Nenne zwei Vorteile.
