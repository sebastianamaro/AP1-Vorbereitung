# MVC (Model-View-Controller)

## Lernziele

Nach diesem Kapitel solltest du:
- Die drei MVC-Komponenten und ihre Aufgaben erklären
- Das Zusammenspiel (Zugriffe/Benachrichtigungen) beschreiben
- Vorteile des Musters nennen

---

## Kerninhalt

**MVC** ist ein **Architekturmuster**, das eine Anwendung in drei Verantwortlichkeiten trennt:

| Komponente | Aufgabe |
|------------|---------|
| **Model** | Daten und Geschäftslogik (kennt weder View noch Controller) |
| **View** | Darstellung/Oberfläche (zeigt Daten des Models) |
| **Controller** | Steuerung: nimmt Benutzereingaben entgegen, ändert das Model, wählt die View |

### Zusammenspiel

```
   Benutzer
      │  Eingabe
      ▼
 ┌────────────┐  aktualisiert   ┌────────┐
 │ Controller │────────────────►│ Model  │
 └─────┬──────┘                 └───┬────┘
       │ wählt/aktualisiert          │ benachrichtigt / liefert Daten
       ▼                             ▼
    ┌──────┐   ◄───────────────  ┌──────┐
    │ View │      liest Daten     │ View │
    └──────┘                      └──────┘
```

- Benutzer bedient die **View**, der **Controller** verarbeitet die Eingabe und ändert das **Model**, das Model informiert die **View** über Änderungen (oft per **Observer**).

### Vorteile

- **Trennung der Zuständigkeiten** (Separation of Concerns)
- **Wartbarkeit** und **Testbarkeit**
- **Wiederverwendbarkeit** (mehrere Views auf ein Model)
- parallele Entwicklung (UI vs. Logik)

---

## Wichtige Begriffe

| Begriff | Erklärung |
|---------|-----------|
| **Model** | Daten + Geschäftslogik |
| **View** | Darstellung/UI |
| **Controller** | Vermittler, verarbeitet Eingaben |
| **Separation of Concerns** | Trennung von Verantwortlichkeiten |

---

## Prüfungstipps

- Die **drei Komponenten und ihre Aufgaben** sicher benennen.
- Das **Zusammenspiel** beschreiben (Controller ändert Model, Model benachrichtigt View).
- MVC lässt sich mit **Sequenz-/Klassendiagramm** veranschaulichen (Buch-Aufgabe).
