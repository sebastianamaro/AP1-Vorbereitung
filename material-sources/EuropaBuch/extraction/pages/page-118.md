### Ihr Use-Case-Diagramm:

### Pizza Bestellsystem

**[VISUAL: BLANK USE CASE DIAGRAM WORKSPACE]**
A blank workspace where students should draw a UML Use Case Diagram for a pizza ordering system (Pizza Bestellsystem).

Expected elements:
**Actors (Akteure):**
- Kunde (Customer)
- Pizzabäcker/Koch (Pizza chef)
- Lieferant (Delivery person)
- System/Kasse (System/Register)

**Use Cases (Anwendungsfälle):**
- Pizza bestellen (Order pizza)
- Pizza auswählen (Select pizza)
- Belag wählen (Choose toppings)
- Bezahlen (Pay)
- Pizza zubereiten (Prepare pizza)
- Pizza liefern (Deliver pizza)
- Bestellung stornieren (Cancel order)

The diagram should show actors as stick figures outside a system boundary box, with use cases as ovals inside, connected by lines showing which actors interact with which use cases.
