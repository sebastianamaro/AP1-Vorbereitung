Informationen zum Thema Datenintegrität (Fortsetzung)

Information 5: Zu Wahrung der referentiellen Integrität in einer Mitarbeiter-
tabelle möchte der Kunde eine entsprechende Regel (SQL-Befehl CONSTRAINT) in die
Tabelle einfügen. Die Mitarbeitertabelle hat einen Fremdschlüssel Abt_ID, der auf einen gültigen Primär-
schlüssel A_ID in einer Abteilungstabelle verweist. Beim Löschen einer Zeile aus der Abteilungstabelle
soll der eventuell betroffene Fremdschlüsselwert aus der Mitarbeitertabelle auf einen Nullwert gesetzt
werden. Es steht eine Auswahl von Regeln zur Verfügung. Wählen Sie die Regel, die den Anforderungen
entspricht.

D CONSTRAINT Mitarbeiter_FK FOREIGN KEY (Abt_ID)
REFERENCES Abteilungen (A_ID) ON DELETE CASCADE

D] CONSTRAINT Mitarbeiter_FK FOREIGN KEY (A_ID)
REFERENCES Abteilungen (Abt_ID) ON DELETE SET NULL

DJ] CONSTRAINT Mitarbeiter_FK FOREIGN KEY (Abt_ID)
REFERENCES Mitarbeiter (A_ID) ON DELETE SET NULL

XI CONSTRAINT Mitarbeiter_FK FOREIGN KEY (Abt_ID)
REFERENCES Abteilungen (A_ID) ON DELETE SET NULL
