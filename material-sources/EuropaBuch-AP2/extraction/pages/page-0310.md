2.8.6 IPv6 Subnetting

Ausgangsszenario:

Ein Kunde Ihrer Firma ConSystem GmbH erhält von seinem Internet Service Provider eine Anbindung
über IPv6. Der Kunde möchte sein IPv6-Netzwerk in mehrere Subnetzwerke unterteilen, um verschie-
dene Unternehmensbereiche voneinander abzugrenzen.

Aufgabenstellung:
Als erfahrene/-r Auszubildende/-r erhalten Sie den Auftrag zur Planung der IPv6-Subnetze.

das folgende IPv6-Netz zugewiesen bekommen:

Aufgabe 1: Der Kunde möchte insgesamt vier gleich große IPv6-Subnetze bilden.
Berechnen Sie, welche Präfixlänge für die vier Subnetze zu verwenden ist.

Ihre Berechnung:

Ausgangspunkt ist eine Präfixlänge von 56. Um vier Subnetze unterscheiden zu können,
müssen zwei Bits als Subnetzbits genutzt werden.

Somit ist in diesem Fall eine Präfixlänge von 58 notwendig.
56+2=58 Neue Präfixlänge: /58
