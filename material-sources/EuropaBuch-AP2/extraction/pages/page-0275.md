Aspekt 2: In der bisherigen Anwendung wurden Daten in Textdateien gespeichert. Für die neue An-
wendung sollen die Daten in einer Datenbank gespeichert werden. Bei der Auswahl einer geeigneten
Datenbank braucht der Kunde Ihre Unterstützung - beantworten Sie ihm folgende Fragen.

Frage 1: Was ist der Unterschied zwischen einer relationalen Datenbank und einer
NoSQL-Datenbank?

Eine relationale Datenbank basiert auf Tabellen und deren Beziehungen. NoSQL-Datenbanken
(not only SQL-Datenbanken) sind hingegen eine Sammlung von nicht relationalen Daten-
bankmanagementsystemen (DBMS), die in der Regel kein SQL verwenden.

Frage 2: Die bisherige Anwendung hat eine große Menge von Datenblättern in einzel-
nen Textdateien gespeichert. Wie ist Ihre Meinung zum Einsatz einer objektorientierten
Datenbank oder alternativ zum Einsatz einer dokumentenorientierten Datenbank für
die neue Anwendung?

Eine objektorientierte Datenbank könnte für die Speicherung von Textseiten in Form

von Objekten durchaus genutzt werden. Entsprechende Methoden könnten dann auch
Funktionalitäten für die Verarbeitung der Textseiten zu Verfügung stellen. Die bessere Wahl
wäre allerdings die dokumentenorientierte Datenbank, die auf das Speichern und Verwalten
von sehr vielen Dokumenten (natürlich auch Textseiten) sehr gut vorbereitet ist.

Aspekt 3: Die Entwicklung der neuen Anwendung soll mit einer Versionsverwaltung unterstützt werden.
Dem Kunden wurden die beiden Systeme Git und Subversion (SVN) empfohlen.

Erläutern Sie dem Kunden den Unterschied der beiden Systeme:

Die beiden Systeme unterscheiden sich in der Art der Verwaltung. Subversion (SVN) gehört
zu den zentral verwalteten Systemen in Form eines Client-Server-Systems. Die Versionierung
erfolgt dabei in einem zentralen Repositorium. Git gehört zu den verteilten Systemen — es
gibt kein einziges zentrales Repositorium, sondern jeder Nutzer hat eine lokale Kopie, die
über das Git-System synchronisiert werden kann. Dadurch ist Git unabhängiger vom Netz, da
lokal gearbeitet werden kann. Allerdings wird es komplizierter, wenn mehrere Nutzer lokal an
denselben Dateien arbeiten und es später zusammengeführt werden muss. Bei SVN hingegen
darf immer nur ein Nutzer an einer Datei arbeiten (sie wird dann für die anderen gesperrt).
Allerdings ist die Abhängigkeit von Netz oder Internet viel größer.
