Aufgabe 4: Der folgende PHP-Code-Ausschnitt zeigt eine typische Anmeldung an GmbH
einer Webseite mit Datenbankabfrage.

Was versteht man unter SQL-Injection und wie könnte das in dem obigen Code-Ausschnitt eingesetzt
werden?

Ihre Antwort:

SQL-Injection nutzt eine Schwachstelle in Anwendungen mit Datenbankzugriff (meistens
Webanwendungen) aus, indem kritische SQL-Befehle in eine Datenbank-Abfrage ein-
schleust (injiziert) werden. Damit könnte unerlaubter Zugriff auf Daten möglich sein oder
im schlimmsten Fall der Eingriff in das System mit Administrator-Rechten.

In dem obigen PHP-Code könnte das sehr einfach mit den folgenden Anmeldedaten
geschehen:

Benutzername: admin" --

Passwort: in diesem Fall beliebig

Damit entsteht die folgende SQL-Abfrage im PHP-Programm:

SELECT * FROM Benutzer WHERE Name = "admin' --' AND PWD = "beliebig"

Der Doppelstrich „--“ kommentiert den Rest der Zeile aus, inklusive der Passwortprüfung.

Damit prüft die Datenbank nur, ob es einen Benutzer mit dem Namen "admin" gibt —
das Passwort wird ignoriert.

Im schlimmsten Fall hat der Angreifer nun Zugriff mit Administratorrechten.
