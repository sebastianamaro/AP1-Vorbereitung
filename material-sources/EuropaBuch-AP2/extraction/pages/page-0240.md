2.3.2 Softwarequalität

Ausgangsszenario:

Der Leiter der Abteilung Entwicklung möchte eine Präsentation zum Thema ISO/IEC 25000 (interna-
tionale Norm für Qualität von Softwareprodukten) halten. Dazu hat er bereits einige Informationen
zusammengetragen.

Aufgabenstellung:
Als Auszubildende/-n der Firma bittet er Sie, die Informationen zu ergänzen.

Qualität von Softwareprodukten nach ISO/IEC 25000
Aufgabe 1: Die ISO/IEC 25000 kurz dargestellt:

ConSystem
GmbH

Die ISO/IEC 25000 ist eine internationale Normenreihe, die sich mit der Qualität von Software-
produkten beschäftigt. Neben den Grundlagen, die in der ISO/IEC 25000 erläutert werden, gibt
es einige weitere Fortführungen, die sich mit weitergehenden Themen befassen. Im Bereich

der Anwendungsentwicklung ist vor allem die ISO/IEC 25010 zu erwähnen, die Methoden zur
Bewertung der Qualität von Softwareprodukten und softwarebasierten Leistungen bietet. Die
Empfehlungen der ISO 25010 sind deshalb auch eine gute Grundlage für die zu verwendende
Softwarearchitektur.

Aufgabe 2: Wesentliche Qualitätsmerkmale kurz erklärt:

Kurze Erklärung

Die Software erfüllt die festgelegten oder vorausgesetzten Anforderungen.

Die Software zeigt die Fähigkeit das Leistungsniveau unter festgelegten Bedingungen über
einen festgelegten Zeitraum zu bewahren.

Die Software kann das Leistungsniveau und ihre Funktionalität mit möglichst wenigen
Ressourcen umsetzen.

Die Software kann möglichst einfach angepasst werden bzw. ist für eventuelle
Anpassungen vorbereitet.

Die Software kann von einer Umgebung an andere Umgebungen angepasst werden.

Aufgabe 3: Text über die Entwicklung der Norm ISO/IEC 25000 in Englisch:

“ISO/IEC 25000 is the result of the evolution of several other standards; specifically from ISO/IEC 9126,
which defines a quality model for software product evaluation, and ISO/IEC 14598, which defines the
process for software product evaluation.”

Ihre Übersetzung:

Die Norm ISO/IEC 25000 ist das Ergebnis der Weiterentwicklung mehrerer anderer Standards;
insbesondere der Norm ISO/IEC 9126, die ein Modell für die Evaluation von Softwareprodukten
definiert und der Norm ISO/IEC 14598, die den Prozess für die Evaluation von Softwareprodukten
definiert.
