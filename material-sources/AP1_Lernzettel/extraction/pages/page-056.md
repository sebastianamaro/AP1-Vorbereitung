# Page 56 - EPK-Symbole (Fortsetzung)

## EPK-Symbole (Fortsetzung)

| Symbol | Beschreibung |
|--------|--------------|
| Rechteck | **Objekt** - Ein Objekt kann ein Informationsobjekt, ein Material, eine Ressource oder ein Produkt sein. |
| Pfeil (→) | **Kontrollfluss** - Gerichteter, zusammenhängender Graph, dessen Knoten Ereignisse, Funktionen und Verknüpfungsoperatoren sind. |
| Doppelpfeil (↔) | **Informationsfluss** - Gibt an, dass anlässlich einer Funktion Informationen von einem Informationsobjekt gelesen oder auf ein Informationsobjekt geschrieben werden. ODER **Material-/Ressourcenfluss** - Gibt an, dass in einer Funktion Material bzw. Ressourcen verbraucht werden. |
| Linie | **Zuordnung** - Gibt an, welche Organisationseinheit oder welche Objekte einer Funktion zugeordnet sind. |
| ∧ (AND) | **Konnektor "UND"** - Eine Funktion wird ausgeführt, wenn mehrere Ereignisse eingetreten sind. ODER Nach einer Funktion treten mehrere Ereignisse ein. ODER Ein Ereignis tritt ein, nachdem alle direkt vorangestellten Funktionen ausgeführt wurden. |
