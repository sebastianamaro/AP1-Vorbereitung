# Page 83 - Klassendiagramm Symbole

## Symbole des Klassendiagramms

| Symbol | Name |
|--------|------|
| Rechteck mit 3 Bereichen (Name, Attribute, Methoden) | **Klasse** - Enthält Klassenname, Attribute (+/-) mit Typ, und Operationen mit Parametern und Rückgabewert |
| Rechteck mit «Interface» und Methodenliste | **Schnittstelle / Interface** - Definiert Methodensignaturen ohne Implementierung |
| Durchgezogene Linie mit Pfeil → | **Assoziation** - Beziehung zwischen Klassen |
| Durchgezogene Linie mit leerer Dreiecksspitze ▷ | **Vererbung** - Kindklasse erbt von Elternklasse |
| Gestrichelte Linie mit leerer Dreiecksspitze - -▷ | **Implementation** - Klasse implementiert Interface |
| Gestrichelte Linie mit offener Pfeilspitze - -→ | **Abhängigkeit** - Eine Klasse nutzt eine andere |
| Linie mit leerer Raute ◇ | **Aggregation** - "hat ein"-Beziehung, Teile können unabhängig existieren |
| Linie mit gefüllter Raute ◆ | **Komposition** - Starke "hat ein"-Beziehung, Teile können nicht unabhängig existieren |

### Sichtbarkeiten in Klassendiagrammen

- `+` = public (öffentlich)
- `-` = private (privat)
- `#` = protected (geschützt)
